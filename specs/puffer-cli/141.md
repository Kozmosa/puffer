# Tmux Mock HTTP Read Robustness

## Summary

The tmux agent-loop integration test mock now forces accepted provider
connections back into blocking mode before reading HTTP request bytes.

## Design

- Keep the listener nonblocking so the mock server can poll with its existing
  deadline.
- Normalize each accepted `TcpStream` to blocking mode before applying the
  existing read timeout.
- Preserve the current request parser and SSE response contract.

## Compatibility

This is test-harness only. It removes a platform-dependent `WouldBlock` race
seen when macOS accepts a connection before request bytes are ready, without
changing runtime provider behavior.
