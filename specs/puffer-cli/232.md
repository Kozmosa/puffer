# Media capability axis contract

The daemon media capability list exposes image and video configuration through
ordered `axes`. Each axis includes its logical id, display label, role, control,
request field, and wire type so clients can render provider settings without
knowing provider-specific parameter shapes.

Media capability responses must not include legacy top-level `parameters` or
`defaults` fields. Existing model and adapter identifiers remain capability
metadata only: `modelId` is the logical model id clients save as
`logicalModelId`, and `adapter` describes runtime handling without becoming
part of persisted desktop media settings.

Daemon tests must assert representative video capabilities include expected
axes and omit the legacy fields.
