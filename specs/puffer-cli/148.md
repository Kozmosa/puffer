# Gmail Browser Auth-Required State

## Summary

The Gmail-browser subscriber now persists account-scoped auth-required status
when a browser poll lands on Google's sign-in flow, allowing the connection
auth monitor to degrade the connection instead of leaving inbox polling silent.

## Design

- `gmail-browser-accounts/<connection>/auth_state.json` stores only accounts
  whose last poll returned `auth_required`.
- A successful poll for an account clears that account from the auth state.
- Saving Gmail-browser config clears the auth state so a repaired connection
  starts clean.
- Runtime config changes prune auth-required entries for accounts that are no
  longer configured.
- The built-in connection auth checker reports Gmail-browser auth as unhealthy
  when any currently configured account is marked auth-required.

## Compatibility

Gmail-browser `auth_required` events remain control events and still do not
match workflows. Existing subscription auth refresh logic owns the degraded
connection notice and repair instruction.
