# Tmux hot-reload wait budgets

## Scope

`puffer-cli` tmux hot-reload integration coverage now uses wider wait budgets for
startup, slash-command rendering, and resource reload acknowledgement.

## Design

The hot-reload test still drives a real TUI inside tmux and verifies that a new
workspace skill appears without `/reload-plugins`. Only the polling timeouts
changed:

- TUI startup waits up to 30 seconds.
- Slash-command echo waits up to 30 seconds.
- Resource reload acknowledgement waits up to 20 seconds.

## Compatibility

This does not change product behavior. It makes the existing integration test
match the slower timing envelope seen when the full workspace suite runs tmux
tests after heavy Cargo builds or target cleanup.
