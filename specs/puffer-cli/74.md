# Workflow Binding Create RPC

## Design

The daemon exposes `workflow_binding_create` for deterministic workflow action
binding creation from desktop clients. The first supported action is
connection-triggered `file_append`, matching the common "messages containing X
to /tmp/file" workflow without requiring the user or model to construct action
YAML.

## Contract

- Input accepts `connection_slug`, optional `connector_slug`, optional `slug`,
  optional `description`, optional regex `pattern`, `file_append_path`, and
  optional `enabled`.
- Existing connections are validated against their stored connector and must
  support workflow trigger events.
- Planned connections may be used when `connector_slug` names a connector that
  can produce workflow trigger events; the binding becomes active once the
  connection exists and the subscription manager refreshes consumers.
- `pattern` is stored as a case-insensitive regex filter. Empty strings and
  `.*` mean no filter.
- `file_append_path` is validated by the existing subscription action rules:
  safe relative paths or absolute `/tmp` paths only.
- The RPC upserts by slug and returns the refreshed workflow snapshot.

## Compatibility

Native workflow definition save/toggle behavior is unchanged. Snapshot
workflow binding entries now include `action_path`, `action_format`, and
`filter_pattern` so GUIs can render non-monitor actions without reparsing the
persisted binding file.
