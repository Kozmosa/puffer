# Contact Store Version Default

## Summary

The contacts runtime store now writes the current schema version for new and
legacy files instead of preserving the derived `0` default.

## Design

- `ContactStoreFile::default()` initializes `version` to the current contacts
  store schema version.
- Loading a store whose `version` is missing or zero normalizes it to the
  current version before callers receive the snapshot.
- Saving clones and normalizes the store snapshot before writing
  `.puffer/runtime/contacts.json`, so callers cannot accidentally persist a
  zero-version contacts file.
- Existing non-zero future versions are left intact by the normalizer.

## Compatibility

Existing contacts data remains JSON-compatible. This changes only the persisted
metadata version for missing or zero-version files; saved contacts, inferred
contacts, and legacy proposal rows keep their current shapes.
