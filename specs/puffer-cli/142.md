# Connector Delete Monitor Cascade

## Summary

Deleting a connector connection now also deletes monitor configuration for that
connection.

## Design

- The daemon `workflow_connection_delete` RPC first removes the connection from
  the connection store.
- It then deletes enabled or paused monitor workflow bindings whose
  `connection_slug` matches the deleted connection.
- Monitor detection covers the canonical `monitor-<connection>` slug and
  triage-agent bindings whose description identifies them as monitors.
- The connection's monitor memory file under
  `.puffer/runtime/monitors/<connection>.md` is removed if present.
- Existing monitor task records and workflow history are retained as historical
  data.

## Compatibility

Non-monitor workflow bindings are not deleted by this cascade. Missing monitor
bindings or missing monitor memory files are treated as already-cleaned-up
state.
