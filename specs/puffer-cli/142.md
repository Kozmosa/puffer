# Tmux Connector Command Input Robustness

## Summary

The connector-monitor tmux tests now allow more time for slash-command input to
render before pressing Enter.

## Design

- Introduce a dedicated `PROMPT_INPUT_TIMEOUT` for command-entry confirmation.
- Keep the prompt guard so tests still fail with pane context if input is lost.
- Increase the guard from 3 seconds to 20 seconds to tolerate full-suite CPU
  contention and slash popup redraw latency.

## Compatibility

This is test-harness only. Runtime connector commands, prompts, and workflow
generation are unchanged.
