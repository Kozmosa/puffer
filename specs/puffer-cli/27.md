# 27. Immediate Daemon Turn Cancellation

## Summary

Desktop `cancel_turn` now marks the session interrupted immediately instead of
waiting for the provider loop to unwind.

## Design

- In-flight daemon turn handles retain the session id, event channel, and a
  shared cancellation-report flag.
- `cancel_turn` flips the cancel token, drains pending prompts, persists an
  interrupt system message, and publishes a `turn-error` with category
  `cancelled` right away.
- Cancellation reporting first preserves the submitted user prompt if the worker
  has not reached its normal persistence point yet.
- The worker checks for cancellation before persisting a user prompt and skips a
  duplicate cancellation event if the RPC already reported it.

## Compatibility

Existing provider-loop cancellation stays in place. Clients still receive a
`turn-error` event for cancelled turns, but now receive it promptly from the
cancel RPC path.

## Tests

The daemon smoke tests cover cancelling while the mock OpenAI response is still
delayed, then assert the desktop session timeline contains the submitted prompt
plus interrupt and the grouped session activity is already `idle`.
