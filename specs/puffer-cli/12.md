# Desktop auth provider aliases

Daemon RPCs that mutate provider credentials now canonicalize desktop-facing
aliases before touching the provider registry or auth store.

`codex` is treated as `openai`, and `claude` is treated as `anthropic` for API
key login, OAuth login, external credential import, and logout. The canonical
provider id is also used when refreshing default routing after authentication.

This keeps legacy or desktop UI clients from storing credentials under alias
keys or failing reconnect flows with `unknown provider` after the rest of the
session and model-selection RPCs have already accepted those aliases.
