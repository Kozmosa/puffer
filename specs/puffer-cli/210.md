# Contact Store Saved Row Repair

## Summary

Contact store sanitization now drops legacy saved-contact rows that no longer
contain any valid contact IDs, and repairs blank saved-contact names or local IDs
when a row still has usable routing data.

## Design

- `load_store` continues to normalize saved contact IDs before returning a
  contacts snapshot.
- Saved contacts whose normalized `contact_ids` list is empty are discarded
  because they cannot route monitor scopes or contact context requests.
- Legacy rows with valid contact IDs but blank names use the primary normalized
  contact ID as a display fallback.
- Legacy rows with valid contact IDs but blank local IDs receive a stable
  deterministic local ID derived from their primary normalized contact ID.

## Compatibility

Normal `contacts_save` records are unchanged because the save path already
requires a non-empty name and at least one valid contact ID. This repair only
affects older or manually edited `contacts.json` files that violate the current
saved-contact contract.
