# Desktop Proxy Connectivity Test

## Design

The daemon proxy test now checks generic network connectivity instead of
provider discovery availability.

## Architecture

- `test_proxy` keeps accepting a saved `proxyId` or inline endpoint draft.
- The daemon probes public, no-auth connectivity URLs through the selected proxy.
- `puffer-core::test_proxy_endpoint` returns latency plus the observed HTTP
  status code to the daemon.

## Logic

- The primary target is `https://www.gstatic.com/generate_204`.
- Fallback targets are `https://cp.cloudflare.com/generate_204` and
  `https://www.cloudflare.com/cdn-cgi/trace`.
- Any HTTP response from a target is treated as connected.
- Transport failures still return `ok=false` with a scrubbed, truncated error.

## Contracts

- `ProxyTestResultDto` includes `statusCode`.
- Successful proxy tests report `ok=true`, `latencyMs`, `statusCode`, and a
  message naming the connectivity target.
- Provider auth errors no longer make proxy connectivity appear failed.

## Compatibility

No legacy provider-discovery proxy-test behavior is retained.
