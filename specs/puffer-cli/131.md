# Parallel Monitor Agent Triage

Monitor triage and ignore-analysis agent actions no longer take the
process-wide workflow runner mutex. Each action builds an independent
background `AppState`, session, and auth-store clone, so the global lock was
only serializing otherwise independent monitor work.

Native workflow DAG runs and tool actions still use the workflow runner mutex
because they can mutate shared workflow/tool state. Agent-backed monitor
processing can now proceed concurrently up to the subscription router's action
processor limit, improving catch-up throughput after subscriber restarts.
