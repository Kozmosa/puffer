# Shared internal tool descriptors

## Purpose

Avoid duplicating internal CLI tool metadata in `puffer-cli`.

## Design

`puffer internal-tool aliases` now reads internal CLI tool ids, aliases, and
skill names from `puffer-tools::internal_tools`. Browser remains the only
current descriptor, but future internal tools can be added without updating a
CLI-local metadata trait.

## Contract

The hidden internal-tool command dispatch remains CLI-owned because it parses
tool-specific command arguments. Metadata used for shell setup is shared.

## Verification

Compiler and focused tool tests verify that CLI alias output still builds
against the shared descriptor API.
