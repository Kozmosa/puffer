# Contact Daemon Search and Telegram Fixes

- `contacts_search` now invokes connector-native search operations when available
  and falls back to filtered saved contacts plus locally inferred candidates. It no
  longer aliases list behavior for every query.
- Connector-native candidates are merged only after their contact ids normalize and
  match the connector family that produced them.
- Telegram diagnostics ranking requires handle-backed contact ids and skips bot
  chats or bot senders. Numeric Telegram ids are intentionally not surfaced as
  user-facing contact ids.
- Personal-chat reply scoring only counts the next message in the same chat when
  that next message is outgoing. Later unrelated replies no longer inflate an
  incoming message's importance.
- Telegram diagnostics ranking lives in a dedicated module so daemon contact API
  code stays focused on JSON-RPC request handling and connector candidate merging.
