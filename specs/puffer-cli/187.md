# Contact Inference OpenAI Error Status

## Design

OpenAI-backed contact inference treats non-2xx HTTP responses as request
failures before attempting to parse completion JSON.

## Architecture

`daemon_contacts_infer` validates the blocking `reqwest` response status in a
dedicated response reader and only parses the JSON body after
`error_for_status` succeeds. Error JSON from OpenAI therefore drives the
existing failed-tool trace and local heuristic fallback instead of being
interpreted as a successful response with zero proposals.

## Compatibility

The `contacts_infer` RPC shape, trace event schema, no-key heuristic behavior,
and OpenAI request payload stay unchanged. This only makes API error handling
explicit and covered by a loopback HTTP regression test.
