# Workflow Snapshot Connector Context

## Summary

The daemon `workflow_list` RPC now augments the native workflow snapshot with a
compact connector catalog and current connection records for desktop workflow
editing.

## Behavior

- `workflow_list` still returns the persisted workflow and run snapshot from
  `WorkflowStore`.
- When the subscription manager is installed, the response also includes:
  - `connectors`: connector slug, description, skill, capability flags, and
    action slugs.
  - `connections`: connection slug, connector slug, description, lifecycle
    state, consumer state, and auth-notice flag.
  - `connector_error`: `null` unless connector context could not be refreshed.
- If the subscription manager is unavailable, the RPC still succeeds with empty
  connector arrays and a diagnostic `connector_error` string.

## Compatibility

The original `workflows` and `runs` fields are unchanged. The extra fields are
optional for older clients and avoid embedding full action schemas so the
desktop picker stays fast and provider-schema details do not leak into the UI
contract.
