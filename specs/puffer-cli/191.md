# Contact Inference History Context Labels

History-backed contact inference context now labels Telegram snippets before they
are added to candidates.

Workflow history fallback snippets use `history_message` and format Telegram
payload text as `[history][dest: ...][from: ...] ...`, using cached trigger
payload fields such as `chat_title`, `chat_username`, `sender_name`, and
`sender_username`. Raw connector payloads remain excluded from inference unless
the caller explicitly requested payload retention through existing context
options.

Telegram diagnostics context keeps its existing `recent` and `interacted`
labels. The change only makes workflow-history fallback context auditable in the
same prompt and trace surfaces.
