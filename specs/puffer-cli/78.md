# Tmux Hot Reload Test Synchronization

The tmux hot-reload test now verifies the user-visible `/skills` result instead
of blocking on the transient reload status line. It closes the skills overlay
between attempts, nudges the TUI loop, and retries until the newly dropped
workspace skill appears.

Runtime behavior is unchanged: the watcher still raises the reload signal, the
TUI still reloads resources, and `/skills` remains the observable contract for
newly available skill manifests.
