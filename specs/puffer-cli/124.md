# Gmail Browser Daemon Setup

- `start_connector_setup` routes `gmail-browser` setup through a daemon-owned flow instead of the generic `/connect` question executor.
- The daemon opens Google account chooser/sign-in pages in the shared Puffer managed Chrome profile and emits connector setup question events with `browserSessionId`, `browserTabId`, and `browserUrl` metadata.
- Gmail setup discovers already signed-in Google accounts from the Puffer profile, lets the user sign in to additional accounts in the same browser session, then saves selected accounts to the per-connection Gmail browser config.
- Completing setup creates or refreshes the `gmail-browser` connection record as authenticated and refreshes connection consumers/auth state so monitor streams can start using the configured accounts.
