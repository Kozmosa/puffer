# OpenAI Realtime client-secret daemon RPC

## Summary

The daemon now exposes `create_openai_realtime_client_secret` for desktop clients
that need a browser-safe Realtime API token. The daemon reads the existing OpenAI
API-key credential from the auth store, mints a short-lived client secret, and
returns only the ephemeral value plus metadata.

## Contract

- Params accept optional `providerId`, `model`, `voice`, `reasoningEffort`, and a
  flexible `session` object.
- Defaults are `gpt-realtime-2` and `marin`; `reasoningEffort` is opt-in.
- The generated session config is forced to `type: realtime` and includes
  `audio.input.transcription: null` so the path does not implicitly enable a
  separate transcription model.
- OAuth credentials are rejected for now because the Realtime client-secret
  minting path is API-key based.
- The handler runs off the tokio runtime and does not perform provider discovery.
