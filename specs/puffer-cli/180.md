# Telegram Candidate Name Fallback

## Design

Telegram contact candidates now avoid first-name-only labels when local
diagnostics include a handle. One-token display names are shown as
`Name (@handle)`, while multi-token names remain unchanged.

## Architecture

The Telegram diagnostics reader derives candidate display names from the
diagnostic display name plus the normalized contact username. This stays in the
local diagnostics path and does not add live Telegram calls during contact
inference.

## Compatibility

Existing diagnostics remain readable. Candidates with full display names, such
as `Jeff Liu`, keep those names; candidates with only a short display name gain
the handle for disambiguation.

Self messages are not contact candidates. Telegram group messages sent by the
local account still remain available as reply evidence, while direct self-chat
diagnostics and local group sender rows are excluded from `CollectContacts`
results.
