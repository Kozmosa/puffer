# Sessionless Connector Setup RPC

- The daemon exposes `start_connector_setup` for deterministic `/connect` setup without a persisted session.
- Requests accept `setupId`/`setup_id` plus a `/connect <connector> <connection>` message; the setup id becomes the turn id and is limited to ASCII alphanumeric, dash, and underscore characters.
- Connector setup events stream on `connector-setup:<setupId>:event` and reuse the existing `turn-start`, `user-question-request`, `turn-complete`, and `turn-error` payload shapes.
- User-question answers and cancellation still route through the normal `turnId` handlers because the sessionless setup is registered in the in-flight turn table.
- Sessionless connector errors publish only to the setup channel and do not emit `workspace:sessions:changed`.
- The RPC builds a synthetic in-memory `AppState` for `execute_connect_flow`, so connector records and tool effects still occur while no session metadata or transcript is written.
