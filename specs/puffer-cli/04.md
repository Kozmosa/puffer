# Browser ref resolution update

## Purpose

Agent browser actions now resolve `@eN` refs against the live DOM using the
ref snapshot metadata, instead of relying only on the original viewport point.
This prevents stale-ref failures after navigation, scroll, or reflow.

## Design

`browser_agent snapshot` still records the same ref payload:

- `ref`
- `role`
- `name`
- `tag`
- `href`
- `x`
- `y`

The action scripts for click, double-click, hover, focus, fill, select, upload,
check, uncheck, and scroll-into-view now resolve the ref through a shared
helper that:

- matches the live element by role, tag, name, and href when possible
- falls back to the nearest visible candidate when the old point is stale
- preserves the existing `@eN` user-facing ref ids

## Compatibility

- Snapshot output format is unchanged.
- Existing refs remain valid when the page still contains a matching element.
- Stale refs now fail with a direct "no matching browser ref" error instead of
  a hard `document.elementFromPoint` miss.
