# Daemon desktop turn options

## Purpose

Desktop clients send model, reasoning, fast-mode, and permission choices as
structured turn options. The daemon must honor those fields, not only the older
`modelOverride` compatibility field.

## Behavior

- `run_agent_turn` parses `providerId`, `modelId`, `thinkingOptionId`,
  `fastMode`, and `permissionMode`.
- `modelOverride` remains supported and takes precedence for older clients.
- Provider/model selection is applied to the per-turn `AppState` before runner
  selection.
- Thinking and fast-mode options update the same per-turn model preferences
  used by slash commands.
- Permission mode maps to the turn sandbox mode; `full-access` also grants the
  per-session allow-all state for that turn.

## Compatibility

When none of the desktop fields are present, daemon turns keep the session and
workspace defaults exactly as before.
