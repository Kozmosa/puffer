# Desktop Verified Skill Runtime Preflight

Desktop Verified Skill import and readiness now use the core Lambda host
catalogue runtime validator. A library is not saved as runtime-ready unless each
generated host catalogue parses, declares concrete tool bindings, and provides
concrete input contracts for those bindings.

Readiness snapshots report runtime validation failures per skill as not
gate-ready. That keeps Desktop from showing a Verified Skill as model-invocable
when the runtime would later reject ordinary host calls because of unsupported
refinement predicates or missing concrete input contracts.

The repository ignores `.DS_Store` files so macOS metadata does not appear in
future commits.
