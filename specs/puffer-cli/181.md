# Telegram History Peer Names

## Design

History-derived Telegram contact candidates now choose names that match the
contact id source. Direct user chats use `chat_title` because the contact id is
the peer's `chat_username`; group messages continue to use `sender_name`.

## Logic

`name_from_payload` checks `chat_kind == "user"` before the generic sender-name
fallback. This prevents outgoing direct messages from naming the peer candidate
after the local account.

## Compatibility

Payload shape and normalized contact ids are unchanged. The change only improves
candidate display names derived from existing Telegram history payloads.
