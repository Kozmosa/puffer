# Lambda Skill Library Desktop RPC

## Scope

The local daemon exposes the Lambda Skill library manifest layer to desktop
without moving external libraries into the Puffer repository.

## Change

- `list_lambda_skill_libraries` returns workspace/user manifest directories,
  parsed manifest records, the Lambda Skill doctor summary, and warning lines.
- `save_lambda_skill_library` writes one YAML manifest under either the
  workspace or user `resources/lambda_skill_libraries` directory.
- Folder-only saves infer the default generated host catalogue
  `out/host.json` and concrete tool allow-list when generated catalogues are
  already present in the selected library.
- Saved manifests normalize empty strings, trim list values, reject unsafe ids,
  and preserve typed `allowed_tools`, global `host_tool_bindings`, per-skill
  host bindings, `user_invocable`, and `disable_model_invocation`.

## Compatibility

The RPC only edits declarative config files. Resource loading, readiness checks,
model-invocable gating, and concrete tool execution continue to use the
existing Lambda Skill loader and doctor paths after reload or restart.
