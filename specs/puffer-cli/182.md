# Telegram Peer Cache Contact Names

## Summary

Contact collection now reads Telegram account `peer-cache.json` files before
ranking diagnostics-backed candidates.

## Design

- Each account under `telegram-accounts/<account>/` may contain
  `peer-cache.json`.
- The cache is local durable metadata written by the Telegram subscriber.
- `CollectContacts` remains offline and does not call live connector methods in
  inference mode.

## Logic

- Telegram diagnostic rows still provide ranking, context, message recency, and
  the normalized `telegram@handle` contact id.
- When a cached user record matches that handle, the cached name is preferred
  over `chat_title` / `sender_name`.
- Name merging keeps the most complete visible value, so a cached
  `Rin Tohsaka` replaces an abbreviated diagnostic `Rin`.
- Bot records are ignored.

## Compatibility

- Existing diagnostics without a peer cache continue to use the previous
  diagnostic fallback.
- The cache is optional and parse failures are non-fatal.
