# Native CEF Target Reuse

The daemon Browser root now treats a CEF remote debugging endpoint as an
attachment surface only. When the desktop native CEF bridge is active, the
daemon may attach to an existing page target exposed by the embedded view, but
it no longer creates replacement CEF targets through DevTools `/json/new`.

This prevents agent Browser recovery from opening a CEF-created browser window
outside the Puffer Desktop GUI after a user manually closes the embedded tab.
Managed screencast Chrome behavior is unchanged: roots owned by the daemon still
create fresh page targets when they need another tab.

If a CEF endpoint is reachable but no embedded page target exists, agent actions
now fail with a clear missing-target error instead of manufacturing a native CEF
page. The desktop should open or focus a native CEF tab first; the daemon then
attaches to that real embedded target.
