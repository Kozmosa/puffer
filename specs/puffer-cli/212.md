# Contact Search Dead Path Cleanup

## Summary

`contacts_search` no longer carries an unreachable connector-native search
branch. Bulk contact list and search responses remain local/cache-backed so
they cannot block on live connector commands.

`contacts_list`, `contacts_search`, and `contacts_infer` now report malformed
parameter objects instead of silently falling back to defaults. Missing fields
still use the existing default values.

## Compatibility

`contacts_context` still uses connector-native context methods when full
context is requested. Request and response shapes are unchanged.
