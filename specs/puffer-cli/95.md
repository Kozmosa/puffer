# Verified Skill Concrete Tool Registry Support

Desktop Verified Skills import validation now checks concrete tool names against
the normal Puffer `ToolRegistry` built from loaded resources instead of a
separate hardcoded allowlist.

## Design

`daemon_lambda_skills::import_validation` owns a small `LambdaConcreteToolSupport`
wrapper backed by `ToolRegistry::from_resources` over `load_tool_resources`, a
tools-only resource load that preserves the normal resource layering without
scanning skills, providers, or plugins. The host catalogue shape is parsed
through `puffer-core`'s Lambda host catalogue helper rather than a second
daemon-local JSON schema. Import validation and the desktop readiness snapshot
both use those runtime-aligned helpers when checking `concreteTools`, so
workspace or user tool resources that Puffer can actually execute are accepted
automatically.

## Contracts

This removes the duplicate static concrete-tool catalogue from the daemon. A
Verified Skill import is considered concrete-tool supported only if the
currently loaded Puffer tool registry can resolve the named concrete tool.
Unsupported concrete tools still fail before a manifest is written.

## Verification

`cargo test -p puffer-cli desktop_lambda_skill_library_save_uses_tool_registry_for_concrete_tool_support`
covers a workspace-defined runtime tool used as a Verified Skill concrete tool.
