# Contact Save Identity Replacement

## Design

`contacts_save` now treats normalized contact IDs as unique routing keys across
saved contacts. Saving a contact absorbs any existing saved contact whose
normalized IDs overlap the saved record, preserving the union of connector IDs
before writing `contacts.json`.

## Compatibility

The RPC request and response shapes are unchanged. Existing contacts with
distinct IDs are preserved, while duplicate records for the same connector
identity collapse to the most recently saved user-curated record without
discarding non-overlapping connector IDs from the replaced records.
