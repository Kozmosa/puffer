# Secret Description RPC Metadata

The daemon and desktop API expose secret descriptions in the existing secrets
settings RPC surface.

- `save_secret` accepts optional `description` and writes it through
  `SecretUpsert`.
- `SecretSummaryDto` includes optional `description` for CLI desktop API and
  native Tauri paths.
- The Chrome sync RPC name remains `import_chrome_secrets`; its behavior is
  implemented by the vault's sync path for compatibility with existing clients.
