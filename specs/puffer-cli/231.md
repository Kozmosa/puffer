# Contact inference default model routing

Contact inference must reuse the configured OpenAI/Codex default model when no
explicit inference model is supplied. Model selectors may be scoped as
`openai/<model>` or `codex/<model>` and must be normalized before sending the
OpenAI Responses request body.

If the configured default provider is not OpenAI/Codex and the model selector is
unscoped, contact inference must ignore that default and keep the existing local
fallback model. This keeps contact inference aligned with conversation routing
without sending Anthropic or other provider model ids to the OpenAI endpoint.
