# Desktop Verified Skill Precompiled Catalogues

The desktop daemon no longer discovers or persists `lskillc` when a user adds a
Verified Skills folder. Saving a folder infers readiness only from checked-in
`out/host.json` catalogues and `out/GENERATED.SKILL.md` descriptors.

The lightweight snapshot still accepts older manifests that contain
`compiler_path`, but that field is not used to mark a skill ready. A folder with
only `.lskill` source and generated prompt text remains visible as recognized
but not model-invocable until precompiled host catalogue output is present.

Validation:
- `cargo test -p puffer-cli desktop_lambda_skill -- --nocapture`
