# Browser Agent Debug Trace

The daemon Browser runtime now supports an opt-in diagnostic trace controlled
by `PUFFER_BROWSER_DEBUG=1`. When enabled, Browser agent requests and tab
registry operations emit compact stderr markers for action routing, tab open,
tab close, page-session close, tab metadata removal, and backend-state
recording.

The trace is designed for native CEF crash debugging from the desktop app. It
shows whether a `browser_agent close` request reached the daemon, whether a live
page worker existed for the requested backend session, and whether daemon tab
metadata was actually removed. This is especially useful for distinguishing a
native CEF crash from a local-only desktop tab whose daemon record never
existed.

The diagnostics are passive and do not change Browser tab behavior. Normal
daemon runs remain quiet unless the environment variable is present.
