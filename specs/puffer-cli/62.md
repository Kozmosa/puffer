# Telegram message listing CLI

- The hidden `puffer internal-tool telegram` surface now supports
  `list-messages` with the alias `messages`.
- `list-messages` accepts `--peer`, `--limit`, `--before-id`, and
  `--succint`/`--succinct`. It maps to the Telegram workflow action
  `list_messages`.
- `search-messages` now defaults `--context` to `0`. This preserves explicit
  context support while making the normal search path avoid expensive context
  scans.
- Existing connection selection flags and login/import commands are unchanged.
