# Workflow Binding Toggle RPC

## Summary

The desktop daemon workflow snapshot now includes subscription workflow
bindings and exposes a `workflow_toggle` RPC.

## Behavior

- `workflow_list` returns `workflow_bindings` and `workflow_binding_error`
  alongside native workflow definitions, connector context, and monitor tasks.
- Binding rows include slug, connection, connector, enabled state, action type,
  monitor classification, and the conventional monitor memory path.
- `workflow_toggle` accepts `{ "slug": string, "enabled": boolean }` and toggles
  either a native workflow definition or a subscription workflow binding.
- Native workflow toggles also synchronize a matching `workflow-<slug>` binding
  when one exists.

## Compatibility

Existing workflow list, save, run-list, and run-show RPC contracts are
unchanged. Clients that do not read `workflow_bindings` keep seeing the same
workflow snapshot shape they used before.
