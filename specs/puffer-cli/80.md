# puffer-cli update 80

CLI session writers preserve tool invocation metadata.

## Design

Non-interactive runs and daemon stream persistence copy non-null
`ToolInvocation.metadata` into `TranscriptEvent::ToolInvocation`.

## Contracts

- Existing transcript readers remain compatible because metadata is optional.
- CLI timeline and activity views ignore metadata unless a future view chooses
  to render it.
- Tool output text and activity classification are unchanged.

## Compatibility

This keeps Lambda Skill trace annotations durable across CLI-created sessions
without changing the desktop API response shape.
