# Tmux Interaction Harness

## Summary

The tmux-backed CLI integration tests now drive slash commands and prompt text
through the same visible states a user sees in the TUI.

## Design

`puffer-test-support` sends ordinary text to tmux as literal input and keeps
special keys such as `Enter`, `Escape`, and arrows as key names. Timeout errors
include the last pane capture so failures show the actual TUI state.

Tests that use slash command suggestions now wait for the visible popup row,
confirm the completed command is in the prompt, and then submit it. The
hot-reload test waits for the reload system message before reopening
`/skills`, avoiding races where a resource reload clears in-flight input.

The tmux agent-loop fixture sets an explicit default Anthropic model so the
mock provider flow reaches the prompt instead of stopping at model selection.

## Contracts

- Literal tmux input must preserve normal prompt text exactly.
- Slash-command tests must wait on visible TUI state instead of fixed sleeps.
- Hot-reload verification must wait for the reload event before typing the next
  command.
- Agent-loop tmux tests must start with a resolvable default model.
