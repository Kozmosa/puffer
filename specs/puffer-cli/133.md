# Gmail Browser And Email ConnectorAct Routing

## Summary

`puffer-cli` now routes email-style connector actions to subscriber children:
`email_act` for the IMAP/SMTP subscriber and `gmail_browser_act` for the
Gmail-browser subscriber.

## Gmail Browser Subscriber

The Gmail-browser subscriber now reads stdin control commands while preserving
its existing polling loop. It continues to stream newly observed Gmail inbox
rows, and also accepts list/search, mark-read, draft, send, and delete actions
against the daemon-managed global Puffer browser profile.

## Routing

The connector action executor starts the required subscriber if it is not
already running, sends the custom action command, waits for the action-specific
terminal event, and returns the subscriber payload to `ConnectorAct`.

## Compatibility

The Gmail path does not ask the agent to open a separate browser window. It
uses the account list from `gmail-browser-accounts/<connection>/config.toml`
and the global Puffer-owned browser profile, so connector authentication no
longer depends on selecting or copying a desktop Chrome profile.
