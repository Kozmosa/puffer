# Subscription Autostart State Resolution

## Summary

Connector-backed monitor bindings now instantiate their subscriber manifest from
the binding's `connector_slug` even when the corresponding connection record is
missing from `connections.json`.

## Design

- Autostart still prefers a real connection record when one exists.
- If the connection row is absent, autostart checks enabled workflow bindings for
  the requested connection slug and resolves the connector template from the
  binding.
- The resolved template is passed through the normal
  `connection_subscriber_manifest` path with a synthetic authenticated
  connection record, preserving per-connection state roots such as
  `telegram-accounts/<connection>`.
- Direct subscriber manifest fallback remains available for legacy
  non-connector subscriptions.

## Compatibility

This keeps existing direct subscribers working while preventing connector-backed
monitors from falling back to bundled resource-local state directories. It is
especially important for personal-account subscribers, where session files and
delivery cursors must stay in the user config state root rather than under
`resources/`.
