# Daemon model thinking options

## Purpose

Desktop clients need model reasoning choices from the real daemon so the chat
composer can expose the same effort controls that slash commands support.

## Behavior

- `list_provider_models` returns `thinkingOptions` for models with
  `supportsReasoning=true`.
- Options are derived from the provider preference family so Anthropic, OpenAI,
  and generic providers expose the right effort vocabulary.
- `defaultThinkingOptionId` marks the provider-family default.
- Models without reasoning support omit thinking choices.

## Compatibility

Existing model descriptor fields are unchanged. Older clients that ignore
`thinkingOptions` continue to receive the same model list shape.
