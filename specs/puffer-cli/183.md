# Telegram Contact Context Sections

## Design

Telegram-backed contact context is now emitted as marked message sections instead
of isolated entropy-ranked snippets. Each Telegram candidate can carry up to ten
recent message rows plus ten interaction pairs, where each pair includes the
contact message and the user's reply.

## Architecture

- `contacts_list` remains lightweight and does not include context payloads.
- `contacts_context` and contact inference use the same diagnostics-backed
  Telegram context builder.
- Telegram context rows include destination metadata in text and payload:
  direct chats use the user/contact label, and group chats use the group title.
- Interaction rows are marked separately from recent rows with
  `telegram_interaction_contact_message` and `telegram_interaction_user_reply`.

## Compatibility

The `ContactContext` wire shape is unchanged. New section, destination, sender,
and reply-pair metadata is added inside the existing `payload` object, while the
human-readable `text` field is prefixed with stable section markers for model
prompts and UI rendering.
