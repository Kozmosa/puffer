# Browser Network Idle Filtering

- Browser navigation readiness waits release when CDP reports
  `Page.domContentEventFired`, so default open/screenshot/console flows use DOM
  readiness instead of waiting for full page load or background network quiet.
- BrowserAction `waitNetworkIdle` now tracks only meaningful page-load network
  work instead of every CDP request.
- Non-render-blocking request types such as `Ping`, `Preflight`, `EventSource`,
  `WebSocket`, reporting, and beacon-like traffic do not keep the idle gate
  open.
- Known telemetry, analytics, RUM, tracking, and Shopify monorail-style hosts or
  paths are ignored for network-idle accounting.
- Unknown document, fetch, XHR, script, image, font, and other ordinary
  application requests remain tracked until CDP reports `loadingFinished` or
  `loadingFailed`.
- The timeout error's active-request count now represents active meaningful
  requests, not background telemetry noise.
