# Tmux Codex Alias Turn Coverage

## Scope

The native TUI must accept `codex` as the user-facing OpenAI provider alias and
complete a first prompt through the real tmux-driven agent loop.

## Change

- Adds a tmux agent-loop regression with `default_provider = "codex"` and
  `default_model = "codex/gpt-5"`.
- The test uses a workspace OpenAI provider override that points to an
  in-process Responses API SSE mock.
- The pane must render the mock assistant reply and the captured request must
  hit `/v1/responses` with the typed prompt.

## Compatibility

No runtime behavior changes. This is coverage for the existing alias contract
across the real TUI input path.
