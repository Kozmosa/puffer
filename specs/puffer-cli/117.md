# Workflow Background Session Tagging

- Workflow-triggered agent execution creates sessions with the internal `puffer:background` tag.
- Subscription triage agents, ignore-analysis agents, tool-action app state, cron workflow agents, and DAG workflow agents all share the same tagged session creation path.
- Background workflow sessions still persist transcripts for audit/debugging through the session store, but normal session listing APIs hide them from desktop workspace/session surfaces.
- Model selection behavior is unchanged: explicit workflow action models still override the authenticated provider fallback for the temporary app state.
