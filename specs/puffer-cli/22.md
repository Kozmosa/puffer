# 21. Subscription Runtime Async-Safe Shutdown

The CLI subscription manager owns a dedicated Tokio runtime. Dropping that
wrapper from an existing Tokio worker must not panic with Tokio's blocking
runtime shutdown error.

Requirements:

- `SubscriptionRuntime` owns its runtime through an optional slot so shutdown
  can consume the runtime before Rust's default field drop runs.
- Manager shutdown still stops the router and subscriber supervisors before the
  runtime is released.
- If shutdown is requested while already inside a Tokio runtime, manager
  shutdown runs on a plain OS thread so its internal `block_on` calls do not try
  to enter a runtime from within another runtime.
- The Tokio runtime is released with `shutdown_background`, avoiding the
  blocking drop path that panics in async contexts.

Verification adds a regression test that constructs a subscription runtime,
enters a Tokio context, drops the wrapper, and asserts that no panic occurs.
