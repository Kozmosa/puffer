# puffer-cli 233: WorldRouter image settings capabilities

The daemon media capability contract now treats WorldRouter as a mixed image
provider.

When connected, `list_media_capabilities(kind = image)` exposes:

- `gpt-image-2` with `adapter = images_json`;
- `gemini-2.5-flash-image` with `adapter = gemini_generate_content`;
- `gemini-3-pro-image-preview` with `adapter = gemini_generate_content`;
- `gemini-3.1-flash-image-preview` with `adapter = gemini_generate_content`.

Older tests that hid WorldRouter from image settings are obsolete. The daemon
should continue filtering by connected executable capability, but no longer
assumes every cloud image model uses `images_json`.
