# Telegram History Contact Names

## Design

History-derived Telegram contact candidates now use the offline Telegram peer
cache before falling back to the older workflow payload display name.

## Architecture

`daemon_contacts_telegram.rs` exposes the existing peer-cache username-to-name
map for all configured Telegram accounts. `daemon_contacts.rs` loads that map
once while collecting workflow-history candidates and applies it only to
normalized `telegram@...` ids.

## Compatibility

No live connector calls are added to contact inference or contact listing. The
RPC shapes are unchanged; only Telegram candidate display names become more
complete when cached peer metadata is available.
