# Workflow binding delete RPC

The desktop daemon now exposes `workflow_binding_delete` for removing
subscription-backed workflow actions from Corbina.

## Contract

- Request shape: `{ "slug": "<binding-slug>" }`.
- The handler rejects missing or blank slugs before touching storage.
- The handler deletes the matching workflow binding through the subscription
  store, refreshes connection consumer state, and returns the same refreshed
  workflow snapshot shape as `workflow_list`.
- Deleting a missing binding surfaces the subscription store's not-found error.

This RPC is intentionally scoped to workflow bindings. Native workflow
definition deletion is a separate workflow-store capability and is not implied
by this endpoint.
