# Saved Contact Proposal Filtering

## Summary

Stored contact proposals are now filtered against saved contacts when loading
the contacts store and when saving fresh inference results.

## Design

- `load_store` removes proposals whose contact ids overlap any saved contact.
- `save_proposals` prunes incoming inference proposals against saved contacts,
  persists the filtered list, and returns the persisted proposals to callers.
- `contacts_infer` returns the same filtered proposals that were written to
  `contacts.json`.

## Compatibility

Existing `contacts.json` files with stale proposals are sanitized in memory on
load. Users can rerun inference for new suggestions, but already-saved contacts
will not reappear as proposals in the refreshed snapshot.
