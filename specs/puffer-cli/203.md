# Contact Delete ID Trimming

## Summary

`contacts_delete` now trims the requested contact id before matching saved
contacts.

## Design

- Delete requests reject empty IDs after trimming.
- Matching and not-found errors use the trimmed contact id.
- Save behavior is unchanged; saved contact IDs are still generated or trimmed
  through the existing save path.

## Compatibility

Existing callers that pass exact saved contact IDs behave the same. Callers with
accidental surrounding whitespace now delete the intended contact instead of
getting a false not-found error.
