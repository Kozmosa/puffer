# Desktop Proxy Settings API

## Design

The desktop settings snapshot now includes sanitized provider proxy settings.
Desktop clients can save proxy configuration and run a connectivity test through
the daemon without receiving stored proxy passwords.

## Architecture

- `desktop_api_types` owns the proxy DTOs shared by snapshot, save, and test
  responses.
- `desktop_api::load_settings_snapshot` maps `PufferConfig.network.proxy` into a
  credential-free `networkProxy` object.
- The daemon exposes `save_proxy_settings` and `test_proxy` RPC methods.
- The desktop TypeScript API mirrors the Rust DTOs and adds typed helpers for
  saving proxy settings and testing a proxy endpoint.

## Logic

- Save requests parse string schemes into `ProxyScheme`, validate the resulting
  `ProxyConfig`, persist it to user config, reload daemon config, and return a
  fresh settings snapshot.
- `keepPassword` preserves the existing password for the same endpoint id when
  the save request omits `password`.
- Test requests accept either `proxyId` or an inline endpoint draft, choose a
  provider discovery URL when available, and return `ok`, message, and latency.
- Error messages are truncated and scrub the configured password before they are
  sent back to the client.

## Contracts

- Snapshot proxy endpoints include `hasPassword` and `uri` but never a password.
- Supported proxy schemes are `http`, `https`, `socks5`, and `socks5h`.
- Enabled proxy settings must have a selected endpoint id that exists in the
  saved endpoint list.
- The daemon remains the single writer for persisted proxy settings.

## Compatibility

This intentionally adds a required `networkProxy` field to desktop settings
snapshots. Desktop mocks and TypeScript types were updated with no legacy
fallback field.
