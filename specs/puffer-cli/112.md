# Daemon Local Model RPC

- The standalone Puffer daemon now exposes `local_model_status` and `install_local_model` so the Vite/browser desktop preview has the same MiniCPM5 setup surface as the Tauri backend.
- `local_model_status` accepts `modelId`/`model_id` and returns the MiniCPM5 desktop status contract: support, install/config/run state, endpoint, paths, logs, and diagnostics.
- `install_local_model` accepts the same model id, starts or joins the in-process MiniCPM5 installer, and emits `local-model:minicpm5:event` progress payloads on the daemon WebSocket event bus.
- The daemon implementation embeds the MiniCPM5 shim and provider resource, creates the isolated `~/.puffer/venvs/minicpm5` runtime, downloads `openbmb/MiniCPM5-1B-MLX`, registers the provider YAML, and starts the OpenAI-compatible shim on `127.0.0.1:8088/v1`.
- Unsupported local model ids are rejected before any install work begins. The status path is safe and read-only except for localhost health probing.
