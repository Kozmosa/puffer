# Desktop provider display ids

Desktop and Tauri clients may pass user-facing provider names such as
`OpenAI` or `Anthropic` when creating sessions or sending turns.

Daemon provider normalization now treats those display-case ids as the
canonical registry ids `openai` and `anthropic`, matching the existing
`codex` and `claude` alias behavior. Model prefixes are canonicalized through
the same path, so `OpenAI/gpt-5` and `Anthropic/claude-sonnet-4-5` resolve to
the stored provider models instead of producing `unknown provider` errors.

Unknown provider ids remain unchanged after trimming so custom registry entries
can still be addressed directly.
