# Contact Candidate Preview Payloads

## Summary

`contacts_list` and `contacts_search` now return connector candidates as
lightweight previews without inline avatar data. Saved contacts are still
enriched with avatar values when the workspace has Telegram peer-cache metadata.

## Compatibility

The `contacts_infer` and `contacts_context` paths still use full candidate
context internally. Desktop contact pickers keep candidate id, name, and score
fields, while avoiding large base64 avatar payloads that can slow or disrupt
WebSocket reconnects.
