# Gmail Browser Account Resolution

## Summary

Gmail-browser connector actions now distinguish the connector connection slug
from the Gmail account address. `ConnectorAct` injects `connection_slug` into
the action payload so the subscriber can route to the configured connection;
that field must not be interpreted as the account to read.

## Behavior

When no `account` or `account_slug` is provided, Gmail-browser actions use the
first configured account from `gmail-browser-accounts/<connection>/config.toml`.
When an explicit account is provided, it must match one of the configured
accounts after normalization.

## Compatibility

This preserves existing multi-account selection through `account` and
`account_slug`, while allowing the common `ConnectorAct` shape
`connection_slug=gmail-browser` to list or search the default configured Gmail
account instead of failing with an unconfigured account error.
