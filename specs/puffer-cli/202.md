# Contact RPC Parameter Module Split

## Summary

Desktop contact RPC request parameter structs now live in
`daemon_contacts_params.rs` instead of the main contacts RPC module.

## Design

- `daemon_contacts.rs` imports typed parameter structs from the new module.
- The new module owns list, save, delete, context, and infer request shapes.
- Field defaults, aliases, and save-description compatibility remain unchanged.

## Compatibility

There is no RPC contract change. This keeps the main contact module farther
from the Rust line-count contract while preserving the existing deserialization
behavior.
