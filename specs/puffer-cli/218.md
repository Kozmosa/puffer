# Media Internal CLI And Timeline Attachments

The CLI exposes media generation through existing internal tool plumbing.

- Hidden subcommands `puffer internal-tool image-generation` and
  `puffer internal-tool video-generation` use explicit arguments and aliases
  `imagegen` and `videogen`.
- Media internal commands require the parent Bash broker endpoint. Outside that
  broker they fail with the existing required-endpoint error.
- CLI argument parsing builds the workflow input JSON at the command boundary:
  image uses `prompt`, `count`, optional `aspect`, `promptReference`, `purpose`,
  and `retryFromError`; video uses `prompt`, optional `purpose`, and
  `parameters`.
- Desktop timeline attachment synthesis now accepts only successful `Bash`
  invocations whose input command is a supported media internal command or
  helper alias. The Bash output must parse as JSON and its `stdout` must parse
  as the media result JSON.
- Arbitrary Bash output, even when shaped like media JSON, is ignored.
