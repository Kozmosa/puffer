# Desktop Proxy Save Normalization

Updates the desktop daemon `save_proxy_settings` RPC to normalize proxy enabled
and selected state before validation and persistence.

- Saving `enabled=true` with an empty proxy list persists `enabled=false` and
  `selected=null`.
- Saving `enabled=true` with endpoints but no selected ID chooses the first
  endpoint before validation.
- Password preservation and credential redaction behavior are unchanged.
- The RPC no longer returns a stale missing selected-ID error for empty proxy
  lists.
