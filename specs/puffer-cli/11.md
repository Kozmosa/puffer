# Real daemon turn smoke test

The daemon must support the same flow Desktop depends on: create a session,
send one user instruction, receive a completion event, and persist the assistant
reply in session detail.

Compatibility contract:

- The smoke test runs the real `puffer daemon` binary over WebSocket.
- Provider calls go to a local OpenAI Responses mock, not external services.
- Desktop-facing provider aliases such as `codex` are accepted on both
  `create_session` and `run_agent_turn`.
- The daemon emits `turn-complete` with assistant text and persists the same
  assistant message into the session timeline.

Regression coverage lives in `crates/puffer-cli/tests/daemon_turn_smoke.rs`.
