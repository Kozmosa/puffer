# Browser Composite Agent Actions

## Summary

The daemon browser agent endpoint supports two verified-skill composite actions:
`openScreenshot` and `openConsoleLogs`.

## Runtime Contract

- `openScreenshot` requires `url`, navigates a managed browser tab, waits for
  the load state to settle, then captures the same screenshot payload as
  `screenshot`.
- `openConsoleLogs` requires `url`, clears old logs for the target tab before
  navigation, waits for load completion, then returns the page console log
  payload.
- Only these actions treat `url` as a navigation target. Plain `screenshot` and
  `consoleLogs` continue to inspect the current tab, even when permission
  context injects the current URL.

## Compatibility

Existing BrowserAction behavior is unchanged. Verified Lambda skills can bind
host calls that explicitly mean "open this URL and inspect it" without relying
on a sequence of unguarded browser actions.
