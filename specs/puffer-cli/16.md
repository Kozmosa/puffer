# Failed Turn Settlement

## Summary

Daemon turn failures now settle desktop-visible session state instead of leaving
the workspace sidebar in a running state.

## Design

`run_agent_turn` publishes a shared `turn-error` envelope for all setup and
provider execution failures. The envelope is followed by a
`workspace:sessions:changed` event with reason `turn_error` so desktop boards,
session lists, and sidebars refresh even when the active session view is no
longer subscribed to the session event channel.

Provider execution failures occur after the user message is persisted. Those
failures now append the friendly error text as a transcript `SystemMessage`
before broadcasting the error. This gives later `load_session_detail` and
`list_grouped_sessions` calls a terminal non-user event, so activity status
classification returns to `idle`.

## Contracts

- Every asynchronous turn error emits a session `turn-error` event.
- Every asynchronous turn error emits `workspace:sessions:changed` with
  `reason = "turn_error"` and the affected `sessionId`.
- Provider execution failures persist the same friendly error text shown in the
  live event as a system transcript item.
- Failed sessions with a persisted user prompt are listed as idle after the
  failure settles.
