# puffer-cli update 189

Configured OpenAI contact inference failures now surface as `contacts_infer`
errors instead of falling back to local heuristic proposals.

The local heuristic proposal path is still used when no OpenAI API key exists,
preserving offline/no-key inference behavior. When an API key is configured,
request, status, or response parsing failures are treated as provider failures:
the trace records the failed OpenAI tool event and the RPC returns the error so
the desktop can show it to the user.

This avoids presenting heuristic contacts as if model-backed inference
succeeded and keeps non-2xx API error handling visible at the source.
