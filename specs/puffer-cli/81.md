# Lambda Skill Doctor Summary

The non-interactive `puffer doctor` command now prints the same Lambda Skill
harness status exposed by `/doctor`: loaded verified skills, checked-in host
catalogues, compile-on-demand source count, compiler stats, gate mode, resolved
`lskillc`, and the first source that would be compiled.

When external Lambda Skills are loaded without checked-in host catalogues,
`puffer doctor` also prints `lambda_skill_warnings` so users can see whether the
session is prompt-only or strict-gated through `PUFFER_LAMBDA_SKILL_GATE=compile`.

`command_surface.rs` was split so the CLI command surface stays below the
repository line-count limit. Top-level basic commands now live in
`basic_commands.rs`; MCP and plugin management remain in `command_surface.rs`.
