# Desktop provider aliases at daemon ingress

Desktop may present Claude and Codex as product-facing provider labels, but the
daemon runtime registry stores those providers as `anthropic` and `openai`.

Compatibility contract:

- Daemon RPCs accept `claude` as an alias for `anthropic`.
- Daemon RPCs accept `codex` as an alias for `openai`.
- `create_session`, `list_provider_models`, and `run_agent_turn` model
  selection persist and return canonical provider ids.
- Alias-prefixed model ids such as `claude/claude-sonnet-4-5` and
  `codex/gpt-5.4` resolve through the same canonical provider validation as
  direct `anthropic/...` or `openai/...` ids.

Regression coverage lives in `crates/puffer-cli/src/daemon.rs`.
