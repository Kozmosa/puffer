# Telegram direct-connect account state

The one-shot `puffer connect telegram` REPL now stores every Telegram personal
account under `telegram-accounts/<connection>`, including the default
`telegram-user` connection.

Previously the default connection used `subscribers/telegram-user/state` while
other Telegram connection paths used `telegram-accounts/telegram-user`. That
split allowed two different `telegram.session` files for the same account and
could make login, subscriber auth probes, and reconnect checks disagree.

Regression tests cover both default and alternate direct-connect state paths.
