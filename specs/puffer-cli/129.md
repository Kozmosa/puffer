# Google Calendar Browser Connector

`gcal-browser` is a hidden subscriber bundled in `puffer-cli`, parallel to
`gmail-browser`, and uses the daemon-managed global Puffer Chrome profile.

- `puffer __subscriber gcal-browser` loads per-connection config from
  `gcal-browser-accounts/<connection>/config.toml`.
- Config records the workspace root and monitored Google account emails.
- The subscriber opens Google Calendar agenda tabs with `authuser=<account>`,
  polls every 30 seconds, and emits newly observed visible event rows with
  stable dedup keys.
- Initial snapshots emit only the top observed event to avoid backfilling an
  entire calendar into new monitors.
- Auth-required and poll-error states are emitted as control events.
- Connector actions are handled by a dedicated `gcal_browser_act` subscriber
  command and support `get_detail`, `accept`, and `deny`.
- Actions can target events by `event_id`, `title`, or `url`, and execute only
  through the configured Puffer browser session.
- Daemon connector setup recognizes `/connect gcal-browser [connection]`,
  opens Google account sign-in when needed, discovers logged-in accounts, and
  saves selected accounts as an authenticated connection.
