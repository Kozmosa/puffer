# Contact Inference Candidate Prompt Formatting

OpenAI-backed contact inference now formats the candidate prompt as pretty JSON
before sending it and before recording it in the trace.

Each entry still includes `id`, `name`, `score`, and bounded context snippets,
but snippets are now structured as `kind`, `text`, and `timestamp_ms` fields
instead of bare strings. Raw connector payloads remain omitted. The trace
preview remains bounded by the existing prompt limit, but the JSON is
line-broken and indented so operators can manually inspect which candidates and
snippets the model saw.

This does not change candidate collection, ranking, OpenAI tool schema,
zero-proposal handling, or the no-key local heuristic fallback.
