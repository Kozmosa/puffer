# 25. Custom Provider Slash Model Routing

## Summary

Desktop daemon routing now preserves slash-delimited model ids for custom
providers such as OpenRouter.

## Design

- Provider/model prefix stripping is limited to built-in desktop aliases:
  OpenAI/Codex and Anthropic/Claude.
- Custom providers keep model ids like `openrouter/owl-alpha` intact when
  creating sessions and when sending explicit turn routing.
- Slash model override resolution still accepts the provider/model form first,
  then falls back to a model id whose literal value includes the provider
  namespace.

## Compatibility

Existing Codex/OpenAI and Claude/Anthropic alias behavior is unchanged. Custom
provider catalogs that use namespaced model ids no longer fail with
model-not-found errors after the GUI sends a selected model.

## Tests

Daemon unit tests cover both create-session routing and explicit turn options
for an OpenRouter model id containing a slash.
