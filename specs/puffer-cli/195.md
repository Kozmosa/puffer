# Contact Relationship Descriptions

Contact inference descriptions now summarize the user's relationship with each
contact instead of following the previous justification template.

The system prompt still requires grounded two-sentence descriptions and still
filters spam-like candidates, but it no longer asks the model to dedicate one
sentence to why the contact matters and one sentence to why the contact is not
spam. It explicitly discourages stock phrases such as `This contact matters
because` and `This is not spam`.

The local heuristic fallback uses the same relationship-summary tone so fallback
proposals do not reintroduce the removed phrasing. The RPC shape and
CreateContact tool schema are unchanged.
