# Desktop Session Page Endpoint

## Purpose

Prevent the desktop daemon from sending the full historical session archive to
the app on every workspace refresh.

## Design

The daemon exposes `list_grouped_sessions_page` alongside the existing
`list_grouped_sessions` RPC. The new endpoint accepts `offset` and `limit`,
clamps the page size to a daemon maximum, groups only that window of sessions,
and returns page metadata.

## Logic

The endpoint uses `SessionStore::list_sessions_page`, then applies the same
folder grouping, activity status, project tags, and routing enrichment used by
the full session list. Existing callers of `list_grouped_sessions` keep their
current full-list behavior.

## Contracts

- Default page size is 30 sessions.
- Maximum page size is 200 sessions.
- Responses include `groups`, `offset`, `limit`, `returnedSessions`,
  `totalSessions`, and `hasMore`.
- Page windows are ordered by the same newest-first session ordering as the
  full list.
