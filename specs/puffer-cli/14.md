# Auth Provider Aliases

## Summary

CLI auth commands now accept the same provider aliases as the desktop daemon.

## Design

Auth command provider arguments are canonicalized before OAuth start/exchange,
OAuth refresh, API-key storage, clear, login, and logout operations. `codex`
stores and refreshes `openai` credentials; `claude` stores and refreshes
`anthropic` credentials.

OAuth bundle creation resolves the actual registry provider id before deciding
which provider-family URL builder to use, so alias inputs and canonical inputs
share the same flow.

## Contracts

- `puffer auth oauth-start codex` returns an OpenAI OAuth bundle.
- `puffer auth set-api-key codex ...` stores credentials under `openai`.
- `puffer auth clear codex` removes `openai` credentials.
- Existing custom provider ids continue through the provider registry lookup
  path.
