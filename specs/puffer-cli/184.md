# Contact Inference Prompt Trim

## Design

The model-backed contact inference system prompt no longer includes the
selection guidance examples for strong contacts, weak contacts, organization
handling, or the explicit zero-tool-call instruction.

## Architecture

The change is limited to `daemon_contacts_infer.rs`. Existing hard constraints
remain in place: proposals are capped, descriptions stay grounded in chat
history, spam-like senders are discouraged, and descriptions keep the two
sentence contract.

## Compatibility

The RPC shape, trace events, fallback heuristic path, and CreateContact tool
schema are unchanged. Only the system prompt text sent to OpenAI changes.
