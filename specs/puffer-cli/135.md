# Long-term Puffer browser profile cleanup

## Summary

Browser connector sessions use a Puffer-owned Chrome user-data directory as
their long-term auth home. Puffer no longer inspects, selects, launches, or
copies from the user's desktop Chrome profiles.

## Behavior

Puffer creates a marked managed profile root and launches Chrome with that
Puffer-owned `--user-data-dir`. Browser-backed connectors can be signed in once
inside that managed profile and reused across daemon restarts without touching
desktop Chrome state.

Managed profile cleanup, marker writing, and orphan process termination remain
limited to Puffer-owned profile roots.

## Compatibility

Puffer must not copy cookies, token databases, preferences, local storage, or
other auth/session material from desktop Chrome into the connector profile.
Users authenticate browser-backed connectors inside the Puffer-owned long-term
profile. Existing managed profile roots without the current marker are reset to
purge legacy cloned auth state, while marked profile roots are preserved. The
global profile reuse contract is documented in `specs/puffer-cli/123.md`.
