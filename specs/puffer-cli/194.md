# Contact Inference Provider Requests

Contact inference now builds OpenAI requests from the daemon's provider registry
and auth store instead of posting directly to the public Chat Completions URL.

The inference path uses the configured OpenAI provider base URL, headers, query
params, auth mode, proxy settings, and Responses endpoint path. OAuth credentials
are sent as bearer credentials and refreshed once after a 401 using the existing
OpenAI OAuth refresh helper; refreshed credentials are saved back to the auth
store.

This keeps contact inference aligned with the rest of Puffer's OpenAI routing,
including Codex-imported relay settings such as custom base URLs and query
params. Local heuristic proposals remain the fallback when no OpenAI provider
credential is configured.
