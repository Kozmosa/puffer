# 22. Daemon Turn Smoke Mock Streams

The daemon turn smoke tests run local HTTP mock servers for OpenAI-compatible
and Anthropic-compatible provider calls. Those mock connections must be stable
on platforms where sockets accepted from a nonblocking listener can briefly
behave as nonblocking streams.

## Contract

- Mock provider handlers switch accepted TCP streams back to blocking mode
  before reading request bytes.
- Existing read timeouts remain in place so broken test clients do not hang the
  smoke test indefinitely.
- The daemon/provider logic remains unchanged; this only hardens the test
  harness used by desktop alias and turn-routing smoke coverage.

## Coverage

`cargo test -p puffer-cli --test daemon_turn_smoke` exercises OpenAI, Codex
alias, Anthropic, Claude alias, session-routing, and turn-error smoke flows
against the local mock provider servers.
