# Global Browser Profile For Browser Connectors

## Summary

Daemon browser roots now use one global Puffer-owned Chrome profile instead of
configurable per-user or per-root Chrome profile selections.

## Behavior

`BrowserRegistry` launches a single root under `puffer-global` in the daemon
browser profile directory and reuses that root for normal browser tabs and
browser-backed connectors. Closing a session drains the matching page worker and
only shuts down Chrome when no browser sessions remain. Idle pruning drains all
page sessions attached to the global root before stopping Chrome.

The managed browser profile is created and owned by Puffer. It does not inspect,
select, or copy from desktop Chrome profiles, and it does not accept a
configured `chromeProfile` override from daemon calls or browser-agent requests.

## Gmail Browser

The Gmail browser connector writes only the workspace root and selected Gmail
accounts to its connection config. Browser actions use the daemon-managed global
profile, so users sign in once inside Puffer's long-term browser profile and the
same profile is reused across daemon restarts.

## Compatibility

Older config files may still contain browser profile fields, but the daemon no
longer reads or exposes them. Existing marked Puffer-owned profile roots are
preserved; unmarked managed roots continue to be reset before launch.
