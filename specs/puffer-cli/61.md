# Connection Auth Repair Notice

The background connection auth monitor now points users at the deterministic
connector repair command.

- Auth failure notices are rendered as:
  `run /connect <connector-slug> <connection-name> to repair it`.
- The notice uses the failing connection's stored connector slug and connection
  slug, so users can run the command without guessing the right auth surface.
- The monitor behavior, interval, subscriber startup, and auth check routines
  are otherwise unchanged.

This aligns CLI-visible repair guidance with the new `/connect` auth flow.
