# Streaming Workflow Agents

- Daemon workflow action runners now execute background agent prompts through the streaming runtime instead of the blocking runtime.
- This covers monitor `triage_agent` actions, native workflow agent nodes, and ignore-analysis prompts.
- Streaming events are drained because these agents run headlessly; the returned assistant text and tool effects remain the only persisted contract.
- The change preserves existing workflow locking and session creation, but avoids providers that reject non-streaming OpenAI/OAuth requests with `Stream must be set to true`.
- Direct tool actions still execute through `execute_tool_action_once` and are unaffected.
