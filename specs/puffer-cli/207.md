# Browser Client Test Isolation

Browser client unit tests now build explicit temporary `ConfigPaths` instead of
calling `ConfigPaths::discover` for temp workspaces.

## Behavior

- Stable CLI browser session id tests keep both workspace and user config state
  under the test temp directory.
- The tests no longer depend on process-wide `PUFFER_HOME` or `HOME` values
  that other parallel tests may temporarily adjust.
- Browser daemon test support and pending CDP response bookkeeping are split
  into purpose-specific submodules so the browser source files stay under the
  repository's 1000-line limit.

## Compatibility

Runtime path discovery and browser behavior are unchanged. This only makes the
daemon browser client unit tests deterministic under the full workspace test
runner while preserving the repo contract checks.
