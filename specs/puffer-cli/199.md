# Persisted Contact Proposals

## Summary

Contact inference now saves the latest proposal list in the workspace contacts
store and returns that list with every contacts snapshot.

## Design

- `contacts.json` gains a defaulted `proposals` array containing the most
  recent inferred `ContactProposal` values.
- `contacts_infer` replaces the stored proposal array after a successful run.
- `contacts_list`, `contacts_search`, and save/delete refresh responses include
  the stored proposals alongside saved contacts and connector candidates.

## Compatibility

Existing contacts stores without a `proposals` field deserialize with an empty
proposal list. Saved contacts remain independent user-curated records; rerunning
inference only replaces the transient proposal cache.
