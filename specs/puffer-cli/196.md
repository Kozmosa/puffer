# Telegram Contact Avatars

Telegram contact collection now reads the peer cache as metadata instead of
name-only data. Cached `avatar` values are attached to ranked Telegram
`ConnectorContact` candidates when the diagnostic payload resolves to the same
normalized Telegram contact id.

OpenAI-backed contact inference does not include avatar data in the model
prompt. After parsing CreateContact tool calls, the daemon fills any missing
proposal avatar from the matched local candidate contact id. This preserves the
existing RPC schema while keeping base64 avatar payloads out of model input.

Saved contacts are enriched at read time as well: when a saved contact has no
explicit avatar, `contacts_list` and `contacts_search` fill the response avatar
from the first matching cached Telegram contact id without rewriting the
contacts store.
