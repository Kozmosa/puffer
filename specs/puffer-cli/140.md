# Browser CEF backend and CDP attach

## Summary

The daemon now reports Browser renderer capability through
`browser_backend_status`. Desktop clients can request either `cef` or
`screencast` and receive the daemon active renderer plus fallback reason before
opening tabs. When the desktop app starts a native CEF view, the daemon can also
attach CDP browser sessions to that CEF remote debugging endpoint.

## Design

- `daemon_browser::cef` owns CEF runtime discovery and keeps it separate from the
  Chrome/CDP session code.
- Discovery checks `PUFFER_CEF_PATH`, `PUFFER_CEF_ROOT`, `CEF_PATH`, and then
  local Tintin CEF outputs under `~/chromium_tintin/src/out/Release_GN_arm64`,
  `~/chromium_tintin/src/out/Release`, and packaged CEF distributions in
  `~/chromium_tintin/output/cef_binary_*/Release`.
- macOS CEF discovery requires
  `Chromium Embedded Framework.framework/Chromium Embedded Framework`; Windows
  requires `libcef.dll`; Linux requires `libcef.so`.
- `~/chromium_tintin/src/out/Release/Chromium.app` is detected as the Tintin
  Chromium fork, but is explicitly not treated as a CEF runtime.
- The daemon Browser display backend remains `screencast` because native CEF is
  owned by the desktop Tauri process. The status payload tells clients why the
  daemon path is still a fallback.
- `PUFFER_CEF_REMOTE_DEBUGGING_PORT` advertises the desktop CEF DevTools
  endpoint to spawned daemons. `BrowserRootSession` first tries to attach to
  that existing endpoint and reuses its page target; if the endpoint is not live
  yet, it falls back to launching managed headless Chrome as before.
- CEF-attached roots do not own targets and do not close the page target on
  worker shutdown; lifecycle is controlled by the desktop CEF host.

## Compatibility

The managed Chrome/CDP path now prefers the Tintin Chromium executable at
`~/chromium_tintin/src/out/Release/Chromium.app/Contents/MacOS/Chromium` on
macOS before system Chrome candidates. Existing `PUFFER_CHROME` overrides still
win. Existing screencast Browser behavior is unchanged unless the desktop app
has already published a live CEF DevTools endpoint on the configured port.
