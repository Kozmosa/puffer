# MiniCPM5-1B — local macOS provider + on-device behavior analysis

## Summary

On macOS, puffer can run a small on-device model (MiniCPM5-1B, 4-bit MLX) as a
local provider — no API key, no Docker, runtime is just the `mlx-lm` pip package
plus the model weights. Its first use is **real-time user-behavior analysis**:
the local model reads what the user is doing (session transcript) and produces a
rolling structured read of their intent/state/activity — cheap and private enough
to run continuously, where a frontier model would be wasteful.

## Pipeline

```
scripts/minicpm5-install.sh   (macOS-gated)
  → isolated venv at ~/.puffer/venvs/minicpm5  (avoids PEP 668)
  → pip install mlx-lm + huggingface_hub
  → download openbmb/MiniCPM5-1B-MLX (~589MB) → ~/.puffer/models/minicpm5-1b
  → install shim → ~/.puffer/bin/minicpm5-shim.py
  → register provider → ~/.puffer/resources/providers/minicpm5.yaml
scripts/minicpm5-serve.sh [--bg]   → OpenAI-compatible server on 127.0.0.1:8088
  → puffer 'minicpm5' provider (model minicpm5-1b), auth_modes: []
scripts/minicpm5-behavior.py [--watch] <session.jsonl>
  → tails a session transcript (user msgs + tool invocations = behavior)
  → local model (No-Think → clean JSON) → {intent, state, activity, signals, suggestion}
  → rolls into ~/.puffer/behavior/<session>.jsonl
```

## Components
- `resources/providers/minicpm5.yaml` — local provider (base_url 127.0.0.1:8088/v1,
  openai-completions, `auth_modes: []`), following the ollama/lmstudio pattern.
- `scripts/minicpm5_shim.py` — OpenAI-compatible mlx server. Honors per-request
  `enable_thinking` (thinking on for agentic/tool use; off for structured analysis),
  converts MiniCPM's native XML tool calls to OpenAI `tool_calls`. Model path via
  `MINICPM5_MODEL`.
- `scripts/minicpm5-install.sh` / `minicpm5-serve.sh` — guided install + launch.
- `scripts/minicpm5-recommend.sh` — pure detection; emits a JSON decision
  (`recommend` true only on macOS + Apple Silicon + not-yet-installed) the
  desktop onboarding renders as a "you're on macOS — install a local model?" card.
- `scripts/minicpm5-behavior.py` — the behavior analyzer (consecutive-duplicate
  collapsing so repeated reads/test-runs become a counted signal, not noise).
- Desktop onboarding card (`apps/puffer-desktop`): `minicpm5_recommend` /
  `minicpm5_install` Tauri commands run the scripts above; on macOS the
  Onboarding screen calls recommend on mount and, when `recommend` is true,
  shows an Install button. Install streams `minicpm5://install-log` lines and a
  final `minicpm5://install-done`. This is the in-app "recommend → install →
  becomes a provider" path; the scripts stay the single source of truth.

## Findings / limits
- Verified end-to-end on a fresh install: install → serve → puffer reaches the
  provider; behavior analysis on a realistic debugging session correctly inferred
  intent/activity (incl. "stuck" after repeated failed test runs) + a useful
  suggestion.
- **MiniCPM5-1B is not a general agent model.** Under puffer's full system prompt
  + tool advertising it degenerates (runaway repetition) — same as observed under
  other heavy harnesses. Its role here is **narrow, focused analysis** (No-Think,
  short prompt, structured output), not the main driving model. Pathological
  transcripts (very long, highly repetitive tool tails) can still make the 1B
  parrot; the duplicate-collapsing mitigates but does not fully eliminate this.
