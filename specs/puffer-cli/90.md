# Verified Skill Gate Stream Events

The desktop daemon now includes tool metadata in live `tool-invocations`
events. When a tool invocation carries `lambda_skill` metadata, the daemon also
posts a `lambda-gate` event on the same session channel so the Desktop UI can
show host-call admission, commit, and recoverable rejection decisions while the
turn is still running.

Persisted session timelines also surface the same gate metadata as a concise
system event adjacent to the tool invocation. This keeps reload/replay behavior
aligned with the live event stream.

Validation:
- `cargo test -p puffer-cli lambda_gate_stream_payload_maps_tool_metadata -- --nocapture`
- `cargo test -p puffer-cli timeline_surfaces_lambda_gate_metadata_as_system_event -- --nocapture`
