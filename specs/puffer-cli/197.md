# Telegram Saved-Contact Hydration Env

## Summary

Telegram subscriber launches now include the workspace `.puffer` directory in
`PUFFER_WORKSPACE_CONFIG_DIR` so the authenticated subscriber process can read
saved contact IDs and resolve Telegram profile metadata directly.

## Design

- `/connect` Telegram login sessions populate `SkillEnv.workspace_config_dir`.
- Daemon-supervised subscriber manifests receive `PUFFER_WORKSPACE_CONFIG_DIR`
  before `SubscriptionManager::start_subscriber`.
- Telegram action-launched subscribers set the same env on their generated
  connection manifest.

## Compatibility

Existing subscriber manifests keep working. Subscribers that do not read this
env ignore it, and standalone Telegram subscriber runs continue without saved
contact username hydration when the env is absent.
