# 26. Desktop Transcript Rewrite Projection

## Summary

Desktop session detail and activity status now replay append-only transcript
rewrite events before projecting chat history to the GUI.

## Design

- `TranscriptRewritten::Clear` clears previously projected desktop timeline
  items and pending assistant text.
- `TranscriptRewritten::PopLast` removes the latest projected transcript item
  before subsequent system messages are rendered.
- Activity status derives from the rewritten event stream so old user turns do
  not keep a cleared or rewound session marked as running.

## Compatibility

The append-only session log format is unchanged. TUI/Core restore already
honored transcript rewrites; this aligns the desktop projection with that
existing behavior.

## Tests

Daemon desktop unit tests cover timeline projection and activity status for
both clear and pop-last rewrite events.
