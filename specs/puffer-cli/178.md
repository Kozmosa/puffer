# Contact Infer Selection Prompt

## Design

Contact inference now uses stricter selection rules for meaningful contacts.
The system prompt tells the model to skip uncertain candidates, reject spam-like
senders, ground descriptions in chat history, deduplicate obvious duplicates,
and return zero tool calls when no candidate is worth saving.

## Architecture

The OpenAI request keeps the `CreateContact` tool schema but changes
`tool_choice` from `required` to `auto`. This allows the model to return no tool
calls when every candidate is weak, matching the prompt contract.

## Compatibility

Successful `CreateContact` tool calls still parse into the same
`ContactProposal` shape. Empty model output now returns an empty proposal list
instead of triggering the heuristic fallback.
