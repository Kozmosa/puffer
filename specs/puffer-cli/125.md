# Browser Native Key Input

- Browser key input sent through Chrome DevTools includes `nativeVirtualKeyCode` alongside `windowsVirtualKeyCode` when a virtual key can be derived.
- Enter keeps browser-like text payloads on key-down events by sending carriage return text and unmodified text while omitting text on key-up.
- Backspace, printable keys, and key-up events keep their existing behavior, with unit coverage asserting the native key code field.
