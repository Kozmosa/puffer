# Telegram Auth Probe Availability

Telegram connection auth monitoring now distinguishes an unavailable subscriber
probe from an explicit failed auth result.

- The `telegram-login` auth checker still sends `TelegramAuthOk` to the
  connection-owned subscriber when it is running.
- An `auth_ok` event with `ok: true` keeps or restores the connection as
  authenticated.
- A terminal `login_error` or `auth_ok` with `ok: false` remains an explicit
  failed auth result.
- Transport-level probe failures such as subscriber restarts, command-channel
  timeouts, or temporarily disconnected stdin are treated as indeterminate and
  leave the stored connection auth state unchanged.

This prevents transient supervised subscriber restarts from printing
user-facing "auth is no longer functioning" notices for credentials that have
not actually been rejected by Telegram.
