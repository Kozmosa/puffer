# puffer-cli Update 04

## Summary

Adds the hidden `puffer non-interactive` command used by the `/genskill`
Ladybird replay evaluation harness.

## Design And Architecture

The command is implemented in `crates/puffer-cli/src/non_interactive.rs` and
registered as a tuple variant in the top-level CLI command enum. Keeping the
logic in its own module avoids growing the already large CLI entrypoint.

The command creates a normal `AppState` and `SessionStore`, auto-allows
permissions for unattended execution, and reuses `puffer-core`'s streaming
runtime entrypoint. It does not introduce a second agent loop.

## Logic

The command supports:

- `--user-message` for a single unattended model turn.
- `--load-transcript` for preloading session JSONL, or using markdown as
  prompt context.
- `--load-skill` for prepending one or more `SKILL.md` files as guidance.
- `--run-command` for local slash commands such as `/genskill`.
- `--output`, `--transcript-out`, and `--emit-artifact` for eval artifacts.
- `--provider`, `--model`, `PUFFER_PROVIDER`, and `PUFFER_MODEL` overrides.

For `/genskill`, the command copies the generated skill file to `--output`
when the local command returns `Skill written to <path>`.

When both `--run-command` and a non-empty `--user-message` are present, the CLI
records the user message in `AppState` and the session transcript before
dispatching the slash command. This lets commands such as `/genskill` consume
explicit unattended context while keeping the normal user-turn path unchanged.

## Contracts

- The command is additive and does not alter the interactive TUI path.
- Session JSONL loaded through `--load-transcript` preserves structured tool
  invocations so `/genskill` can see real tool-call history.
- Markdown transcripts are treated as textual context for direct generation,
  not as structured `/genskill` evidence.
- Replay artifacts use the schema expected by `puffer-genskill-eval`.
- Common provider environment variables are honored for containerized evals.

## Verification

- `cargo test -p puffer-cli non_interactive`
- `cargo run -p puffer-cli -- non-interactive --help`
- `cargo build -p puffer-cli --release`
- `cargo test -p puffer-cli` currently has unrelated local failures in auth
  callback permissions and benchmark permission expectations.
