# Detached desktop session creation

Desktop `create_session` RPCs may include a provider or model override from the
New Agent provider picker. Resolving that override loads runtime inputs and can
run provider model discovery, which uses `reqwest::blocking` internally.

Contract:

- `create_session` must run through the daemon's off-runtime handler path.
- Provider and model validation still happen before the session is returned.
- Session creation without a provider override keeps the same response shape.
- The daemon must not drop blocking reqwest internals on a Tokio worker thread.

Regression coverage lives in
`create_session_provider_routing_runs_off_tokio_runtime`.
