# Internal tool resource reporting

## Purpose

Expose internal tool resource counts separately from provider-callable tool
counts so diagnostics match the new registry boundary.

## Design

`puffer resources --json`, desktop resource counts, command surface summaries,
`/help` resource counts, `/doctor`, and command summaries now report
`internal_tools` separately from `tools`.

Executable registry counts also distinguish normal executable tools from
internal executable tools.

## Contract

Moving Browser to `resources/internal_tools/` reduces the normal tool count and
adds one internal tool count. Consumers that inspect resource counts should use
`internal_tools` for CLI-only internal surfaces and `tools` for provider-callable
tools.

## Verification

CLI and core test runs validate that resource summaries and API DTOs compile
with the new count field.
