# External Runtime Actor Metadata

## Purpose

The daemon and desktop API expose actor metadata so external clients using
Puffer as a runtime can tell whether a message came from the root agent or a
specific subagent.

## Design

Daemon and desktop live turn stream payloads for assistant text, thinking, tool
calls, tool results, usage, retries, permission requests, user questions,
reflection checkpoints, and turn completion include the current assistant actor.
Persisted daemon transcript events include user, assistant, and tool actors.

Desktop session timeline DTOs preserve optional actor metadata on user,
assistant, system, command, tool, and permission items. Structured tool
invocations are rendered directly into timeline tool cards so their actor
metadata survives instead of being lost through system-message parsing. Tool
subjects are populated when the tool output identifies another agent; ordinary
tool calls leave `subject` unset.

## Compatibility

Clients may ignore the new actor fields. Existing event type names, turn ids,
tool payloads, and transcript ordering are unchanged.
