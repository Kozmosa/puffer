# puffer-cli Update 165: Native CEF Target Reuse Reset

Native CEF prewarm targets are reset before returning to the daemon Browser
pool. The daemon still does not own CEF targets, but release now connects to the
page target, stops loading, navigates it back to
`about:blank#puffer-cef-slot=<slot id>`, and clears navigation history before
the target can be allocated to another subagent tab.

This preserves daemon-owned Chrome behavior: owned screencast targets and
`close_on_release` targets continue to use the existing DevTools close path.
If a reusable CEF target cannot be reset, it is not returned to the pool, which
prevents stale URL, loading, or history state from leaking into a later tab.
