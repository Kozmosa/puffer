# Optional Contact Save Descriptions

## Summary

`contacts_save` now accepts save requests that omit `description` and stores an
empty description string.

## Design

- `ContactSaveParams.description` defaults during deserialization.
- The save path still trims any provided description before persisting the
  saved contact.
- Existing validation for non-empty names and valid contact ids is unchanged.

## Compatibility

Desktop callers that already send `description` are unaffected. Scripted and
future callers can now save contacts with only a name and contact ids.
