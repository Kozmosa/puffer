# Desktop Verified Skills Import Sanity Checks

The desktop daemon now validates a Verified Skills folder before writing the
`lambda_skill_libraries` manifest. The selected root must contain at least one
`skill.lskill` or `main.lskill` directory, and every recognized skill must have
the generated descriptor and precompiled host catalogue expected by the current
manifest paths.

Default folder imports continue to infer `host_catalogue_subpath: out/host.json`
and `allowed_tools` from checked-in host catalogues. Import fails before config
is written when any skill is missing `out/GENERATED.SKILL.md`, missing
`out/host.json`, has invalid host JSON, or has a host tool without
`concreteTools` bindings. This prevents Desktop from saving half-ready folders
that later appear as permanently Adding or non-invocable.

Snapshots also normalize legacy manifests in memory. When old configs only have
`compiler_path` or large `host_tool_bindings`, the daemon infers the default
`out/host.json` path from files already on disk and omits host binding maps from
the Desktop payload. This keeps the settings response small and avoids duplicate
workspace/user reads when both config roots are the same directory.

Validation:
- `cargo test -p puffer-cli desktop_lambda_skill -- --nocapture`
