# Monitor History RPC

- The daemon exposes `task_monitor_history_list` and `monitor_history_list`.
- The RPC returns recent monitor workflow history from the subscription manager history store, filtered to monitor bindings and monitor action names.
- Each returned message includes the workflow run id, envelope id, connection, connector, received timestamp, succinct display summary, raw text, payload, status, and action log.
- Agent-backed history actions include token usage when the provider reports it; token spend is the non-cached input plus output total.
- Monitor task ignore analysis now writes async agent output, errors, and token usage back into the ignored task metadata for UI inspection.
- Monitor triage prompts require new tasks to copy the workflow trigger `envelope_id` into `monitor_envelope_id` metadata so future history views can link messages to ignored tasks.
- Monitor triage remains task-only: prompts explicitly forbid memory edits, subscription filter edits, and ignore-filter metadata writes.
