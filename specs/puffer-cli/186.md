# Telegram Peer Cache Robustness

## Design

Telegram peer-cache name enrichment now tolerates malformed individual peer
entries instead of discarding the entire cache file.

## Architecture

The CLI peer-cache reader parses the top-level cache JSON first, then attempts
to deserialize each peer entry independently. Invalid entries are skipped while
valid user peers still contribute normalized `telegram@...` display names.

## Compatibility

No RPC shape or connector behavior changes. Contact listing and inference stay
offline for Telegram enrichment and become less sensitive to cache schema drift
or partial cache corruption.
