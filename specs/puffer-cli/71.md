# Desktop Workflow Save RPC

The desktop daemon now exposes `workflow_save` for Corbina's Pipelines editor.

- The RPC accepts `{ workflow }` and also tolerates `{ definition }` for older
  callers.
- The payload is validated through the native `WorkflowStore::register_json`
  path, so desktop saves keep the same slug, trigger, DAG, cron, and unsupported
  AgentFlow feature checks as `WorkflowCreate`.
- Successful saves upsert the definition and return the same enriched snapshot
  shape as `workflow_list`, including connectors, connections, monitor tasks,
  and runtime errors.
- Invalid workflow definitions return a daemon RPC error and leave persisted
  workflow storage unchanged.

This is an additive desktop RPC. Existing workflow listing, run listing, run
inspection, connector setup, and monitor-task actions keep their current
contracts.
