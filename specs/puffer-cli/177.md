# Telegram Contact Names and Bot Filtering

## Design

Telegram contact candidates now prefer the most complete display name observed
for a handle. If older diagnostics contain only a first name and newer
diagnostics contain a full name, the full name replaces the shorter value.

## Logic

`daemon_contacts_telegram` treats usernames ending in `bot` as bots even when
older diagnostic records lack `chat_is_bot` or `sender_is_bot`. This keeps bot
accounts out of Telegram contact candidates without relying on inference-time
post-processing.

## Compatibility

The candidate id format remains `telegram@username`, and existing diagnostics
continue to load. The fallback bot check only filters Telegram handles that
match Telegram bot username conventions.
