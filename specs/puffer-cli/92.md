# Verified Skill Mixed-Readiness Import

## Summary

Desktop Verified Skill folder import performs structural checks for generated
descriptors and host catalogues, but it no longer rejects the whole folder when
one catalogue is not runtime-ready.

## Behavior

- Missing generated descriptors, missing host catalogues, malformed host JSON,
  and host tools without `concreteTools` still reject import.
- Runtime contract validation remains per skill in the desktop skill snapshot
  and doctor readiness path.
- Skills whose host catalogue fails runtime validation are imported as
  recognized Verified Skills, marked not ready, and blocked from model
  invocation until their concrete contracts are fixed.

This lets users add a mixed translated skill library once and then enable the
gate-ready skills without losing visibility into the blocked ones.
