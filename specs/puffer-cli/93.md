# Verified Skill Concrete Tool Import Check

Desktop Verified Skills imports validate concrete tool names before saving a
library manifest. A host catalogue is rejected when any formal host tool binds a
concrete tool that is not part of Puffer's callable tool surface.

This keeps generated host catalogues from presenting a skill as usable when the
model would later try to bridge to an unavailable concrete tool. Skills that need
external MCP tools must bind through `McpToolCall` with explicit `server`,
`tool`, and `args` input contracts rather than naming the external tool directly.

The same concrete-tool check is used by the desktop readiness path so legacy
manifests already on disk are marked not ready instead of model-invocable.
