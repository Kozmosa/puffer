# Desktop turn option scope

## Purpose

Desktop chat turns can carry provider, model, effort, fast-mode, and permission
options from the composer. Applying those options must not rewrite user config
or make a normal send depend on `~/.puffer/config.toml` being writable.

## Behavior

- `run_agent_turn` applies desktop turn options to the active in-memory
  `AppState`.
- Provider/model selections still resolve against the loaded provider registry.
- The selected provider, model, effort, and fast mode are available to the turn
  runtime through the session state.
- Persisted user defaults are updated only through explicit settings or command
  flows, not every chat turn.

## Compatibility

Existing desktop clients can keep sending the same option fields. The change
only narrows the persistence boundary for per-turn requests.
