# Connector and monitor TUI e2e coverage

Adds tmux-backed integration coverage for the connector and monitor flows that
must be exercised through the Rust TUI rather than by directly calling command
helpers.

## Design

- `tmux_connector_monitor.rs` launches the real `puffer` binary in a temporary
  workspace with the no-alt-screen tmux test configuration.
- The serve-mode connector test drives `/connect` through AskUserQuestion for
  `discord-bot` and `matrix-bot`, then verifies the resulting
  `.puffer/connectors.toml` entries.
- The email monitor test installs a workspace-local email subscriber manifest,
  drives `/connect email email-e2e`, verifies the connection store and subscriber
  state config, then drives `/monitor email-e2e`.
- `/monitor email-e2e` is validated by checking rendered TUI output plus the
  persisted monitor memory file and workflow binding in `.puffer/subscriptions.json`.
- A gated live Slack app test drives `/connect slack-app slack-live` through the
  TUI when `PUFFER_E2E_SLACK_BOT_TOKEN` and `PUFFER_E2E_SLACK_APP_TOKEN` are set.
  It verifies that Slack `auth.test` hydrated workspace and user metadata, then
  confirms `/monitor slack-live` remains rejected because Slack app subscriptions
  are not event-capable yet.

## Contracts

- These tests cover local deterministic TUI behavior without requiring real
  third-party credentials or network access.
- The Slack app live test is opt-in only and never embeds credentials in source;
  CI and local default runs skip it unless the explicit environment variables are
  present.
- The email subscriber manifest is workspace-local so subscriber state is kept in
  the test workspace instead of the repository's bundled resources directory.
- Each temporary workspace stores a dummy Anthropic API key before launching the
  TUI, so connector slash commands are not blocked by provider onboarding.
- The tmux process prepends the built test binary directory to `PATH` so subscriber
  manifests that invoke `puffer` resolve the same test binary.

## Compatibility notes

- Lark, Telegram account login, Discord live gateway, Matrix live room, Slack
  login, and real platform callbacks still require external credentials and
  platform-owned test targets before they can be claimed as fully live e2e.
- HTTP webhook connector presets were removed from the built-in catalog, so this
  TUI layer only covers the surviving local bot and email connector flows.
