# CEF DevTools Page Target Recovery

## Scope

The daemon browser backend now handles CEF remote debugging endpoints that are
available before any page target appears in `/json/list`.

## Design

- CEF root discovery waits briefly for an existing page target after
  `/json/version` is reachable.
- If CEF still publishes no page, the daemon creates a CEF DevTools page with
  `/json/new?about:blank` and attaches CDP to that target.
- Targets created through `/json/new` are marked as daemon-owned and closed
  when their worker exits. Existing CEF targets remain owned by the native
  embedder and are not closed by daemon cleanup.
- If CEF cannot publish or create a page target, root selection falls back to
  the screencast Chrome backend instead of failing the tool call.

## Compatibility

The native CEF visual embed remains the default renderer in the desktop app.
Agent/browser tools no longer fail just because CEF's target list is empty at
startup; they either attach to an existing CEF page, create a CEF page, or use
the existing screencast fallback path.
