# Desktop proxy latency integer width

The desktop API no longer exposes proxy connectivity latency as `u128`.
`ProxyTestResultDto.latency_ms` is now `Option<u64>`, and the daemon saturates
the core `u128` timing measurement at `u64::MAX` before serialization.

This keeps the Rust network timing path unchanged while ensuring the desktop
command surface only contains integer widths supported by the current
frontend/IPC serialization path. A daemon unit test covers normal values,
`u64::MAX`, and overflow saturation.
