# Session Routing For Agent Turns

## Summary

Daemon turns now honor desktop session provider routing even when a turn request
does not include an explicit model id.

## Design

`run_agent_turn` loads persisted desktop session routing before building the
runtime `AppState` when the request omits provider and model overrides. The
turn option layer also supports provider-only requests by selecting that
provider's default model through the same model resolution used for session
creation.

## Contracts

- A session created with `providerId` can run a turn without resending
  `providerId` or `modelId`.
- A turn request with only `providerId` selects that provider's default model.
- Explicit model overrides and provider/model pairs still take precedence.
- Permission, thinking, and fast-mode turn options continue to apply after
  routing is resolved.
