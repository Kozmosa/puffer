# puffer-cli update 38

The hidden `puffer internal-tool` surface now includes subscriber-backed
`email` and `telegram` commands in addition to `browser`.

`puffer internal-tool email configure` collects IMAP/SMTP settings, username,
from-address, optional sender filters, and the password either from
`--password` or `--password-stdin`.

`puffer internal-tool telegram` exposes:

- `login-start <phone>` with optional `--api-id` and `--api-hash`
- `login-submit-code <code>`
- `login-submit-password --password-stdin` or `--password <value>`

These commands do not execute subscriber logic in the child process. They send
an internal execution request back to the parent Puffer runtime and print the
parent workflow output. Without the parent internal-tool callback endpoint they
fail closed, matching the hidden Browser internal-tool behavior.
