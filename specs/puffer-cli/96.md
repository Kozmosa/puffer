# Cancelled Turn Progress Persistence

## Scope

The desktop daemon now keeps per-turn streaming progress in the in-flight turn
handle so cancellation can persist what already happened before the interrupt.

## Contract

- Text deltas are accumulated as assistant progress for the current turn.
- Requested tool calls are tracked until matching tool invocation events arrive.
- Completed tool invocations are retained for cancellation persistence.
- On cancel, the daemon appends, in order, the user prompt if needed, the
  streamed assistant text, completed tool invocations, synthetic failed tool
  invocations for still-pending tool calls, and the interrupt system message.
- After cancellation is reported, later stream events from the cancelled worker
  are ignored and a racing provider outcome does not persist a second completed
  or failed turn.

## Compatibility

Normal completed turns still persist from the final `TurnExecution`. The progress
buffer is only consumed on cancellation, so non-cancelled daemon behavior remains
unchanged.
