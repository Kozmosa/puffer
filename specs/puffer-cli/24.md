# Desktop Transcript Actor Compatibility

## Problem

After transcript events gained optional actor metadata, the desktop daemon and
activity-status helpers still had a few actor-less event constructors and
patterns. Once the gRPC runner could compile without a system `protoc`, these
old call sites blocked `puffer-cli` tests before daemon create-session coverage
could run.

## Design

- Daemon turn errors persisted to the transcript carry the current assistant
  actor.
- Desktop activity status logic ignores actor metadata when only text/output is
  relevant.
- Activity-status unit fixtures include explicit `actor` and `subject` fields
  for actor-aware transcript variants.

## Coverage

- `cargo test -p puffer-cli create_session_ -- --nocapture` compiles the
  desktop daemon path far enough to exercise provider-routed session creation.
