# Monitor Task Agent Models

- `task_monitor_create` accepts an optional `model` field for monitor triage agents.
- A missing `model` preserves the existing monitor model when a monitor is resumed.
- `model: null` or an empty string clears the explicit monitor model so the task agent inherits the daemon default provider/model.
- New monitor bindings persist the model on the `triage_agent` action, which is already consumed by the workflow runtime.
- Workflow snapshots expose the triage action model as `workflow_bindings[].model` so desktop clients can display and update monitor task-agent routing.
- Refreshing a monitor still rebuilds the prompt and connector metadata, but it preserves ignore filters and the current model unless an explicit model update is sent.
