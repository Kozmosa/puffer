# Chat Completions malformed tool arguments

## Summary

OpenAI Chat Completions tool calls now preserve malformed `function.arguments`
as a raw JSON string instead of failing response extraction for the whole turn.
This matches the retry behavior used by the Responses/SSE path.

## Compatibility

Valid JSON arguments are still parsed into structured values. Invalid argument
strings become the tool-call argument payload, so normal tool execution can
return a failed invocation and the model can retry instead of losing the turn.
