# Request Test Module Split

## Summary

The OpenAI request builder tests now live in `request/tests.rs` instead of the
main `request.rs` implementation file.

## Design

- Keep production request-building types and helpers in `request.rs`.
- Load the existing test suite through `#[cfg(test)] mod tests;`.
- Preserve test behavior while bringing the implementation file back under the
  repository 1000-line contract.

## Compatibility

No public API, request serialization, auth header, URL, or test assertion
behavior changes.
