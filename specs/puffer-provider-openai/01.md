# Responses Include Normalization

## Summary
- The OpenAI request boundary normalizes legacy Responses API reasoning include selectors before serializing JSON.
- `reasoning.encrypted_content` and `reasoning.content` are rewritten to the current `reasoning.encryptedcontent` selector.

## Compatibility
- The rewrite only applies to Responses endpoint paths.
- Non-reasoning include values are preserved.
- This is a defensive boundary fix for provider paths that may construct a raw JSON body outside the primary runtime helper.
