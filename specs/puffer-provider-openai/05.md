# Realtime client-secret request builder

## Summary

`puffer-provider-openai` exposes a typed builder for the Realtime API client-secret
minting endpoint. The builder targets `/v1/realtime/client_secrets`, reuses the
existing ordered OpenAI request construction, and keeps credentials in headers
instead of payloads.

## Contract

- Callers pass an `OpenAIRealtimeClientSecretRequest` with a flexible `session`
  JSON object so the eventual voice UI can iterate without another provider API
  refactor.
- The builder applies normal OpenAI auth, custom headers, query params, and base
  URL normalization.
- No API key or client secret is stored in resources or specs.
