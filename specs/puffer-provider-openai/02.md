# Responses include sanitizer

## Summary

- The OpenAI `/responses` request boundary now sanitizes `include` selectors
  against the currently supported compact selector names before JSON
  serialization.
- Legacy underscore spellings such as `reasoning.encrypted_content` and
  `web_search_call.action.sources` are rewritten to compact forms.
- Duplicate selectors and unknown values are dropped so stale callers cannot
  trigger provider-side `include[0]` validation failures.

## Compatibility

- The sanitizer only applies to `/responses` endpoint paths.
- Supported selectors keep their canonical compact spelling, matching the error
  contract returned by current OpenAI Responses validation.
- Request builders that already emit compact selectors keep the same wire body.
