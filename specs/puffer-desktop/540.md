# Verified Skills Generated Output Copy

The Desktop Verified Skills settings copy now states that a selected folder must
already include generated verification output and host catalogues. The folder
picker remains the only user-facing input; path fields and numeric readiness
details stay hidden.

When the daemon rejects an incomplete import, the existing settings error note
shows the concrete missing generated descriptor, host catalogue, invalid JSON,
or missing `concreteTools` reason and the Add folder button returns to its idle
state.

Validation:
- `npm run check`
