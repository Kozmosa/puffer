# Native CEF containment and recovery

- Native CEF now cancels popup browser creation in `OnBeforePopup` and loads a
  concrete popup target URL in the existing embedded browser view. This prevents
  CEF-created windows from escaping the Tauri window while preserving normal
  navigation intent.
- CEF slot state now carries an optional error and reports `connected: false`
  when the browser closes unexpectedly, the renderer process becomes
  unresponsive, or the renderer process terminates. The Tauri command surface
  adds `browser_cef_native_state` so the Svelte Browser pane can inspect the
  active native slot without issuing a navigation command.
- The Browser pane probes active native CEF state every 30 seconds and routes
  native resize, reload, history, and address-bar command failures through a
  native reopen attempt before demoting to screencast. Screencast remains the
  compatibility fallback only after native CEF is unavailable or cannot be
  reopened.
- Native reopen treats `CreateBrowser` as asynchronous. The first
  `browser_cef_native_open` response may report `connected: false` while CEF is
  still creating the child browser, so the Browser pane polls
  `browser_cef_native_state` briefly before declaring recovery failed.
- Native CEF initialization is lazy. App startup and
  `browser_cef_native_status` only discover the packaged runtime; helper
  processes are not spawned until the Browser pane explicitly opens an embedded
  native browser. `browser_cef_native_state` and close calls before
  initialization return disconnected/no-op results instead of warming CEF.
- The desktop daemon launcher distinguishes owned child processes from
  already-running host daemons. If a bundled `puffer daemon` exits before
  printing a handshake because the host singleton lock is already held, the
  launcher reads the existing workspace/user `daemon.handshake`, verifies the
  workspace root and local WebSocket reachability, then attaches without taking
  ownership of that process. This keeps packaged app relaunches from failing
  with an empty-handshake startup error.
- Close is callback-owned once a browser exists: `browser_cef_native_close`
  requests `CloseBrowser`, hides the container, and lets `OnBeforeClose` remove
  the native slot. This avoids deleting a slot while CEF still holds lifecycle
  callbacks for it.

Compatibility: existing screencast behavior is unchanged. Native CEF remains
macOS-only and still uses the existing Tauri command and remote debugging port
contract.

Verification caveat: the Objective-C++ bridge is compiled only when local CEF
headers and the runtime are discoverable through `PUFFER_CEF_PATH`,
`PUFFER_CEF_ROOT`, `CEF_PATH`, or `target/puffer-cef-runtime`. The regular
desktop Playwright suite exercises the screencast browser path unless a Tauri
invoke shim or native CEF runtime is provided.
