# Browser Error Disconnect State

## Scope

The desktop Browser pane must stop accepting navigation, pointer, keyboard, and
cursor-probe input once the backend reports a Chrome/CDP state error for the
active tab.

## Change

- Active-tab `BrowserState.error` events now update the pane-level connected
  flag in addition to the tab record.
- Browser startup failures also clear the pane-level connected flag.
- Disconnect transitions clear pending cursor probes and active pointer state
  so delayed input cannot target a failed browser session.
- Playwright coverage now verifies that a backend state error disables the URL
  field and prevents canvas clicks from sending `browser_input`.

## Compatibility

Successful state events continue to restore the connected flag and keep normal
navigation, resizing, and input behavior unchanged.
