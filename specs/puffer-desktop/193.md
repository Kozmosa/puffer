# Connect Project Session Fallback

## Scope

Create Project and remote connect flows must open the freshly created agent
session even when grouped session history has not caught up yet.

## Change

- Connect Project now returns the full `create_session` payload to `App.svelte`
  instead of only the session id.
- `App.svelte` shares the New Agent fallback opener for all newly created
  sessions, synthesizing a temporary `SessionListItem` when history refreshes
  late.
- The fallback preserves the daemon-returned provider/model routing so the
  composer can send the first prompt immediately.

## Compatibility

No daemon RPC shape changes. This only changes renderer handoff state after an
existing `create_session` response.
