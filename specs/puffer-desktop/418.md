# OpenRouter Auto Model Preservation

## Context

The chat composer restores provider and model routing from session metadata.
Some provider-native model ids contain a slash and are not always returned by
the provider catalog.

## Change

- Provider/model prefix stripping now applies only to built-in OpenAI and
  Anthropic alias ids.
- The OpenRouter virtual route `openrouter/auto` is treated as a custom model
  id, so the composer can send it even when it is absent from the advertised
  model list.
- Stale non-custom OpenRouter models still fall back to the first advertised
  agent-capable model before a turn starts.

## Compatibility

Existing OpenAI and Anthropic provider-prefixed session ids keep normalizing to
their provider-native model ids. The chat turn request continues to send the
selected `providerId`, `modelId`, thinking option, fast mode, and permission
mode as before.
