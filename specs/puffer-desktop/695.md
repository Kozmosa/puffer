# Contact Inference Trace UI

## Design

The contacts inference modal now shows a collapsible trace above the proposal
list. It starts expanded for a new run, so users can inspect the prompt and
ongoing model/tool activity, then collapse it to focus on proposals.

## Architecture

`desktop.ts` exposes `subscribeContactInferEvents(traceId, handler)` and passes
`trace_id` through `inferContacts`. `Contacts.svelte` generates a run id,
subscribes before starting inference, and adapts trace events into existing
conversation renderers.

## Reused Components

Prompt/status events render through `MessageBody`. Tool events render through
`ToolCard` as synthetic `ToolTimelineItem`s, preserving the same collapsible
tool-call behavior used by agent conversations.

## Compatibility

The inference result flow is unchanged: proposals are still returned by
`contacts_infer` and can be used to prefill the create-contact popup. Closing
the modal remains available while inference is running.
