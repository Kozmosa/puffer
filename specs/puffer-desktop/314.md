# 314. Composer Routing Is Session Scoped

## Summary

Composer provider and model selections are now remembered per session instead
of reverting to project defaults when switching between active sessions.

## Behavior

- Selecting a provider or model stores that routing choice under the current
  session id.
- Returning to a session restores its provider, model, placeholder label, and
  turn request routing.
- Other sessions continue to show their own default or previously selected
  routing state.

## Compatibility

Existing session metadata and project defaults still seed composer routing when
a session has no local routing preference.
