# 297. PDF Preview Legibility Controls

## Summary

The desktop Files pane PDF preview now treats rendered pages as the primary
document view and keeps fallback text useful when renderer assets are loading or
unavailable.

## Behavior

- PDF previews expose zoom-out, reset, and zoom-in controls above the rendered
  page stack.
- The page-limit status, such as `Showing first 20 of 29 pages.`, uses
  foreground text on an explicit surface instead of blending into the preview
  background.
- Extracted PDF text fallback filters common internal font and CMap tokens such
  as `CIDInit` and `TeX-T1-0`, plus isolated control-glyph noise.
- If the daemon supplies only noisy PDF text, the preview falls back to clean
  text extracted from rendered page streams when available.

## Compatibility

This only changes desktop document preview presentation. It does not alter file
read limits, PDF page count limits, raw file editing, or Office/Markdown/CSV
preview behavior.
