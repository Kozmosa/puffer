# Permission Prompt Guardrails

## Scope

The desktop client no longer presents a broad all-actions permission choice in
live, cached, fallback, or mock permission dialogs.

## Design

Generic permission choices are now limited to approve once, always allow the
current request, and deny. Browser prompts keep the bounded browser-context
approval.

The desktop action mapper no longer maps arbitrary labels containing "all" to
`allow_all_session`. The wire type still accepts that value for backend
compatibility, but the desktop UI does not emit it from normal prompt buttons.
