# Verified Skills Settings

## Scope

Desktop Settings has a Verified Skills pane for adding external verified Lambda
Skill libraries without making users edit manifest fields by hand.

## Change

- The Settings navigation labels the pane `Verified Skills`.
- Users add a library through the native folder picker. The UI does not expose
  raw manifest paths, doctor counters, allowed-tool lists, or host binding JSON.
- Choosing a top-level translated library folder is supported; the daemon fills
  in compiler and host binding config when it can discover them from the folder.
- The pane shows added folders by friendly folder name and a compact readiness
  label. Detailed failure text remains available through `/skills doctor` and
  `/doctor`.
- New folders are saved as workspace-scoped config and model invocation remains
  enabled when the daemon can infer generated verification output.

## Compatibility

The desktop UI writes the same YAML manifests that TUI users can create. It
does not copy or vendor skill libraries, and it does not bypass Lambda Skill
readiness checks or the host-call gate.
