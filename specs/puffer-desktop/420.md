# Background Stream Delta Preservation

## Context

Users can keep one session running while reading or working in another session.
The desktop shell already caches background approval and question prompts, but
plain assistant stream deltas were dropped until the transcript later persisted.

## Change

- Background `text-delta` events now update the cached transient conversation
  state for that session.
- Switching back before transcript persistence restores the streamed assistant
  text instead of showing an empty running conversation.
- Background thinking, reflection, and retry events now refresh cached turn
  state so the composer and sidebar retain the right in-flight status.

## Compatibility

Selected-session streaming behavior is unchanged. Completion and error events
still clear background runtime state through the existing settled-turn path, so
persisted transcripts remain the source of truth after a turn finishes.
