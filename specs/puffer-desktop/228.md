# 228. Browser Pane Follows Agent Recordings

## Summary

The desktop Browser pane now consumes live root-session browser recording events
for the active tab, so agent Browser actions can update the side pane even when
the pane misses or already has an older backend frame.

## Behavior

- The pane still subscribes to the active backend tab's frame, state, and
  devtools streams.
- It also subscribes to `browser:<session>:recording` while a root session is
  active.
- A recording frame is rendered only when its root session, tab id, and backend
  session still match the active pane.
- Applying a matching recording also refreshes the URL, title, loading, status,
  connected state, favicon, and retained tab frame.

## Compatibility

No daemon protocol changes are required. Existing recording snapshots and live
recording events are reused as a UI synchronization path.

## Tests

`desktop-ui.spec.ts` covers a Browser action recording for the currently active
tab after the pane has already rendered an older blank frame, and verifies the
canvas updates to the recorded frame instead of staying blank.
