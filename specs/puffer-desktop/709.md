# Remote Browser Daemon Params

## Summary

Browser-preview URLs that provide only `pufferRemoteBackend` style parameters now
work as the active daemon connection for ordinary desktop RPCs.

## Design

- `configuredBrowserDaemonHandshake` accepts remote backend, token, protocol, and
  workspace-root query/env aliases after explicit local daemon aliases.
- Explicit local aliases such as `backend`, `token`, and `workspaceRoot` keep
  precedence.
- A protocol regression opens the UI with only remote-style params and verifies
  the local daemon client connects to that WebSocket.

## Compatibility

This preserves existing local and Tauri handshakes while restoring browser
preview links that historically used `pufferRemoteBackend`.
