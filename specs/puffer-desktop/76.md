# Daemon-backed desktop settings

Desktop settings and auth operations use the active daemon client whenever a
daemon handshake is configured or Tauri can provide one.

Compatibility contract:

- Browser-preview and Tauri-hosted Desktop use the same daemon RPC surface for
  settings snapshots, provider login, logout, and external credential imports.
- Explicit SSH remote settings calls keep using the Tauri remote invoke path.
- Provider/auth state shown in Desktop must match the daemon that handles
  `create_session` and `run_agent_turn`, so connected providers do not drift
  between the Tauri shell and the runtime daemon.

Regression coverage lives in `apps/puffer-desktop/tests/settings-ui.spec.ts`.
