# 368. Provider Settings Reconnect Recovery

Settings provider routing state now keys local model-picker cache invalidation
on daemon URL, workspace root, authenticated provider ids, provider ids, and
default routing config. A reconnect that restores the same default provider but
changes auth/provider inventory resets stale model cache and loading flags.

When the Providers pane is active and a refreshed snapshot exposes authenticated
agent providers while no provider is selected, the pane selects the default
route provider and loads its models. This keeps the model selector rendered
after reconnect-driven settings refreshes.
