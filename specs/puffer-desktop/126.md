# OAuth Login Duplicate Guard

## Scope

Provider OAuth login can be started from the Settings Providers pane by clicking
the OAuth connect button.

## Change

- The desktop shell now ignores OAuth login requests while any auth request is
  already in flight.
- The login view routes OAuth clicks through the same local busy check used by
  API key submission.
- The fake desktop daemon supports `login_with_oauth` for UI regression tests.
- A Playwright regression test delays the daemon OAuth response, triggers the
  button twice in the same browser task, and verifies only one
  `login_with_oauth` request is sent.

## Compatibility

Normal first-click OAuth login behavior is unchanged. The guard only removes
duplicate provider connection attempts caused by rapid or repeated activation
while the first request is still pending.
