# Terminal Create In-Flight Guard

## Scope

The desktop Terminal pane opens PTY-backed tabs through the daemon. Repeated
activation of New terminal while the first PTY create request is still in flight
must not create duplicate terminal tabs.

## Change

- `TerminalPane.svelte` tracks whether a PTY create request is currently in
  flight.
- Repeated create requests return before sending a second `pty_open`.
- Switching sessions resets the guard with the rest of the terminal restore
  state.
- A Playwright regression test delays the second `pty_open` and verifies rapid
  repeated New terminal activation only sends one create request.

## Compatibility

The daemon PTY protocol is unchanged. Automatic first terminal restore and
normal user-created terminal tabs still use the same `pty_open` flow.
