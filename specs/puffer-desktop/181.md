# Preserve First Prompt During Initial Load

## Context

Opening a session starts an initial `load_session_detail` request. If that
request is slow and the user sends the first prompt before it returns, the
desktop UI already has an optimistic user row and live turn state for the
selected session.

## Change

- Detect transient conversation state when an initial session load resolves.
- Reuse transient message ids and keep live turn state when the same session
  has started a turn while the load was in flight.
- Keep the previous fresh-session reset behavior when no transient state exists.

## Compatibility

Persisted timelines remain authoritative. This only changes how a delayed
initial detail response is merged with already-visible local rows, so the first
submitted prompt no longer disappears, flashes, or changes DOM identity while
the turn is still running.

## Verification

- Added a Playwright regression that delays the initial `load_session_detail`,
  sends the first prompt, then verifies the optimistic user message row remains
  stable after the persisted copy arrives with a different id.
