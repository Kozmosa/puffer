# Turn Scoped Prompt IDs

## Scope

Live permission and user-question prompt ids are UI identities. Backend
request ids can be reused by later turns, so dismissing one prompt must not hide
a distinct prompt from a new turn.

## Change

- Live permission prompt ids now include both `turnId` and `requestId`.
- Live user-question prompt ids use the same turn-scoped identity scheme.
- A Playwright regression test approves a permission prompt, starts a later turn
  with the same backend request id, and asserts the second prompt remains
  visible.

## Compatibility

The daemon request id is still preserved in the lookup table used for
`resolve_permission` and `resolve_user_question`. Replayed events within the
same turn still replace the existing prompt instead of duplicating it.
