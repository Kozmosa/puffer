# WebKit desktop UI regression entrypoint

The desktop UI test surface needs an explicit WebKit pass because macOS Tauri
uses WebKit-backed WebViews, and Chromium-only Playwright coverage can miss
renderer and event-loop differences.

- `npm run test:desktop-ui:webkit` runs the same desktop UI and `plus-*`
  regression suite as `test:desktop-ui`, but forces Playwright's WebKit browser.
- The setup docs install both Chromium and WebKit so contributors can run the
  default browser pass and the WebView-adjacent pass without rediscovering
  Playwright browser-install errors.
- Production verification now lists both desktop UI passes alongside Svelte,
  Vite, Tauri, and Codex Playwright checks.

This does not change runtime UI behavior; it makes the existing desktop
automation framework cover the browser engine closest to the shipped macOS app.
