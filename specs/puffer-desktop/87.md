# Workspace session history visibility

The Desktop workspace board now keeps all daemon sessions in its searchable
project data model instead of truncating each project to the six newest
sessions before search and rendering.

Project rows report total sessions rather than a capped agent count, and older
sessions remain visible/searchable after new sessions are created. This keeps
session history discoverable from the workspace instead of making older
transcripts look like they disappeared when a newer agent is opened.
