# 351. Browser Tab-List Timestamp Grace

Browser tab-list events are authoritative for daemon tab membership, active-tab
selection, URL, and connected state, but genuinely stale tab-list events must
not overwrite newer local browser state.

## Contract

- Tab-list events older than the local tab by more than a short grace window are
  treated as stale and keep the local tab state.
- Tab-list events that arrive in the same browser-open/reconnect turn are still
  accepted even if local state was stamped a few milliseconds later.
- Missing active tab ids never create requests for synthetic backend tab ids;
  the browser chooses a real connected tab from the event instead.
- Disconnected active tab events remain visible while the reconnect request is
  pending, so navigation controls are disabled and the address bar stays
  editable.

## Coverage

`tests/desktop-ui.spec.ts` covers missing active ids, disconnected active tabs,
reconnect-pending toolbar state, and truly stale URL tab-list events.
