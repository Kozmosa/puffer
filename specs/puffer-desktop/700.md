# Tasks List Lazy Rendering

The desktop Tasks screen now renders filtered task rows in progressive batches.
The first batch is capped at 40 rows, and the next batch loads when the bottom
sentinel nears the viewport or when the user activates the visible load-more
button.

Filtering, counts, selection, task details, and action handlers still operate
against the full task snapshot. Only the row DOM is windowed, so existing daemon
`workflow_list` responses and task mutation RPCs remain unchanged.

The Tasks Playwright coverage now includes a large 65-task snapshot to verify
that only the first 40 rows render initially and that the remaining rows appear
after loading the next batch.
