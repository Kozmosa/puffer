# 295. Auto-Recap Timer Scoped to Session at Blur Time

## Summary

The auto-recap idle timer now captures the session ID when the window loses focus
and only fires `/recap` if the same session is still selected when the timer
expires.

## Behavior

- `armRecapBlurTimer` records `selectedSession.id` at the moment the window blurs.
- When the timer fires after `RECAP_IDLE_MS`, it checks that `selectedSession.id`
  still matches the captured session ID before submitting `/recap`.
- If the user switched sessions during the idle period, the timer is a no-op.
- `cancelRecapBlurTimer` clears both the timer and the captured session ID.

## Compatibility

No protocol or config changes. The recap feature continues to gate on
`config.recap.enabled` inside puffer-core.

## Tests

`chat-session-ui.spec.ts` verifies that switching sessions during the blur window
prevents `/recap` from being sent to the new session.
