# Browser Special-Key Dispatch

The Browser pane forwards keyboard input from the Svelte canvas to the Rust
managed-browser backend in both the desktop-local backend and the spawned
`puffer daemon` backend. Printable keys continue to use `keyDown` events with
their text payload.

Non-printable keys now include Chrome DevTools Protocol virtual key and location
fields when the backend calls `Input.dispatchKeyEvent`. Enter is serialized with
virtual key code 13 and carriage-return text on key-down events so focused page
controls can submit. Backspace, Delete, Tab, Escape, and arrow keys include
their virtual key codes without text payloads so focused fields can edit or
navigate normally.

Key-up events never include text. This preserves the existing frontend contract
while making the backend CDP payload match Chromium's expected special-key
shape.
