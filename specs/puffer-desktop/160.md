# Active Agent Live Sidebar State

## Summary

The desktop sidebar now keeps live agent state per session instead of tying it
only to the currently opened chat session.

## Design

`App.svelte` stores a small sidebar-only overlay keyed by session id. Submitting
a prompt marks that session as `thinking`, streamed turn events can update it to
`running` or `awaiting`, and turn completion workspace notifications clear the
overlay. Transcript live-stream state remains scoped to the selected session.

## Contracts

- A running agent remains visible as `thinking` or `running` after the user opens
  another session.
- A live sidebar overlay can keep a recently selected session visible even when
  grouped history is stale.
- The overlay is removed on `workspace:sessions:changed` with `turn_complete` or
  `turn_error`, so completed agents fall back to grouped session metadata.
- The sidebar rendering contract stays unchanged: it still receives an
  `ActiveAgent[]` with a concrete `state`.
