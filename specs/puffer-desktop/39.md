# Complete onboarding repo step

## Purpose

The signed-in onboarding repo picker must not be a dead end. Users who click
Continue or Skip for now should land in the workspace shell.

## Design

The onboarding screen now exposes an explicit finish callback for the repo
selection step. The app records that onboarding was completed locally so a
forced onboarding URL cannot immediately reopen the screen after the user exits
it.

## Logic

- Continue and Skip for now both call the parent completion handler.
- The parent hides onboarding, switches to the workspace screen, persists the
  local skip flag, and refreshes pins plus session groups.
- Provider login/import flows also mark onboarding completed so later settings
  refreshes do not resurrect forced onboarding.
- Playwright covers both repo-step exits with a mocked signed-in daemon.

## Contracts

No daemon contract changes. This is UI flow state only; provider auth and
session list RPCs remain unchanged.
