# Keep session turn events scoped to the active turn

Desktop session streams can replay or deliver late events after a turn has
already completed. The Agent view now records settled turn IDs and ignores any
late event for a settled turn, or any event whose turn ID differs from the
currently active turn.

The transcript reload kicked off by `turn-complete` or `turn-error` also carries
the completed turn ID. If a newer turn starts before that reload returns, the
older reload no longer replaces `liveStreamItems` or `submittedMessages`.

Compatibility contract:

- Replay events for an in-flight turn still update the active stream.
- Text deltas that arrive before `turn-start` can still establish the active
  turn.
- Stale completion reloads cannot clear or overwrite the visible answer for a
  newer active turn.
- Settled turn IDs are reset when a different session is opened.

Regression coverage lives in `chat-session-ui.spec.ts` and interleaves an old
turn completion reload with a newer streamed answer in the same session.
