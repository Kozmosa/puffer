# Desktop turn-control coverage

The desktop e2e suite now covers more of the normal personal-agent control
surface beyond browser-specific flows.

Compatibility contract:

- Composer turn requests preserve selected provider, model, reasoning, fast
  mode, and permission-mode fields when those controls are available.
- The Stop control sends `cancel_turn` for the active daemon turn id without
  relying on a browser-only path.
- Session title editing calls `rename_session`, refreshes the workspace list,
  and updates the open agent header from the daemon response.
- The Playwright fake daemon implements `rename_session` and `cancel_turn` so
  tests exercise the same request names as the desktop daemon protocol.

Regression coverage lives in `chat-session-ui.spec.ts`, using a mocked daemon
session with an explicit GPT-5 model for fast-mode availability plus a title
rename fixture.
