# Empty-session first prompt fallback

Opening an agent now installs the target session metadata before the transcript
detail request resolves. If the initial `load_session_detail` call is slow or
fails, the chat view still has a concrete `sessionId`, provider, model, and cwd.

This keeps the first-message composer usable on a newly-created empty session
instead of showing the empty transcript state with a disabled input.

The persisted transcript still replaces the fallback once `load_session_detail`
succeeds, and switching sessions clears live turn state before the async load.
