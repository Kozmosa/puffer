# Browser toolbar command guards

Browser toolbar actions now keep one in-flight navigation-class command per
backend tab. URL submits, Back, Forward, and Reload disable the current tab's
navigation controls until the daemon response settles.

Repeated submits or fast double clicks are ignored while a command is pending,
so the daemon receives one command for the user's single visible action. Pending
toolbar state is cleared when the Browser pane switches sessions or when daemon
tab state removes all tabs.

Regression coverage dispatches repeated URL submits and Reload clicks while the
fake daemon holds the first response, then asserts only one daemon request is
sent and the toolbar is disabled during the pending command.
