# Background Turn Error Preservation

## Context

Background sessions can fail while the user is viewing another session. The
selected-session path surfaces inline agent errors, but background turn errors
were cleared from transient state before the transcript could persist an error
item.

## Change

- Background `turn-error` events now cache an inline system error row with the
  daemon error text.
- Settled background errors clear pending approval and question prompts while
  preserving useful non-prompt live items until persistence catches up.
- Replay events for the settled failed turn remain ignored.

## Compatibility

Selected-session error handling is unchanged. Persisted transcript reloads
remain authoritative once the daemon records the failed turn.
