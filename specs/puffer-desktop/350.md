# 350. PDF Preview Controls Contrast

The Files preview PDF renderer keeps zoom controls and page-limit status readable
when the preview pane is narrower than the full desktop window.

## Contract

- The PDF controls row uses a solid surface that remains distinct from the PDF
  preview background in light and dark themes.
- The page-limit status pill uses high-contrast foreground/background colors so
  messages such as `Showing first 20 of 29 pages.` cannot blend into the shell.
- Narrow preview panes use container-based wrapping so zoom buttons, the reset
  value, and the zoom slider stay visible and pointer-accessible even when the
  overall app window is still wide.

## Coverage

`tests/files-workspace-ui.spec.ts` exercises a long PDF in a squeezed Files
preview pane, checks the status contrast, and verifies both the zoom button and
slider remain the top pointer targets before zooming the rendered canvas.
