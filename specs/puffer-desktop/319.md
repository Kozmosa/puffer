# OpenAI Fallback Model Freshness

## Contract

The desktop model picker must treat runtime model discovery as authoritative,
and its bundled OpenAI fallback list must not steer a user toward stale default
models while discovery is still loading or unavailable.

## Behavior

- The bundled OpenAI provider list starts with the current public Responses API
  family instead of the older `gpt-5` entry.
- Fallback metadata uses current large-context and 128k-output limits so GUI
  context estimates and runtime defaults are less misleading before discovery
  completes.
- Runtime discovery still replaces the bundled list whenever the provider
  returns an available model list.

## Verification

Resource parsing coverage asserts that `openai.yaml` starts with `gpt-5.5`,
keeps 128k output limits, and includes the current nano entry.
