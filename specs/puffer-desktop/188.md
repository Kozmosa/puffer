# Files LSP Location Jumps

## Context

The Files pane parses LSP definition and reference results into file, line, and
character locations. Clicking one of those locations opened the file but dropped
the parsed position, so users stayed wherever the editor already was.

## Change

- Preserve line and character data when opening a clicked LSP location.
- Focus the editor at the parsed one-based line and character after the target
  file is active, including already cached tabs.
- Extend the desktop fake daemon so tests can seed file contents and non-default
  LSP locations.

## Compatibility

Existing file links, tab preview behavior, and pinned tabs are unchanged. The
new character handling only affects callers that provide a parsed location.

## Verification

- Added a Playwright regression that opens a symbol popup, clicks an LSP result
  at `src/lib.rs:30:5`, and asserts the editor focus and cursor position.
