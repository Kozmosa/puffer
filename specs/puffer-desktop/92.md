# IME-safe composer submit

The desktop agent composer must not treat Enter as a send shortcut while a text
input method editor is composing. `ConversationView` now ignores keydown events
with `KeyboardEvent.isComposing` or the legacy `keyCode` 229 marker before
checking the Enter shortcut.

This keeps Chinese, Japanese, Korean, and other IME candidate confirmation
flows inside the textarea instead of submitting and clearing a partial draft.
Regular Enter still sends the prompt, and Shift+Enter still remains available
for newlines.

Playwright coverage in `chat-session-ui.spec.ts` dispatches both modern and
legacy composing Enter events and asserts no `run_agent_turn` request is sent.
