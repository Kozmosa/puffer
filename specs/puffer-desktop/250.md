# File Preview Formats

## Scope

The desktop Files pane previews common user-facing document and data files
instead of treating every binary payload as an opaque blob. The feature covers:

- PDF files through an embedded `application/pdf` viewer sourced from the
  daemon's base64 file payload.
- Markdown and CSV files through local structured rendering from UTF-8 file
  content.
- Office Open XML files (`.docx`, `.pptx`, `.xlsx`, `.xlsm`) through client-side
  ZIP/XML text extraction.
- Legacy Office binary extensions (`.doc`, `.ppt`, `.xls`) with a visible
  type-specific fallback explaining the modern rich-preview boundary.

## Architecture

The daemon protocol remains unchanged: `read_file` continues returning either
UTF-8 text or base64 content with size and truncation metadata. Preview
selection and rendering live in the desktop frontend so local and remote daemon
backends share the same behavior.

`filePreview.ts` owns format detection and extraction. Office previews parse the
ZIP central directory, decompress deflated entries with the browser
`DecompressionStream`, and read only the relevant XML parts:

- `word/document.xml` paragraphs for DOCX.
- `ppt/slides/slide*.xml` text runs for PPTX.
- workbook metadata, shared strings, and worksheet XML for Excel.

## UI Contract

Known preview formats disable the code editor surface and render a document
view in the existing right-hand viewer. Regular UTF-8 source files keep the
editable code path and LSP interactions unchanged. Unknown binary files retain
the existing opaque binary message.

## Tests

Playwright coverage seeds Markdown, CSV, PDF, DOCX, PPTX, and XLSX files through
the fake daemon, opens each from the Files tree, and asserts the corresponding
preview surface renders expected content or the PDF embed.
