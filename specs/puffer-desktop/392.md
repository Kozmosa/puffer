# 392. Canonical File Preview Paths

Files opened through symlinks or other canonicalized paths must still render in
the tab the user selected.

## Contract

- The Files pane keeps the clicked path as the logical tab and active-view key.
- A daemon `read_file` response may return a different canonical `path`.
- Successful reads from canonicalized paths populate the selected tab instead
  of leaving the viewer blank.
- Rich previews, raw editing state, size updates, and reloads all stay keyed to
  the user's visible tab path.

## Verification

`tests/files-workspace-ui.spec.ts` seeds a file whose fake daemon response
returns a different canonical path, opens the visible `link-readme.md` row, and
asserts the Markdown preview renders while the logical tab remains selected.
