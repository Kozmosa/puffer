# MCP Add Server Duplicate Guard

## Scope

The Settings MCP Servers pane can add or replace a server through the daemon
`add_mcp_server` endpoint.

## Change

- The MCP save handler now ignores calls while an add request is already in
  flight.
- A Playwright regression test delays `add_mcp_server`, triggers the Add server
  button twice in one browser task, and verifies only one add request reaches
  the daemon.

## Compatibility

Normal first-click add behavior is unchanged. The guard only removes duplicate
MCP server writes caused by repeated activation while the first add request is
still pending.
