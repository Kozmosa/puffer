# 395. Hidden Turn Completion State

When a non-selected session finishes a turn, the desktop client must clear both
the live sidebar overlay and any cached turn runtime state for that session.
The user may switch back before the final persisted transcript has been loaded,
so the cached `currentTurnId`, cancel state, pending permission/question
lookups, and live stream cards cannot be allowed to revive a stale running
state.

The cache cleanup preserves locally submitted user messages so a quick switch
back does not cause composer history to disappear while the daemon finishes
writing the transcript. Once the persisted detail arrives, normal transcript
deduplication still reconciles the local message with the stored event.

Regression coverage starts a turn in one session, switches to another session,
emits a completion workspace refresh before the hidden session detail changes,
then reopens the original session and asserts that both the sidebar and detail
remain idle with no Stop button.
