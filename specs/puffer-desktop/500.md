# Monitor Task Triage Details

## Summary

The Pipelines connector pane now renders monitor tasks as triage rows with
message context and explicit ignore choices.

## Behavior

- Monitor task rows show the task description beneath the subject so the user
  can decide from the GUI without opening the task detail first.
- Up to two monitor task actions remain visible as direct agent-start buttons.
- Suggested ignore reasons are separate buttons that dispatch deterministic
  `/tasks ignore <task-id> <reason>` commands.
- If a task has no suggested ignore reasons, the generic ignore button remains
  available.
- Search indexing already includes task descriptions, action prompts, and
  ignore reasons, so the new visible fields follow existing query behavior.

## Compatibility

The daemon protocol is unchanged. Existing `monitor_tasks` snapshots continue
to work; tasks without descriptions or ignore reasons simply omit the new
description/choice rows.
