# Puffer Desktop Update 627

## Summary

The CEF BrowserWidget compatibility patch now forces the default constructor
declaration into Chromium's `BrowserWidget` header when the old declaration is
missing.

## Design

- `scripts/release.sh` no longer relies on a broad sentinel for the default
  constructor declaration.
- The patch replaces the exact public constructor block, which is idempotent
  because the original block no longer exists once the declaration has been
  inserted.

## Compatibility

This keeps CEF's `ChromeBrowserWidget` constructor aligned with the matching
out-of-line default constructor definition during Linux release builds.
