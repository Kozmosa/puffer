# 367. Pipeline Refresh Draft Preservation

Pipeline refreshes now preserve unsaved workflow node edits. Dirty workflows are
kept by slug and merged over the latest daemon `workflow_list` snapshot instead
of replacing the editor state wholesale.

Workflow refresh uses a generation guard so late responses cannot update editor
state after a newer refresh has started. Existing selected nodes are preserved
when still present in the merged workflow, avoiding unnecessary graph and
inspector churn after delayed refreshes.
