# New agent provider routing

## Purpose

When users create a new Personal Agent from the workspace screen, the provider
they choose must be the provider used by the first turn.

## Behavior

- The new-agent provider picker accepts real provider ids such as `openai` and
  `anthropic`, while keeping the Codex/Claude/Puffer fallback labels.
- After `create_session`, the desktop app opens the refreshed session with the
  provider/model echoed by the daemon.
- FakeDaemon now supports `create_session`, stores the created session, and
  returns provider/model routing for e2e coverage.
- Desktop e2e coverage creates an Anthropic session and verifies the first
  `run_agent_turn` carries `providerId=anthropic`.

## Compatibility

Existing workspaces and sessions still load without provider/model routing and
fall back through the existing settings defaults.
