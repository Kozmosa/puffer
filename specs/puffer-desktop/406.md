# 406. Background Approval Requests

Session events for non-selected agents now cache live approval and question
requests in the same transient conversation store used for submitted turns.
When a background agent moves to `awaiting`, returning to that chat restores
the pending approval or question instead of showing an idle transcript with no
way to continue the turn.

The cached request keeps the daemon turn/request identifiers so approving,
denying, or answering after switching back still calls the original
`resolve_permission` or `resolve_user_question` endpoint.
