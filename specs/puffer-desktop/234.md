# 234. Custom Model Providers In GUI Pickers

## Summary

The desktop GUI now treats provider resources as the source of truth for agent
provider eligibility instead of hard-coding the pickers to OpenAI, Anthropic,
and Puffer.

## Behavior

- Provider pickers include authenticated custom model providers such as Groq.
- New Agent, Connect Project, Model Picker, Settings default routing, onboarding,
  and composer auth checks share the same provider capability rules.
- Built-in aliases still map `codex`/`openai` and `claude`/`anthropic`.
- Alias-equivalent sessions use the same provider descriptor id for model
  loading, selection state, and thinking-option lookup.
- Non-model auth such as GitHub remains excluded from agent routing.

## Compatibility

No daemon protocol changes are required. The GUI uses the existing provider
summary fields `defaultApi`, `modelCount`, and provider id aliases.

## Tests

`files-workspace-ui.spec.ts` covers New Agent and Connect Project flows with an
authenticated Groq provider and verifies that `create_session` receives
`providerId: "groq"`.
