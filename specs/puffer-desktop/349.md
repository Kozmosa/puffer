# 349. Browser URL Race Recovery

## Summary

Browser tab and address-bar state now reject stale tab-list updates and preserve
the user's navigation intent across transient command failures.

## Behavior

- Browser tabs track an `updatedAtMs` value, and tab-list events older than the
  current local tab state cannot replace the active tab URL.
- Failed navigation keeps the typed URL visible in the address bar so the user
  can retry or edit it.
- A pending navigation records its backend session and URL; duplicate submits
  for the same URL are ignored until the navigation settles or times out.

## Compatibility

No daemon protocol changes. The existing `updatedAtMs` field from tab-list
events is used when present, while older events without timestamps remain
accepted for compatibility.

## Tests

`browser-ui.spec.ts` covers stale URL rejection, typed URL preservation after
failure, duplicate navigate suppression, disconnected-tab recovery, stale empty
tab-list handling, and navigation loading fallback.
