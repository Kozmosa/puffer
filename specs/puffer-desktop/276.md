# 276. File Tab Close Target Labels

## Summary

Files tab close buttons now identify the file by path instead of only by
basename. Repositories commonly contain duplicate names such as `main.rs` in
different folders, so a generic `Close main.rs` control could target the wrong
open tab.

## Design

- Close controls use `Close <relative path>` when a file is inside the current
  workspace root.
- Files outside the root fall back to the full path.
- The visible tab label remains the basename to preserve compact tab layout.
- Close controls remain native buttons and keep the existing close behavior.

## Compatibility

This changes only file-tab button metadata and tooltip text. File opening,
preview, save, dirty-state, and tab-persistence contracts are unchanged.

## Tests

- Existing keyboard close coverage now targets `Close src/lib.rs`.
- A new Playwright regression opens `src/main.rs` and `tests/main.rs`, asserts
  the generic `Close main.rs` button is gone, and closes only
  `Close tests/main.rs`.
