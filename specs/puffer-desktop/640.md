# Linux CEF Optional Runtime Packaging

## Scope

Make the Linux CEF release package robust when Chromium's output directory omits optional runtime files or directories such as `swiftshader`, `chrome-sandbox`, or `cefclient`.

## Design

The release script now copies optional Linux CEF runtime entries through explicit `if [[ -e ... ]]` guards instead of `[[ -e ... ]] && cp ...` expressions. This preserves the intended optional-copy behavior while avoiding a nonzero function status when the final optional entry is absent.

## Compatibility

Required runtime validation is unchanged: Linux CEF packaging still requires `libcef.so` before creating the package. Optional runtime assets are included when present and skipped when absent, matching the previous artifact contents without letting a missing optional directory fail an otherwise successful build.

## Verification

The fix addresses a Linux package failure where CEF linked successfully and `cefsimple` built, but `make build-release-cef` returned nonzero because the final optional runtime copy expression evaluated false for a missing `swiftshader` directory.
