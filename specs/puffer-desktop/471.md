# Pipeline Trigger Readiness

## Summary

The Pipelines editor now respects connector trigger-readiness metadata from the
daemon snapshot instead of treating every connector or connection as a valid
workflow trigger candidate.

## Behavior

- Connector and connection rows read optional `can_trigger_workflow` fields.
- Older daemon snapshots fall back to the previous `can_subscribe` behavior.
- Rows that cannot start workflow triggers remain searchable and visible, but
  are disabled and labeled `no trigger`.
- Selecting a disabled connection or connector cannot mutate the draft workflow
  trigger.
- Event-capable connectors still stage the deterministic `/connect` command
  when no connection exists.

## Compatibility

Existing snapshots without readiness fields continue to work. The new disabled
state only applies when the daemon explicitly reports a connector or connection
as not trigger-capable.
