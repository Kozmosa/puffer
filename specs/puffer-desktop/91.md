# Workspace session history

The desktop workspace now renders an explicit Session history region above the
project rows. It is built from the same daemon `list_grouped_sessions` response
as the workspace board, sorted by most recent session update.

Starting a new agent must not hide older sessions. After a new session opens and
the user backs out to Workspace, older sessions remain available in Session
history and can be reopened with their persisted transcript.

Workspace search applies to Session history as well as project rows, so old
sessions remain discoverable by title, cwd, project label, or project path.
