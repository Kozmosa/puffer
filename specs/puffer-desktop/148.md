# Workspace Switch Mode Status Isolation

## Scope

The Workspace switch modal exposes Current, Local, and Remote modes. Status and
error messages describe the current mode's pending or failed action and should
not leak after the user changes modes.

## Change

- Workspace switch mode changes now route through a shared helper.
- Switching modes clears transient error and status text before rendering the
  new panel.
- The helper preserves typed local and remote fields so users can still compare
  modes without losing work.
- Playwright coverage now forces a failed local daemon restart, switches to
  Remote, and verifies the stale local error is gone while the SSH target field
  is visible.

## Compatibility

The daemon restart and SSH connection contracts are unchanged. Busy operations
still lock mode changes until the in-flight request settles.
