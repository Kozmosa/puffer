# Provider Switch Failure Rollback

## Context

The model picker shows a pending provider while its model list is loading.
During rapid provider changes, that pending route could become the rollback
baseline for a later failed switch, leaving the session on an intermediate
provider instead of its last stable provider and model.

## Change

- Provider-switch rollback now captures the last stable provider/model before
  the first pending switch in a rapid sequence.
- A failed latest switch restores that stable route even if an earlier pending
  provider later returns models.
- Picking a model, changing sessions, or completing a switch clears the rollback
  baseline.

## Compatibility

The picker still shows the pending provider while models load. Successful
switches continue to select the provider default model, while failed rapid
switches keep the session sendable on the previous stable route.
