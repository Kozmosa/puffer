# Background Tool Activity Preservation

## Context

Users often leave one session running while reviewing another. Background
sessions already preserved assistant text, approvals, and questions, but tool
request and invocation events were still dropped until the transcript
persisted.

## Change

- Background `tool-calls-requested` events now cache pending tool cards in the
  session's transient conversation state.
- Background `tool-invocations` events now update or create cached tool cards
  with the final status, input, output, and parsed input JSON.
- Returning to the running session before persistence restores the same tool
  activity that would have been visible if the session had remained selected.

## Compatibility

Selected-session tool rendering is unchanged. Settled turn events still clear
background runtime state through the existing completion path, keeping the
persisted transcript authoritative once the turn finishes.
