# Browser Extension Settings

## Summary

The desktop Settings screen adds a Browser tab for extension loading and
captcha solver configuration. The Browser tab has two subtabs:

- Extensions: lists bundled solver extensions and user-added custom extension
  folders, with add/remove/enable controls for custom extensions.
- Captcha solvers: selects and configures one built-in solver at a time.

The built-in solver catalog contains NopeCHA and 2Captcha.

## Configuration

- `PufferConfig.browser.extensions_enabled` controls whether bundled browser
  extensions may be loaded into compatible browser sessions.
- `PufferConfig.browser.extensions` stores user-added unpacked extension
  folders with IDs, display names, paths, and enablement.
- `PufferConfig.browser.captcha.enabled` enables captcha solver extension use.
- `PufferConfig.browser.captcha.selected_solver` selects the default solver.
- Per-solver config stores `enabled`, `base_url`, and `api_key_secret_id`.
- Unknown solver IDs are dropped during normalization; selected solver falls
  back to `nopecha` when unset or invalid.
- Solver enablement is single-choice: normalization and save handling mark only
  `selected_solver` as enabled and force every other built-in solver inactive.

## Desktop API

- Settings snapshots include `browser`.
- `save_browser_settings` validates built-in solver IDs, HTTP(S) base URLs, and
  custom extension `manifest.json` files, writes user config, refreshes daemon
  state, restarts active browser roots when launch settings change, and returns
  a fresh snapshot.
- API keys are saved separately through the existing encrypted secret vault.
  Browser config stores only secret IDs.

## Resources

- MIT-licensed NopeCHA 0.6.0 and 2Captcha 3.7.2 Chrome extension payloads are
  bundled under `resources/browser_extensions/`.
- `resources/browser_extensions/catalog.toml` records source URLs, versions,
  hashes where available, license status, and bundled status.

## Runtime

- Screencast-managed Chrome receives `--disable-extensions-except` and
  `--load-extension` for enabled custom extensions and the selected built-in
  captcha solver extension.
- Built-in solver API keys and base URLs are seeded into extension local
  storage through CDP after extension workers become visible.
- The selected built-in solver is the only captcha extension loaded when
  captcha solving is enabled.
- Native CEF receives the same extension directory list before CEF
  initialization through `OnBeforeCommandLineProcessing`; CDP seeding also runs
  when the daemon attaches to a CEF DevTools endpoint.
- Captcha extensions may still depend on browser behavior outside Puffer's
  control, but the settings, launch flags, and storage seeding are wired for the
  screencast path and fresh native CEF initialization.
