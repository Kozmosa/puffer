# Browser Action Recording Window

## Change

- Agent Browser `open` actions now arm their recording window before opening or reusing the managed tab.
- When the action does not specify a tab id, the backend resolves the active tab or reserves the next tab id before launch so first screencast frames are retained.
- The desktop Browser pane and Browser tool card can hydrate from those retained frames instead of staying blank after an agent action.

## Compatibility

- Explicit `tabId` actions keep targeting the requested tab.
- Existing tab reuse behavior is preserved for `open`; CLI daemon `new` continues to force a fresh tab while pre-arming that tab's recording window.
