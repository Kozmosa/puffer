# 256. Browser Navigation Pending State Survives Tab Refreshes

## Summary

Browser navigation commands now keep a tab-scoped pending state so stale tab
list refreshes cannot clear user-visible loading feedback before the active
page state event arrives.

## Behavior

- Address submit, reload, and back/forward mark the active backend browser
  session as navigation-pending.
- Browser tab list refreshes preserve `Loading` for that backend session while
  the pending marker exists.
- Browser state events and command failures clear the pending marker.
- Empty tab-list events and session switches clear pending navigation state.

## Compatibility

The daemon protocol is unchanged. This only changes desktop client state
merging for existing Browser tab-list and Browser state events.

## Tests

`browser-ui.spec.ts` covers a reload followed by a stale `browser:<session>:tabs`
event and asserts the status remains `Loading`.
