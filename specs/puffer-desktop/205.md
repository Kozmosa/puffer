# Workspace Switch Entry Restore

## Context

The desktop workspace picker is opened from the workspace header. During the
mainline rebase, `App.svelte` still passed the `onOpenWorkspacePicker` handler,
but `Workspace.svelte` no longer accepted or rendered the header control. The
workspace picker interaction tests therefore waited indefinitely for the Switch
workspace entry.

## Change

- Restore the `onOpenWorkspacePicker` prop on the Workspace screen.
- Render the default workspace path as a header button with
  `title="Switch workspace"` and an accessible label.
- Re-add compact subtitle styling for the header path button.

## Compatibility

Project search, Create Project, and empty-workspace New agent flows are
unchanged. The header button appears only when a default workspace and picker
handler are available.

## Verification

- The existing workspace picker tests cover duplicate local switch submission
  and mode-switch error clearing through the restored entry point.
