# 259. Ignored Turn Completion Settling

## Summary

The chat event loop now records ignored completion events as settled turns when
they arrive for a non-current turn in the selected session.

## Behavior

- A late `turn-complete` or `turn-error` for an older turn no longer mutates the
  visible transcript while a newer active turn is running.
- The ignored older turn is still marked settled so later deltas for that turn
  remain ignored after the active turn finishes.
- Current-turn completion behavior and transcript refresh are unchanged.

## Compatibility

No daemon event schema changes. The desktop client only tightens local stale
event bookkeeping for selected-session turn events.

## Tests

`chat-session-ui.spec.ts` covers a current turn receiving an older turn's late
completion before the current turn completes, then verifies a later older delta
does not render.
