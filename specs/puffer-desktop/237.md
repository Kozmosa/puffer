# 237. Session Scoped Composer Controls

## Summary

Composer routing controls are now remembered per session instead of globally
across the project.

## Design

- Permission/access mode and fast mode are persisted under session-specific
  localStorage keys.
- Thinking level is persisted per session, provider, and model. When the same
  session is reopened, the saved thinking option is restored if the current
  model still advertises that option; otherwise the model default is used.
- Switching sessions no longer leaks access mode or thinking effort from the
  previous session into the newly selected session.

## Compatibility

This change only affects desktop UI preference storage. Agent turn request
fields and daemon contracts are unchanged.

## Tests

Playwright coverage now opens two sessions with the same model, changes
thinking/access in one session, verifies the second session keeps its defaults,
and verifies switching back restores and submits the first session's controls.
