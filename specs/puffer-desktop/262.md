# Binary document preview reads

PDF and Office previews now bypass the generic UTF-8 sniff in both desktop file
backends. The daemon and Tauri fallback return `.pdf`, `.doc`, `.docx`, `.ppt`,
`.pptx`, `.xls`, `.xlsx`, and `.xlsm` content as base64 even when the first
bytes look like plain text.

The frontend also accepts a UTF-8 PDF payload from older daemons and converts it
back to bytes before handing it to pdf.js. This keeps ASCII-heavy PDFs readable
during daemon/UI version skew.

Regression coverage exercises a PDF fixture that would previously be sniffed as
UTF-8, plus Rust unit coverage for the extension-based binary read policy.
