# Linux Tauri Bundle Default

## Scope

Make Linux desktop release packaging independent of AppImage tooling by default.

## Design

Linux Tauri builds now read `LINUX_TAURI_BUNDLES`, defaulting to `deb`. The release tarball packaging already accepts either a `.deb` or an AppImage, so the default build no longer requires `linuxdeploy` and its downloaded AppImage plugins to succeed before producing Puffer release artifacts.

## Compatibility

Callers that still need AppImage output can set `LINUX_TAURI_BUNDLES=deb,appimage` or any Tauri-supported bundle list. The default release path remains compatible with the existing artifact layout because `pack-linux-local` copies whatever `.deb` or AppImage files exist into the desktop tarball.

## Verification

This change addresses a remote Linux failure where Tauri successfully built the app and `.deb`, then failed only while running `linuxdeploy` for the AppImage bundle.
