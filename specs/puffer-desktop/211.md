# Tauri Provider-Safe First Turn Routing

## Context

The desktop app can use the WebSocket daemon or the local Tauri fallback
backend. Both frontends send canonical provider ids such as `openai` and
`anthropic`, while the local CLI backend launches `codex` and `claude`.

Newly created sessions can still be empty when the first prompt is sent. In
that state, the first turn may override the session's stored provider.

## Change

Tauri fallback turn routing now resolves provider and model together. A model
from the session record or desktop config is reused only when its provider
matches the selected backend provider. Provider-prefixed model ids are
normalized when the prefix matches the selected provider.

## Contract

- Empty first-turn sessions may switch provider from the selected UI provider.
- Switching provider does not reuse a stored model from another provider.
- Provider-prefixed model ids such as `openai/gpt-5.4` and
  `anthropic/claude-sonnet-4-5` are stripped only when the prefix matches the
  selected provider.
- Sessions with existing transcript events keep their stored provider for later
  turns, preserving transcript-provider continuity.

## Verification

- Added Tauri backend routing regressions for first-turn provider overrides,
  locked-session provider continuity, and provider-prefixed explicit models.
