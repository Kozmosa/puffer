# Closed Session Reconnect Memory

## Scope

Desktop session restore should honor a user's explicit close action across a backend reconnect.

## Behavior

- Closing an agent detail view clears `puffer-desktop:remembered-session`.
- A later daemon reconnect refreshes settings, pins, and grouped sessions.
- Reconnect reloads session detail only when that same session detail view is still open.
- A closed session remains visible in history but is not written back as the remembered restore target.

## Compatibility

This keeps remembered-session restore unchanged for currently open agent detail views, while preventing stale in-memory selection from reviving a session the user already closed.
