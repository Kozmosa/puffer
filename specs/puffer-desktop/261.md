# Lazy PDF preview loading

The desktop shell must not load the PDF rendering engine during normal chat,
browser, terminal, or text-file workflows. PDF support is still available when a
PDF tab is opened, but pdf.js is imported on demand from the preview component.

- `PdfDocumentPreview` dynamically imports `pdfjs-dist` and configures its
  worker only after a PDF preview is mounted.
- The Files UI test records browser requests and asserts Markdown/CSV previews
  do not load pdf.js before opening a PDF.
- Existing canvas rendering and text fallback behavior are unchanged.
