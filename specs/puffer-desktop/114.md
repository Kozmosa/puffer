# Turn Scoped Live Tool IDs

## Scope

Live tool cards use UI identities while a turn is still being streamed or while
the transcript reload after completion is still pending. Backend tool call ids
can be reused by a later turn, so the desktop UI must keep each turn's live
tool cards distinct.

## Change

- Live tool cards now use `turnId` and `callId` together for their UI id.
- Pending `tool-calls-requested` cards and completed `tool-invocations` cards
  share the same turn-scoped id, preserving in-place updates within one turn.
- A Playwright regression test starts two turns with the same backend `callId`
  and asserts both live tool cards remain visible.

## Compatibility

Persisted transcript tool ids are unchanged. Replay and completion events still
dedupe tool cards within the same turn, while later turns no longer overwrite
earlier live cards during delayed transcript reloads.
