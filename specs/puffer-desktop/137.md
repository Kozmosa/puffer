# Terminal Close Failure Preservation

## Scope

Closing a desktop Terminal tab requires daemon acknowledgement. If `pty_close`
fails, the UI must not drop the tab and leave the user thinking the terminal was
closed successfully.

## Change

- `TerminalPane.svelte` now removes a tab only after `closePty` succeeds.
- Failed close attempts keep the tab visible and show the daemon error in the
  terminal pane.
- The close in-flight guard still releases after failure so the same tab can be
  retried.
- A Playwright regression test forces `pty_close` to fail and verifies the tab
  stays visible.

## Compatibility

The daemon PTY protocol is unchanged. Successful tab close behavior remains the
same; only failed closes preserve the retryable UI state.
