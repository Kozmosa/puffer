# Sessionless Connector Setup

- Settings connector creation now calls the daemon `start_connector_setup` RPC instead of creating a regular session and dispatching `/connect`.
- Each setup generates a client-side setup id and listens on `connector-setup:<setupId>:event`.
- The UI continues to handle connector questions, completion, cancellation, and errors through the existing turn event shape.
- Connector setup no longer creates visible sessions such as `Connector connection configured` in the session list.
- The existing session-backed `dispatchSlashCommand` API remains available for explicit slash-command execution elsewhere.
