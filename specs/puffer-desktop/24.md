# puffer-desktop UI Test Suite Coverage

## Purpose

The default desktop UI test command must run every user-flow UI spec, not only
the original browser-focused file.

## Design

`npm run test:desktop-ui` now targets `tests/*ui.spec.ts`. This includes the
browser, chat/session, files/workspace, and terminal interaction suites while
leaving specialized non-UI harness specs on their own script.

## Logic

- `desktop-ui.spec.ts` remains the browser-heavy regression suite.
- `chat-session-ui.spec.ts`, `files-workspace-ui.spec.ts`, and
  `terminal-ui.spec.ts` are included by the shared suffix.
- `codex-playwright-mcp.spec.ts` remains excluded from this command because it
  is not a desktop user-flow UI spec.

## Contracts

Adding another desktop interaction spec should use the `*-ui.spec.ts` suffix so
the default desktop UI test command picks it up automatically.
