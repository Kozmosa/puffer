# Session History Stale Refresh Fallback

## Context

The desktop workspace keeps a daemon-provided grouped session list and a local
fallback map for sessions that have been opened before the daemon's grouped
history catches up. That fallback previously covered newly created sessions
that were missing from grouped history from the start.

## Change

Workspace refresh now also promotes the currently selected session into the
fallback map when a later `list_grouped_sessions` response omits it. This
preserves an opened session across stale, filtered, or temporarily incomplete
daemon history snapshots even after the user opens another session.

## Contract

- A session opened from history remains visible in Session history if a later
  grouped-history refresh omits it.
- Once grouped history includes the session again, the fallback entry is pruned.
- The fallback remains local UI state only; daemon session metadata stays the
  authoritative persisted record.

## Verification

- Added a Playwright regression where session Alpha starts in grouped history,
  a later grouped refresh omits Alpha, and opening Beta must not make Alpha
  disappear from Session history.
