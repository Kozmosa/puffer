# 396. Browser Recording URL Action Matching

Browser activity details continue to match recorded frames by backend session id
and tab id when those fields are present. URL-only Browser tool calls do not have
a stable tab identity, so the detail pane now prefers frames whose recorded URL
matches the action URL.

If no frame URL matches exactly, the pane keeps the existing structural fallback.
This preserves redirected or final-page recordings while preventing unrelated
later browser actions from overwriting the selected action's preview.

Live recording subscriptions apply the same URL preference before appending new
frames for URL-only actions. The regression test covers two URL-only Browser open
actions and verifies that selecting each action shows only its own recording.
