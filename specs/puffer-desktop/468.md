# Settings Permissions Load Retry

## Summary

The Permissions settings pane now treats the initial `list_permissions` request
as a single auto-load attempt. A failed load leaves the error visible and exposes
an explicit Retry action instead of immediately reissuing the same request.

## Behavior

- Opening Settings -> Permissions still lazy-loads permission policy data from
  the active daemon.
- A successful load populates the editable permission rows and stored path.
- A failed load records the error, stops the loading state, and marks the
  automatic attempt as complete.
- Retry starts a fresh `list_permissions` request against the same daemon.
- Changing the settings source resets the permission load state so the next
  Permissions visit auto-loads data for the new daemon/workspace.

## Compatibility

This only changes desktop Settings UI state management. The daemon RPC contract
for `list_permissions` and `save_permissions` is unchanged.
