# puffer-desktop Update 669: Daemon Reconnect Endpoint Recovery

## Summary

Local desktop reconnect now reacquires a fresh daemon handshake before opening a
new WebSocket. This fixes the case where the native daemon restarts on a
different port and the UI previously kept retrying the stale endpoint.

Remote SSH recovery remains explicit: remote connect and workspace switch flows
continue to use their existing handshakes and do not gain background polling.

## Frontend Contract

- `DaemonClient` remains endpoint-specific. Its handshake is immutable for the
  lifetime of the instance.
- `ensureLocalDaemonClient()` reuses the current shared client when one exists.
  When it must create a client, Tauri desktop asks `ensure_local_daemon` before
  consulting browser-configured local backend URLs.
- `reacquireLocalDaemonClient()` always obtains a new local handshake, connects a
  candidate client, and publishes it as the shared client only after the
  connection succeeds.
- Failed reacquire attempts leave the previous shared client reference in place
  for diagnostics while the app connection state stays closed.
- `switchDaemonClient()` follows the same candidate-first publication rule for
  workspace and remote switches.
- The app reconnect button clears listeners from the old client, reacquires a
  local daemon client, then attaches listeners to the connected candidate so the
  open-state refresh path reloads pins, settings, groups, and active
  subscriptions.
- The workspace header exposes the existing workspace picker through a compact
  `Switch workspace` icon button wired to `onOpenWorkspacePicker`.

## Tauri Launcher Contract

- Cached spawned local daemons are reusable only when the child process is still
  running and the cached handshake port accepts a TCP probe.
- Attached local daemons still use reachable-handshake liveness because there is
  no child process owned by the launcher.
- `ensure_started()` mutably inspects the cached child so it can call
  `Child::try_wait()` directly. Exited or unreachable spawned daemons are
  discarded before the launcher starts or attaches to another local daemon.
- Singleton attachment behavior is unchanged: existing daemon handshakes are
  reused only for matching workspaces with reachable ports.

## Tests

- Playwright covers changed-port reconnect with two fake daemon endpoints and a
  Tauri `ensure_local_daemon` sequence.
- Playwright covers Tauri startup preferring native daemon acquisition over
  stale browser-configured local backend URLs.
- Existing backend reconnect tests continue to cover same-endpoint reconnect,
  session event resubscription, pending prompts, and remembered-session behavior.
- Workspace picker harnesses that mock `restart_local_daemon` also mock
  `ensure_local_daemon`.
- Rust unit tests cover reachable attached handshakes, stale ports, mismatched
  workspaces, spawned children with unreachable handshakes, and exited spawned
  children.

## Compatibility Notes

No backward-compatible stale-client fallback is retained for local desktop
reconnect. Local Tauri reacquire is authoritative over browser local backend
configuration. Browser preview still uses configured WebSocket handshakes when
Tauri invoke is unavailable. Remote SSH recovery is intentionally unchanged.
