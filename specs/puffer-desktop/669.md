# Browser CEF Crash Diagnostics

The desktop Browser pane now records a bounded debug trace for tab lifecycle
events in native CEF mode. In dev builds, or when
`localStorage.pufferBrowserDebug` is set to `1`, Browser tab activation,
local CEF tab creation, native CEF open/close calls, daemon tab close
reconciliation, stale-generation exits, and window-level JavaScript errors are
written to the console and to `localStorage.puffer-browser-debug-log`.

The Tauri native CEF command layer also logs begin/end markers for status,
open, resize, navigate, reload, history, close, and hide commands. These markers
include the session id and a compact state summary so crashes during two-tab
close flows can be localized to either the Svelte lifecycle, the Tauri command
boundary, or the native CEF bridge.

Local daemon stderr is forwarded in debug builds and when
`PUFFER_BROWSER_DEBUG=1` is set. This keeps daemon Browser diagnostics visible
in the desktop dev terminal while preserving quiet stderr handling for normal
non-debug launches.
