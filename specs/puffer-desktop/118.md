# Session-Scoped Chat File Links

## Scope

Chat messages can contain absolute local file links. Clicking one switches the
agent detail view to the Files tab and asks the Files pane to reveal that path.
That pending file-open request belongs to the session where the user clicked
the link.

## Change

- Agent detail now clears the pending linked-file path whenever the selected
  session id changes.
- A linked file opened from one session can no longer be replayed when the user
  later switches to another session and opens its Files tab.
- The regression test clicks a file link in one session, returns to Chat,
  switches sessions, opens Files, and asserts the old path is not read again.

## Compatibility

Files tabs restored from the daemon remain session-owned through the existing
`load_file_tabs` state. This change only clears the transient chat-link intent
that was stored in the agent detail component.
