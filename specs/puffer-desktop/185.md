# Remote Provider Auth Gate

## Context

The Connect Project modal rendered provider choices from the local settings
snapshot before remote SSH connection. After switching to the remote daemon it
loaded the remote settings snapshot, but provider resolution still fell back to
`openai` when the remote had no authenticated agent provider. That allowed a
remote session to be created even though the first turn would fail for missing
credentials.

## Change

- Treat an empty provider option list as an error during provider resolution.
- Use a remote-specific error message after loading the remote settings
  snapshot.
- Keep the existing remote fallback among authenticated remote providers when
  the locally selected provider is not available on that host.

## Compatibility

The daemon protocol is unchanged. Local submission is still blocked by the
existing disabled button state when no local provider is authenticated. Remote
submission now fails before `create_session` and rolls back to the previous
daemon client when the remote host lacks authenticated agent providers.

## Verification

- Added a Playwright regression that connects to a remote daemon with no
  provider auth, verifies no `create_session` request is sent, and confirms the
  UI restores the previous daemon client.
