# Permission Response In-Flight Guard

## Scope

Pending desktop approval prompts should send at most one permission response
while a previous `resolve_permission` call for the same prompt is still awaiting
the daemon response.

## Change

- `App.svelte` tracks permission ids that are currently resolving.
- Repeated choices for the same permission id return before touching the daemon.
- The in-flight marker is cleared after success or failure so failed approvals
  remain retryable.
- Session and daemon-scope resets clear the marker to avoid stale guards after
  workspace or session changes.

## Compatibility

The daemon protocol is unchanged. This only prevents duplicate renderer-origin
`resolve_permission` requests caused by rapid repeated clicks.
