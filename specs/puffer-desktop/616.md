# Automatic CEF Checkout

## Scope

This update lets `make build-release-cef` work against Chromium-only
`chromium_tintin` checkouts such as the Linux build host's
`/mnt/lvm_data/chromium_tintin`.

## Design

The release script now has `CEF_REPO` and `CEF_BRANCH` environment variables.
When `src/cef` is missing, it clones `CEF_REPO` into the Chromium checkout. If
`CEF_BRANCH` is not set, the script derives the branch from Chromium's
`chrome/VERSION` `BUILD` value, matching CEF's branch naming convention such as
`7680`.

Existing CEF checkouts are preserved. Clean checkouts on the inferred branch are
fast-forwarded; dirty, detached, or differently named checkouts are fetched only
so local fork work is not overwritten.

Linux output directory selection now prefers existing `out/Linux`,
`out/LinuxNoOzone`, `out/Release_GN_x64`, or `out/Release` directories before
falling back to `out/Release`. The script also bootstraps the checkout-local
`third_party/depot_tools` before invoking its `autoninja`, which avoids relying
on an uninitialized global depot_tools installation. The bootstrap guard checks
both `python-bin/python3` and `python3_bin_reldir.txt`, since Chromium's
`autoninja.py` requires both.

CEF project generation runs from `src/cef`, matching CEF's script assumptions.
This ensures `tools/gclient_hook.py` resolves inside the CEF checkout and
generates CEF targets before ninja runs. Generation and ninja execution also
prepend Chromium's `third_party/depot_tools` to `PATH`, which CEF's Python
helpers require.

For Linux CEF builds against an existing Chromium `out/Release` directory, the
script enforces CEF-required GN args before regeneration:
`enable_widevine = true` and `clang_use_chrome_plugins = false`. It then runs
`gn gen` for the selected output directory before invoking ninja so assertions
fail early and the CEF targets are present.

## Compatibility

The macOS checkout already includes `src/cef`, so this path is mainly for Linux
remote packaging. It keeps the Chromium checkout's existing modified files
intact and only manages the nested CEF checkout required by CEF project
generation.
