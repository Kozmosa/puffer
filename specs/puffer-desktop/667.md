# puffer-desktop Update 667: Chat Scroll-To-Bottom Button

## Summary

Desktop chat shows a floating down-arrow button when the transcript is
scrollable and the user is away from the latest message.

## Behavior

- The button is scoped to the active chat conversation.
- It is hidden when the transcript has no meaningful overflow.
- It is hidden when the transcript is within 100 px of the bottom.
- It appears when the user scrolls farther than 100 px from the bottom.
- Clicking it hides the button immediately and smooth-scrolls the thread to the
  bottom.
- Session switching clears stale button visibility, preserves the existing
  initial top scroll, then measures the selected thread after the DOM settles.

## Boundaries

- No backend, session-store, transcript DTO, provider, or daemon changes.
- No unread count, new-message pill, generic scroll abstraction, observer, or
  agent-sdk state machine port.
- Composer autosize bottom anchoring remains the existing local behavior.

## Verification

Playwright coverage should verify visible, hidden, and click-to-bottom states,
and the existing composer autosize anchoring test should continue to pass.
