# puffer-desktop Files Tab Keyboard Close

## Purpose

Files tab close buttons must work with keyboard activation, not only pointer
clicks.

## Design

The file tab container keeps Enter/Space activation for the tab itself, but it
no longer handles key events that originate from nested controls. This lets the
native close button receive its default keyboard click behavior.

## Logic

- The tab keydown handler returns early when the event target is not the tab
  element itself.
- Enter/Space on a focused file tab still activates that tab.
- Enter/Space on a focused close button closes the matching file tab.

## Contracts

Nested controls inside a `role="tab"` file tab must remain keyboard-operable.
The Files pane may not prevent default keyboard behavior for those controls.
