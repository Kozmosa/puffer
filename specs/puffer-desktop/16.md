# Chat Streaming Completion Reload Guard

## Purpose

The desktop chat transcript must not flicker or briefly remove streamed
assistant text when a turn-complete event arrives before the daemon has flushed
the final transcript to persistent session detail.

## Design

Turn completion still triggers a session detail reload, but live streamed items
are retained until the reloaded persisted timeline contains matching content.
This makes stale reloads harmless and lets a later fresh reload naturally
replace local live placeholders.

## Logic

- Before reloading session detail, the app snapshots current live stream items
  and locally-submitted user messages.
- After the reload resolves, each pending item is compared against the persisted
  timeline by kind and body text.
- Items already present in persisted history are dropped to avoid duplicates.
- Items missing from the stale reload remain in the live layer so the transcript
  does not flash empty.
- Turn-error placeholders keep the same preservation behavior, filtered through
  the same persisted-content check.

## Contracts

Streaming text must remain visible across a stale completion reload. The
Playwright desktop harness emits `turn-start`, `text-delta`, and `turn-complete`
events while the fake daemon intentionally returns the old transcript, then
asserts that the streamed assistant text stays visible.
