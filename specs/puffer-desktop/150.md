# Connect Project Provider Availability Guard

## Scope

Connect Project should only start a project when the current workspace has an
authenticated personal agent provider available. If the settings snapshot has
auth for an unrelated provider, the modal can still open, but it must not submit
with an implicit Codex/OpenAI fallback.

## Change

- Connect Project now treats an empty provider option list as a non-submittable
  state.
- The provider section shows a direct Settings hint when no usable provider is
  available.
- Playwright coverage opens Connect Project with only an unrelated authenticated
  provider, fills the local directory, and verifies Start agent stays disabled
  and no `create_session` request is sent.

## Compatibility

Authenticated Codex/OpenAI, Anthropic/Claude, and Puffer provider flows are
unchanged. The guard only affects workspaces that have no usable personal agent
provider in the current settings snapshot.
