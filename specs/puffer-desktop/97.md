# Sidebar Filter Self-Healing

## Scope

The desktop sidebar's Active Agents filters should not strand the user on an
empty list after workspace/session refreshes remove the currently selected
project.

## Change

- The project filter remains local UI state while the selected project still
  exists in the active agent list.
- If a refresh removes that project from the available filter options, the
  sidebar resets the project filter to `all`.
- The state filter is guarded the same way, so invalid persisted or future
  values also fall back to `all`.

## Compatibility

This does not alter daemon session grouping or pin behavior. It only keeps the
renderer's filter state consistent with the currently available sidebar rows.
