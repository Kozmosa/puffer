# Custom-Only User Questions

## Scope

Desktop question prompts must handle user-question requests that provide a question but no predefined options.

## Behavior

- Question normalization keeps valid question records even when their option list is empty.
- The prompt displays the question text and custom-answer input.
- `Send answer` stays disabled until the user enters a non-empty custom answer.
- The daemon receives the typed answer under the original question text.

## Compatibility

Option-based single-select and multi-select questions keep the existing behavior. This only removes the normalization filter that converted custom-only questions into empty prompt payloads.
