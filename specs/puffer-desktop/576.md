# Telegram Connector Setup Feedback

## Change

The desktop Settings connector setup flow now gives Telegram-specific progress
and failure feedback while `/connect telegram-login ...` is running.

## Contract

- Telegram setup questions explain whether the UI is waiting for method, QR,
  phone, code, or 2FA password input.
- Missing, unreadable, and startup-failed subscriber manifest cases are rewritten
  into user-facing guidance that points at the new `[telegram-connect]` daemon logs.
- Non-Telegram connector setup messages keep the existing generic behavior.

## Compatibility

The UI still dispatches the same daemon slash command and uses the existing
AskUserQuestion event contract. This change only affects visible copy and error
presentation for Telegram setup.
