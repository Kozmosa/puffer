# MCP Add Server Edit Lock

## Scope

Adding an MCP server is a daemon round trip that clears the add-server form on
success. Users must not be able to edit that form while the save is in flight,
because those edits can be discarded by the delayed success response.

## Change

- The MCP add-server form now disables ID, name, transport, scope, command/URL,
  arguments, and description fields while MCP servers are loading, saving, or the
  daemon is unreachable.
- The Add server button uses the same disabled predicate as the form fields.
- A Playwright regression test delays `add_mcp_server` and verifies every form
  control is locked while the save request is pending.

## Compatibility

The MCP daemon protocol is unchanged. Successful add behavior still refreshes the
server list and clears the form after the daemon confirms the new server.
