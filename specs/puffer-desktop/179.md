# Desktop Settled Turn Sidebar State

## Context

The desktop shell preserves live sidebar overlays for agents that keep running
after the user opens a different session. A fast `turn-complete` event followed
by a delayed `run_agent_turn` response could lose the settled marker during the
session switch, then revive the old sidebar row as `thinking`.

## Change

- Track settled turn ids by `(session id, turn id)` instead of only the
  currently opened detail view.
- Keep that settled-turn memory across detail-session switches.
- Skip live sidebar resurrection when a delayed start response belongs to an
  already-settled turn.

## Compatibility

Running turns still stay visible in the sidebar after switching sessions. The
new guard only suppresses stale start responses for turns that already emitted
`turn-complete` or `turn-error`.

## Verification

- Added a Playwright regression that completes one session before its
  `run_agent_turn` RPC returns, opens another session, then verifies the first
  sidebar row remains idle after the delayed RPC response arrives.
