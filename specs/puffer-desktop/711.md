# Contact Save Selection Normalization

## Design

After saving a contact, the contacts screen compares user-entered contact IDs
with backend-returned saved contact IDs using the same canonical shape for known
contact prefixes. This keeps the newly saved contact selected when the backend
normalizes case or redundant `@` characters.

## Compatibility

The save request payload is unchanged. The frontend still sends the user's
trimmed IDs to the daemon, and only the local post-save selection lookup uses
the canonical comparison key.
