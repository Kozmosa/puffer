# Verified Skills Folder Readiness Labels

The Desktop Verified Skills settings page treats a folder as ready only when the
daemon reports precompiled host catalogue configuration. It no longer marks a
folder ready because a compiler path exists in an older manifest.

Users still add a folder through the normal folder picker. Recognized skills are
listed with per-skill enable switches, and source-only or compiler-only folders
remain visible with a "Needs verification output" status until generated host
catalogues are present.

Validation:
- `npm run check`
