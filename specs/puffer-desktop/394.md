# 394. PDF Preview Control Contrast

PDF previews keep the zoom controls and page-limit status in a fixed,
high-contrast toolbar independent of the active document background. The
toolbar uses a dark control rail in both light and dark themes, while the
`Showing first N of M pages.` status uses an amber badge so it cannot blend
into the surrounding preview.

The PDF shell uses responsive padding so narrow Files panes leave enough hit
area for the zoom buttons and range input after the project sidebar or file
tree has been resized. The existing zoom behavior remains immediate: buttons,
the range input, keyboard shortcuts, control-wheel, and gesture zoom all update
the already-rendered canvases without re-rendering the PDF.

Regression coverage asserts that long PDF previews keep the limit badge
readable, that zoom controls receive pointer events in compact layouts, and
that the canvas width changes after zooming.
