# Browser blank-canvas recovery

## Change

- BrowserPane now reconnects the active daemon tab when a tab-list event arrives for the same connected tab but the canvas has no recorded frame yet.
- The reconnect path resubscribes to frame/state/devtools streams and issues a resize, which prompts the daemon-backed browser worker to restart screencast delivery.

## Compatibility

- Existing active-tab switches, disconnected-tab recovery, and empty-tab clearing keep their current behavior.
- The condition only adds recovery work when the visible tab is blank, so normal connected tabs with a frame do not churn subscriptions.

## Tests

- Added a Playwright regression where `browser_open` intentionally sends no frame, a later agent-style tab event keeps the same active tab, and `browser_resize` delivers the first frame into the canvas.
