# Empty provider setup fallback

- Onboarding and Settings no longer dead-end when the daemon returns an empty provider registry.
- The renderer shows built-in setup cards for OpenAI, Anthropic, and OpenRouter so a first-time user can connect credentials or refresh resources.
- Connected fallback providers count as agent-ready for onboarding and default-route selection until the daemon provider registry reloads.
- New-session and connect-project provider choices share the same fallback catalog, keeping provider recovery behavior consistent across entry points.
