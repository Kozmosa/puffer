# Canceled Session Reopen State

## Change

- Reopening a session after `cancel_turn` no longer revives stale local turn runtime when the daemon reports the session as idle.
- The desktop shell clears cached `currentTurnId`, cancel state, pending live stream items, and optimistic submitted messages for canceled turns that are no longer active upstream.

## Compatibility

- Active turns still keep their live sidebar state while another session is selected.
- Normal completed-turn transcript reloads continue to deduplicate optimistic chat rows against persisted transcript items.
