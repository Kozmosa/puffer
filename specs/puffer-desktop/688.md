# Native CEF Agent Slot Handoff

- Desktop CEF prewarm pages now load `about:blank` with a `puffer-cef-slot`
  fragment that identifies the native slot behind each DevTools target.
- The daemon preserves that slot id on the Chrome target, browser session, and
  tab metadata as `nativeCefSessionId`.
- BrowserPane keeps backend session ids for BrowserAction/CDP routing, but uses
  `nativeCefSessionId` for native CEF open, resize, navigation, history, reload,
  hide, and state calls when the daemon controls a prewarmed target.
- Closing or demoting daemon-owned CEF tabs hides the native slot and lets the
  daemon release the CDP target instead of closing the visible native browser
  directly.
- Existing Chrome fallback and non-CEF tab metadata remain unchanged because the
  native slot field is optional and omitted when no desktop CEF slot is known.
