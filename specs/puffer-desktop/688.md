# Video Settings Multi-Model Capability Rendering

Date: 2026-06-09

## Scope

Desktop video settings tests now prove the existing capability-driven modal can
render multiple video models for one provider without provider-specific UI.

## Behavior

- A single available Relaydance provider renders as a read-only provider field.
- Multiple Relaydance video capabilities render a model selector.
- Resolution-specific single-value video parameters render as read-only fields.
- Saving after switching models persists the selected model id, adapter, and
  normalized parameter defaults.

## Compatibility

No Svelte implementation change is required; the modal remains a pure consumer
of `list_media_capabilities(kind = video)`.
