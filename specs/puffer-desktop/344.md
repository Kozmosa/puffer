# 344. Pipeline Refresh Busy Guard

Pipeline refresh controls must not allow concurrent workflow snapshot loads.

Contracts:

- The Pipelines `Refresh` button is disabled while a workflow snapshot request
  is in flight.
- The button exposes `aria-busy="true"` during the load and returns to
  `aria-busy="false"` after the request resolves or fails.
- The loading guard prevents repeated clicks from issuing duplicate
  `workflow_list` requests.

Playwright coverage delays the fake daemon `workflow_list` response, opens
Pipelines, checks the busy disabled state, waits for recovery, and verifies only
one workflow snapshot request was sent.
