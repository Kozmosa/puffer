# Real Daemon Provider Alias Smoke Coverage

## Summary

Desktop real-daemon UI tests now cover Codex and Claude provider aliases when
they are stored in config but resolved through canonical OpenAI and Anthropic
daemon providers.

## Design

`real-daemon-ui.spec.ts` starts the actual `puffer daemon` with alias defaults
(`codex` and `claude`) while auth and provider resources are canonical. The UI
opens the new-agent modal, creates a session, sends a prompt, and asserts the
provider mock receives exactly one request and renders the streamed reply.

## Contracts

- Alias defaults must not leave the composer in a reconnect-blocked state.
- The new-agent modal must resolve alias defaults to a usable authenticated
  provider option.
- Sending from the created session must reach the canonical provider backend.

## Verification

- `npm run test:desktop-ui -- tests/real-daemon-ui.spec.ts -g "real daemon UI can create a .* alias agent and render a reply"`
