# Puffer Desktop Update 633

## Summary

Linux CEF release builds now restore the Browser and BrowserView hooks needed
by CEF's hosted Chrome views path.

## Design

- `scripts/release.sh` adds a BrowserView compatibility patcher to the CEF
  release patch chain.
- BrowserView can be constructed before a Browser exists, then initialized via
  `InitBrowser` when CEF wires the hosted browser view.
- BrowserView exposes CEF's toolbar override and teardown hooks, including a
  guarded browser-window deletion path for externally owned widgets.
- Browser exposes `ReleaseBrowserWindow` so CEF can transfer hosted view
  ownership back to the embedding view hierarchy during native widget teardown.

## Compatibility

The normal Chromium browser window path still constructs BrowserView with a
Browser and runs initialization immediately. The CEF hosted path can defer that
initialization until the CEF widget and Browser are both available.
