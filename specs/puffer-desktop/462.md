# Terminal Closing Input Guard

## Summary

The Terminal pane now suppresses keyboard input for a PTY while that PTY is in
the middle of a close request.

## Behavior

- Closing a terminal tab marks that PTY as closing until the close request
  resolves.
- Input events from the attached xterm instance are ignored while its PTY is
  closing.
- A failed close keeps the tab retryable and restores normal input once the
  closing marker is cleared.
- Closing an inactive tab does not block input to the active terminal.

## Compatibility

The daemon PTY RPC API is unchanged. This only prevents stale `pty_write`
requests from the desktop UI during an in-flight `pty_close`.
