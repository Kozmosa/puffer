# 346. Browser Error Address Lock

Browser tabs with a Chrome/CDP error must not leave the URL field editable.
The rest of the disconnected-browser behavior remains unchanged: tabs that are
temporarily reconnecting but have no error keep their address draft editable.

## Contract

- The URL input is disabled when the active browser tab has an error.
- Pending navigation commands still disable the URL input until the command
  settles.
- Disconnected tabs without an error keep the URL input enabled so users can
  prepare or edit the next navigation during reconnect.

## Coverage

The existing desktop Playwright coverage now exercises both sides: the browser
state-error test expects the URL input to disable, while the reconnecting-tab
test expects the address field to stay editable without an error.
