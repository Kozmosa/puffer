# Browser recording and error-state UI

- Root browser recording events may introduce a previously unknown tab. The browser pane now creates that tab, selects it, paints the recorded frame, and persists the tab metadata instead of discarding the frame.
- Browser error states keep the address bar editable for recovery while disabling navigation controls and canvas input until the tab reconnects.
- Desktop UI fake-daemon fixtures include the `browser_backend_status` response so browser regression tests exercise renderer fallback state without surfacing fixture errors.
- The desktop Vite dev server constrains dependency scanning to `index.html`, preventing generated Tauri/CEF HTML under `src-tauri/target` from becoming test entries after local CEF builds.
- Playwright dev dependencies are pinned to one `1.59.1` runner version so `npm run test:desktop-ui` uses the same framework instance as the tests import.
