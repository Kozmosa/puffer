# Desktop longhunt UI race fixes

This update closes the accepted longhunt R3 desktop regressions across the
browser pane, chat turn lifecycle, permissions/questions, files, settings, and
daemon event transport.

## Browser pane

- Browser address input remains editable after state-error events so users can
  recover from failed navigation.
- Live recording events no longer create or activate unknown stale tabs.
- Successful local tab open/close responses are not discarded when racing
  tab-list pushes update local version counters.
- Recent local tab opens and closes are protected from stale list pushes for a
  short grace window.

## Chat and prompts

- Workspace `turn_complete` and `turn_error` events immediately clear the
  active session's running state before transcript reload finishes.
- Pending permission approvals are disabled while a turn is no longer
  cancelable, including after Stop is requested.
- User-question answers use disambiguated keys when multiple visible questions
  share the same text.

## Files and settings

- File save success only resets a draft when the draft still matches the saved
  snapshot; edits typed while save is in flight remain visible and dirty.
- Provider credential success only navigates to Workspace when completing
  onboarding. Settings > Providers stays on the settings screen.

## Daemon event transport

- The daemon supports explicit `subscribe_event` and `unsubscribe_event` RPCs.
- High-volume `browser:*:frame` and `browser:*:recording` events are forwarded
  only to subscribed websocket clients.
- The desktop daemon client replays active event subscriptions after websocket
  reconnect, preserving terminal PTY output and other subscribed streams.

## Regression coverage

- `tests/longhunt-ui-regressions.spec.ts` covers browser state-error recovery,
  stale recording frames, tab-list races, duplicate question text, workspace
  completion clearing, Stop/permission disabling, in-flight file save edits,
  and settings provider credential navigation.
- `puffer-cli` daemon unit coverage verifies that high-volume browser events
  require explicit subscription.
