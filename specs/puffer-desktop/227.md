# 227. Replace Stale Catalog Models Before Submit

## Summary

The desktop composer now replaces stale provider catalog models after live model
discovery returns an authoritative list for the current provider.

## Behavior

- If the current model is present in the live provider model list, it remains
  selected.
- If the current model is missing and looks like a custom model id, such as an
  OpenAI fine-tune id, it remains selected so users can run unadvertised custom
  models.
- If the current model is missing and is a normal catalog model id, the composer
  switches to the provider's live default model or first live model before a turn
  can be submitted.

## Compatibility

Provider/model payloads sent to `run_agent_turn` are still selected in the UI
before submission. No daemon protocol changes are required.

## Tests

`chat-session-ui.spec.ts` covers an OpenAI session restored with stale `gpt-5`
while live discovery only returns `gpt-5.4`, and asserts the submitted turn uses
`gpt-5.4`.
