# Files Watch Reroot Teardown

## Context

Switching agent sessions changes the Files pane root and rebuilds the recursive
filesystem watch. The old watch must be released, but releasing it is a daemon
RPC that can be delayed if the backend is busy or disconnecting.

The pane previously awaited `fs_unwatch` before starting the new root watch. A
slow stale unwatch could leave the new session temporarily unwatched, dropping
the first filesystem changes after a fast session switch.

## Change

- Tear down the old Files watch locally first by removing the event listener and
  clearing the current watch id.
- Release the old daemon watch as a best-effort background `fs_unwatch` request.
- Start the new root watch immediately, preserving the client-generated watch
  id behavior for early events while the new `fs_watch` response is pending.

## Compatibility

The `fs_unwatch` RPC remains best-effort and idempotent. Stale events from the
old watch are still ignored because the old listener is removed and the current
watch id is cleared before the new watch is installed.

## Verification

- Added a Playwright regression that delays `fs_unwatch`, switches Files roots,
  and verifies the next `fs_watch` is sent without waiting for the old unwatch
  response.
