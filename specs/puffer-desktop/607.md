# Connector setup loading modal

Connector setup questions now keep their modal open after answers are submitted
until the underlying connector flow completes, fails, or emits the next setup
question. The submitted question request is no longer cleared immediately after
`resolve_user_question` returns.

While the daemon is checking auth or performing connector work, the modal body
switches to a loading state. A subsequent `user-question-request` replaces the
loading state with the new questions, while `turn-complete` and `turn-error`
retain the existing completion and failure handling.
