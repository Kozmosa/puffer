# Desktop provider alias canonicalization

Desktop creation flows canonicalize user-facing provider aliases before sending
session creation requests to the daemon.

Compatibility contract:

- `codex` is accepted as a UI/auth alias but `create_session` receives
  `openai`.
- `claude` is accepted as a UI/auth alias but `create_session` receives
  `anthropic`.
- Canonical provider ids such as `openai`, `anthropic`, and `puffer` pass
  through unchanged.
- New Agent and Connect Project use the same boundary normalization.
- Existing provider labels and filtering behavior stay unchanged.

Regression coverage lives in `files-workspace-ui.spec.ts`.
