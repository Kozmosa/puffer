# New Agent Create Guard

## Scope

The desktop New Agent modal creates a session in an existing workspace. Repeated
activation of Start agent while the first create request is pending must not
create duplicate sessions or duplicate agent rows.

## Change

- The parent modal callback now returns immediately when a new-agent create is
  already in flight.
- A Playwright regression test delays `create_session`, double-activates Start
  agent, and asserts the daemon records exactly one create request.

## Compatibility

Normal provider selection and first-turn behavior are unchanged. The guard only
blocks repeated submit callbacks that can arrive before the disabled button
state has rendered.
