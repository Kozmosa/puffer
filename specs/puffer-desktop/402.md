# 402. Query-Specific Browser Recording Matching

Browser tool recordings are scoped by the action URL query/hash when the action
provided one. Host and path compatibility still allow `www.` and redirected
subpath matches, but two actions on the same path with different query strings
no longer share the same recorded frame.

Both activity detail rendering paths use the same rule so URL-only browser
actions such as `/results?q=alpha` and `/results?q=beta` show their own
screenshots instead of whichever frame happened to be most recent.
