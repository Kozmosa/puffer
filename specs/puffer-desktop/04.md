# Browser Empty Tab State

## Scope

The renderer must handle daemon tab-list events that intentionally remove every
managed Browser tab.

## Change

- Initial daemon tab sync still ignores an empty list so local saved tabs remain
  a migration fallback.
- Real-time daemon tab events and close responses now opt into empty-list
  handling.
- Empty tab handling clears local tabs, active tab state, canvas subscriptions,
  and status text.

## Compatibility

This preserves startup fallback behavior while fixing stale UI after the daemon
or another client closes the final Browser tab.
