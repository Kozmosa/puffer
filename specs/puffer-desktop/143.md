# Global Provider Auth Busy Lock

## Scope

Provider login is a global single-flight operation in the desktop shell. While
one provider is opening OAuth or submitting an API key, other provider cards
must not appear interactive because their clicks are ignored by the app-level
guard.

## Change

- LoginView now disables all provider OAuth buttons, API key fields, and API key
  connect buttons while any provider login is in flight.
- The in-flight provider still shows its provider-specific busy label, while
  other provider cards remain visually disabled without starting extra requests.
- A Playwright regression test delays Codex OAuth and verifies Anthropic API key
  controls are disabled during that pending login.

## Compatibility

The daemon protocol and login request payloads are unchanged. This only aligns
the settings UI affordance with the existing app-level single-flight behavior.
