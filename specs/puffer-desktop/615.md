# Staged CEF Build Runtime

## Scope

This update makes macOS release builds independent from a developer's
`~/chromium_tintin` checkout after the CEF runtime has been downloaded from the
GitHub release.

## Design

`make build-tauri`, `make build-macos`, and `make pack-macos` now stage the
selected CEF runtime at `apps/puffer-desktop/src-tauri/target/puffer-cef-runtime`
before invoking Tauri. The path is a symlink to either the downloaded release
runtime or the local fallback runtime.

The Tauri build script checks that staged path in addition to explicit
`PUFFER_CEF_PATH`, `PUFFER_CEF_ROOT`, and `CEF_PATH` values. It canonicalizes
runtime roots before walking to the packaged `cef/` headers and `libcef_dll`
wrapper sources, so a symlinked `Release` directory still resolves to the
self-contained downloaded release package.

The CEF release package prefers Chromium's generated `output/cef_binary_*`
directory for headers and `libcef_dll` wrapper sources when it is available.
That generated package contains `include/cef_config.h`, which the Tauri build
script requires before enabling native CEF.

## Compatibility

The staged path lives under `target/`, so it is not committed. The local
`~/chromium_tintin` fallback remains available for bootstrapping the first CEF
release artifact, but downloaded CEF releases are now sufficient for subsequent
macOS Tauri builds.
