# ToolCard Documentation docs

## Design

The ToolCard Storybook catalog now includes a `Documentation` story that models
a documentation-oriented tool call and an MDX docs page at
`Agent/ToolCard/Documentation`. The fixture uses a document path, title,
section, and structured result so the existing generic structured renderer can
show documentation metadata and excerpt content without adding a bespoke
component branch.

The MDX page mirrors the existing `Design/Buttons/Documentation` docs pattern:
it imports Storybook docs blocks, references existing ToolCard stories through
`Canvas`, and documents structure, renderers, behavior, and accessibility
expectations. The docs page lists every current `Agent/ToolCard` story, grouped
by file/terminal, search/source lookup, web/browser, interaction/task,
connector/workflow, media/app actions, MCP, and lifecycle states.

Each ToolCard story exposes a Storybook Controls-only `prompt` arg. The prompt
is a copyable chat instruction for manually reproducing the same ToolCard in
the conversation UI during visual testing. A small story harness owns that
extra arg so the runtime `ToolCard` component continues to receive only
`item`, `sessionId`, and `defaultCollapsed`.

The docs page renders that same prompt before every ToolCard canvas. This keeps
the documentation catalog usable as a chat UI test checklist without duplicating
prompt text outside the story args. Prompt text renders at 12px and each prompt
block includes a copy button with clipboard and textarea fallback behavior.

## Compatibility

Runtime transcript contracts are unchanged. The ToolCard icon classifier now
treats documentation-style tool names as file-like so these rows use the shared
file icon rather than the fallback bolt.

Storybook docs pages opt out of the desktop app root scroll lock through a
preview-only CSS override. The production app keeps `html`, `body`, and `#app`
fixed at full height with hidden overflow, while docs pages with
`#storybook-docs` restore auto height and native overflow so long MDX catalogs
remain scrollable inside the preview iframe.

## Tests

The story and docs imports are covered by Storybook rendering and TypeScript
compilation through the desktop app's existing `npm run check` path. The
Storybook static build also covers the MDX docs entry and preview CSS import.
Agent-browser verification confirmed that the ToolCard docs iframe exposes
auto overflow with a scroll height larger than the viewport and renders all 32
current ToolCard stories. Storybook controls verification confirmed that a
story such as `ReadFile` exposes the expected prompt text in the addon panel.
The ToolCard docs page was also verified to render 33 prompt blocks for 33
ToolCard canvases, including the duplicated top-level Bash example. The first
prompt block was verified to compute to 12px text and switch its copy button to
`Copied` after activation.
