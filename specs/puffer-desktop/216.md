# Remembered Session Detail Fallback

## Problem

The desktop shell restored the remembered last session only when that session
was present in the currently loaded grouped history. A filtered, paginated, or
temporarily stale grouped list could therefore make a valid remembered session
look missing and leave the user on an unrelated session or the empty workspace.

## Design

- Keep the grouped-history fast path unchanged when the remembered session is
  already visible.
- If the grouped list does not contain the remembered id, call
  `load_session_detail` directly by id.
- Adopt the returned session detail as the active agent and register it as a
  sidebar fallback so the user can still see and navigate the restored session.
- If the direct detail load fails, return to the normal startup fallback flow
  without clearing the remembered preference.

## Test Coverage

- `settings-ui.spec.ts` now hides the remembered session from
  `list_grouped_sessions`, seeds the remembered-session preference, and asserts
  the desktop still opens the agent detail through `load_session_detail`.
