# Daemon-Running Session Events

## Scope

Desktop must keep daemon-reported active sessions wired to their live event streams, even when the current renderer did not start those turns.

## Behavior

- Sidebar sessions with daemon `activityStatus` of `running` or `awaiting` are subscribed for session events.
- Existing local live overlays remain subscription targets.
- Background permission and user-question events from daemon-running sessions can update the sidebar state and are cached for when the user opens that session.
- Idle sessions are not subscribed unless they have a local live overlay.

## Compatibility

This preserves the existing selected-session subscription path and only broadens background subscriptions to sessions the daemon already reports as active.
