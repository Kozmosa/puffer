# Prompt Resolution Busy Controls

## Summary

Permission approvals and user-question prompts now receive their resolve-in-flight
state from the app shell and render their controls disabled until the backend
response returns.

## Behavior

- `App.svelte` keeps the existing session-scoped resolve guards and passes the
  active scoped ids through `AgentDetail`, `AgentDetailContent`, and
  `ConversationView`.
- `ConversationView` maps each pending prompt to the current session id before
  deciding whether it is resolving, matching the dismiss/retry scoping used by
  the app shell.
- `Approval.svelte` disables all choices while the selected response is being
  delivered.
- `QuestionPrompt.svelte` disables options, the custom-answer input, and the
  send button while the answer is being delivered.

## Compatibility

Backend request semantics are unchanged. Failed permission or question
responses keep the prompt visible and retryable after the in-flight state is
cleared.

## Tests

`chat-session-ui.spec.ts` covers delayed permission and question responses,
asserting that controls disable during submission, suppress duplicate attempts,
and re-enable after failures.
