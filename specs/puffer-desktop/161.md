# Final-Only Completion Fallback

## Summary

Desktop chat now renders assistant text from a `turn-complete` event
immediately, even when no `text-delta` event streamed before completion.

## Design

When a turn completes, `App.svelte` already builds a fallback assistant item
from `assistantText`. That fallback is now written to `liveStreamItems` before
the delayed transcript reload starts. The reload still reconciles live rows with
persisted transcript items afterward.

## Contracts

- Final-only assistant text appears before `load_session_detail` returns.
- Once visible, the final-only assistant row does not disappear during the
  transcript reload window.
- Persisted transcript reload continues to replace the live fallback with the
  durable assistant message.
- Streamed assistant rows keep their existing no-flash behavior.
