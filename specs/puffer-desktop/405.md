# 405. Provider Switch Load Failure Rollback

The chat model picker now treats a provider switch as tentative until that
provider's model list loads successfully. While the load is pending, the
composer can show the pending provider state, but a daemon, network, or provider
failure rolls the picker back to the prior provider and model.

This prevents a usable chat session from getting stuck in a half-switched
`provider + empty model` state with a disabled send button. Providers that load
successfully with an empty model catalog still remain selected so the user can
see that the provider has no available agent-capable models.
