# Agent Provider Filtering

## Scope

Desktop settings can contain credentials for integrations that are not usable
agent runtimes, such as GitHub OAuth. Chat provider switching and empty-session
recovery must not treat those credentials as a way to run `run_agent_turn`.

## Change

- Provider id utilities now expose a shared agent-provider predicate for Codex,
  OpenAI, Claude, Anthropic, and Puffer aliases.
- New Agent, Connect Project, the chat model picker, composer recovery gating,
  and the app-level submit guard all use the same agent-provider filter.
- Playwright coverage verifies GitHub is not offered in the chat model picker
  and that a disconnected empty session remains blocked when only GitHub is
  authenticated.

## Compatibility

Authenticated Codex/OpenAI, Claude/Anthropic, and Puffer flows keep their
existing alias behavior. Non-agent credentials remain visible in Settings but
are no longer selectable for agent turns.
