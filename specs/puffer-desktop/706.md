# Contact Proposal Save Refresh

## Summary

The desktop Contacts regression daemon now mirrors backend proposal pruning
when a proposal is saved as a contact.

## Design

- The fake daemon removes stored proposals whose contact ids overlap the saved
  contact ids in `contacts_save`.
- The Contacts browser test covers the full modal flow: infer a proposal, use
  it, save the contact, reopen inference, and confirm the saved proposal no
  longer appears.

## Compatibility

The visible Contacts UI contract is unchanged. The refreshed snapshot from the
daemon remains the source of truth for proposal rows.
