# Permission response switch cleanup

- A successful `resolve_permission` response applies to the session that owned the approval, even if the user switched away before the daemon replied.
- The session-scoped dismissed permission id is recorded before checking the currently selected session, so reopening the original session cannot show a stale approval card.
- Cached transient state for that session drops the resolved permission item and clears its awaiting-approval hint while preserving the active turn state.
- Delayed permission failures remain scoped to the original response path and still leave the approval retryable when appropriate.
