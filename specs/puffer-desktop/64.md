# Logged-out provider turn blocking

Historical sessions keep their original provider and model routing, but the
desktop composer must not start a new turn for a provider that is no longer
authenticated in the current workspace.

Compatibility contract:

- Existing transcript history remains readable after a provider is signed out.
- Composer input, model controls, and the send action are blocked when the
  selected session routes to a signed-out provider.
- The app-level submit path also rejects stale provider routing before issuing
  `run_agent_turn`, so keyboard shortcuts or future composer variants cannot
  bypass the guard.
- Sessions can continue normally once the provider is reconnected.

Regression coverage lives in `chat-session-ui.spec.ts`.
