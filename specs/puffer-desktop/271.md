# Codex Playwright Server Isolation

Desktop UI automation should fail fast on stale dev-server state instead of
silently reusing a Vite process left behind by an interrupted run.

- Playwright still reuses an existing local server for ordinary manual runs.
- `CI` and `CODEX_CI` runs now both start their own configured web server, so a
  stale server on port 1420 is reported as an environment problem rather than
  producing misleading daemon/onboarding failures against old frontend state.
- Static regression coverage keeps the Codex automation guard in
  `playwright.config.ts`.
