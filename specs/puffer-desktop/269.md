# Resilient PDF and Legacy Word Previews

The Files pane must keep document content readable even when the embedded PDF
renderer is slow or unavailable, and old Word-family files should not fall back
to the generic binary message.

- PDF previews now show the extracted text fallback while pdf.js is still
  loading, not only after a hard renderer failure.
- Frontend PDF text extraction is bounded to the first scan window and a fixed
  number of streams so large PDFs cannot block the preview before the renderer
  mounts.
- Legacy Word preview routing now includes `.dot` templates and standalone
  `.rtf` files in addition to `.doc`; desktop reads also ask macOS `textutil`
  for native text previews for those old Word-family formats.

Regression coverage delays the PDF renderer asset and expects the text fallback
to appear immediately, while the common document preview flow now opens `.dot`
and `.rtf` fixtures as Legacy Word previews.
