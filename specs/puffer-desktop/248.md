# Browser Recording Fallback

## Summary

The desktop Browser pane now treats recorded browser frames as recoverable tab
state when the daemon tab list is empty. This prevents sessions with persisted
Browser actions from opening a fresh `about:blank` tab when the tab list event
was missed or the daemon has not replayed tab metadata yet.

## Behavior

- `BrowserPane` still prefers daemon tab state when `browser_agent list` returns
  one or more tabs.
- If the only local state is the default disconnected `tab-1/about:blank`
  fallback, the pane reads `browser_recording` for the agent root session.
- The latest recorded frame per tab is converted into a Browser tab using the
  recorded root session, tab id, backend session id, URL, title, and frame data.
- The most recently recorded tab becomes active and connects through its
  recorded backend session id, so toolbar and canvas interactions continue to
  target the agent-created browser tab.

## Compatibility

Saved non-blank local tabs are not overwritten by recordings. Empty recordings
retain the previous local fallback behavior.
