# Contacts Snapshot Refresh Coverage

The Contacts screen refresh path now requests the normal 80-item contact
snapshot instead of a one-candidate snapshot. Saved contacts are still filtered
client-side in the redesigned list/detail view, while the daemon response keeps
the broader candidate set available for inference and future contact controls.

The Contacts Playwright coverage now targets the modal-based flow. It verifies
that closing the detail pane stays dismissed until a contact is selected again,
and that creating a contact through the dialog selects the saved record returned
by the daemon.
