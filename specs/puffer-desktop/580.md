# Task Monitor Model Configuration

- The Tasks configuration dialog now includes a monitor agent model field.
- Creating or updating a monitor sends `model` through `task_monitor_create`; blank input clears the explicit model and inherits the daemon default.
- Existing monitor bindings show their current model from `workflow_bindings[].model` in both the edit form and filter-rule details.
- The model input uses known provider model selectors as datalist suggestions when discovery succeeds, while still accepting a manually typed provider/model selector.
- Model discovery reuses the settings provider/auth snapshot and `list_provider_models`, filtering to agent-capable providers and models.
