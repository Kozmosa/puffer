# Background Completion Text Preservation

## Context

Background sessions can finish while the user is viewing another session. The
completion event may carry final assistant text before the transcript reloads
or persists that message.

## Change

- Background `turn-complete` events now cache the final assistant text in the
  session's transient conversation state.
- Settled background completions clear pending approval and question prompts
  while preserving useful assistant text and tool activity until persistence
  catches up.
- Replay events for a settled background turn remain ignored, so stale prompts
  do not reappear after completion.

## Compatibility

Selected-session completion handling is unchanged. Persisted transcript reloads
still replace transient completion text once the daemon records the completed
turn.
