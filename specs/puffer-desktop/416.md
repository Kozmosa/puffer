# Settled Background Start Responses

## Scope

Desktop sessions can finish a turn before the delayed `run_agent_turn` RPC response returns, while the user has already opened another session.

## Behavior

- If the delayed RPC returns for a turn already marked settled, the background session is not cached as running again.
- Reopening that session keeps the composer enabled and does not show `Stop turn`.
- Unsettled delayed starts still cache a background running state so the sidebar and reopened session reflect the active turn.

## Compatibility

This preserves existing optimistic send behavior and only skips the stale cache write when the turn completion or error has already been observed.
