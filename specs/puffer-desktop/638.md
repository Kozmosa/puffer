# CEF Originating Process Network Observer Patch

## Summary

The release pipeline now patches CEF network-service code that still includes
Chromium's removed `originating_process.h` helper.

## Design

Chromium 146 exposes browser-process initiated URL-loader ownership through
`network::mojom::kBrowserProcessId`. CEF 7680 still calls
`network::OriginatingProcess::browser()` when creating URL loader network
observers for browser-owned requests. The release patcher now rewrites both
CEF call sites in `browser_urlrequest_impl.cc` and
`resource_request_handler_wrapper.cc` to include
`network_context.mojom.h` and pass `network::mojom::kBrowserProcessId`.

## Compatibility

The patch is release-time only and idempotent. It is limited to CEF's
network-service browser-process observer setup, preserving the existing frame
and navigation observer paths.

## Verification

Linux CEF packaging should progress past `browser_urlrequest_impl.cc` and the
matching resource request handler compile after this compatibility rewrite.
