# 298. Browser Tab Close Race with Navigation Error Handler

## Summary

The `submitUrl` error handler in `BrowserPane.svelte` now checks whether the
target tab still exists before applying error state, preventing stale navigation
errors from appearing on the pane after a tab has been closed.

## Behavior

- If a navigation is in progress and the user closes the tab before the
  navigation response arrives, the error handler now bails out silently.
- The `closeTab` handler already applies the authoritative tab state from the
  daemon; the navigation error handler should not override it.

## Compatibility

No protocol changes. Pure frontend race condition fix.

## Tests

`browser-pane.spec.ts` verifies that closing a tab during a pending navigation
does not leave an error message on the pane.
