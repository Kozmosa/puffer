# Sidebar Live Agent State

## Scope

The desktop sidebar's Active Agents list should reflect the live state of the
currently opened session, not only the last activity status returned by grouped
session metadata.

## Change

- Sidebar agent mapping now overlays the selected session's pending approvals,
  questions, and in-flight turn state onto the daemon metadata.
- A Playwright regression test starts a turn while `run_agent_turn` is delayed
  and asserts the selected sidebar row shows `thinking`.

## Compatibility

Non-selected sessions still use grouped session activity metadata. The live
override is limited to the selected session because that is the only session
with authoritative in-memory turn state in the desktop shell.
