# Turn Error Prompt Preservation

## Scope

When a desktop agent turn starts and then fails, the chat should preserve the
prompt the user submitted even if the daemon reload does not yet include that
user transcript event.

## Change

- Turn-error transcript reloads now reconcile submitted user rows against the
  persisted transcript just like successful turn reloads.
- If the prompt has been persisted, the optimistic row is removed.
- If the prompt is missing from the refreshed transcript, the optimistic row
  remains visible alongside the inline error.

## Compatibility

This keeps the persisted transcript canonical while preventing a provider or
daemon failure from making the user's sent prompt disappear from the UI.
