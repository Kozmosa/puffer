# Empty session provider recovery

Empty Desktop agent sessions no longer lock the composer when their saved
provider is disconnected while another provider is authenticated.

Before a conversation starts, the model picker remains usable even if the
session's current provider is not authenticated. The composer also accepts a
draft so users can type while switching to a connected provider. Sending still
requires the selected provider to be authenticated.

This preserves provider immutability for sessions that already have transcript
history while preventing a blank session from becoming impossible to use after a
stale provider choice or auth-state race.
