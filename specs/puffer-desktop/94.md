# Workspace refresh freshness

Workspace session-group refreshes are now generation-gated. If an older
`list_grouped_sessions` request resolves after a newer refresh, its result is
ignored instead of replacing the current workspace board and sidebar state.

This prevents a slow reconnect or event-triggered refresh from hiding a newly
created session after the create flow has already refreshed and opened it.
Loading state and error messages are also applied only by the latest refresh.

Playwright coverage delays an older workspace refresh, creates a new agent, and
asserts the new session remains visible after the delayed response arrives.
