# Turn-start response ownership

Submitting a chat turn now captures the active session id before calling
`run_agent_turn`.

If the user switches agents while the daemon is still accepting the turn, the
late response no longer appends the old prompt, turn id, or error state into the
newly selected agent view.

The already-started daemon turn still belongs to its original session; the
desktop UI simply ignores its local optimistic state unless that same session is
active when the start response returns.
