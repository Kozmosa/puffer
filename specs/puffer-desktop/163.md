# Same Session Live Reopen

## Summary

Reopening or refreshing the currently selected session no longer clears active
live turn state.

## Design

`App.svelte` now treats same-session opens as transcript refreshes by default.
Different-session opens still reset transient chat state, while same-session
opens preserve submitted rows, streamed assistant text, and turn controls so
the refreshed daemon transcript can reconcile them.

## Contracts

- Clicking the currently active running agent keeps streamed text visible.
- The Stop control remains active after a same-session reload while the daemon
  turn is still running.
- Switching to a different session still clears transient live state.
- Explicit `resetLiveState` options remain authoritative for special refresh
  paths.
