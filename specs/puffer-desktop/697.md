# Contact Infer Rerun Refresh

## Design

Opening the Contacts infer modal always starts a fresh inference run. Existing
proposals are cleared immediately so the modal cannot present stale proposals
while a new candidate set is being collected.

## Architecture

`Contacts.svelte` calls `runInference()` every time `openInferDialog()` opens
the modal. `runInference()` resets proposals, trace state, and error state
before subscribing to the new trace id and invoking `contacts_infer`.

## Compatibility

The backend `contacts_infer` contract is unchanged. Manual reruns through the
modal Run button keep using the same fresh collection path.
