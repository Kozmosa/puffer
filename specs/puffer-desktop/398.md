# 398. Browser Recording Redirect URL Scoping

URL-only Browser activity details now treat redirected or final page URLs as the
same action when the recorded frame keeps a compatible host and path. This
handles common cases such as `https://example.com` rendering as
`https://www.example.com/final` while still separating different Browser
actions in the same session.

The action details pane and inline tool card both apply the same URL matching
rules. If a URL-only action has multiple structurally matching frames but none
match the action URL, the UI no longer falls back to every frame because that can
show a later unrelated page for the selected action.

The regression covers two URL-only Browser open actions whose recorded frames
use redirected final URLs, and verifies that selecting each action only shows
its own frame.
