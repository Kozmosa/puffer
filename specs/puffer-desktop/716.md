# Contact Inference Fixture Dedupe

## Summary

The desktop fake daemon now mirrors production inferred-contact persistence when
multiple connector candidates normalize to the same contact id in one inference
batch.

## Design

- Fake `contacts_infer` tracks contact ids saved earlier in the same batch.
- Later inferred rows that overlap existing or just-inferred ids are skipped.
- Contacts UI coverage verifies a duplicate candidate batch saves one contact
  and reports one inferred contact.

## Compatibility

This is test-support behavior only. Production daemon RPC contracts and
desktop request shapes are unchanged.
