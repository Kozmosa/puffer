# Linux CEF Blink Heap TLS Link Fix

## Summary

Linux CEF release builds now force Chromium's
`blink_heap_inside_shared_library = true` GN argument.

## Design

Chromium's non-component Linux default uses Blink heap thread-local storage with
the `local-exec` TLS model. That is valid for code linked directly into an
executable, but CEF links Blink into `libcef.so`, where the linker rejects
`R_X86_64_TPOFF32` relocations against `blink::g_thread_specific_`. The release
script now sets Blink's shared-library heap flag before regenerating CEF GN
files, causing Blink to use the shared-library-safe TLS access path.

## Compatibility

The setting is Linux-only in the release pipeline and does not alter macOS
packaging. It keeps the build non-component while matching CEF's shared library
link model.

## Verification

Linux CEF packaging should progress past the `libcef.so` link step instead of
failing on Blink heap TLS relocations.
