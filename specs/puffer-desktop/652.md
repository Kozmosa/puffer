# Native CEF Pane Teardown

Switching away from the Browser pane hides native CEF views instead of closing
their renderer processes.

- `browser_cef_native_hide` hides the native container without destroying the
  CEF browser slot, preserving browser state across app-tab switches.
- The Browser pane calls native hide during Svelte teardown; explicit Browser
  tab close still calls native close and destroys the slot.
- Native CEF slots track asynchronous `CreateBrowser` work. If close arrives
  before `OnAfterCreated`, the slot is kept alive, hidden, and closed once CEF
  reports the browser instance.
- If CEF creates a browser after the slot disappeared, the native client closes
  that browser immediately instead of leaving an orphaned child view.

Compatibility: existing native open, resize, navigation, history, reload, and
close command payloads are unchanged. The new hide command is a Tauri-only
renderer lifecycle command and is a no-op before native CEF initialization.
