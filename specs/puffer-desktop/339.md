# 339. Deployment Add Secret Draft

Deployment secret controls must not expose an enabled Add action that drops
clicks.

Contracts:

- The Secrets pane `Add secret` button opens a focused local form.
- The form requires a secret key and masked preview value before submission.
- Added keys are normalized to uppercase environment-variable syntax and
  inserted at the top of the selected deployment's secret list.
- Newly added secrets remain masked until the existing per-key reveal control is
  used.
- The pane live status reports the added key and selected deployment.

Playwright coverage opens the Secrets tab, creates a build-scope secret, checks
the updated key count, masked row, live status, and reveal behavior.
