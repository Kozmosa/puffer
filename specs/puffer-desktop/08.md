# puffer-desktop Deterministic E2E Agent Entry

## Purpose

Desktop interaction tests need a stable way to enter an agent detail screen
when the same session appears in both the sidebar and the workspace project
card.

## Design

The Playwright desktop harness now opens the regression session through the
sidebar active-agent list. That scope matches the persistent app navigation and
avoids ambiguity with workspace board cards that intentionally expose the same
session title.

## Logic

- `openRegressionAgent` scopes the accessible button lookup to
  `.pf-sidebar-agents-list`.
- `openAgentPanel` scopes Browser and Files tab selection to `.pf-agent-tabs`.
- Tests continue to use role-based lookup inside that scoped region.
- Browser and Files tests share the helper so future title or card changes only
  need one harness update.

## Contracts

The e2e harness may assume the fake daemon provides a sidebar-visible active
agent named `Browser regression`.

Workspace cards and sidebar agent buttons may share user-facing titles, but
tests must scope their intent to the navigation region they are exercising.
