# Desktop Browser Toolbar Failure Scoping

## Context

Browser toolbar commands are asynchronous. New-tab, navigation, close, and focus
paths already guard against stale responses, but Back, Forward, and Reload
failures wrote directly to the pane error state. If a user clicked Reload in one
agent and immediately switched agents before the daemon returned an error, the
new agent's Browser pane could show the old agent's failure.

## Change

- Capture the active backend browser session id and session generation before
  running toolbar history/reload commands.
- Route Back, Forward, toolbar Reload, and shortcut Reload through scoped helper
  functions.
- Suppress command failures when the pane was disposed, the session generation
  changed, or the active backend browser session no longer matches the request.

## Compatibility

Failures from the currently active Browser tab still render normally. The change
only drops failures that belong to an older agent or an older active tab target.

## Verification

- Added a Playwright regression that delays an Alpha agent reload failure,
  switches to Beta before the failure resolves, and asserts Beta remains
  connected without rendering Alpha's Browser error.
