# Files symbol inspect ownership

Files symbol inspection now carries a request generation plus the file, root,
and session identity that started the `lsp_inspect` call.

Switching files or agents clears the visible symbol state and invalidates any
older inspect request. Late language-server responses are ignored unless they
still match the active file, workspace root, and session.

This prevents hover, definition, and reference results from a previous file from
reappearing over the current editor after a quick file or agent switch.
