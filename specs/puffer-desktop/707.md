# Contacts Fake Inference Saved Filtering

## Summary

The desktop fake daemon now filters inferred contact proposals that overlap
saved contacts.

## Design

- Fake inference builds the set of saved contact ids from the contacts snapshot.
- Candidate proposals whose contact ids overlap that saved set are omitted.
- The Contacts browser test covers the default Alice-saved, Bob-unsaved fixture
  so future UI regressions use the same proposal contract as the backend.

## Compatibility

This changes only desktop test support. Production contact inference filtering
continues to be owned by the daemon.
