# 589 - Proxy Test Status Copy

Updates Settings -> Network proxy list status rendering for connectivity tests.

- Successful tests render under the endpoint as `connected` or
  `connected (ping: N ms)`.
- Failure labels include common actionable causes such as timeout, connection
  refused, SOCKS handshake, and TLS errors.
- The full daemon message stays in the status title for diagnostics.
- `ProxyTestResult` includes `statusCode` from the daemon response.
- Status formatting lives in a small TypeScript helper so UI tests can cover it
  without mounting the full settings screen.
