# MCP List/Add Race Guard

## Scope

Settings loads MCP servers lazily when the MCP pane opens. Users can add a new
server while that initial list request is still in flight.

## Change

- MCP list requests now carry a local generation guard.
- Adding a server invalidates any older list response before writing the new
  server list returned by the daemon.
- Switching daemon/workspace state also invalidates pending MCP list responses.
- A Playwright regression test delays the initial MCP list, adds a GitHub
  server, and verifies the late list response does not remove the new card.

## Compatibility

Normal MCP listing and add behavior is unchanged. The guard only prevents stale
responses from replacing fresher user-initiated state.
