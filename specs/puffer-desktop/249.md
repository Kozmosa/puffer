# Terminal Close Pending State

## Summary

The desktop Terminal tab close button now reflects the in-flight close request
state instead of staying visually clickable while duplicate close attempts are
being ignored.

## Behavior

- Each terminal tab close button is disabled while its PTY id is present in the
  close-in-flight set.
- The existing duplicate-request guard remains in place, so delayed clicks still
  cannot issue multiple `pty_close` RPCs.
- Disabled close buttons use the same visual pending treatment as other terminal
  controls: reduced opacity and a not-allowed cursor.

## Compatibility

The daemon `pty_close` contract is unchanged. This is a renderer-only feedback
fix for pending close requests.
