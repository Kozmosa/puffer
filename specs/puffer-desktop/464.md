# UI Fuzz Bug Ledger

## Summary

The desktop UI fuzz harness now includes a main-agent-owned markdown bug ledger
for confirmed and candidate interaction findings.

## Behavior

- `apps/puffer-desktop/tests/fuzz/BUGS.md` stores the shared bug list with
  `pending`, `fixed`, `duplicate`, `rejected`, and `out-of-scope` statuses.
- `puffer-fuzz.mjs bug-list` creates the ledger when missing.
- `puffer-fuzz.mjs bug-list --append` appends a normalized row and detail
  section for a new finding under a file lock.
- `puffer-fuzz.mjs bug-list --set-status` updates an existing finding status
  and records a status note.
- Subagent-facing docs and prompts now instruct subagents to emit a
  `BUG_LIST_APPEND` block instead of editing the central ledger directly.

## Compatibility

This only changes the fuzz harness and documentation. Product desktop behavior,
fake daemon behavior, and Playwright regression specs are unchanged.
