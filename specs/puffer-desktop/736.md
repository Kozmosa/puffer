# Media Attachments From Bash Internal Tools

Desktop timeline rendering matches the CLI daemon media attachment contract.

- Generated media attachments are synthesized only from successful `Bash` tool
  events whose input command is `puffer internal-tool image-generation`,
  `puffer internal-tool video-generation`, `imagegen`, or `videogen`.
- The stored Bash output must parse as JSON and its `stdout` field must parse
  as the media result JSON containing a non-empty `jobId` and artifacts.
- Image artifacts use generated media sidecar metadata when present and fallback
  artifact metadata when sidecars are missing.
- Video artifacts become generated video attachments only when the sidecar
  resolves to a supported video MIME type. Local availability is derived from a
  canonical artifact path under the generated video/artifact roots, not from
  Bash stdout paths.
- Trusted video sidecar metadata may carry `remoteSourceUrl` through to the
  attachment DTO.
- Plain Bash commands that print media-shaped JSON do not create attachments.
- Frontend fake-daemon media fixtures use `Bash` tool events and the Shell
  command activity surface; direct `ImageGeneration` tool fixtures are not kept
  as current behavior.
