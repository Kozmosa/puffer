# Browser Tab Focus Error Scoping

## Summary

Browser tab focus failures no longer leak into a different agent after the user
switches sessions.

## Design

`selectTab` captures the root session id and session generation before sending
the daemon focus request. A late focus failure updates the visible Browser
error only when the same session generation, root session, and active tab are
still current.

## Contracts

- A failed focus request in agent A cannot display an error in agent B.
- Switching tabs within the same Browser pane still reports a focus failure if
  that tab remains active.
- Existing tab activation, resize, and browser-open behavior is unchanged.

## Verification

- `playwright test tests/desktop-ui.spec.ts -g "late Browser tab focus failures do not leak into a switched agent"`
