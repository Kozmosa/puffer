# Codex include retry guard

- Desktop Codex/OpenAI turns omit absent `effort` and `serviceTier` fields from
  the `turn/start` app-server request instead of sending explicit JSON nulls.
- If the Codex app-server returns the OpenAI Responses `include[0]` validation
  error while a reasoning effort is selected, the adapter starts a fresh
  app-server attempt without a reasoning effort and keeps the original desktop
  session transcript to a single user turn.
- The retry is intentionally scoped to OpenAI include validation failures so
  unrelated provider, model, auth, and tool errors still surface unchanged.

## Tests

- Unit coverage asserts that `turn/start` request params do not contain null
  effort/service-tier values, include selected effort when present, and classify
  the OpenAI include validation error as retryable.
