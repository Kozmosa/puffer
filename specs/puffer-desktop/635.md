# Puffer Desktop Update 635

## Summary

Linux CEF release builds now restore CEF's Browser delegate bridge in Chromium's
desktop Browser type.

## Design

- `scripts/release.sh` adds a Browser delegate compatibility patcher.
- Browser creation parameters regain CEF's opaque `cef_params` and popup
  `opener` fields.
- Browser owns a `cef::BrowserDelegate` when CEF is enabled and exposes it via
  a typed accessor.
- Browser construction now creates the CEF delegate from those creation
  parameters before tab-strip model setup.

## Compatibility

Non-CEF Chromium builds keep the existing Browser layout. CEF-enabled release
builds can propagate hosted-popup and DevTools opener state through
`Browser::CreateParams`, matching CEF's Chrome runtime sources.
