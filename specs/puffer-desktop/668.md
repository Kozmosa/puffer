# CEF Tab Close Daemon Cleanup

Manual Browser tab close in native CEF mode now closes both sides of the tab
lifecycle. The Svelte Browser pane first closes the embedded CEF view through
the Tauri native command, then sends `browser_agent` `close` for the same root
session and tab id.

This keeps the daemon tab registry aligned with the GUI after a user closes a
CEF tab. Follow-up agent Browser actions no longer see a stale connected tab
record that points at a closed CEF page target.

The close flow preserves screencast behavior and still uses the existing daemon
tab close state response to reconcile local tab state. A regression test stubs
native CEF commands and verifies that closing the CEF tab also emits the daemon
close request for `session-browser:browser:tab-1`.
