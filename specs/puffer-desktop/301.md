# 294. Pipeline Provider Switcher State

## Summary

The pipeline editor now exposes the selected provider for the active agent node
as a real radio group instead of visual-only selected buttons.

## Behavior

- The selected-agent provider switcher reports `role="radiogroup"`.
- Each provider option reports `role="radio"` and `aria-checked`.
- The active provider remains tabbable while inactive providers use roving
  tabindex.
- Left/right/up/down arrows move between adjacent providers.
- Home and End jump to the first and last provider.
- Changing providers still updates the selected node agent, model, tools, and
  prompt defaults.

## Compatibility

This only changes the provider switcher semantics and keyboard navigation. It
does not change workflow loading, daemon contracts, graph layout, or draft save
behavior.
