# Real daemon-backed chat smoke

Desktop UI coverage now includes a browser-rendered flow backed by the real
`puffer daemon` binary:

- The test starts a temporary daemon with isolated `PUFFER_HOME`.
- OpenAI traffic is routed to a local mock server via `OPENAI_BASE_URL`.
- The UI opens through daemon WebSocket query parameters, creates a new agent
  from the empty workspace state, sends one instruction, and waits for the
  assistant reply to render in the conversation.
- The test asserts the mock provider received the exact user instruction.

The smoke spec lives in `apps/puffer-desktop/tests/real-daemon-ui.spec.ts`.
It runs when `target/debug/puffer` exists or when `PUFFER_DESKTOP_TEST_DAEMON`
points at a daemon binary.
