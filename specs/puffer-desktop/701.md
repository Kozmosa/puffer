# Contacts List Lazy Rendering

The desktop Contacts screen now renders saved contact rows in progressive
batches. The first batch is capped at 40 rows, and additional rows load when the
bottom sentinel nears the viewport or when the user activates the visible
load-more button.

Search filtering, summary counts, selection, contact details, editing, deleting,
and proposal-saving still operate against the full contacts snapshot. Only the
saved-contact row DOM is windowed, so existing contacts RPC shapes remain
unchanged.

The Contacts Playwright coverage now includes a 65-contact snapshot to verify
that only the first 40 rows render initially and that the remaining rows appear
after loading the next batch.
