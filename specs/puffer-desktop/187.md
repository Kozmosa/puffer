# Model Picker Alias Selection

## Context

Desktop sessions and auth snapshots can use either `openai` or the user-facing
`codex` alias for the same daemon provider. The model picker already treats
those ids as equivalent when finding the current provider and rendering provider
buttons, but model rows used raw string equality for their selected state.

## Change

- Mark a model row as current when its provider id is alias-equivalent to the
  session provider id and its model id matches the session model.

## Compatibility

Provider switching and turn submission are unchanged. This only fixes the
visual and accessibility selected state in the model picker.

## Verification

- Added a Playwright regression for a session whose provider is `openai` while
  the authenticated provider list exposes the same route as `codex`.
