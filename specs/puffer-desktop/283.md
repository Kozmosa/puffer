# 283. Files Tree Expanded State

## Summary

Files tree directory and symlink rows now expose their expanded state through
`aria-expanded`. File rows remain plain file-opening buttons and do not report
expand/collapse state.

## Behavior

- Collapsed folders report `aria-expanded="false"`.
- Expanded folders report `aria-expanded="true"`.
- The attribute updates when the user clicks the row to expand or collapse it.
- Existing row click, double-click, cache, and file-opening behavior is
  unchanged.

## Compatibility

This is a semantic update for the existing Files tree controls. It does not
change directory listing RPCs, file reads, watcher invalidation, or tab restore
contracts.
