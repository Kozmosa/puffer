# Contacts Fake ID Trimming

## Summary

The desktop fake daemon now trims contact IDs on save and delete requests.

## Design

- Fake `contacts_save` trims caller-provided saved-contact IDs before matching
  existing contacts.
- Fake `contacts_delete` trims the requested ID and rejects empty IDs.
- This mirrors the backend contact save/delete behavior used by production RPCs.

## Compatibility

This changes only desktop test support. Existing UI flows send exact IDs and
continue to behave the same.
