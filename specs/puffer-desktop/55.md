# Restore daemon after failed remote project creation

Remote project setup temporarily switches the shared desktop daemon client to
the SSH-backed daemon before cloning or creating the session. If clone or
session creation fails after that switch, the modal must restore the previous
daemon handshake so the existing local workspace remains usable.

Compatibility contract:

- Native Tauri remote setup still uses `start_ssh_daemon` and then runs clone
  and `create_session` against the connected remote daemon.
- Browser preview and Playwright tests can provide a remote daemon handshake
  with `pufferRemoteBackend`, `pufferRemoteToken`, and
  `pufferRemoteWorkspaceRoot` query parameters without pretending the whole app
  is running inside Tauri.
- On post-connect failures, rollback is best-effort and preserves the original
  error message for the modal status.
- A failed remote connect before daemon switching does not trigger rollback.

Regression coverage lives in `files-workspace-ui.spec.ts`, which drives the
Connect Project remote tab, injects a failing `create_session`, verifies the
modal error, and confirms the active daemon token returns to the local
handshake.
