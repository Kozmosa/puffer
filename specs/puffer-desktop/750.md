# Task monitor add and contact inference fixes

The Tasks and Workflows screens must treat a non-empty `monitor_command` as an
explicit monitor affordance and otherwise fall back to `can_trigger_workflow`.
Nullable snapshot fields such as `monitor_command: null` must not hide
trigger-ready connections from the monitor creation form.

The task monitor configuration regression test must click the Add button and
assert that `task_monitor_create` is sent with the selected connection, model,
and contact scope. Styling-only checks are not sufficient for the monitor add
path.

When the selected monitor-capable connection is degraded or has a recorded auth
failure, the monitor form must keep the account selectable, explain that auth
repair is required, and route the primary action to the existing reconnect
command instead of presenting a disabled Add button or silently returning.
