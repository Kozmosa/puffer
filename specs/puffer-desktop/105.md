# Sidebar Active Agent Highlight

## Scope

The sidebar should highlight an agent only while that agent detail is actually
open. Returning to the workspace board keeps the last loaded session in memory
for fast reopen and transcript state, but that hidden selection must not keep
the sidebar row visually active.

## Change

- Sidebar active state now follows `openAgentSessionId` instead of the last
  loaded `selectedSession`.
- The workspace regression test asserts that returning from agent detail clears
  active agent highlighting while preserving the workspace board and session
  history.

## Compatibility

Opening an agent still highlights its sidebar row. Fallback sidebar entries for
newly created sessions continue to work because new-agent flows set
`openAgentSessionId` before displaying the detail view.
