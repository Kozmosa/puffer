# Browser Printable Key Events

## Scope

The managed Browser tab must forward keyboard input through Chrome DevTools
Protocol in a way that matches real browser interaction.

## Change

- Printable key presses now emit `browser_input` key events with `eventType:
  keyDown` and `text` instead of using a separate text insertion RPC.
- Existing key-up handling is preserved, so pages receive a balanced
  keydown/keyup pair for normal character input.
- Paste still uses the text insertion path because clipboard text can contain
  multiple characters and does not represent a single physical key.

## Compatibility

This keeps the Browser tab compatible with editors and web apps that listen for
`keydown` while still allowing Chrome to insert the printable character from the
CDP key event payload.
