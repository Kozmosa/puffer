# Preserve Live Answer Across Next Turn

## Scope

When a turn completes before the transcript reload finishes, its live assistant
answer is the only visible copy of that response. Starting the next turn must
not clear or merge that previous live answer.

## Change

- Live streaming assistant rows are keyed by `turnId` instead of by timestamp.
- A new non-replay `turn-start` no longer clears prior live items that may still
  be waiting for persisted transcript reconciliation.
- A Playwright regression test completes one turn with a delayed reload, starts
  a second turn, and asserts the first live answer remains visible.

## Compatibility

Persisted transcript reconciliation still removes live rows once matching
daemon history arrives. The change only preserves unresolved live rows between
settled turns and prevents later deltas from updating the wrong assistant row.
