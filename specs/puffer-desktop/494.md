# Pipeline Workflow Saving

Corbina Pipelines can now persist edited workflow definitions through the
daemon instead of keeping every change local-only.

- The editor header includes a Save action that is enabled only for the
  selected dirty workflow.
- Field, trigger, connector, node, and wiring edits mark the selected workflow
  dirty and show a save reminder.
- Saving calls `workflow_save` with the typed workflow envelope currently shown
  in the editor.
- Successful saves refresh the workflow snapshot, clear the dirty state for the
  saved slug, keep other dirty drafts overlaid, and show a saved notice.
- Failed saves surface the daemon validation or transport error in the existing
  save notice and do not clear the dirty state.

Connector search, monitor task actions, repair commands, and workflow run
inspection are unchanged.
