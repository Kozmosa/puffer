# Puffer Desktop Update 631

## Summary

Linux CEF release builds now restore the content-main entrypoints required by
CEF's browser main runner.

## Design

- `scripts/release.sh` adds a CEF content-main compatibility patcher.
- The patcher restores split `ContentMainInitialize`, `ContentMainRun`, and
  `ContentMainShutdown` functions on top of Chromium's current content main
  implementation.
- `ContentMainParams` regains CEF's POSIX signal-handler suppression flag.
- `ContentMainRunnerImpl` regains the UI-thread shutdown hook that CEF calls in
  multi-threaded message-loop mode.
- `ChromeMainDelegate` regains a CEF cleanup hook used during shutdown.

## Compatibility

The monolithic Chromium `ContentMain` path still delegates through the restored
split functions, so Chrome's default behavior remains intact while CEF can
initialize, run, and shut down the embedded runtime explicitly.
