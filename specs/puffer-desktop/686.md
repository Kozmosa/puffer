# Browser New Tab Surface

- Blank browser tabs continue to use `about:blank` as the internal browser URL so CEF target reset and recycled-tab safety remain unchanged.
- `BrowserPane.svelte` presents active blank tabs with an app-owned new-tab overlay instead of exposing the raw black `about:blank` canvas.
- The overlay shares URL draft state with the toolbar address field, auto-focuses for newly opened tabs, and submits through the existing navigation path.
- URL drafts are normalized before navigation: empty and `about:blank` stay blank, existing schemes pass through, local host targets use `http://`, and bare domains use `https://`.
- Overlay pointer, wheel, keyboard, paste, and context-menu events are stopped before they reach the browser canvas, while non-blank pages continue sending input to the active browser target.
- Playwright coverage verifies the overlay is visible on blank tabs, does not leak input to the daemon, navigates typed URLs, yields back to the canvas after navigation, and preserves typed drafts during blank-tab state refreshes.
