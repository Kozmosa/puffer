# Terminal Focus Session Guard

## Context

The Terminal pane awaited `pty_focus` before attaching xterm to the selected
PTY. If the user switched agents while the focus request was in flight, the
late response could attach the old session's PTY into the new visible agent
view.

## Change

- Guard PTY activation with the current session id and restore generation.
- Pass the restore context through terminal attachment and replay setup.
- Stop registering terminal input/replay handlers when the session changes
  while an attach is in flight.

## Compatibility

The PTY daemon protocol is unchanged. Existing same-session tab focus and
creation flows still use the same `pty_focus`, `pty_replay`, `pty_resize`, and
`pty_write` calls.

## Verification

- Added a Playwright regression that delays the alpha session's `pty_focus`,
  switches to beta, types into the visible terminal, and asserts the write goes
  to beta's PTY rather than alpha's stale PTY.
