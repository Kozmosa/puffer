# Desktop Settings Model Provider Scoping

## Context

The Providers settings pane lazy-loads model lists per provider. Switching away
from a provider while its model request was still pending could let that
provider's late failure render as the current provider error. Switching back to
a provider whose models were already cached could also leave the model picker
blank because the cached path returned before restoring a default model.

## Change

- Restore the provider's default model when the selected provider already has a
  cached model list and the current model field is empty.
- Apply model-load errors only if the failing provider is still selected.
- Keep per-provider loading flags unchanged so in-flight requests still
  complete and cache normally.

## Compatibility

The selected provider still displays its own load failures. Cached providers now
recover their default model consistently when reselected.

## Verification

- Added a Playwright regression that switches from Codex to Anthropic while the
  Anthropic model list fails late, switches back to Codex, and asserts the
  Codex model stays selected without showing Anthropic's stale error.
