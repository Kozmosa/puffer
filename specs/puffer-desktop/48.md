# Persist remember-last-session preference

The Settings checkbox for remembering the last session is now backed by
desktop renderer storage and restores the selected agent detail on launch when
the remembered session still exists in the active workspace.

Compatibility contract:

- Desktop preferences are stored under `puffer-desktop:preferences`.
- The remembered session is stored with its workspace root so switching daemon
  roots cannot accidentally reopen an unrelated session with the same id.
- Turning the preference off or resetting desktop preferences clears the
  remembered session.
- The default launch behavior is unchanged when the preference is off: the
  workspace board remains the first visible screen.
- When the preference is on, the shell opens the remembered session and enters
  the agent detail view after groups load.

Regression coverage lives in `settings-ui.spec.ts`, toggling the preference,
opening an agent, reloading the app, and asserting that AgentDetail reopens.
