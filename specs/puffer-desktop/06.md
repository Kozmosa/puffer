# Browser Disconnected Active Tab Recovery

## Scope

The Browser pane should recover when the daemon reports that the active tab
exists but its backing Chrome session is disconnected.

## Change

- Real-time tab-list handling now reconnects when the active tab is unchanged
  but marked disconnected.
- Reconnect uses the tab's current URL, preserving the visible browser context
  instead of leaving the user on a dead canvas.
- The e2e suite covers a disconnected active-tab event and expects a second
  `browser_open` request for that backend tab.

## Compatibility

Active-tab switch behavior from the prior spec remains unchanged. This only
adds recovery for disconnected active tabs.
