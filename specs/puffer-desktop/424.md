# Background Completion Tool Preservation

## Context

Background turns may finish while the user is reading another session. The
sidebar state handler cleared the cached transient conversation state when it
observed `turn-complete`, which removed tool activity that had not yet been
persisted into the transcript.

## Change

- Sidebar event handling now only updates sidebar live-state for settled turns.
- Background completion caching remains responsible for clearing prompts and
  preserving non-prompt live activity until transcript persistence catches up.
- The regression covers a background tool invocation followed immediately by a
  completion event before the session is reopened.

## Compatibility

Selected-session completion behavior is unchanged. Background permission and
question prompts are still cleared by the completion cache instead of being
replayed after a turn has settled.
