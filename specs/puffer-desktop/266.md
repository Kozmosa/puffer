# PDF fallback and legacy DOC extraction

The Files pane must keep document previews useful when the WebView PDF renderer
or old Word binary parser cannot handle a real file.

- `PdfDocumentPreview` resolves the pdf.js worker through Vite's asset URL
  pipeline and catches renderer import/setup failures so extracted text is still
  shown instead of leaving the preview in a loading state.
- Daemon `read_file` responses may include `textPreview` lines. The frontend
  uses those lines before its JavaScript fallback parser for PDFs and legacy
  Office previews.
- Local desktop `.doc` reads try macOS `textutil` for native Word text
  extraction. If `textutil` is unavailable or fails, the existing base64 payload
  and frontend CFB/RTF/HTML fallbacks remain unchanged.

This is additive to the file transport contract: callers that ignore
`textPreview` continue to receive the same `encoding`, `content`, `size`, and
`truncated` fields.
