# 352. Terminal Active Tab Reattach Guard

The Terminal pane must not recreate the xterm instance or replay PTY output when
the user clicks the already-active terminal tab.

## Contract

- Selecting a different terminal tab still focuses and attaches that PTY.
- Selecting the active terminal tab is a no-op once the current xterm instance
  and daemon data subscription are attached.
- The guard avoids duplicate `pty_focus` and `pty_replay` requests, preventing
  visible terminal flicker and unnecessary daemon work.

## Coverage

`tests/terminal-ui.spec.ts` clicks the active terminal tab repeatedly and checks
that no additional focus or replay requests are sent for the active PTY.
