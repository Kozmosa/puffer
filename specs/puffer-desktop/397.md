# 397. Composer Thinking Default Clearing

The composer thinking picker treats `Default` as an explicit no-override state.
When a model exposes thinking options, the select opens on `Default` unless the
current session/model has a saved concrete option.

Choosing a concrete option such as `High` stores that session/model preference.
Choosing `Default` removes the saved preference, keeps the select on `Default`
when the user leaves and returns to the session, and sends `thinkingOptionId:
null` with the next turn.

Fast mode and permission mode remain session-scoped independently from thinking
level. The regression covers two sessions sharing the same model so a cleared
thinking override cannot leak back from local storage.
