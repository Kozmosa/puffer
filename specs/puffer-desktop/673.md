# macOS Tauri Runtime Bundle

`make build-tauri` must leave the macOS `.app` bundle directly runnable.

- The macOS `build-tauri` release path still stages the selected CEF runtime
  and compiles Tauri with `PUFFER_CEF_PATH` and `PUFFER_CEF_ROOT`.
- After Tauri bundling, the produced `.app` now embeds the release `puffer`
  binary, bundled resources, the CEF framework, and all CEF helper apps.
- The app executable receives the `@executable_path/../Frameworks` rpath and
  the completed bundle is re-signed after CEF files are copied.
- `pack-macos` reuses the same embedding function for its staged zip payload
  so the build and package paths cannot diverge.

Compatibility: this changes only macOS release packaging. The generated app no
longer links against a missing CEF framework when launched immediately after
`make build-tauri`.
