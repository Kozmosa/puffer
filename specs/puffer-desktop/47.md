# Require auth before honoring onboarding skip

The desktop onboarding skip flag records that a signed-in user chose to skip
the repo setup step. It is not an authentication grant. The shell now checks
provider authentication before honoring the skip flag so logout cannot reopen
the workspace with no usable provider.

Compatibility contract:

- A settings snapshot with zero authenticated providers always shows the login
  onboarding path.
- `skipOnboarding` still bypasses repo setup for snapshots that include at
  least one authenticated provider.
- `forceOnboarding` still forces the repo step for authenticated test and
  screenshot flows.
- Logging out the final provider clears the workspace view immediately and a
  later settings refresh cannot bypass the login screen through localStorage.

Regression coverage lives in `onboarding-ui.spec.ts`, using the desktop
skip flag with an empty-auth daemon fixture.
