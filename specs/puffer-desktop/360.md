# 360. Backend Reconnect Failure Feedback

The desktop shell must make backend reconnect attempts observable when the
daemon remains unavailable.

## Contract

- When the daemon WebSocket closes, the shell shows the disconnected backend
  banner with a native `Reconnect backend` button.
- Clicking reconnect clears any prior retry error, disables duplicate retry
  clicks while the attempt is in flight, and attempts to reuse the configured
  daemon client.
- If reconnect fails, the banner stays visible and reports the concrete failure
  instead of silently swallowing it.
- If reconnect later succeeds, the banner disappears and the stale retry error
  is cleared.
- The reconnect flow does not replace the daemon identity or session state
  unless the existing daemon client opens successfully.

## Verification

`tests/daemon-protocol-ui.spec.ts` uses the real daemon WebSocket framing,
forces the fake daemon to drop existing sockets and reject new connections,
asserts the reconnect failure is visible, then allows connections again and
verifies that a retry clears the banner.
