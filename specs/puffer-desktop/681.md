# puffer-desktop Update 681: Session List Initial Page Size

The workspace board now requests 100 sessions per page from the Puffer daemon
instead of 30. This keeps real user sessions visible when recent zero-event
temporary sessions occupy the top of the recency-sorted list.

The daemon pagination contract is unchanged. The desktop still tracks
`hasMore` and keeps the existing "Load more" flow for stores larger than the
initial page.
