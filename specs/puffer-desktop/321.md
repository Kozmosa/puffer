# Browser Navigation Loading Recovery

Browser toolbar navigation commands must not leave a tab permanently stuck in
`Loading` when the daemon accepts the command but no follow-up state event
arrives.

## Requirements

- Reload, back, forward, and address-bar navigation mark the active browser tab
  as loading immediately so the user gets responsive feedback.
- A later browser state event remains authoritative and clears the pending
  navigation state.
- If no state event follows an accepted navigation command, the desktop UI
  clears the pending loading state after a short fallback interval and restores
  the tab status to `Connected` or `Disconnected` based on the current tab.
- Switching sessions, clearing all tabs, or destroying the pane cancels pending
  fallback timers so stale commands cannot update a different session.

## Verification

The browser UI regression suite covers a reload command whose fake daemon never
emits a state event. The status must first show `Loading`, then recover to
`Connected` without user action.
