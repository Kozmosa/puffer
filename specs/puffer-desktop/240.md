# 240. Session Detail Close Control

## Summary

Agent detail now exposes an explicit Close session button in the header. Closing
the detail view returns to the workspace board and does not delete or hide the
session from history.

## Design

- The existing back action remains available for navigation.
- The new close control uses the same parent `onBack` contract, so it only
  clears the open detail view.
- Session records stay in the workspace/history data model and can be reopened
  from the Session history list after closing.

## Compatibility

No daemon API changes are required. This is a renderer affordance over the
existing workspace navigation state.

## Tests

Workspace UI coverage opens a stored session, closes it through the explicit
Close session button, verifies the history row remains visible, and reopens the
same transcript.
