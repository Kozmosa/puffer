# Default desktop UI regression coverage

The desktop UI test command must run every checked-in interaction regression
that protects normal desktop behavior. The issue-specific `plus-*` specs were
valid Playwright UI regressions, but the default `test:desktop-ui` command only
matched `tests/*ui.spec.ts`, so those tests could pass locally only when called
by filename.

- `npm run test:desktop-ui` now runs both `tests/*ui.spec.ts` and
  `tests/plus-*.spec.ts`.
- The Codex Playwright MCP smoke test remains outside this command and keeps
  its dedicated `test:codex-playwright` entrypoint.
- The expanded command covers the active-agent grouping/collapse, provider
  selection layout, project tab activation, and connect-project modal regressions
  alongside the larger desktop UI suite.

No production daemon or UI contract changes are included in this update.
