# 257. Settings Default Model Recovery

## Summary

The Settings provider page now replaces an invalid configured default model
with the selected provider's loaded default model.

## Behavior

- Provider model loading still leaves the model picker disabled while the
  request is pending.
- When models load, the picker keeps the configured model only if it exists in
  the loaded model list.
- If the configured model is empty or belongs to another provider, the picker
  selects the provider default, or the first listed model.

## Compatibility

No settings schema or daemon RPC shape changes. The desktop client only avoids
presenting or re-saving a cross-provider default route.

## Tests

`settings-ui.spec.ts` covers `defaultProvider=openai` with a stale Claude model
and verifies the picker and saved config use the OpenAI model.
