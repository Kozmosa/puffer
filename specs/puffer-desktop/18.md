# puffer-desktop Browser Pointer Reset

## Purpose

Browser canvas pointer state must not stay stuck when the managed browser tab
set is cleared or disconnected while a pointer button is held.

## Design

The browser pane now treats tab clearing and disconnected active-tab state as a
pointer cancellation boundary. Mouse release events that arrive after
disconnection also clear local pointer bookkeeping even though no daemon input
is sent.

## Logic

- Empty tab-list events clear the active tab, canvas, cursor timer, and pointer
  state.
- Non-empty tab-list events that leave the active tab disconnected also clear
  pointer state.
- Mouse release while disconnected resets pointer bookkeeping instead of
  returning with stale pressed-button state.
- Restored tabs can immediately run hover cursor probes after a mid-drag clear.

## Contracts

The daemon must not receive synthetic input for a disconnected tab, but local
UI state must still settle so later connected tabs start from a neutral pointer
state.
