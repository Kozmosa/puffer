# Provider API Key Clearing

## Summary

Provider API key inputs now clear after a successful API-key login or update
round trip. This prevents the submitted secret from remaining visible in the
settings form after the credential has been accepted.

## Behavior

- Submitting an API key records the provider as a pending key update.
- The input is only cleared after that provider enters and then leaves the busy
  state without an auth error.
- Failed submissions keep the typed value available for correction or retry.
- Other providers' API key drafts are not modified by the completed update.

## Compatibility

This is limited to the desktop login form. The `login_with_api_key` RPC payload,
provider registry behavior, and credential storage flow are unchanged.
