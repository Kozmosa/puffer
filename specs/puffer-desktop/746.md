# Conversation Transient Turn Ordering

## Summary

Conversation rendering now keeps delayed live assistant output attached to the
turn that produced it when a user starts the next turn before transcript reload
finishes.

## Design

- Submitted user messages and live assistant/tool placeholders are merged as one
  transient sequence before being reconciled with the persisted transcript.
- Live placeholders receive a creation timestamp when they are first rendered,
  and replacement events preserve the earlier timestamp.
- Persisted transcript items still win once `load_session_detail` catches up,
  so live-only rows disappear after matching saved events arrive.

## Compatibility

Persisted transcript order is unchanged. The change only affects the temporary
desktop state shown while a completed turn is waiting for session-detail reload
and a newer prompt has already been submitted.
