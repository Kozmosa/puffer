# Legacy Word Class Style Preservation

## Design

`textutil` often exports old `.doc`, `.dot`, and `.rtf` documents as HTML with
class-based CSS rules in the document head. The Files preview now inlines safe
class rules into the body before sanitizing and rendering the preview, so basic
legacy Word styling survives even though the head stylesheet is not rendered.

## Contracts

- Only class selectors exported by the preview HTML are considered.
- Scriptable markup, event handlers, external resources, and URL-bearing CSS are
  still removed before the preview reaches `{@html}`.
- Safe typography, spacing, color, and alignment declarations are preserved as
  inline styles.

## Compatibility

Plain text fallback remains unchanged. Existing DOCX, PPTX, XLSX, Markdown, CSV,
and PDF preview paths do not use this class-style inlining path.
