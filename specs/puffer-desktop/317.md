# 317. Modal Focus Trap

## Summary

Desktop workspace modals now move keyboard focus inside the active dialog on
open and keep Tab navigation contained while the dialog is mounted.

## Behavior

- New agent, Create Project, and Switch workspace dialogs focus their first
  available control after opening.
- Tab and Shift+Tab cycle through focusable controls inside the active dialog.
- If focus is moved outside the active dialog while it remains mounted, it is
  redirected back into the dialog.
- Closing the dialog restores focus to the opener when it is still connected.

## Compatibility

No daemon RPC or modal submit behavior changes. This is a client-side keyboard
and assistive-technology interaction fix shared by the affected modal dialogs.

## Tests

`files-workspace-ui.spec.ts` covers New agent and Create Project focus behavior.
`workspace-ui.spec.ts` covers Switch workspace focus behavior.
