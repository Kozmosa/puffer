# Pending Tool Transcript Reconciliation

## Scope

Live pending tool cards can appear before the daemon emits a completed
`tool-invocations` event. If that completion event is missed, the later
transcript reload still contains the completed persisted tool call, and the UI
must not keep showing the stale pending card.

## Change

- Pending live tool cards with no output now reconcile against persisted tool
  calls by normalized `toolName` and input payload.
- JSON tool inputs are compared with stable key ordering, so equivalent input
  objects match even when whitespace or object key order differs.
- A Playwright regression test covers a `tool-calls-requested` event followed
  directly by `turn-complete` and a persisted completed tool in the reload.

## Compatibility

Tool cards with output still reconcile through existing body matching. Empty
pending tools only disappear when the persisted transcript has the same tool
name and normalized input, so unrelated concurrently displayed tools remain
visible.
