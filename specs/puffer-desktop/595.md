# Task history delivery diagnostics

Task History now shows Telegram delivery diagnostics when the subscriber
includes them in the event payload. Each received-message row displays the
event kind, delivery source (`live` or `catch-up`), event lag computed from
`payload.date_ms` versus `received_at_ms`, and the history row number.

The selected message detail keeps the message text visible and always renders
the structured payload below it. This makes connector delivery metadata
inspectable without hiding the raw payload whenever text content is present.
