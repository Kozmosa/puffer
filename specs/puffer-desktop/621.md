# Linux CEF Ozone Buildflag Compatibility

## Summary

CEF branch 7680 still checks `BUILDFLAG(SUPPORTS_OZONE_X11)` in Linux browser
and base headers. The local Chromium 146 checkout now generates
`BUILDFLAG(IS_OZONE_X11)` from `ui/base:ozone_buildflags`, so the stale CEF
buildflag fails before Linux CEF can compile.

## Design

`apply_cef_compatibility_patches` now detects the current Chromium
`IS_OZONE_X11` flag shape from `ui/base/BUILD.gn`. When the old
`SUPPORTS_OZONE_X11` flag is absent, it rewrites CEF's stale buildflag checks
under `src/cef` to `IS_OZONE_X11`.

The rewrite is idempotent and release-build local to the Chromium checkout. It
does not mutate non-CEF Chromium code and only runs when CEF still contains the
old flag name.

## Compatibility

The generated Ozone header already exposes `IS_OZONE_X11` with the same
platform enablement value used by this checkout. This keeps CEF's Linux X11
paths compiled when Chromium enables the X11 Ozone platform, while preserving
older checkouts that still generate the `SUPPORTS_OZONE_X11` buildflag.
