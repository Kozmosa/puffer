# Reconnect UI/UX Regression Coverage

## Summary

Adds regression coverage for reconnect races found by the UI fuzzing harness. The desktop app must keep active chat-turn state usable across a backend reconnect and must clear the reconnect banner before narrow-viewport workspace navigation resumes.

## Contracts

- Permission requests emitted after a backend reconnect for the selected active session render as actionable approval prompts.
- User-question requests emitted after a backend reconnect for the selected active session render as actionable question prompts.
- Turn cancellation errors emitted after reconnect clear the running/canceling composer state and restore the Send button.
- Turn completion events emitted after reconnect append the final assistant response and restore the Send button.
- At narrow viewport widths, a successful backend reconnect removes the connection banner and leaves workspace session navigation clickable.

## Compatibility

The tests exercise the existing fake daemon WebSocket protocol and do not change product behavior or daemon contracts.
