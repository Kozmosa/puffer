# Puffer Desktop Update 625

## Summary

The Linux CEF release build now maps CEF's persistent storage content-setting
enum to Chromium's current durable storage enum.

## Design

- `scripts/release.sh` detects Chromium checkouts where
  `ContentSettingsType::PERSISTENT_STORAGE` has been removed and
  `ContentSettingsType::DURABLE_STORAGE` is present.
- In that case, CEF's `setting_helper.cc` keeps the public
  `CEF_CONTENT_SETTING_TYPE_PERSISTENT_STORAGE` value but translates it to and
  from Chromium's `DURABLE_STORAGE`.
- The patch is sentinel-based and only runs when the old Chromium enum is
  missing, preserving compatibility with older checkouts.

## Compatibility

CEF API consumers keep using the existing CEF enum value. The patch only adapts
the Chromium implementation enum name during release packaging.
