# Disable unwired project memory editing

Project Detail memory entries are currently synthesized summaries, not editable
files loaded through the Files pane. The Memory `Edit` control no longer acts
like an enabled command until a real edit path exists.

Compatibility contract:

- Memory browsing and selection are unchanged.
- The `Edit` button is disabled so clicks are not silently dropped.
- Future work can replace the disabled state with a real handoff into an
  editable file tab once Project Detail owns that route.

Regression coverage lives in `workspace-ui.spec.ts`, opening Project Details,
switching to Memory, and asserting the edit control is disabled.
