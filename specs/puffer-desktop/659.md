# CT desktop browser runtimes

Desktop-managed headless CDP browser sessions now use only Puffer CT Chromium.
The Tauri browser module resolves explicit `PUFFER_CHROME`/`PUFFER_CT_CHROME`,
packaged `browser-runtimes`, cached `berabuddies/ct` release assets, or a local
Tintin checkout. If none are present, it downloads the current-platform
`chromium-tintin-chrome-*` release asset into the CT runtime cache.

Native CEF discovery now also understands the CT release cache. Status checks do
not download large CEF artifacts, but native initialization can fetch and
extract `puffer-cef-<platform>-<arch>.tar.gz` from `berabuddies/ct@ct` when no
env, bundled, compiled, or cached CEF runtime is available.

Compatibility implications:

- Desktop Browser screencast/headless CDP no longer launches system Google
  Chrome or generic Chromium.
- `PUFFER_BROWSER_RUNTIME_DIR`, `PUFFER_CT_REPO`, and `PUFFER_CT_RELEASE_TAG`
  control the release cache/source for both Chromium and CEF runtime downloads.
- Native CEF remains gated by the existing compiled bridge, helper executable,
  and remote debugging port contracts.
