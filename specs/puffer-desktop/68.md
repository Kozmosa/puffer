# Desktop provider alias send flow

Desktop auth checks treat user-facing provider aliases and daemon provider ids
as the same provider before allowing a turn to start.

Compatibility contract:

- `codex` auth satisfies sessions and requests routed through `openai`.
- `claude` auth satisfies sessions and requests routed through `anthropic`.
- New Agent, Connect Project, model picking, composer gating, and App-level
  turn submission use the same provider-id equivalence.
- Existing sessions may store model ids as `provider/model`; when the provider
  prefix matches the session provider, the composer strips that prefix before
  resolving fast-mode, thinking options, and `run_agent_turn` options.
- Provider prefixes that do not match the selected provider are preserved so
  router-style model ids remain intact.

Regression coverage lives in `chat-session-ui.spec.ts` and
`files-workspace-ui.spec.ts`.
