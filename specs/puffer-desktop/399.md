# 399. Daemon-Backed Composer Routing Persistence

Explicit provider/model selections sent with a desktop turn are persisted to
daemon desktop session routing after the resolved provider/model is applied.

Thinking level, fast mode, and permission-only changes do not overwrite routing.
The persisted model id is stored without the provider prefix so list/detail DTOs
match create-session routing.

The desktop fake daemon updates session metadata on `run_agent_turn` when
provider/model params are present, allowing the e2e to clear localStorage and
reload while still recovering the selected route from daemon-backed session
state.
