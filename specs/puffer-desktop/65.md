# Canonical new-agent provider ids

The desktop New Agent and Connect Project provider fallbacks use daemon
provider identifiers even while the settings snapshot is still loading.

Compatibility contract:

- Codex is displayed as the user-facing OpenAI lane, but the daemon receives
  `openai` as the provider id.
- Anthropic is displayed as the user-facing Claude lane, and the daemon
  receives `anthropic` as the provider id.
- Fallback providers are only used before a settings snapshot supplies the real
  provider list.
- Provider filtering continues to hide unauthenticated providers once auth
  state is available.
- Connect Project and New Agent share the same canonical fallback ids.

Regression coverage lives in `files-workspace-ui.spec.ts`.
