# Browser Navigation Failure Scoping

## Summary

Browser URL navigation failures now stay attached to the tab that submitted the
navigation request.

## Design

`submitUrl` captures the active tab id, backend browser session id, URL, and
session generation before sending `browser_navigate`. The optimistic loading
state and any late failure response are applied to that captured tab only. The
visible toolbar error is updated only when the submitted tab is still active.

The fake daemon supports delayed failure responses so Playwright can reproduce
late daemon errors after the user switches tabs.

## Contracts

- A failed navigation in tab A cannot show an error while tab B is active.
- A late navigation failure after a session switch is ignored.
- Successful navigation behavior and optimistic loading state still use the
  submitted URL.

## Verification

- `playwright test tests/desktop-ui.spec.ts -g "late Browser navigation failures stay scoped to the submitted tab"`
