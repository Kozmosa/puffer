# Fresh Model Picker Lists

The agent model picker must treat daemon model discovery as the source of truth
each time the menu is opened or a provider segment is selected. Cached models
may remain available only as previously rendered state; they must not suppress a
fresh `list_provider_models` RPC for the active provider.

The picker guards concurrent provider switches with generation counters so a
late response from an older open or provider selection cannot replace the
currently selected provider's model list or loading state. A refreshed list that
removes a stale model, such as an unavailable `gpt-5`, should remove that model
from the visible menu on the next open.

Regression coverage lives in the desktop Playwright chat-session suite and uses
the fake daemon to change a provider's model list between two picker openings.
