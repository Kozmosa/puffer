# Closed Session Ignores Stale Workspace Updates

## Context

Closing a session clears the active detail pane and remembered-session restore
target, but the previously selected session remains in memory for workspace
navigation. A delayed workspace update for that same session could reload its
detail and save it as remembered again.

## Change

- Workspace session update events only refresh session detail when that session
  is still the open agent detail.
- Closed sessions continue to appear in history without being reopened or
  remembered by stale title or rename events.
- The regression covers closing a remembered session, receiving a delayed
  `workspace:sessions:changed` event, and reloading the app.

## Compatibility

Open sessions still refresh on completion, error, title, and rename events.
Closed sessions keep their history rows and can still be reopened explicitly by
the user.
