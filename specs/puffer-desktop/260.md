# PDF and legacy Office previews

The Files pane must render user documents from the desktop daemon rather than
depending on browser plugins or the 256 KiB text-file read default.

- Rich preview formats now request up to the daemon's 24 MiB preview ceiling.
  Text editing keeps the smaller default read path.
- PDF previews render pages with pdf.js on a canvas and keep extracted text only
  as a renderer failure fallback.
- Legacy binary Office previews inspect OLE Compound File streams before falling
  back to whole-file readable string extraction. This keeps old `.doc`, `.ppt`,
  and `.xls` files useful when the readable text is stored away from the header.
- Desktop daemon read limits are raised for previews only; write limits remain
  at the previous 5 MiB ceiling.
