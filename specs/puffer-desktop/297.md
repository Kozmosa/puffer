# 297. Cap Dismissed Permission and Question ID Lists

## Summary

The `dismissedPermissionIds` and `dismissedQuestionIds` arrays are now capped at
200 entries, preventing unbounded memory growth during long-running sessions with
many permission or question prompts.

## Behavior

- Each time a permission or question is dismissed, the array is trimmed to the
  most recent 200 entries using `.slice(-DISMISSED_IDS_CAP)`.
- Older dismissed IDs that fall off the cap can theoretically re-appear in the
  UI, but this is harmless since the underlying turn data is already resolved.
- The arrays are still fully cleared on session switch (existing behavior).

## Compatibility

No protocol changes. Pure frontend memory optimization.

## Tests

`chat-session-ui.spec.ts` verifies that after dismissing more than 200
permissions, the array length stays at the cap.
