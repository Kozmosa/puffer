# 310. Live Browser Recording Tab Adoption

## Summary

The desktop Browser pane now adopts live recording frames for new agent-created
tabs instead of leaving the visible browser on a stale blank tab.

## Behavior

- Browser recording events for an unknown tab create a matching UI tab.
- The newly recorded tab becomes active immediately, matching the already
  existing cold-start restore behavior that opens the latest recorded tab.
- Recording events for known inactive tabs update their stored frame without
  stealing focus.
- The active tab keeps using the frame's daemon backend session id for future
  resize, input, navigation, and close commands.

## Compatibility

Existing daemon tab-list events remain authoritative when they arrive. This only
fills the live-recording gap where an agent Browser action produces frames
before or without a fresh tab-list event.
