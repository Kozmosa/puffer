# External Credential Import Duplicate Guard

## Scope

Provider credentials discovered from local Claude or Codex config can be
imported from the Settings Providers pane and the onboarding provider view.

## Change

- The desktop shell now ignores external credential import requests while an
  import is already in flight.
- LoginView routes import clicks through a local busy check and disables all
  credential import buttons while one import is pending.
- The fake desktop daemon can now list and import external credential fixtures.
- A Playwright regression test delays the daemon import response, triggers the
  same import button twice in one browser task, and verifies only one
  `import_external_credential` request is sent.

## Compatibility

Normal first-click import behavior is unchanged. The guard only removes
duplicate credential adoption attempts caused by repeated activation while the
first request is still pending.
