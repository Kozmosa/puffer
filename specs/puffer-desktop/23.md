# puffer-desktop Terminal Session Switching

## Purpose

The Terminal pane must follow the selected agent session. Users commonly keep
Terminal open while moving between personal agents; the pane must not keep the
old session's PTY attached.

## Design

Terminal restoration now runs from the reactive `sessionId` and `cwd` props
instead of only on component mount. Each restore captures a generation token so
late list/open responses from an older session cannot attach to the current
pane.

## Logic

- Session or cwd changes clear the active terminal, tab list, sequence cache,
  and xterm attachment.
- The pane lists PTYs for the new session and opens one when the session has
  not initialized terminal state yet.
- Pending restore/create responses are ignored when their captured generation
  or props no longer match.
- The fake daemon now supports minimal PTY list/open/focus/replay/write/resize
  calls for desktop interaction tests.

## Contracts

Terminal ownership is scoped by agent session id. Switching sessions while the
Terminal tab is visible must issue PTY calls for the newly selected session,
not continue sending input or resize events to the prior session.
