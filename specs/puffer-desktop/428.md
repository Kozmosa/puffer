# Composer Draft Survives Workspace Return

## Context

Composer drafts were preserved while switching directly between sessions, but
the draft cache lived inside `ConversationView`. Returning to the workspace
board unmounted that component, so reopening the same session showed an empty
composer and lost the user's half-written prompt.

## Change

- Session drafts now persist through the existing per-session preference key
  namespace.
- Clearing or successfully submitting the composer removes the saved draft.
- The regression covers writing a draft, returning to the workspace board, and
  reopening the same session.

## Compatibility

Drafts remain scoped per session. Submitted prompts still clear the saved draft
so completed turns do not repopulate the composer.
