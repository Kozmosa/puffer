# Linux CEF Resource Bundle Delegate Compatibility

## Summary

CEF branch 7680 expects `ChromeMainDelegate` to expose a virtual
`GetResourceBundleDelegate()` hook so CEF can provide its resource bundle
delegate during browser and subprocess resource initialization. In the local
Chromium 146 checkout, the CEF patch changed the resource helper signature but
the main-delegate hook and call-site changes did not apply cleanly.

## Design

`apply_cef_compatibility_patches` now repairs that API surface when missing:

- adds the `ui/base/resource/resource_bundle.h` include to
  `chrome_main_delegate.h`;
- adds a protected virtual `GetResourceBundleDelegate()` defaulting to
  `nullptr`;
- passes `GetResourceBundleDelegate()` into browser-process `LoadLocalState`;
- passes the same delegate into subprocess `ResourceBundle` initialization.

The patch is idempotent and only mutates the local Chromium checkout used for
release CEF builds.

## Compatibility

Chrome's default behavior is unchanged because the base hook returns `nullptr`.
CEF's derived `ChromeMainDelegateCef` can compile its override and provide
`CefResourceBundleDelegate`, preserving CEF resource loading for embedded
browser processes.
