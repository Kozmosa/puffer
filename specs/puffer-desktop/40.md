# Keep Connect Project provider choices complete

## Purpose

The Connect Project modal should offer the same personal-agent provider set as
the New Agent modal. Authenticated Anthropic and OpenAI providers must not be
hidden when a user connects a fresh local or remote project.

## Design

Connect Project now uses the shared personal-provider allowlist:
`codex`, `openai`, `claude`, `anthropic`, and `puffer`. The existing default
provider selection logic remains unchanged and still respects the daemon's
settings snapshot when that provider is visible.

## Logic

- Snapshot providers are filtered through the personal-agent allowlist.
- Fallback providers remain available for preview and offline test mode.
- The selected provider is sent unchanged to `create_session` for local and
  remote project connection flows.
- Playwright coverage verifies Anthropic is visible in Connect Project and is
  passed through to the daemon when selected.

## Contracts

No daemon API or session metadata contract changes. This is a desktop UI
filtering fix that restores existing provider ids to the project-connection
workflow.
