# 393. Default Routing Refresh For Open Sessions

Desktop sessions may omit both `providerId` and `modelId` to inherit the
current configured default route. The frontend must preserve that null route
instead of coercing it to Codex while normalizing session metadata.

For an already-open session with no explicit route, a refreshed settings
snapshot must update the composer model picker, placeholder provider label, and
next `run_agent_turn` request to the new default provider/model. A saved manual
per-session model selection still wins over the refreshed default. A session
with explicit stored provider/model metadata still uses that stored route.

Regression coverage uses a fake daemon session with null routing fields,
changes the daemon defaults from Codex to OpenRouter, reconnects the desktop
client, and asserts that the visible picker plus submitted turn request both
follow the refreshed OpenRouter default.
