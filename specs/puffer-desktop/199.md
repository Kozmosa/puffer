# Workspace Pin In-Flight Guard

## Summary

Workspace pin save guards now survive opening another agent while the save
request is still in flight.

## Design

`desktopPinInFlightKeys` remains daemon-scoped UI state, not chat live-turn
state. Session changes still clear local chat overlays, but they no longer
clear pending pin request keys.

## Contracts

- Workspace pin/unpin buttons stay disabled until the daemon save response
  resolves.
- Opening an agent during an in-flight pin save must not enable a second pin
  request for the same workspace.
- Daemon reconnect/reset still clears all pin in-flight keys.

## Verification

- `npm run test:desktop-ui -- tests/workspace-ui.spec.ts -g "workspace pin save stays guarded after opening an agent"`
