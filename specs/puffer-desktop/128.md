# Default Model Save Duplicate Guard

## Scope

The Settings Providers pane can save the default provider and model through the
daemon `update_config` endpoint.

## Change

- The default model save handler now ignores repeated calls while a save is
  already in flight.
- A Playwright regression test delays `update_config`, triggers the Save default
  button twice in one browser task, and verifies only one config update request
  reaches the daemon.

## Compatibility

Normal first-click save behavior is unchanged. The guard only removes duplicate
config writes caused by repeated activation while the first save is still
pending.
