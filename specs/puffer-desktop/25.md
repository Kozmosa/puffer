# puffer-desktop Resolved Permission Prompts

## Purpose

Resolved transcript permission dialogs must not reopen as live approval prompts
when a user loads an older desktop session.

## Design

The desktop shell now derives `pendingPermissions` only from permission timeline
items whose normalized status or dialog state is `pending`. `AgentDetail`
derives its awaiting status from the already-filtered pending permission and
question props instead of scanning every historical permission item.

## Logic

- Live permission requests still use `status: "pending"` and remain actionable.
- Historical `allowed` or `denied` permission transcript entries are not shown
  in the live approval slot.
- The agent status pill no longer reports `awaiting` solely because the
  transcript contains an old resolved permission dialog.

## Contracts

`chat-session-ui.spec.ts` covers this by loading a session with an `allowed`
permission dialog and asserting that no `Approval needed` prompt appears.
