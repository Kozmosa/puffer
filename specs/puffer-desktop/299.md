# 299. File Tab Dirty Indicator Clears After External Save

## Summary

`reloadActiveFile` no longer skips the reload when the tab has unsaved edits.
Instead it reloads the file cache without resetting the draft, allowing the dirty
indicator to clear naturally when the on-disk content matches the draft.

## Behavior

- When a filesystem change event fires for the active file:
  - If the tab is clean: reload file and reset draft (existing behavior).
  - If the tab is dirty: reload file cache only, preserving the user's draft.
    The dirty indicator re-evaluates against the new file content.
- If an external process (or the agent) writes the same content as the user's
  draft, the dirty dot disappears immediately on the next fs event.
- If the external content differs from the draft, the dirty indicator remains
  and the user can still save their version.

## Compatibility

No protocol changes. Pure frontend behavior fix.

## Tests

`files-pane.spec.ts` verifies that the dirty indicator clears when an external
save writes content matching the user's draft.
