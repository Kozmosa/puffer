# Onboarding Agent Auth Gate

## Scope

Desktop onboarding decides whether the user can enter the workspace through the
first-run screen, skip flag, and login/import callbacks. That gate must only be
satisfied by providers that can run agent turns.

## Change

- Onboarding completion now checks authenticated agent providers rather than
  any stored credential.
- The onboarding ready state reports connected agent providers so GitHub or
  other integration credentials are not presented as runnable agent auth.
- Login, OAuth, external import, refresh, and logout paths all use the same
  agent-provider predicate when deciding whether onboarding can close.
- Playwright coverage verifies a persisted skip flag cannot bypass provider
  login when the only stored credential is GitHub.

## Compatibility

GitHub and other non-agent credentials remain visible in settings and related
integration surfaces. Existing Anthropic, OpenAI/Codex, and Puffer auth aliases
continue to satisfy onboarding.
