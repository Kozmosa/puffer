# Paged Session Loading

## Purpose

Keep the desktop workspace responsive for large session histories by loading
only the first page of sessions on refresh and fetching more on demand.

## Design

The desktop API client calls `list_grouped_sessions_page` with a 30-session
page size. `App.svelte` tracks loaded count, total count, and `hasMore`, then
merges subsequent pages into the existing grouped session state.

## Logic

Workspace refresh replaces the current groups with page zero. The workspace
screen receives pagination metadata and renders a footer that shows progress
plus a load-more action. Loading another page appends sessions into existing
groups by ID while preserving recency sorting and avoiding duplicates.

## Contracts

- Initial workspace render hydrates at most 30 daemon sessions.
- Load-more requests fetch the next page using the number of sessions already
  returned by the daemon.
- Selected/fallback sessions still appear even when they are outside the
  currently loaded page.
- Search operates over the loaded window and reports that scope.
