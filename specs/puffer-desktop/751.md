# Browser Preview Handshake Fallback

- The desktop Vite dev server now serves `/__puffer/daemon-handshake` from the
  first available daemon handshake among an explicit environment path, the
  repo-local `.puffer` path, and the user-level `~/.puffer/daemon.handshake`.
- This keeps the in-app Browser preview reconnectable when Tauri launches the
  daemon for a workspace outside the repo checkout, such as `/Users/shou`.
- The direct `ws://127.0.0.1:1421/ws` preview path remains the first connection
  attempt; the handshake endpoint is still only a recovery fallback for stale
  or unavailable dev-default sockets.
