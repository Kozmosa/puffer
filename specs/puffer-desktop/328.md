# 328. Fuzzed Interaction Race Guards

## Summary

Desktop chat, pin, and Browser controls now recover from rapid activation,
reconnect, stale tab events, and failed Browser commands without leaving core
controls stuck or silently dropping the next user action.

## Behavior

- Chat submit uses a synchronous per-session guard in both the composer and app
  submit handler, so rapid clicks or duplicate Enter events produce one turn.
- Reconnect refresh reopens the selected session with live-state reconciliation,
  allowing idle session detail to clear stale turn-running UI.
- Browser URL input remains editable for disconnected recovered tabs and can
  reopen the backend tab with the typed URL.
- Browser command failures clear loading state and re-enable the address bar.
- Stale empty or shrinking Browser tab lists are ignored while local tab open,
  close, or navigation transitions are still settling.

## Compatibility

No daemon protocol changes. Existing Browser navigation buttons still require a
connected active tab; only the address submit path can recover a disconnected
active tab by reopening it.

## Tests

`chat-session-ui.spec.ts` covers duplicate rapid send. `browser-ui.spec.ts` and
`desktop-ui.spec.ts` cover failed navigation recovery, disconnected-tab address
submit, stale tab-list handling, and reconnect toolbar behavior.
