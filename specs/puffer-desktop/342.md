# 342. Ask Puffer Save To Memory

Ask Puffer diagnostic actions must not expose a Save button that drops clicks.

Contracts:

- The `Save to memory` action on the seeded diagnostic answer writes a local
  memory draft for the selected deployment.
- The saved memory persists across the Ask Puffer and Memory tabs in the same
  deployment detail view.
- The saved item records the diagnostic source, pitfall kind, high confidence,
  useful tags, and a concise remediation body.
- The action reports a live status and becomes disabled after saving to prevent
  duplicate local drafts.

Playwright coverage clicks `Save to memory`, checks the live status and disabled
button state, switches to Memory, and verifies that the saved diagnostic note is
visible with source metadata and tags.
