# Stabilize chat row identity during remote streaming

Conversation rows now anchor their generated row keys to each timeline item's
stable `id` before falling back to a content hash. This prevents streamed
assistant rows from being destroyed and recreated every time their body grows.

The important case is a live stream that appears without a locally submitted
user row, such as a turn started from another client or replayed by the daemon.
Those rows previously used a body-derived key, so every `text-delta` changed the
key and forced Svelte to remount the visible agent message.

Compatibility contract:

- Existing duplicate-key protection remains through per-build row key counts.
- Timeline items with stable IDs keep DOM identity as their text, summary, or
  child activity updates.
- Content-hash fallback remains for malformed or future items without IDs.

Regression coverage lives in `chat-session-ui.spec.ts` and verifies that a
streamed agent row remains connected in the DOM across multiple deltas even
when no local user row exists.
