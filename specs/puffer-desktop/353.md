# 353. PDF Page Limit Badge And Viewer Zoom Focus

Long PDF previews must keep the page-limit notice visually separate from the
document surface and make keyboard zoom available after the user clicks the
rendered pages.

## Contract

- The long-document status badge uses amber foreground, background, and border
  colors that differ from the PDF paper, preview shell, and control strip.
- The PDF viewer is focusable as a document region so keyboard zoom shortcuts
  can be routed after ordinary interaction with the rendered pages.
- Clicking inside the PDF pages focuses the viewer without blocking the slider,
  zoom buttons, or page scrolling.
- Existing zoom buttons, range slider, and Ctrl/Command-wheel behavior stay in
  place.

## Coverage

`tests/files-workspace-ui.spec.ts` verifies that the page-limit badge has an
amber background and that clicking a PDF page enables Ctrl-key zoom without
first focusing a zoom-control button.
