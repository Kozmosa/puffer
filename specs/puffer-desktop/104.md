# Remote Connect Provider Resolution

## Scope

Connect Project's remote mode first shows provider choices from the local
daemon, then switches the app to the remote daemon before creating the session.
The remote daemon can have a different auth store, so the create-session
provider must be resolved against the remote settings after the switch.

## Change

- Provider option filtering is shared through a helper that accepts an explicit
  settings snapshot.
- Local project creation still resolves the selected provider against the local
  snapshot.
- Remote project creation reloads the connected remote settings snapshot before
  `create_session` and falls back to the remote default/authenticated provider
  if the originally selected local provider is unavailable there.

## Compatibility

The modal UI remains unchanged. Existing local creation behavior is preserved,
while remote creation now follows the provider ids and auth state of the daemon
that will actually own the session.
