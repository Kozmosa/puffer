# Browser New Tab Identity

## Scope

The Browser tab's plus button must create a distinct managed browser tab rather
than reusing the active tab.

## Change

- The renderer now passes a stable `tabId` when the user clicks New tab.
- The Playwright fake daemon mirrors the production `browser_agent open`
  contract: calls without `tabId` reuse the active tab, while calls with
  `tabId` create or reuse that specific tab.
- The desktop UI e2e suite covers the New tab button and asserts that the
  daemon receives `tab-2` for the first user-created tab.

## Compatibility

Agent-oriented browser opens can still omit `tabId` to reuse an existing tab.
The stricter identity is only added to the renderer's explicit New tab action.
