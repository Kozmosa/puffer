# Secret User Questions

Desktop user-question prompts understand `secret: true` on input questions.

- Pending secret questions render as password inputs instead of plain text.
- Answer summaries and read-only answered views show `Secret provided` rather
  than the submitted value or the returned placeholder.
- Generic chat prompts and connector setup prompts both preserve the `secret`
  flag when normalizing daemon question payloads.
- Existing wording-based password detection in connector setup remains as a
  fallback for older connector prompts.
