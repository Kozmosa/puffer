# 365. PDF Page Limit And Gesture Zoom

Long PDF previews must make the render limit obvious and support the zoom
gestures users expect in the desktop WebKit shell.

## Contract

- The page-limit notice, such as `Showing first 20 of 29 pages.`, owns a full
  high-contrast row in the PDF control strip instead of blending into the
  preview background.
- Existing zoom buttons, reset control, range slider, keyboard shortcuts, and
  Ctrl/Command-wheel zoom continue to resize already-rendered canvases in
  place.
- WebKit `gesturestart`, `gesturechange`, and `gestureend` events map to the
  same bounded PDF zoom state so trackpad pinch gestures work in the desktop
  shell.
- The 20-page render cap remains unchanged while zooming.

## Verification

`tests/files-workspace-ui.spec.ts` runs the long-PDF preview under WebKit,
checks that the page-limit notice is visually distinct and full-width, then
zooms the same rendered canvases with the slider, buttons, keyboard shortcuts,
Ctrl-wheel, and synthetic WebKit gesture events.
