# Desktop session provider auth filtering

New desktop session entry points now align their provider picker with the
authenticated provider set from the settings snapshot.

Compatibility contract:

- New agent creation shows only personal-agent providers that are already
  authenticated in the current workspace snapshot.
- Connect project uses the same provider filtering before it creates a local
  or remote session.
- Pure preview mode without a settings snapshot still keeps the built-in
  fallback provider list so static desktop review remains usable.
- Session creation requests cannot be initiated from these pickers with a
  visible but unauthenticated provider.

Regression coverage lives in `files-workspace-ui.spec.ts` with a Codex-only
auth snapshot.
