# Release Build Pipeline

## Scope

This update adds the top-level release Makefile used to build and package
Puffer Desktop, the TUI binary, and CEF runtime artifacts for GitHub release tag
`ct`.

## Targets

- `build-rust` builds the release `puffer` CLI/TUI binary.
- `build-tauri` builds the current platform Tauri bundle. On macOS it first
  attempts to download a `puffer-cef-macos-<arch>.tar.gz` release artifact.
- `build-macos` hard fails off macOS and runs the Rust and Tauri build path.
- `build-release-cef` ensures the Chromium Tintin checkout exists, pulls when
  safe, builds CEF with `autoninja`, packages the runtime plus CEF headers and
  wrapper sources, and uploads `puffer-cef-<platform>-<arch>.tar.gz`.
- `pack-macos` builds macOS, copies the daemon/TUI binary and resources into the
  `.app`, embeds CEF under `Contents/Frameworks`, uploads the app zip, and
  uploads a standalone TUI tarball.
- `build-linux` hard fails off Linux and runs the Rust and Tauri build path.
- `pack-linux` pushes the current branch, builds on `c@65.19.161.135` with
  `/mnt/lvm_data/chromium_tintin`, copies Linux artifacts back, and uploads them
  with the local GitHub credentials.
- `pack-linux-local` is the Linux-only packaging step used by the remote build.

## Artifacts

The pipeline writes generated artifacts under `release/` and working state under
`.release/`. Both paths are ignored by git.

Expected release assets are:

- `puffer-cef-macos-<arch>.tar.gz`
- `puffer-desktop-macos-<arch>.zip`
- `puffer-tui-macos-<arch>.tar.gz`
- `puffer-cef-linux-<arch>.tar.gz`
- `puffer-desktop-linux-<arch>.tar.gz`
- `puffer-tui-linux-<arch>.tar.gz`

## Compatibility

macOS Tauri builds prefer downloaded CEF release artifacts, then fall back to the
local `~/chromium_tintin` runtime so a fresh `ct` release can bootstrap itself.
Linux remote builds run with `NO_UPLOAD=1`; uploads are performed locally so the
remote host does not need GitHub credentials.
