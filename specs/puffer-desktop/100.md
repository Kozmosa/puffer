# Stop Cancellation Debounce

## Scope

The desktop chat composer should send at most one cancellation request for a
running turn while the daemon is still acknowledging the first request.

## Change

- `App.svelte` tracks the turn id currently being cancelled.
- The Stop button is disabled after the first click for that turn.
- Cancellation state is cleared when the turn settles, the session state is
  reset, or the cancel request fails.

## Compatibility

The daemon `cancel_turn` contract is unchanged. The renderer only suppresses
duplicate user clicks while a cancellation request is already in flight.
