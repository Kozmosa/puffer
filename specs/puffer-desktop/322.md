# MCP Browser Activity Recording

Browser activity rendering must treat MCP browser tools as browser actions, not
as generic textual tool output.

## Requirements

- Tool cards render browser recording frames for both the native `Browser` tool
  and MCP browser tools such as `mcp__browser__open`.
- Browser recording frame filters understand both direct action inputs and
  nested MCP `arguments`, including `tabId` and `backendSessionId`.
- The action history trajectory classifies MCP browser tools as browser
  actions and applies the same recording-frame matching rules.
- Existing non-browser MCP tools keep their structured/textual rendering.

## Verification

The agent detail UI suite includes an MCP browser tool call with a saved
recording frame. Expanding the activity details must show the recorded frame
and title instead of the empty browser-recording placeholder.
