# Settled Workspace Turn Reloads

## Context

The desktop shell listens to both session-stream events and workspace session
change events. A turn may settle through the workspace channel before the
selected detail view receives a matching session-stream completion event.

## Change

- Settled `workspace:sessions:changed` events for the selected session now
  trigger a quiet session-detail reload.
- The reload uses live-state reset semantics so canceled or completed local
  turn state cannot keep the composer in a running or cancel-requested mode
  after the daemon reports the session as idle.
- Generated-title and rename workspace events continue to reload the selected
  detail without resetting live turn state.

## Compatibility

Background sessions still update through the existing sidebar overlay and
group refresh path. Selected sessions keep the previous title/rename behavior,
while completion and error broadcasts now converge the detail header, composer,
and sidebar state on the daemon's current session metadata.
