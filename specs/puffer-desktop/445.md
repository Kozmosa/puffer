# Open session routing event refresh

- The desktop listens for `workspace:sessions:changed` events with reason `session_routing`.
- When the currently open session receives that event, the detail reloads without resetting live turn state.
- This keeps the composer model picker, provider label, and next `run_agent_turn` route aligned with provider/model changes persisted by the daemon or another UI surface.
- Settled turn events still use the reset path; title and rename events continue to use the same lightweight reload path.
