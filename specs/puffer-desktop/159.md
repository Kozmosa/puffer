# Onboarding Refresh Dismissal

## Scope

Desktop onboarding should stay dismissed after the user clicks Continue during
the current app load. Settings refreshes and auth refreshes must not re-open the
intro screen unless agent-provider auth is actually missing.

## Change

- `shouldShowOnboarding` now respects the in-memory `onboardingCompleted` flag
  before falling back to the startup `skipOnboarding` value.
- The missing agent-provider auth guard still wins, so completed onboarding does
  not bypass the login requirement.
- Playwright coverage now completes onboarding without the skip flag, refreshes
  Settings, and verifies the user remains on Settings instead of being bounced
  back to the onboarding screen.

## Compatibility

Forced onboarding still appears until the user completes it. Existing skip-flag
behavior and no-auth login protection remain unchanged.
