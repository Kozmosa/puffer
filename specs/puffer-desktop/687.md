# Native CEF Window Handle Fix

- Native CEF smoke and prewarm setup runs only when the macOS CEF bridge is
  compiled.
- Tauri setup discovers the main window as a `WebviewWindow`; smoke and prewarm
  helpers now accept that type directly and read its native macOS window handle.
- The macOS CEF host keeps the native child view focused for browser mouse
  input, keeps CEF message-loop work active during event tracking, and restacks
  the child view above Tauri content after bounds updates.
- Native CEF disables background renderer throttling and, unless explicitly
  overridden, GPU compositing to avoid checkout pages freezing on stale overlay
  surfaces inside the embedded macOS child view.
- The desktop daemon launcher replaces a lock-owning `puffer daemon` on local
  startup conflict before falling back to attachment, so a restarted desktop
  does not keep routing agent browser actions to an orphaned CEF DevTools port.
- This keeps non-CEF builds unchanged while allowing CEF-enabled dev and
  release builds to compile and interact with the smoke/prewarm path.
