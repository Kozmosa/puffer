# Pipeline Connector Connection Names

## Summary

Corbina Pipelines lets users choose the exact connection name used by connector
setup commands.

## Behavior

- Selecting a connector seeds a `Connection name` field from the existing
  connection for that connector, or the daemon-provided suggested connection
  slug when no connection exists.
- Editing the field rewrites the staged deterministic command to
  `/connect <connector_slug> <connection_name>`.
- Trigger-capable connector selections keep the planned workflow trigger in
  sync with the edited connection name while the name is valid.
- The field accepts only lowercase letters, digits, and hyphens so the GUI does
  not dispatch `/connect` commands that would fall back to additional prompts.
- Invalid names disable copy/run actions and keep the command preview explicit.

## Compatibility

Existing workflow snapshots are unchanged. The UI still honors
`connect_command` for the suggested default connection name and only constructs
an alternate command when the user edits the connection name.
