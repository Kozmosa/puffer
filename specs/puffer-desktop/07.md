# Files Tab Close Semantics

## Scope

The Files pane must expose stable, valid tab and close controls for keyboard
users and browser automation.

## Change

- File tab headers are now `role="tab"` containers instead of native buttons.
- File tab close affordances are native `button` elements with accessible names.
- Enter and Space still activate a focused file tab.
- The desktop e2e fake daemon now supports enough file RPCs to render restored
  file tabs and verify the close-control behavior.

## Compatibility

The tab visual styling and close behavior are unchanged, but the DOM no longer
nests one interactive control inside another.
