# Recap Blur Requires Open Agent Detail

## Context

The desktop auto-recap timer used the last selected session as its target. After
the user returned to the workspace board, `selectedSession` still pointed at the
last opened session, so a later window blur could submit `/recap` even though
no chat detail was open.

## Change

- The recap blur timer now arms only when the selected session is still open in
  AgentDetail.
- The timer callback rechecks that the same AgentDetail is still open before
  submitting `/recap`.
- The regression covers opening a chat, returning to the workspace board, and
  blurring past the recap delay.

## Compatibility

Auto-recap behavior is unchanged while an agent detail is visible. Running turns
and composer drafts still block auto-recap as before.
