# Message links and generated artifact paths

Desktop chat rendering now keeps underscore-delimited file names intact by
avoiding underscore emphasis parsing inside word/path tokens.

Markdown links with empty labels now fall back to a visible URL host or local
file basename, so citation-style `[](https://...)` links do not render as
invisible anchors.

Assistant messages also autolink bare absolute local paths, including optional
line suffixes. Structured tool rows that contain local file paths, such as
generated image outputs, now call the same file-opening callback used by chat
links.
