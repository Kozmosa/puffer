# 324. Browser Action Recording Selection

Browser action detail panels reload their recorded frames whenever the selected
tool action changes inside the same activity group. A reused `ToolCard` must not
keep showing the previous Browser action's screenshots after the user selects a
different Browser action.

Contracts:

- The recording key includes the root session, tool call id, tool name, browser
  action, URL, backend session id, and tab id.
- Changing any part of that key clears the old frame list and selected frame.
- Snapshot loads and live recording subscriptions are generation-guarded so
  late responses from a previously selected action cannot overwrite the current
  action.
- Activity details with multiple Browser actions only show frames matching the
  currently selected action.

Playwright coverage opens one activity with two Browser actions, selects the
first recording, switches to the second action, and verifies that only the
second action's recorded frame remains visible.
