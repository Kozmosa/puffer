# Browser close response ownership

Browser tab close responses are now tied to the session generation and tab
state version that issued the close request.

If a user closes a tab and immediately switches to another agent, the delayed
daemon response for the previous agent is ignored instead of applying its tab
list to the current Browser pane.

The fallback backend close call also uses the backend session id captured at the
time of the close request, so cleanup cannot drift to a newly selected agent.
