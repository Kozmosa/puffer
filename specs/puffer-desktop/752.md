# Axis-native media settings

Desktop media settings must persist logical selections instead of provider
adapter payloads. Saved image and video settings use:

- `providerId`
- `logicalModelId`
- `selections`

The settings modal renders daemon media capability `axes` in the daemon-provided
order. Enum axes render as selects when more than one value is available and as
read-only values when fixed. Range axes render bounded numeric inputs. Bool axes
render checkboxes. Malformed axes block saving and surface an inline warning.

Saved selections are normalized against the selected capability axes before
rendering and before save. Unknown saved keys are dropped, invalid values fall
back to valid axis defaults, and old persisted `modelId`, `adapter`,
`parameters`, or `defaults` settings are not retained.
