# Browser Close Failure Preservation

## Scope

Closing a desktop Browser tab requires daemon acknowledgement. If the daemon tab
close fails, the UI must not remove the tab or hide the failure from the user.

## Change

- `BrowserPane.svelte` now waits for `browser_agent close` to succeed before
  removing a tab from the local tab strip.
- Failed close attempts keep the tab visible and surface the daemon error in the
  Browser pane.
- A close in-flight guard disables the tab close button while the close request
  is pending, preventing duplicate close RPCs while the retryable tab remains.
- Legacy daemon fallback is kept only for unsupported tab-close actions.
- A Playwright regression test forces `browser_agent close` to fail and verifies
  the tab remains visible.

## Compatibility

The daemon-owned Browser tab protocol is unchanged. Successful closes still
apply the daemon tab list, while failed closes preserve a retryable UI state.
