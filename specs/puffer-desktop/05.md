# Browser Active Tab Reconnect

## Scope

The Browser pane must follow active-tab changes announced by the daemon.

## Change

- The renderer now compares the previous active tab with each real-time tab-list
  event.
- When the active tab changes, the Browser pane reconnects subscriptions and
  resizes or opens the backing browser session for the new active tab.
- The e2e suite covers daemon-driven active-tab changes and expects a resize
  request for the new backend tab id.

## Compatibility

Manual tab selection is unchanged. This only closes a stale-subscription gap
when the daemon or an agent changes active browser state underneath the UI.
