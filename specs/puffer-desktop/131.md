# Question Answer In-Flight Guard

## Scope

Pending desktop question prompts should send at most one answer request while a
previous `resolve_user_question` call for the same prompt is still awaiting the
daemon response.

## Change

- `App.svelte` tracks question ids that are currently resolving.
- A repeated submit for the same question id returns before touching the daemon.
- The in-flight marker is cleared after success or failure so failed answers
  remain retryable.
- Session and daemon-scope resets clear the marker to avoid stale guards after
  workspace or session changes.

## Compatibility

The daemon protocol is unchanged. This only prevents duplicate renderer-origin
`resolve_user_question` requests caused by rapid repeated clicks or key submits.
