# New Agent Provider Availability Hint

## Scope

The New Agent modal can open in workspaces where the settings snapshot has
authentication for a non-agent integration, but no usable personal provider such
as Codex/OpenAI, Anthropic/Claude, or Puffer. In that state the Start agent
button is disabled and the user needs an explicit recovery path.

## Change

- NewSessionModal now shows a Settings hint when the filtered provider list is
  empty.
- The existing Start agent disabled guard remains in place.
- Playwright coverage opens New agent with only an unrelated authenticated
  provider and verifies the hint is visible, Start agent is disabled, and no
  `create_session` request is sent.

## Compatibility

Provider selection and session creation are unchanged when a usable agent
provider is authenticated. This only adds explanatory UI for the empty provider
state.
