# 275. Browser Tab Close Target Labels

## Summary

Browser tab close buttons now describe the exact tab they close. The desktop
browser previously rendered every tab close control as the same `Close tab`
button, which made multi-tab interaction ambiguous for keyboard users,
assistive technology, and Playwright-based interaction tests.

## Design

- Each close button is labeled `Close tab <index>: <target>`.
- The target uses the full untruncated title or URL, and falls back to
  `blank page` for empty tabs so it does not collide with the New tab button.
- The visible tab title can still be truncated for layout stability.
- The button remains a native `<button>` and keeps the existing close request
  behavior.

## Compatibility

This changes only browser tab button metadata and tooltip text. Browser tab
state, daemon request payloads, and local optimistic close behavior are
unchanged.

## Tests

- Playwright opens a second browser tab and asserts no exact generic `Close tab`
  button remains.
- The same regression closes `Close tab 2: blank page` and verifies the tab
  list shrinks normally.
