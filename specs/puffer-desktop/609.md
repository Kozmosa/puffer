# Realtime client-secret desktop API surface

## Summary

The desktop API exports a small `createOpenAIRealtimeClientSecret` helper and
matching TypeScript types. The helper calls the daemon RPC directly and returns a
browser-safe client secret for future voice UI experiments.

## Contract

- The frontend API accepts optional provider/model/voice/reasoning/session
  fields but never accepts or returns a provider API key.
- The response type contains only `providerId`, `model`, `voice`, `clientSecret`,
  and `expiresAt`.
- There is no visible voice UI in this change; it is only the integration seam.
