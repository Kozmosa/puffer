# Puffer Desktop Update 626

## Summary

The Linux CEF release build now restores the `BrowserWidget` hooks required by
CEF's Chrome Views browser widget.

## Design

- `scripts/release.sh` adds a compatibility pass for the subset of CEF's
  `chrome_runtime_views.patch` that current Chromium rejects.
- The pass adds the CEF-safe default `BrowserWidget` constructor and protected
  setters for the browser view and frame view.
- `BrowserWidget::UserChangedTheme` is made virtual again so CEF's
  `ChromeBrowserWidget` override receives theme-change dispatch.
- `SelectNativeTheme` is exposed to derived widgets, matching CEF's expected
  Views integration surface.
- The destructor guards browser-view teardown for instances initialized through
  the CEF embedding path.

## Compatibility

The patch only touches Chromium source during CEF release packaging. Normal
Chrome widget construction still passes a `BrowserView*`, while CEF widgets can
construct first and attach browser state later.
