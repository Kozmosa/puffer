# Connect Project Mode Status Isolation

## Scope

The Connect Project modal has Local and Remote modes with different actions and
failure modes. A remote creation or SSH failure should not remain visible after
the user switches back to the local project form.

## Change

- Connect Project mode switches now clear transient status text, error text,
  and SSH remediation hints before rendering the next mode.
- Existing typed Local and Remote fields are preserved so users can move between
  modes without losing form input.
- The directory picker close behavior from prior mode isolation remains in the
  same helper.
- Playwright coverage now forces a failed remote project creation, switches to
  Local, and verifies the stale remote error is removed while the local
  directory field is visible.

## Compatibility

Project creation, clone, and remote daemon handoff protocols are unchanged.
Busy operations still lock mode switches until the active request settles.
