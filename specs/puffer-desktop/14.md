# Browser Cursor Probe Disconnect Guard

## Purpose

Mouse-move cursor probing must not emit daemon requests after the active browser
tab disappears during the probe debounce window.

## Design

Cursor probes now capture the backend session id when the pointer move is
scheduled. The delayed probe only runs if the same connected active tab is still
current, and the async response only applies if that session remains active.

## Logic

- Pointer moves store both the viewport point and the current backend session
  id.
- The debounce timer clears the pending point/session together.
- Empty tab state, disconnected tabs, and tab switches cancel the pending
  cursor request.
- Late cursor responses cannot update the active cursor style for a different
  tab.

## Contracts

The Browser pane must never call `browser_cursor` with an empty or stale session
id. The Playwright harness moves over the canvas, clears daemon tabs before the
debounce fires, and asserts that no cursor RPC is sent.
