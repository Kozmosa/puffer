# 404. Cached Model Picker Refresh Failures

The chat model picker preserves the last successfully loaded model list when a
provider refresh fails. Reopening the picker still asks the daemon for a fresh
catalog, but a transient provider, network, or discovery error no longer
replaces a usable cached list with an empty menu.

The picker surfaces the refresh failure as a non-blocking inline error while
keeping the selected chat session on its existing provider and model. First-time
loads with no cache still show an empty model list plus the same error state.
