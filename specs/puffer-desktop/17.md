# Chat Completion Text Fallback

## Purpose

The desktop chat transcript must show final assistant text that arrives only in
the `turn-complete` event, even if no prior streaming text delta was emitted and
the immediate persisted transcript reload is stale.

## Design

Turn completion now materializes non-empty `assistantText` as a live assistant
item when the current live stream does not already contain that text. The item
is still filtered out after a reload once matching persisted assistant content
exists.

## Logic

- Completion text is trimmed and ignored when empty.
- Existing live assistant content suppresses duplicate fallback insertion.
- The fallback item participates in the same persisted-content filtering as
  streamed live items.

## Contracts

Users should never wait for a later manual refresh to see final assistant text.
The Playwright desktop harness emits `turn-start` followed immediately by
`turn-complete` with `assistantText` while the fake daemon returns a stale
session detail, then asserts that the final text is visible.
