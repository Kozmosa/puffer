# Empty workspace new agent

## Purpose

An empty Personal Agent workspace must still let users start a normal agent in
the daemon's default workspace without first connecting or importing a project.

## Behavior

- The empty workspace state exposes a primary New agent button.
- The button opens the same provider-selection modal used by project rows.
- The created session uses the daemon-provided default workspace cwd.
- The button is disabled only when the parent did not provide a new-agent
  handler or the default workspace path is unavailable.

## Compatibility

Existing project rows and Connect project flows are unchanged. Workspaces with
sessions continue to show the normal project list.
