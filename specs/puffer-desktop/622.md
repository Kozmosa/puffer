# Linux CEF Thread Type Compatibility

## Summary

CEF branch 7680 maps `TP_DISPLAY` to `base::ThreadType::kPresentation`.
The local Chromium 146 checkout no longer exposes that enumerator; display
critical work is represented by `base::ThreadType::kDisplayCritical`.

## Design

`apply_cef_compatibility_patches` now detects CEF's stale
`base::ThreadType::kPresentation` usage in `cef/libcef/common/thread_impl.cc`.
When Chromium's `platform_thread.h` lacks `kPresentation` and exposes
`kDisplayCritical`, the release build rewrites CEF's display-priority mapping
to `kDisplayCritical`.

The mutation is idempotent, guarded by Chromium's current enum contents, and
limited to the CEF checkout in the release build tree.

## Compatibility

`kDisplayCritical` is Chromium's current priority class for threads critical to
compositing and presenting foreground content. That preserves the intent of
CEF's `TP_DISPLAY` mapping without weakening the display thread to the default
priority class.
