# Desktop Running Session Composer Guard

## Context

Workspace and sidebar metadata can report an agent session as `running` before
the current desktop window has a local turn id. Reopening that session should
not allow a second prompt to start another concurrent turn.

## Change

- Agent detail status now respects daemon `activityStatus` for `running` and
  `awaiting` sessions.
- The chat composer disables input and send controls while the selected agent
  is busy according to daemon metadata.
- `submitMessage` also rejects sends for daemon-busy sessions so keyboard or
  stale UI paths cannot start another turn.

## Compatibility

Local live turns still use the existing cancellable `Stop turn` path when the
desktop has a turn id. Reopened daemon-running sessions remain visible as busy
but do not expose a cancellable stop action until a live turn id is known.

## Verification

- Added a Playwright regression that opens a daemon-running session and asserts
  the composer and Send control are disabled with no `run_agent_turn` request
  emitted.
