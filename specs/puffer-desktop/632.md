# Puffer Desktop Update 632

## Summary

Linux CEF release builds now restore Chromium's ToolbarView constructor shape
expected by CEF's Chrome runtime view wrappers.

## Design

- `scripts/release.sh` adds an idempotent ToolbarView compatibility patcher to
  the CEF release patch chain.
- The patcher gives `ToolbarView` an optional display-mode constructor argument
  with Chromium's normal `GetDisplayMode(browser)` behavior as the default.
- The constructor implementation now accepts CEF's explicit display mode when
  the CEF toolbar wrapper builds location-only browser views.

## Compatibility

Normal Chromium browser windows keep using the inferred toolbar display mode.
CEF can pass an explicit display mode for embedded browser views without
requiring a stale CEF patch application to land cleanly.
