# 308. Agent-Capable OpenRouter Model Filtering

## Summary

OpenRouter models that explicitly do not support the `tools` request parameter
are no longer allowed to start an agent turn from the desktop composer.

## Behavior

- Provider discovery skips models whose `supported_parameters` array is present
  and does not include `tools`.
- The desktop composer treats `supportsTools: false` model descriptors as not
  send-ready.
- Automatic fallback chooses the default agent-capable model first, then any
  agent-capable model, and otherwise leaves the composer blocked.
- The model picker keeps unsupported rows visible but disabled with a
  `No agent tools` label, making the reason obvious instead of failing later
  with an OpenRouter routing error.

## Compatibility

Models without a `supported_parameters` field are preserved for older providers
and static resource files. Custom fine-tuned model IDs remain allowed as before.
