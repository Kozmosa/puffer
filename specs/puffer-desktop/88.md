# Project detail memory history visibility

Project Detail memory now derives session memory entries from the full sorted
session set instead of the first five sessions.

The Memory tab can therefore expose older sessions after a busy project has
created many agents. The synthetic memory file names keep their current
display-order numbering, while the underlying entry id remains the session id.
This preserves the existing UI contract and fixes the history-discovery gap
without changing daemon storage or transcript persistence.

Session memory detail titles now fall back to the session display name when
the optional secondary title is suppressed as a duplicate. This avoids blank
memory detail headers for normal sessions whose display name and title match.
