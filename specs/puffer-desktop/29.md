# Composer thinking option coverage

## Purpose

The desktop chat composer should let users choose a reasoning level when the
active model supports it, and the selected level must be sent with the turn.

## Behavior

- The FakeDaemon model fixture exposes `thinkingOptions` and a default thinking
  option for the default model.
- Desktop e2e coverage waits for the thinking selector to become enabled,
  changes the selected effort, sends a message, and asserts `run_agent_turn`
  carries `thinkingOptionId`.

## Compatibility

Models without thinking options still render the selector disabled. The test
harness now matches the real daemon's richer model descriptors more closely.
