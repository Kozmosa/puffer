# Settings default model loading

## Purpose

The Settings Providers pane must not let users save a default route while the
selected provider's model list is still loading, because an empty picker value
would clear `default_model`.

## Behavior

- Provider model loads are tracked per provider.
- Switching providers clears the model picker and disables the model select
  plus Save button until models arrive.
- Once models load, an empty model picker is filled with the provider default,
  or the first returned model.
- Saving requires daemon connectivity, a provider, a model, no in-flight load,
  and no in-flight save.

## Compatibility

The settings RPC shape is unchanged. Existing provider lists still render from
the settings snapshot, and providers with no models remain unsavable until a
real model is available.
