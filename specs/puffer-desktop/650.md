# Secrets Sync From Chrome

The desktop Secrets settings tab presents Chrome import as synchronization and
surfaces secret descriptions.

- Manual secret creation uses `Name` plus optional `Description`, username,
  origin, and value fields.
- `save_secret` carries `description` through the desktop API and Tauri backend
  into the encrypted vault.
- Secret rows display description before falling back to origin so Chrome synced
  rows show the site as their description.
- The Chrome action is labeled `Sync from Chrome` and still uses the existing
  `import_chrome_secrets` RPC for compatibility with older daemon naming.
- Fake daemon and settings UI regression data include the new description
  metadata.
