# Browser Paste Disconnect Guard

## Purpose

The managed Browser pane must not send clipboard input to a stale or empty
backend session after daemon tab state clears the active browser tab.

## Design

Paste handling now follows the same connection guard as keyboard, pointer, and
wheel input. Clipboard text is forwarded only when the Browser pane has both an
active tab id and an established backend connection.

## Logic

- Empty tab-list events clear `activeTabId` and mark the pane disconnected.
- Paste events received by the still-mounted canvas are ignored while there is
  no connected active tab.
- Connected tabs still use the text insertion path so multi-character clipboard
  contents reach the backend as a single text event.

## Contracts

The Browser UI must never call `browser_input` with an empty browser session id.
The Playwright desktop harness covers this by clearing tabs, dispatching a real
clipboard paste event to the canvas, and asserting that no daemon input request
is emitted.
