# Active Agent Rename Sync

## Context

Renaming a session updates its primary display name while preserving the old
raw title as secondary detail. The grouped active-agent sidebar rendered the
secondary title first, so a renamed open session still appeared under the old
name in the active agents list.

## Change

- Render the active-agent sidebar row from the primary session name before
  falling back to the secondary title.
- Keep the existing secondary-title behavior in the agent detail header.

## Compatibility

This does not change daemon metadata or session history storage. It only
changes which already-loaded label the desktop sidebar chooses for an active
agent row.

## Verification

- The existing Playwright title-edit regression now checks that the sidebar row
  reflects the renamed session.
