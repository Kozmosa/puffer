# PDF and legacy DOC WebView compatibility

The desktop Files pane keeps PDF rendering inside the app, but the renderer must
work on macOS WebViews that do not expose the newest JavaScript built-ins used
by the standard pdf.js bundle.

- PDF previews load `pdfjs-dist/legacy/build/pdf.mjs` and its matching legacy
  worker so older WebKit builds can render pages without `Promise.withResolvers`.
- The common document preview regression deletes `Promise.withResolvers` before
  app boot and still expects the PDF canvas to render.
- Legacy `.doc` previews now recognize RTF and HTML-as-DOC payloads before the
  binary Compound File fallback. This covers old Word export formats that are
  text-based but commonly carry the `.doc` extension.

The daemon file transport contract is unchanged: document extensions continue to
use base64 payloads so the frontend can choose the appropriate preview parser.
