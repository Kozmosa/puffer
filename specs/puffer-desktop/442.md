# 442. Auth-Free Providers Satisfy Onboarding

## Summary

Desktop onboarding treats auth-free local agent providers as ready providers instead of forcing API key login.

## Design

- The app-level onboarding gate now checks shared provider availability instead of only auth rows.
- The onboarding screen counts agent-capable providers that are either authenticated or explicitly auth-free.
- Logout only returns to onboarding when no available agent provider remains.

## Compatibility

Credentialed providers still require auth before the workspace opens. Non-agent providers such as GitHub do not satisfy onboarding.

## Tests

Playwright covers forced onboarding with an auth-free Ollama fixture and asserts the workspace-ready state appears instead of the Anthropic API key form.
