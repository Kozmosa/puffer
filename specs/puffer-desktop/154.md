# Settings Default Route Agent Providers

## Scope

The Settings Providers pane chooses the default provider and model used when a
session does not already carry routing metadata. That control must not save
credentials for non-agent integrations such as GitHub as the default turn route.

## Change

- Settings default routing now lists only authenticated providers that can run
  agent sessions.
- Stale or imported configs that name a non-agent default provider fall back in
  the UI to the first authenticated agent provider and clear the stale model
  until models load.
- Provider model loading ignores non-agent provider ids.
- Playwright coverage verifies a GitHub-only default is not offered and saving
  persists the recovered OpenAI route instead.

## Compatibility

Git-hosting credentials continue to appear in Settings account and Git/PR
sections. Codex/OpenAI, Claude/Anthropic, and Puffer defaults keep their alias
behavior.
