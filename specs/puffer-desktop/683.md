# Video Settings Provider Options

Date: 2026-06-09

## Scope

The desktop video generation settings modal must expose provider-declared
string enum options beyond the primary aspect ratio and duration controls.

## Behavior

- Video aspect ratio matching accepts `aspect_ratio`, `ratio`, and `size`
  parameter names or request fields.
- Video duration matching accepts `duration` and `seconds` parameter names or
  request fields.
- Request fields with dotted metadata paths normalize for matching, so
  `metadata.resolution` can still render as a user-facing `Resolution` select.
- Non-primary video enum parameters render with the same select/read-only rules
  used by image settings and are saved into the media config parameters map.
- Saved provider-specific primary parameter names are read after capability
  loading, so descriptors that expose `metadata.ratio` through a local
  parameter such as `video_shape` restore the selected aspect ratio.
- Stale saved non-primary video parameters fall back to the provider default in
  the form and are pruned to declared parameter names on save.

## Compatibility

Existing saved `aspect_ratio` and `duration` settings remain readable. Provider
resources can now add video enum parameters without adding bespoke UI controls.
