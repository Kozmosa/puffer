# Contact Context Fake Daemon Scope

## Summary

The desktop fake daemon now mirrors production `contacts_context` scoping by
normalizing requested contact ids and rejecting empty or invalid scopes.

Saved contact list responses also mirror production by returning all matching
saved contacts while applying the limit only to connector candidates.

## Compatibility

No visible Contacts screen behavior changes. This keeps future browser tests
from accidentally treating a malformed context request as an all-contact query.
