# Browser Devtools Stream Guard

## Problem

Browser frame and state streams ignored late events once the active browser tab
changed, but the devtools stream appended console/network events without the
same active backend-session guard. Because tab ids are reused across agent
sessions, stale console events could appear in the currently visible Browser
panel.

## Design

- Apply the same `activeEventSessionId` guard to devtools events that frame and
  state events already use.
- Keep devtools history scoped to the active backend browser session and tab.
- Preserve normal console/network rendering for the current tab.

## Test Coverage

- `desktop-ui.spec.ts` now switches from an alpha Browser agent to a beta
  Browser agent, emits a stale alpha devtools event and a current beta event,
  and asserts only the beta event is visible.
