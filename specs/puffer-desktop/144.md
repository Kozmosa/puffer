# Credential Operation Busy Lock

## Scope

Provider login, logout, and external credential import all mutate the same auth
snapshot. The desktop UI must treat them as one credential operation lane so a
slow import cannot overlap with an API-key login, OAuth flow, or sign-out.

## Change

- LoginView now disables provider OAuth buttons, API-key fields, API-key connect
  buttons, and external credential import buttons while any credential operation
  is in flight.
- The app-level handlers reject provider login, logout, and external import
  attempts while either credential busy flag is active.
- The Settings account sign-out control also observes external import busy
  state.
- Playwright coverage delays import and OAuth requests to verify both directions
  of the UI lock.

## Compatibility

No daemon protocol changes are required. Credential requests still use the same
payloads; the desktop shell now serializes them consistently before they reach
the daemon.
