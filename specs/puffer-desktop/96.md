# First Prompt Reload De-Dupe

## Scope

The desktop chat should not flash duplicate messages when the first submitted
prompt causes the daemon to generate a title and refresh the selected session
while the agent turn is still running.

## Change

- Non-reset session reloads now reconcile persisted transcript rows with the
  optimistic submitted message and live stream rows before updating
  `sessionDetail`.
- Persisted user and assistant messages reuse matching transient DOM ids, so a
  title refresh or rename reload does not remount visible chat rows.
- Transient rows are dropped only when the refreshed persisted transcript
  already contains the same role/body content.

## Compatibility

Daemon event and transcript contracts are unchanged. The renderer still treats
the persisted transcript as canonical; it only prevents equivalent transient
rows from being rendered alongside it during mid-turn refreshes.
