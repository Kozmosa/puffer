# 229. Active Agents Hide Child Sessions

## Summary

The desktop Active Agents sidebar now treats sessions with a `parentSessionId`
as child/subagent sessions and does not render them as independent top-level
agents.

## Behavior

- Folder groups and session detail payloads keep child session metadata intact.
- Active Agents is derived only from top-level sessions.
- Live sidebar fallback rows also ignore child sessions, so streamed subagent
  updates do not create duplicate top-level agent rows.
- A directly selected child session can still load through normal session
  detail APIs, but it is not promoted into the top-level Active Agents list.

## Compatibility

No daemon protocol changes are required. The existing `parentSessionId` field is
used as the UI boundary between root user sessions and child/subagent sessions.

## Tests

`plus-121.spec.ts` covers a running parent session with a running child session
and verifies only the parent appears in the Active Agents sidebar.
