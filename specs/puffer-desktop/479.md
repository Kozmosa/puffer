# Runnable Pipeline Connector Commands

## Summary

Corbina Pipelines now lets users act on the deterministic connector setup
command shown by the connector picker. The command remains visible, and the
picker adds compact copy and run actions next to it.

## Behavior

- Selecting a connector still stages its deterministic `/connect <connector>
  <connection>` command.
- The copy action writes the selected command to the clipboard when the browser
  allows clipboard access and reports a local notice if it cannot.
- The run action sends the selected command through the active agent session.
  If no session is selected, Corbina creates a session in the default workspace
  first, opens it, and then submits the command.
- Starting setup switches back to the workspace so any follow-up
  `AskUserQuestion` prompts emitted by the connector flow are visible in the
  agent transcript.

## Compatibility

This preserves the `/connect` contract and does not add connector-specific
desktop auth handling. Connector setup remains deterministic and flows through
the same agent turn path as manually typed slash commands.
