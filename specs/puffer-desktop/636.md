# Puffer Desktop Update 636

## Summary

Linux CEF release builds now handle Chromium's current touch-selection active
status enum spelling.

## Design

- `scripts/release.sh` adds a touch-selection compatibility patch.
- The patch rewrites CEF's stale `ActiveStatus::kInactive` reference to
  Chromium's current `ActiveStatus::INACTIVE` spelling when the current enum is
  present.

## Compatibility

The rewrite only runs when Chromium lacks `kInactive` and exposes `INACTIVE`,
so newer CEF/Chromium pairings that already agree are left unchanged.
