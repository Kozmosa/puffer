# Add remote binary override to connect project

The Connect Project remote flow can now pass a custom Puffer binary path when
starting an SSH daemon. This makes the existing "binary not found" remediation
actionable from the same modal that produced the error.

Compatibility contract:

- Local connect behavior is unchanged.
- Remote connect still defaults to `puffer` on the remote `$PATH`.
- The optional Remote binary field is passed to `connectSshDaemon` only when
  non-empty.
- The command-not-found hint points users to the visible Remote binary field
  instead of an implementation option name.

Regression coverage lives in `files-workspace-ui.spec.ts`, ensuring the remote
mode exposes and accepts the binary override input.
