# Workspace History Note Search

Workspace search should keep project rows and session history aligned when a
match comes from session metadata rather than the visible title.

- Session history filtering now matches `session.note` in addition to display
  name, generated title, cwd, project label, and project path.
- Project rows already surface the same note as the agent step, so this keeps
  the two workspace sections from disagreeing for searches like approval notes
  or browser replay hints.

Regression coverage searches for a phrase that appears only in a session note
and expects both the project row and the Session history entry to remain
visible.
