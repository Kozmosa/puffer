# Persisted Infer Modal Proposals

The Contacts screen now hydrates inferred proposals from the contacts snapshot
instead of relying only on component-local state. Reopening the screen or
reloading the page keeps the latest inferred proposals visible without starting
another inference run.

The fake daemon mirrors backend behavior by storing proposals after
`contacts_infer` and returning them from `contacts_list`, which gives the UI
regression coverage for page reloads as well as modal reopen within a live
component instance.
