# 441. Auth-Free Local Providers In Agent Flows

## Summary

Auth-free local model providers such as Ollama, LM Studio, vLLM, and llama.cpp are treated as connected agent providers in the desktop GUI.

## Design

- Provider availability now accepts agent-capable providers with an explicit empty `authModes` list.
- Settings default routing, new-agent creation, connect-project provider selection, the model picker, composer gating, and submit guards use the shared availability helper.
- Credentialed providers still require matching auth rows, and non-agent providers stay excluded.

## Compatibility

Authenticated OpenAI, Anthropic, OpenRouter, and similar providers keep the existing connected-provider behavior. Providers without an explicit empty `authModes` list are not implicitly treated as auth-free.

## Tests

Playwright covers both default routing and chat submission for an auth-free Ollama fixture.
