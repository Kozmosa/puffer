# Desktop Files Read Failure Scoping

## Context

The Files pane resets when the selected agent session changes, but in-flight
file reads from the previous session can still resolve afterward. `activateFile`
and active-file reloads only checked `activePath`, which is insufficient when
two sessions restore the same path such as `/tmp/puffer/src/main.rs`.

## Change

- Track a file-read generation inside `FilesPane`.
- Invalidate in-flight reads whenever the pane resets for a new root/session.
- Capture the expected root, session id, and generation before file read and
  reload requests.
- Accept file-read success, failure, and loading-state updates only when the
  generation, active path, root, and session id still match.

## Compatibility

Current-session read failures still render in the viewer. Only stale responses
from a previous session or older file-read generation are ignored.

## Verification

- Added a Playwright regression that delays a read failure in session A,
  switches to session B with the same active file path, and asserts B keeps the
  editor contents instead of showing A's stale error.
