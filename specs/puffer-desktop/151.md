# Terminal Inactive Close Failure Scope

## Scope

Terminal close failures should not turn a recoverable inactive-tab failure into
a full terminal outage. If the active PTY is still usable, the terminal host
must remain visible and interactive.

## Change

- TerminalPane now separates inactive tab close errors from pane-wide terminal
  failures.
- Failed closes for the active or only tab still render the existing full
  Terminal failed state.
- Failed closes for inactive tabs render an inline error below the tab strip and
  keep the active terminal attached.
- Playwright coverage opens two terminal tabs, fails the close request for the
  inactive tab, and verifies the active terminal host remains visible while the
  failed tab stays retryable.

## Compatibility

The PTY open, focus, replay, resize, write, and close protocol is unchanged.
Only client-side error presentation changes for inactive-tab close failures.
