# Browser Fuzz Click Storm Harness

## Purpose

The desktop Browser pane needs deterministic stress coverage for unstable UI
states that only appear after mixed clicking, key input, clipboard input, tab
state races, and daemon reconnect-like events.

## Design

The Playwright desktop suite now includes a deterministic click storm against
the mocked daemon. It alternates user actions with daemon tab-list mutations and
then validates global invariants across every browser RPC.

## Logic

- The storm exercises canvas clicks, pointer moves, printable keyboard input,
  clipboard paste, empty tab snapshots, restored tab snapshots, new-tab button
  clicks, and invalid active-tab snapshots.
- The fake daemon records all requests for post-run inspection.
- The invariant checker fails if any browser request targets an empty,
  missing, or undefined backend tab session.
- Page errors and console errors also fail the storm.

## Contracts

Browser UI stress tests must remain deterministic so failures are reproducible
from a single test name. This harness is intended to grow as new click-path bugs
are found without replacing the narrower regression tests.
