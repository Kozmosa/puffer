# Persisted Verified Skill Activity

Desktop timeline hydration keeps persisted Verified Skill Gate system records in
the surrounding Agent Activity row. Gate records are recognized by their
`Verified Skill Gate` body prefix as well as by title or metadata, so session
reloads and turn-complete refreshes do not show them as standalone SYS/ERR
messages.

MCP rendering only applies to explicit MCP tool names or inputs with a `server`
field. LambdaHostCall inputs may include `tool: "Bash"` as part of the verified
bridge contract, but that does not make the host call an MCP action. Persisted
LambdaHostCall rows therefore stay labeled as LambdaHostCall, while the real
Bash row owns the shell command display.

Validation:
- `npm run test -- --run chat-session-ui.spec.ts -g "verified skill gate events render inside agent activity with check details"`
