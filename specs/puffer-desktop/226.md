# 226. Provider Choice Is Selection-Only

## Summary

Provider choices in desktop creation dialogs now use native radio inputs rather
than action-like buttons.

## Behavior

- Selecting a provider updates only the selected provider.
- Starting a new agent or creating a project still requires the explicit
  `Start agent` or `Create` action button.
- The selected provider remains visible through the existing active styling, and
  keyboard focus now lands on a native radio control.

## Compatibility

The create-session daemon payload is unchanged. Provider ids are still
canonicalized at the explicit creation action boundary.

## Tests

`files-workspace-ui.spec.ts` covers that clicking an agent provider radio keeps
the New Agent modal open, does not open AgentDetail, and does not send
`create_session` until `Start agent` is clicked.
