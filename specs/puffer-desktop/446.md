# OpenAI provider labels

- Desktop chat labels prefer exact OpenAI provider resource metadata over the Codex alias.
- A configured `openai` provider with display name `OpenAI` is shown as `Engineer (OpenAI)`.
- Alias-equivalent sessions still fall back to `Codex` when only the `codex` resource is configured.
- Claude/Anthropic sessions keep the user-facing `Claude` label for reconnect and composer copy.
- The lookup keeps equivalent provider matching for legacy `codex`/`openai` sessions after exact ids fail.
