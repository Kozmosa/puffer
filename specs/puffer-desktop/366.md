# 366. PDF Text Fallback Visibility

Successful PDF rendering must keep extracted text out of the main document
surface so noisy parser output does not appear below the rendered pages.

## Contract

- Rendered PDF canvases are the primary view when pdf.js succeeds.
- Extracted text fallback remains available while the renderer is still loading.
- Extracted text fallback remains available when the renderer fails.
- Noisy PDF text tokens such as `EXTRACTED TEXT`, `CIDInit`, and `TeX-T1-0`
  must not appear in the successful rendered preview.

## Verification

`tests/files-workspace-ui.spec.ts` opens ordinary PDFs, ASCII-sniffed PDFs, and
PDFs with noisy daemon `textPreview` data. It asserts that successful renders
hide the fallback region, then separately forces the renderer to fail and checks
that the cleaned fallback text is still shown.
