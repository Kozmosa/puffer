# Native CEF tab-close lifecycle diagnostics

- Native CEF tab close now marks and hides the slot synchronously without
  calling CEF's destructive `CloseBrowser` path during normal in-app tab close.
- Deferred native close captures the session id and browser handle by value, so
  the scheduled close cannot miss the slot after the Tauri command returns.
- CEF new-tab allocation skips visible, closing, and recently closed tab ids so
  a new tab cannot reuse a native backend id that is still tearing down.
- The Browser pane persists the next local tab number and only moves it upward,
  so daemon tab-list shrinkage or a pane remount cannot reissue a native tab id
  that was already retired in the current desktop process.
- Retired native slots report disconnected state and reject late resize,
  navigation, reload, history, and reopen attempts as closing sessions.
- Frontend native CEF recovery treats `CEF browser session is closing` as stale
  retired-slot work rather than a global renderer failure. If the retired slot
  is still represented by a visible local tab, the pane allocates a fresh backend
  tab id and reconnects without demoting to screencast fallback.
- Empty daemon tab-list events are non-authoritative while local CEF tabs remain,
  unless the close path explicitly asked to shrink to zero tabs. This prevents a
  delayed daemon event from clearing the tab strip after closing only one local
  CEF tab.
- Native CEF open, navigate, reload, recovery, and history results schedule a
  short background state-settle poll. The pane applies the immediate state first,
  then refreshes the tab title, URL, favicon, loading flag, and address bar when
  native CEF reports the settled page state, without requiring the user to switch
  tabs.
- The Objective-C++ bridge emits `[puffer-cef-native]` lifecycle diagnostics when
  `PUFFER_BROWSER_DEBUG` is set, covering close requests, retired close work,
  `OnAfterCreated`, and `OnBeforeClose` slot cleanup.
- This preserves the existing callback-owned slot cleanup contract while making
  multi-tab close exits easier to distinguish from frontend or daemon tab
  metadata races.

Compatibility: the Tauri command surface is unchanged. Close still returns
`{ ok: true }` for missing or retired slots. Hidden retired native slots are
kept for the process lifetime because the current CEF child close path can
invalidate the host Tauri window when multiple embedded browsers exist.
