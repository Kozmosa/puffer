# Failed turn start recovery

Desktop chat submission treats `run_agent_turn` acceptance as the boundary for
clearing local input and rendering the local user transcript row.

Compatibility contract:

- Clicking Send starts the turn request and disables duplicate sends while the
  request is in flight.
- If the daemon accepts the turn, the composer clears and the local user row is
  rendered before streamed assistant activity.
- If the daemon rejects the turn start, the composer draft stays intact, no
  unsent user row is added, and the existing `Agent start failed` system error
  remains visible.
- Provider-auth blocks keep the same user-facing reconnect behavior and also
  report a rejected submission to the composer.

Regression coverage lives in `apps/puffer-desktop/tests/chat-session-ui.spec.ts`.
