# 225. Hydrate Browser Pane From Agent Recordings

## Summary

The desktop Browser pane now restores the most recent retained agent browser frame
when attaching to an already-connected daemon tab that has no local canvas frame.

## Behavior

- `BrowserPane` still subscribes to live tab, frame, state, and devtools events
  for the active backend tab.
- When a tab is already connected and the pane has not seen a live frame yet,
  it requests `browser_recording` for the root session and renders the latest
  matching frame for that backend tab.
- The restore is guarded by session generation, root session id, backend session
  id, active tab id, and existing-frame checks so stale async results cannot
  overwrite a switched session or newer live frame.

## Compatibility

This does not change daemon protocols or browser tab ownership. It only uses the
existing recording snapshot as a renderer-side hydration path for missed live
screencast events.

## Tests

`desktop-ui.spec.ts` covers a connected agent tab whose live frame events were
missed before the Browser pane mounted and verifies the canvas hydrates from the
recorded frame instead of staying blank.
