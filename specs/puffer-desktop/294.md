# 294. Isolate Per-Session Routing Errors in Group Listing

## Summary

A corrupted or unreadable routing state file for a single session no longer
prevents the entire `list_grouped_sessions` response from loading. The session
appears with default routing (no provider/model override).

## Behavior

- `apply_session_routing_to_groups` now catches per-session routing load errors
  and falls back to `provider_id: None, model_id: None` for that session.
- All other sessions in the workspace continue to load normally.
- The `apply_session_routing_to_detail` path (single session load) still
  propagates errors since it affects only one session.

## Compatibility

No protocol changes. Clients see `null` provider/model for sessions with
corrupted routing, which is the same as a freshly created session.

## Tests

`chat-session-ui.spec.ts` verifies that the session list renders even when one
session's routing state is invalid.
