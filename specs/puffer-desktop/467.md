# Longhunt Workflow Race Regressions

## Scope

The desktop UI must keep chat, browser, settings, and workspace interactions
recoverable across session switches, delayed daemon responses, reloads, and
failed daemon switches.

## Change

- Permission and question responses now keep their in-flight disabled state in
  cached per-session transient state, so switching away and back cannot submit
  the same decision twice.
- Submitted user prompts are persisted briefly in local storage while
  `run_agent_turn` is still starting, allowing a reload to recover the prompt
  until the daemon transcript catches up.
- Hidden turn-start failures are cached back onto the affected session instead
  of being dropped when the user is viewing another session.
- Browser shortcuts release the remote Control or Meta modifier before moving
  focus to local UI controls.
- Provider default-model saves are disabled while credential import is pending.
- Workspace search includes visible agent activity status and restores search
  focus when filtering removes the focused result.
- Failed browser remote workspace switches restore the previous daemon client.

## Compatibility

No daemon APIs or event payloads change. The persisted submitted-message cache is
renderer-local, namespaced by session id, and expires after ten minutes.
