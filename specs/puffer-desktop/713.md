# Shared Contact ID Normalization

The desktop contacts and task-monitor contact picker now share one contact ID
normalizer with the desktop fake daemon. The helper mirrors the daemon contract:
known prefixes are required, Telegram usernames are lowercase non-numeric
handles, Google addresses are lowercase, duplicate IDs are removed, and invalid
or blank IDs are ignored.

Contacts form submission now sends the sanitized ID set instead of raw textarea
lines. This keeps the post-save selection matcher aligned with the backend when
the user enters duplicate, invalid, or differently cased IDs alongside valid
ones. Task monitor contact scoping uses the same helper before creating or
updating monitor bindings.

The contacts Playwright regression covers a save with invalid and duplicate
IDs, verifies the sanitized RPC payload, and confirms the newly saved contact is
selected instead of an existing earlier contact.
