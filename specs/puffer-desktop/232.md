# 232. Active Project Collapse Uses Stable Keys

## Summary

The desktop Active Agents sidebar now stores project collapse state by a stable
workspace key instead of the displayed folder label.

## Behavior

- Active agent rows carry both a display project label and a stable project key.
- Sidebar project groups are keyed and persisted by the stable key.
- Same-name folders in different paths no longer inherit each other's collapsed
  state.
- User-facing group labels remain unchanged unless duplicate labels are already
  being disambiguated by path.

## Compatibility

No daemon protocol changes are required. Existing active-agent data is enriched
inside the desktop shell before it is passed to the sidebar.

## Tests

`plus-121.spec.ts` covers a stale `"puffer"` label in local storage and verifies
that a different `/tmp/other/puffer` project opens expanded, while path-keyed
collapse state still restores for the original workspace.
