# Browser Tab Close Pending Scope

## Context

The desktop Browser pane tracked in-flight tab closes by `tabId` only. Since
each agent session starts with `tab-1`, closing a tab in one session could leave
the close button disabled after switching to another session that also had
`tab-1`.

## Change

- Track pending tab closes by `{ sessionId, tabId }`.
- Gate duplicate close requests only within the same session/tab pair.
- Render close-button disabled state against the active session id and tab id.

## Compatibility

The daemon close protocol is unchanged. Late close responses still use the
existing generation/session guards before mutating visible tab state.

## Verification

- Extended the late Browser close regression to assert the switched session's
  close button remains enabled while the previous session's close is still
  pending.
