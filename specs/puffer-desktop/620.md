# Linux CEF Network Context Compatibility

## Summary

CEF branch 7680 expects `ContentBrowserClient::ConfigureNetworkContextParams`
to return `bool` so CEF can cancel network-context initialization during
shutdown. The local Chromium 146 checkout still has the upstream `void` API,
while CEF already generates a `bool ChromeContentBrowserClientCef` override and
patches `StoragePartitionImpl` to check the return value.

## Design

`apply_cef_compatibility_patches` now patches the browser-client API shape that
CEF expects:

- `ContentBrowserClient` base declaration and default implementation return
  `bool`.
- `ChromeContentBrowserClient` returns `true` after normal parameter setup.
- content shell and headless overrides also return `true`.
- CEF's existing override can still return `false` when the system network
  context manager is unavailable during shutdown.

The patch is idempotent and only rewrites exact stale signatures and known
normal-return sites.

## Compatibility

This preserves CEF shutdown behavior instead of weakening the injected
`StoragePartitionImpl` guard. Non-CEF browser clients keep their existing
configuration behavior and report success with `true`. The mutation remains
release-build local to the Chromium checkout.
