# Streaming Text Status Row

## Change

When an active turn has already rendered assistant text in the chat thread,
the desktop conversation view suppresses the standalone generic `Running`
status row.

## Design

The transcript remains the source of truth for visible streaming output. A
separate status row is still shown for pre-output states such as `Thinking`,
explicit hints such as `Retrying`, `Running tools`, `Awaiting approval`, and
`Waiting for answer`, or for a running turn that has no visible assistant text
yet.

## Compatibility

This keeps existing turn state, cancel behavior, tool cards, approval prompts,
and transcript reload behavior unchanged. The only user-facing difference is
that streaming prose no longer appears next to a second generic `Running`
agent row in the chat thread.

## Verification

A Playwright regression in `chat-session-ui.spec.ts` covers a streamed
assistant response and asserts that no standalone `Running` agent row remains
once the text delta is visible.
