# Puffer Desktop Update 624

## Summary

The Linux CEF release build now repairs Chromium/CEF context-menu API drift
after `cef_create_projects.sh` runs.

## Design

- `scripts/release.sh` applies an idempotent compatibility pass for the CEF
  context-menu hooks that may fail to apply cleanly against the Chromium fork.
- The pass restores CEF's menu-created and show-handler callbacks on
  `RenderViewContextMenu`.
- It restores menu-running checks through `ContextMenuDelegate`,
  `RenderViewContextMenuBase`, the views/mac concrete menu delegates, and
  `ToolkitDelegateViews`.
- It restores the accelerator delegation path used by CEF context-menu
  observers.
- It restores `source_web_contents()` access for CEF browser lookup.

## Compatibility

The release build remains compatible with partially patched, dirty Chromium
trees because every insertion uses a sentinel before modifying a file. The
patch only runs inside the CEF release setup and does not change Puffer runtime
behavior outside packaged CEF builds.
