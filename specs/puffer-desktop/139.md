# Provider Logout Busy Guard

## Scope

Settings account sign-out is an auth mutation and must be single-flight. Rapid
clicks on `Sign out` should not send duplicate `logout_provider` RPCs or race
settings snapshots.

## Change

- `App.svelte` now ignores logout requests while any provider auth operation is
  already in flight, matching OAuth, API-key, and external credential import.
- `Settings.svelte` disables account sign-out buttons while provider auth is
  busy and labels the active provider as signing out.
- A Playwright regression test double-clicks `Sign out` while the daemon delays
  `logout_provider` and verifies only one logout RPC is sent.

## Compatibility

The auth protocol is unchanged. Successful logout behavior is unchanged; repeated
clicks during an in-flight logout are treated as no-ops.
