# Pipelines Full Connector Catalog Visibility

## Summary

The Pipelines connector picker now mirrors the complete built-in connector catalog
in desktop test fixtures and exposes a compact result count while users search.

## Design

- The connector picker keeps the search-first layout introduced for workflow
  setup and connection repair.
- A muted summary line under the connector search input reports
  `<visible>/<total> connectors; <visible>/<total> connections`.
- The summary uses the same filtered connector and connection collections as the
  visible picker rows, so it updates deterministically as the user types.

## Fixture Coverage

The desktop fake daemon now includes all built-in connector families returned by
the Rust subscription catalog:

- `telegram-login`
- `telegram-bot`
- `discord-bot`
- `lark-app`
- `lark-login`
- `matrix-bot`
- `slack-app`
- `slack-login`
- `slack-bot`
- `email`
- `webhook`

Action chips in the fake daemon are aligned with the catalog for Slack and Lark
reaction aliases, including `send_reaction`.

## Contracts

- Search still matches connector slug, description, skill, deterministic
  `/connect` command, suggested connection name, setup keywords, and action
  slugs.
- Non-trigger connectors remain selectable for setup and disabled as workflow
  triggers.
- Existing connection repair and monitor buttons keep their current command
  contracts.

## Compatibility

The change is additive UI and test fixture coverage only. It does not change the
daemon API shape, workflow schema, or persisted desktop state.
