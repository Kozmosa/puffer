# Stable chat transcript reloads

## Purpose

Desktop chat should not flash, duplicate, or briefly blank assistant text while
a live turn transitions from streamed placeholder rows to the persisted session
transcript.

## Design

Conversation rows now use stable semantic keys instead of array indexes. A row
created from a local placeholder can keep the same DOM identity when the daemon
later returns the same persisted transcript item. Activity expansion state also
uses the row key so reloads do not collapse controls unnecessarily.

Turn completion reloads now fetch the session detail and prune submitted/live
placeholders in one state update path. That prevents an intermediate render
where persisted transcript rows and matching local placeholders are both visible.

## Logic

- User and system rows are keyed by item kind plus a stable hash of their
  visible text.
- Agent rows after a user message are keyed from the preceding user row, so
  streamed assistant text and its persisted replacement share identity.
- Completion reloads compute missing live/submitted placeholders against the
  freshly loaded timeline before publishing the refreshed transcript state.

## Contracts

The semantic key intentionally ignores backend ids for local placeholders whose
persisted ids are not known yet. Repeated identical messages remain distinct by
their occurrence order within the rendered transcript.
