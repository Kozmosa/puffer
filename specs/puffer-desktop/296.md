# 296. Remembered Session Guard Handles Empty Workspace Root

## Summary

The `openRememberedSessionIfAvailable` guard now correctly rejects a remembered
session when the current workspace root is empty (unknown), preventing a session
from workspace A from being opened in workspace B's context.

## Behavior

- If the remembered session has a `workspaceRoot` but the current workspace root
  is empty (daemon hasn't provided it yet), the remembered session is skipped.
- If the remembered session's `workspaceRoot` differs from the current workspace
  root, the remembered session is skipped (existing behavior).
- If neither has a workspace root, the remembered session is still opened
  (backwards-compatible for legacy sessions).

## Compatibility

No protocol changes. Only affects the desktop client's local session restoration
logic.

## Tests

`chat-session-ui.spec.ts` verifies that a remembered session with a workspace
root is not opened when the current workspace root is empty.
