# Background Settled Turn Replays

## Scope

Desktop must not resurrect approvals or questions for a background turn after that turn has completed.

## Behavior

- Background `turn-complete` and `turn-error` events mark the session/turn as settled.
- Any cached transient prompts for the completed background turn are cleared.
- Later replayed events for the same settled turn are ignored before they update the sidebar or prompt cache.
- Background approval/question prompts still work while the turn is active.

## Compatibility

Selected-session turn handling keeps the existing `markTurnSettled` path. The new background path records only the settled key and cached state for that session, avoiding changes to the active foreground turn.
