# Unique Live Error Rows

## Scope

Live error rows are synthesized client-side for turn failures, provider
disconnects, failed prompt responses, and cancel failures. Multiple failures can
arrive in the same browser millisecond when the daemon flushes buffered events
or a reconnect path reports several errors together.

## Change

- Live error ids now include a monotonic sequence number in addition to the
  error code and timestamp.
- Rapid errors with the same code and timestamp append separate rows instead of
  replacing each other.
- A Playwright regression test fixes `Date.now()` and emits two `turn-error`
  events back to back, asserting both inline errors remain visible.

## Compatibility

The visible error content and metadata are unchanged. The sequence only affects
client-side live-row identity, so persisted transcripts and daemon event payloads
are not affected.
