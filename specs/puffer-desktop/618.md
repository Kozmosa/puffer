# Linux CEF Accessibility Compatibility

## Summary

The release build now patches another CEF/Chromium API drift before building
Linux CEF artifacts. CEF branch 7680 injects
`walk_ax_nodes_with_collapse.inc` into Blink accessibility code and expects an
unqualified `StringToInt(...)` helper returning an optional value. The local
Chromium 146 checkout exposes `AXNodeId` as Blink `String`, where conversion is
available through `String::ToInt(bool*)`.

## Design

`scripts/release.sh` keeps CEF compatibility fixes centralized in
`apply_cef_compatibility_patches`. The existing
`content/public/common/child_process_id.h` shim remains independent, so future
CEF compatibility patches still run even when that header shim is not needed.

The new patch is idempotent and only rewrites CEF's exact stale accessibility
expression:

- from `StringToInt(child_id).value_or(0)`
- to `child_id.ToInt(&child_id_ok)` with an explicit parse-success check

If the expression is present but cannot be replaced, the release build fails
before Ninja starts so the problem is visible and repeatable.

## Compatibility

This change does not modify Puffer runtime behavior. It only mutates the local
CEF checkout during release builds, after CEF project generation and before GN
regeneration/Ninja. It preserves CEF's intent to filter child IDs to serialized
accessibility nodes while matching the current Blink string API.
