# Browser Frame Coalescing

The Browser pane now coalesces rapid screencast frame events before decoding
and painting them onto the canvas.

## Behavior

- Incoming frames replace the pending frame while a decode is already active.
- Only one image decode is in flight at a time.
- After the active decode finishes, the latest pending frame is scheduled for
  the next animation frame.
- Clearing the canvas also drops any pending frame.

## Compatibility

The daemon event protocol and Browser pane UI controls are unchanged. The pane
still renders the newest available frame for the active backend browser tab.

## Tests

`desktop-ui.spec.ts` covers a rapid screencast burst and asserts that the pane
settles on the newest frame dimensions.
