# Browser Backend Tab Routing

## Change

- Browser tabs now retain the daemon-provided `backendSessionId` from tab list responses.
- Toolbar commands, URL navigation, canvas input, cursor probing, recording hydration, and legacy close fallback all target the retained backend tab id.

## Compatibility

- Saved local tab metadata continues to omit backend ids so restored tabs fall back to the deterministic root-session/tab id until the daemon syncs current tab state.
- Existing daemons that return the deterministic id keep the same behavior, while daemons with opaque browser worker ids now receive controls on the correct channel.
