# Project Memory Editing

## Summary
- Project detail Memory entries now load their backing file through the desktop daemon instead of showing only generated placeholder text.
- Loaded UTF-8 memory files can be edited and saved in place from the Memory tab.

## Behavior
- Selecting a memory entry reads its path with `read_file`; loading and save responses are ignored if the user switches entries before they complete.
- Missing or binary memory files keep the generated preview visible and leave editing disabled with an inline status.
- Save uses `write_file` and replaces the rendered body with the daemon-returned UTF-8 content.

## Tests
- `workspace-ui.spec.ts` covers loading a seeded project memory file, editing it, saving it, and verifying the daemon `write_file` payload.
