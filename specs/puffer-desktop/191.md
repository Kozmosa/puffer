# Question Response Session Guard

## Context

`AskUserQuestion` responses resolve asynchronously through the daemon. If a user
answered a question in one session and immediately opened another, a delayed
daemon failure could append a question-response error into the newly selected
chat.

## Change

- Capture the selected session id when sending a question response.
- Ignore success and failure continuations when the selected session changes
  before the daemon replies.

## Compatibility

Same-session failures still keep the question prompt retryable and surface an
inline error. The new guard only suppresses stale completions after a session
switch.

## Verification

- Added a Playwright regression that delays a `resolve_user_question` failure,
  switches sessions, and verifies the delayed error does not appear in the new
  session.
