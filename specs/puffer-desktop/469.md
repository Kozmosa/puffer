# Pipeline Connector Trigger Picker

## Summary

The Pipelines editor can now inspect connector catalog context from the
workflow snapshot and switch a workflow to a native connection trigger through
a searchable local picker.

## Behavior

- `WorkflowSnapshot` accepts optional `connectors`, `connections`, and
  `connector_error` fields from the daemon.
- Workflow trigger editing now includes the native `connection` trigger variant
  in addition to cron and legacy subscription topic triggers.
- The Pipeline panel shows a connector search field that filters existing
  connections and connector templates without round-tripping to the daemon.
- Selecting a connection changes the current workflow trigger to
  `{ type: "connection", connection_slug, pattern }` and updates the graph's
  trigger node immediately.
- Connector runtime errors degrade to a visible unavailable state while
  preserving local draft editing.

## Compatibility

Existing daemon snapshots that only contain `workflows` and `runs` still load.
Legacy subscription triggers remain editable, and the new connection picker only
changes the local draft until daemon-backed save support lands.
