# Terminal Close Error Scoping

## Summary

Terminal close failures no longer leak into a different agent after the user
switches sessions.

## Design

`closeTerminalTab` captures the current session id and restore generation before
awaiting `pty_close`. Both success and failure paths check that the same
Terminal pane generation is still active before mutating tabs, error banners,
or in-flight close ids.

## Contracts

- A failed close request from session A cannot show a terminal failure in
  session B.
- Stale successful close responses cannot remove tabs from a newer session.
- Same-session close failures remain visible and retryable.

## Verification

- `playwright test tests/terminal-ui.spec.ts -g "late Terminal close failures do not leak into a switched session"`
