# Terminal Close In-Flight Guard

## Scope

The desktop Terminal pane closes PTY-backed tabs through the daemon. Repeated
activation of a tab close button while the first close request is still in
flight must not send duplicate `pty_close` requests for the same PTY.

## Change

- `TerminalPane.svelte` tracks closing PTY ids.
- Repeated close requests for the same PTY return before touching the daemon.
- Switching sessions clears the close guard with the rest of the terminal
  restore state.
- A Playwright regression test delays `pty_close` and verifies rapid repeated
  close activation only sends one close request.

## Compatibility

The daemon PTY protocol is unchanged. Normal tab close behavior still removes
the tab after the daemon close request settles.
