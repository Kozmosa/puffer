# 307. Composer Model Validation Gate

## Summary

The desktop composer now waits for the active provider catalog before sending a
turn with a normal saved model id. This prevents stale OpenRouter/OpenAI model
ids from reaching the daemon while the GUI is still refreshing provider models.

## Behavior

- A saved catalog model is send-ready only after the current provider's model
  list has loaded and contains that model.
- If the saved catalog model is stale, the existing fallback logic selects the
  provider default before the user can submit.
- While validation is pending, the composer keeps Send disabled and shows a
  provider-specific loading hint.
- Explicit custom model ids, such as fine-tuned OpenAI model names, remain
  send-ready even when they are absent from the advertised catalog.

## Compatibility

This does not change provider discovery, session routing persistence, or
daemon turn request parameters. It only tightens the desktop send gate for
ordinary catalog-backed model ids.
