# Modal autofocus stability

## Intent

Prevent delayed modal autofocus from stealing focus after a user has already
started filling a form.

## Behavior

- `focusTrap` honors a focusable `[data-autofocus]` or `[autofocus]` descendant
  when a modal mounts.
- The deployment draft dialog marks the service-name field as the initial focus
  target instead of scheduling a later focus call.
- Typing into other deployment fields remains stable immediately after the
  dialog opens; branch input no longer lands in the service-name field.

## Compatibility

Existing modals without an explicit autofocus target keep focusing the first
focusable element in the trap.
