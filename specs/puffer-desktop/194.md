# Active Agent Fallback Sessions

## Summary

Desktop now keeps daemon-created sessions visible in the workspace and Active
agents sidebar while grouped session history is still catching up.

## Design

`App.svelte` tracks loaded fallback sessions by id. A fallback session is
inserted into the derived workspace groups until `list_grouped_sessions`
returns it, then the fallback is pruned. The Active agents sidebar derives from
the same fallback-aware groups, and `onOpenAgent` can reopen a fallback session
directly.

## Contracts

- Grouped daemon history remains authoritative once it contains the session.
- A created session that can be loaded by `load_session_detail` remains visible
  after the user backs out and opens older history.
- Active agents can reopen a fallback-created session before the grouped
  history refresh includes it.
- Daemon switches clear fallback session state with the rest of session-scoped
  UI state.
