# Browser Permission Source Label

## Purpose

Keep desktop permission events aligned with the internal Browser permission
protocol.

## Design

Browser permission payloads now expose two source labels:

- `browser_tool`
- `browser_internal_tool`

The old shell-oriented source label is removed from the desktop API type and
the Tauri event serializer.

## Logic

Desktop clients should treat `browser_internal_tool` as a Browser action
requested by a first-party internal CLI that already supplied structured
Browser input through the runtime permission callback.

## Contract

The desktop event payload must not describe internal Browser requests as shell
command parsing.

## Verification

- Rust serializers compile through `cargo check -p puffer-cli`.
