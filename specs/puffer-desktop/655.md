# Live Agent Activity Text Segments

## Summary

Live assistant text emitted during a tool-running turn now stays inside Agent
activity, interleaved with the tool calls that happened around it.

## Design

- Streaming assistant deltas are split into per-turn segments whenever another
  live turn item appears between text deltas.
- Replayed text compares against the concatenated live segments for the turn,
  keeping reconnect replay idempotent without collapsing segment order.
- Turn completion appends a final assistant fallback only when needed instead
  of replacing the first streamed segment.
- When the last streamed segment is the final answer, completion promotes that
  segment into the final assistant response so it is not duplicated inside the
  activity fold.
- Reloaded transcripts merge missing live text back around matched persisted
  tool anchors so intermediate text remains before the correct tool call even
  when the daemon persists only tools and the final answer.

## Compatibility

The daemon event protocol is unchanged. Plain text-only streaming still renders
as the main assistant response, while tool-bearing turns fold live intermediate
assistant messages into Agent activity before and after completion.
