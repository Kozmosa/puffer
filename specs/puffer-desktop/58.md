# Desktop file editing coverage

The desktop e2e harness now covers the Files panel edit-and-save path used in
normal coding sessions.

Compatibility contract:

- The Playwright fake daemon supports `write_file` and returns the same
  `ReadFileResult` shape as `read_file` after updating fixture file contents.
- Saving a dirty UTF-8 file sends the target path and full draft contents to
  the daemon, clears the dirty indicator, and keeps the saved content visible.
- A failed save leaves the draft in the editor, keeps the tab dirty, and shows
  the daemon error beside the file actions so the user can retry.

Regression coverage lives in `files-workspace-ui.spec.ts`, using the restored
Files tab state for `/tmp/puffer/src/main.rs`.
