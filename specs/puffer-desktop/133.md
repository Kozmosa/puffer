# Workspace Switch Submit Guard

## Scope

The desktop workspace switcher can restart the local daemon or attach a remote
daemon. Repeated activation while the first switch request is still pending must
not send duplicate daemon-start requests.

## Change

- `WorkspacePicker.svelte` now checks `busy` inside both submit handlers before
  starting local or remote switching work.
- The existing disabled buttons remain as the visible affordance, while the
  handler-level guard covers rapid same-task clicks before the DOM disable takes
  effect.
- The local-switch regression test mocks Tauri invoke and verifies a rapid
  double activation only calls `restart_local_daemon` once.

## Compatibility

The Tauri command contracts are unchanged. First-click local and remote
workspace switching behavior is unchanged; only duplicate in-flight submissions
are ignored.
