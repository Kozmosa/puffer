# Chat Reconnect and Navigation State Guards

## Summary

Hardens chat state around backend reconnects and cross-screen navigation. The desktop UI now blocks new chat submits while the daemon WebSocket is not open, re-subscribes the active session event channel after reconnect, and saves transient conversation state before leaving an agent detail view.

## Contracts

- The composer keeps the user's draft and does not send `run_agent_turn` while the backend connection is closed or reconnecting.
- A successful backend reconnect tears down the stale active-session event listener and registers a fresh listener for the selected session.
- Navigating away from an agent detail view saves transient live-turn state before clearing the expanded agent id.
- Submitted message baseline IDs are updated immutably so reactive consumers see failed-start cleanup.
- Settled turn tracking is bounded to avoid unbounded growth during long desktop sessions.
- If the composer submit callback throws, the cleared draft is restored for the target session.

## Compatibility

These changes keep the existing fake daemon and product API contracts. They only affect renderer-side UI state handling and reconnect behavior.
