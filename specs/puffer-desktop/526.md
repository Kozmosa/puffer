# Corbina monitor task action visibility

The Pipelines monitor-task panel now renders every action and ignore reason
provided by a monitor task.

## UX

- Monitor tasks no longer hide extra actions behind passive `+N actions` text.
- Every monitor action is a runnable button that sends the task action prompt to
  an agent session.
- Every ignore reason is a runnable button that sends the corresponding
  `/tasks ignore <task-id> <reason>` command.
- Task metadata still shows compact action and ignore counts for scanning.

This is a Corbina display and interaction change only. It does not change
connector manifests, workflow storage, task creation, or webhook behavior.
