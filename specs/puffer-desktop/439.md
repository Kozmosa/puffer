# OpenRouter Catalog Model Tool Support

## Scope

Desktop chat routing must validate persisted provider catalog models before a
message is sent. OpenRouter model ids can contain colons, such as
`baidu/cobuddy:free`, without being user-entered custom ids.

## Behavior

- A selected model that appears in the loaded provider catalog must use the
  catalog's `supportsTools` value.
- Cataloged models with `supportsTools: false` are treated as unsupported even
  when their id contains a colon.
- User-entered custom model ids may still bypass catalog validation only when
  the loaded catalog does not contain that exact id.
- When a cataloged OpenRouter model is unsupported, the composer falls back to
  the default tool-capable model, or the first tool-capable model, before
  allowing a turn to start.

## Compatibility

This preserves existing custom-model behavior while preventing official
OpenRouter catalog entries that do not support tools from reaching
`run_agent_turn` and failing at provider execution time.
