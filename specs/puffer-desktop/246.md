# Terminal UTF-8 PTY Output

## Change

- The Terminal pane now writes decoded PTY frames to xterm as raw bytes instead of Latin-1-like JavaScript strings.
- UTF-8 output such as Chinese text renders correctly while ANSI control bytes remain handled by xterm.

## Compatibility

- PTY wire format remains unchanged: daemon data frames still carry base64-encoded bytes.
- Malformed base64 frames are still skipped without surfacing a renderer error.
