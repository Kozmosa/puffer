# Browser Navigation Loading Feedback

## Summary

The Browser pane now shows immediate loading feedback when the user clicks
Reload, Back, Forward, or submits a URL in the address bar.

## Behavior

- Clicking Reload, Back, or Forward sets `loading = true` and
  `status = "Loading"` synchronously before dispatching the RPC.
- Submitting a URL in the address bar also sets loading state immediately.
- The daemon's subsequent `browser:state` event resolves the loading state
  to its final value (connected, error, etc.).

## Compatibility

No daemon protocol changes. The loading state is purely a renderer-side
optimistic update that gets reconciled by the next state event.

## Tests

`browser-ui.spec.ts` covers:
- Status bar shows "Loading" immediately after clicking Reload
- Status bar shows "Loading" immediately after clicking Back/Forward
