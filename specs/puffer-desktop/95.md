# Active Agent Fallback

## Scope

The desktop sidebar must show the session currently open in AgentDetail even
when the grouped session list is stale or temporarily missing that session.

## Change

- The sidebar agent list still uses daemon grouped sessions as the primary
  source of truth.
- When `selectedSession` is not present in those groups, the renderer derives a
  fallback active-agent row from the selected session metadata.
- The fallback row is de-duplicated once the grouped list catches up, preserves
  pin state, and uses the session folder basename as the project label.
- Re-opening the fallback row is a no-op drill-in instead of failing because the
  grouped list has not caught up yet.

## Compatibility

This preserves append-only session storage and does not change daemon API
contracts. It only makes renderer state coherent while backend refreshes are
eventually consistent.
