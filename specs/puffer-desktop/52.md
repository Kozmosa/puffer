# Remove fake onboarding repository picker

The authenticated onboarding step no longer presents hard-coded repository
choices. Those choices were not loaded from a provider and were not persisted
anywhere, so selecting them created a false permission model.

Compatibility contract:

- Provider login onboarding is unchanged.
- Authenticated onboarding now shows a workspace-ready step with a single
  continue action.
- No fake repository names or selectable repo cards are rendered.
- Repository connection remains owned by the Workspace Connect Project flow.

Regression coverage lives in `onboarding-ui.spec.ts`, asserting that the ready
step enters the workspace and that the former fake repository names are absent.
