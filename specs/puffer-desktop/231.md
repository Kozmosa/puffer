# 231. Completion Text Replaces Partial Streams

## Summary

The desktop transcript merge path now replaces the live streaming assistant row
with the `turn-complete` assistant text when the completion payload differs
from the partial stream.

## Behavior

- Streaming deltas still update a single live assistant row while a turn runs.
- On `turn-complete`, non-empty `assistantText` replaces that turn's live
  assistant row instead of appending a second fallback row.
- After the persisted transcript reloads, the completed assistant text matches
  the stored assistant message and the old partial stream is dropped.

## Compatibility

No daemon protocol changes are required. The behavior uses the existing turn id
to identify the live assistant row that belongs to the completed turn.

## Tests

`chat-session-ui.spec.ts` covers a partial streamed assistant fragment followed
by a different final assistant text and verifies that the final text remains
visible after reload while the fragment disappears.
