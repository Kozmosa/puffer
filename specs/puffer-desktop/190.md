# Permission Response Session Guard

## Context

Permission prompts resolve asynchronously through the daemon. If the user chose
an approval action in one session and immediately opened another session, a late
daemon failure could append a permission error into the newly selected chat.

## Change

- Capture the selected session id when sending a permission response.
- Ignore success and failure continuations if the selected session has changed
  before the daemon replies.

## Compatibility

Same-session failures still keep the approval prompt retryable and surface an
inline error. The guard only suppresses stale completions after a session switch.

## Verification

- Added a Playwright regression that delays a `resolve_permission` failure,
  switches sessions, and verifies the delayed error does not appear in the new
  session.
