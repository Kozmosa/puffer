# User Question Draft Keys

## Scope

Desktop AskUserQuestion prompts keep local draft state per rendered question block.

## Behavior

- Pending prompt selections and custom answers use a client-local key derived from the prompt id and question index.
- Duplicate visible question text cannot share selected-option or custom-answer state in the UI.
- Submitted answers remain keyed by the original question text so the daemon protocol stays unchanged.

## Compatibility

The runtime still requires unique AskUserQuestion text for valid tool calls. This change only hardens the desktop renderer against malformed, replayed, or test-injected duplicate question blocks.
