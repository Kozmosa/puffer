# 348. Browser Navigation Retry Address

Navigation command failures must not leave the Browser address bar disabled.
Those failures are retryable user actions, unlike fatal Browser state errors
where the active tab is disconnected.

## Contract

- Fatal tab state errors still disable the URL input and canvas input.
- Retryable navigation command errors keep the active tab connected and
  re-enable the URL input after the command settles.
- Reconnecting tabs without a fatal error keep their URL draft editable.

## Coverage

The Browser Playwright suite covers the three states together: fatal state
errors disable the URL input, failed navigation re-enables it, and reconnecting
without an error remains editable.
