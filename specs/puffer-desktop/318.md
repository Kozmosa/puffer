# PDF Extracted Text Noise Filtering

## Contract

PDF previews should prefer real readable document text over native extraction
headers and internal font/CMap artifacts. A `textPreview` made only of
extractor boilerplate must not block the in-app PDF text extractor fallback.

## UI Behavior

- `EXTRACTED TEXT` is treated as PDF extraction boilerplate, not document
  content.
- Existing CMap/font garbage filtering still removes `CIDInit`, TeX font names,
  control characters, and single-character fragments.
- When every native preview line is filtered out, the Files pane falls back to
  text extracted from the PDF bytes while keeping the rendered canvas preview.

## Verification

Playwright coverage seeds a PDF whose native preview contains `EXTRACTED TEXT`
plus TeX/CMap garbage, opens it in the Files tab, and asserts the visible
fallback shows the real PDF text while hiding the noise.
