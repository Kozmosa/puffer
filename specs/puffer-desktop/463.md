# MCP Settings Refresh Reload

## Summary

The MCP Servers pane now has an explicit refresh action that reloads the visible
server list from the daemon.

## Behavior

- Opening the MCP Servers pane still lazy-loads `list_mcp_servers` once.
- Clicking Refresh MCP servers triggers a fresh `list_mcp_servers` request in
  addition to the normal settings snapshot refresh.
- The visible MCP cards update from the latest daemon response.
- Refresh is still ignored while credential operations or MCP saves are busy.

## Compatibility

No daemon protocol changes are required. The existing `list_mcp_servers` RPC is
reused for explicit in-pane refreshes.
