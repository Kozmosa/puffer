# Credential Refresh Busy Lock

## Scope

Settings refresh reloads the provider/auth snapshot from the daemon. While a
credential login, logout, or external import is pending, refresh must not be
interactive because a stale snapshot can overwrite the in-flight operation's
result in the desktop shell.

## Change

- Provider refresh in LoginView is disabled while credential operations are
  busy and ignores direct calls during that window.
- The General account refresh button observes the same busy gate.
- The app-level settings refresh handler returns early while credential
  operations are in flight.
- A Playwright regression test delays OAuth, verifies both refresh controls are
  disabled, and asserts no extra `load_settings_snapshot` request is sent.

## Compatibility

Normal manual refresh behavior is unchanged when no credential operation is
pending. The guard only serializes settings snapshot reloads against credential
mutations.
