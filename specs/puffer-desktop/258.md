# 258. Per-Session Composer Drafts

## Summary

The chat composer now keeps unsent draft text per session instead of dropping it
on every session switch.

## Behavior

- Switching to a session with no draft still shows an empty composer.
- Switching back to a session with an unsent draft restores that session's text.
- Submitting a prompt clears only the submitted session's stored draft.
- If a submitted prompt is rejected by the daemon, the submitted session's draft
  is restored.

## Compatibility

No daemon RPC or persisted session format changes. Drafts remain client-local UI
state and are isolated by session id.

## Tests

`chat-session-ui.spec.ts` covers typing separate drafts in two sessions,
switching between them, and submitting only the active session's draft.
