# Browser canvas shortcut handling

BrowserPane consumes browser-level shortcuts while the page canvas has focus.

Compatibility contract:

- `Cmd/Ctrl+R` reloads the active Browser backend session through
  `browser_reload`.
- `Cmd/Ctrl+L` focuses and selects the URL field instead of forwarding the key
  event to the page.
- `Cmd/Ctrl+W` closes the active Browser tab through the daemon tab API.
- Handled shortcut keyup events are swallowed so the web page does not receive
  a dangling key event after the desktop action runs.
- Existing copy handling still routes through the Browser selection bridge.

Regression coverage lives in `desktop-ui.spec.ts`.
