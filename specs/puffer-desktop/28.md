# puffer-desktop Prompt Resolve Failures

## Purpose

Permission approvals and user-question prompts must remain retryable if the
daemon rejects the resolve request.

## Design

The desktop shell now dismisses live permission and question cards only after
the corresponding daemon ACK succeeds. Failed `resolve_permission` and
`resolve_user_question` calls append an error message but leave the prompt and
its in-flight mapping intact.

## Logic

- Successful permission replies dismiss the approval and drop its lookup entry.
- Successful question replies dismiss the question and drop its lookup entry.
- Failed replies keep the existing card visible so the user can retry.
- Historical prompts without an in-flight mapping preserve the previous local
  dismissal behavior.

## Contracts

`chat-session-ui.spec.ts` simulates daemon failures for both permission and
question resolution and asserts that each prompt remains visible and retryable.
