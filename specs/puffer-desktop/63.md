# Remote connect state adoption

Connect Project remote mode changes the shared daemon client before cloning or
creating the new session. The app shell must adopt that daemon identity before
refreshing workspace-scoped data.

Compatibility contract:

- Local Connect Project behavior is unchanged when the shared daemon client has
  not changed.
- Remote Connect Project success reattaches App-level daemon listeners to the
  new client, resets stale session state, and refreshes settings plus pins from
  the remote daemon before opening the created session.
- Remote Connect Project failure still restores the previous daemon client.
- Settings and permissions panes reflect the remote daemon after a successful
  remote session creation, rather than continuing to read local workspace data.

Regression coverage lives in `files-workspace-ui.spec.ts`.
