# Send Reconnect Recovery

## Scope

Desktop chat should not turn an in-flight send into a durable failure when the daemon WebSocket drops after accepting the request.

## Behavior

- If `run_agent_turn` fails because the WebSocket closed, keep the local user message and running turn state.
- Show a reconnect-oriented status instead of appending an `Agent start failed` timeline item.
- Keep the stop button visible but disabled while the turn id is unknown.
- After reconnect, reload the open session transcript and let persisted daemon events replace the transient pending state.
- Non-reconnect `run_agent_turn` failures still remove the optimistic user row and render the normal start-failed message.

## Compatibility

The recovery path is scoped to WebSocket disconnect errors only, so provider validation, auth, and model request failures retain the existing explicit error behavior.
