# Repeated Prompt Optimistic Rows

## Scope

Desktop chat keeps a local optimistic user row visible while the daemon starts a
turn and later reloads the persisted transcript. Repeated prompts with identical
text must still render as separate turns.

## Change

- Optimistic user rows now remember the transcript item ids that existed when
  the prompt was submitted.
- Transient message matching and id reuse ignore those pre-existing ids, so an
  older identical prompt cannot hide a new local row.
- When a genuinely new persisted row replaces the optimistic row, its baseline
  marker is cleared with the submitted row.
- Playwright coverage now verifies identical prompt text can be submitted again
  one minute after the earlier persisted turn.

## Compatibility

Generated-title reloads, completion reloads, clock-skewed transcript timestamps,
and normal optimistic row replacement still use body-based matching for new
persisted rows.
