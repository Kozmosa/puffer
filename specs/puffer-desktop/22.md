# puffer-desktop Session Completion Race Guard

## Purpose

Turn completion reloads must not preserve one session's live chat placeholders
after the user switches to another session.

## Design

The completion handler still reloads the finishing session so persisted
transcript data can replace streamed placeholders. Before applying fallback
live/submitted items after that reload resolves, it verifies that the same
session is still selected.

## Logic

- `turn-complete` and `turn-error` capture the session that was active when the
  event arrived.
- The persisted detail reload may run without a visible loading state.
- If the user selects another session while that reload is pending, the reload
  callback returns before mutating `liveStreamItems` or `submittedMessages`.
- The normal same-session path still preserves assistant completion text when
  the persisted transcript has not caught up yet.

## Contracts

Session selection is the ownership boundary for live chat placeholders. A
delayed completion reload may refresh only the session it was started for; it
must not append fallback user or assistant messages into whichever session is
currently visible.
