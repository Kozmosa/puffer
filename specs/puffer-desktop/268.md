# Model picker provider visibility

The composer model picker must keep the selected provider visible even when
that provider has not returned any models yet or returns an empty model list.

- The trigger now shows the provider chip whenever there is an active provider,
  not only when a model id is present.
- The trigger title mirrors the same state with the provider and `Pick model`,
  so empty-model and loading states remain inspectable.
- The composer can still block send until a concrete model is available; this
  change only fixes the selected-provider affordance.

This keeps slow or empty provider model fetches from looking like the provider
selection was lost.
