# Files and Browser State Hardening

## Scope

The desktop Files and Browser panes should recover cleanly from failed file
reads and stale browser metadata while keeping native CEF state authoritative.

## Change

- Files tab restore now ignores malformed, out-of-root, and duplicate
  persisted tab records instead of trusting every stored value.
- Directory listings are normalized before caching so malformed entries and
  duplicate names cannot reach keyed tree rendering.
- File read responses are normalized before preview rendering so missing
  fields cannot crash the Svelte view.
- File activation uses a per-read token so late file reads cannot overwrite a
  newer click target.
- Tree, tab, and language-server file opens convert read failures into an
  inline viewer error for the selected file.
- Browser history commands keep active URL and title while command-local
  navigation is pending, so stale tab-list metadata cannot overwrite the
  current tab.
- Screencast history evaluates URL and title after `popstate` or a short
  timeout, then emits that state through the existing browser state path.
- Native CEF history only enters loading state when CEF can actually move
  backward or forward, and the renderer polls native state until the history
  result has a short quiet period before applying it.
- Native CEF state snapshots include `updatedAtMs` from CEF callbacks so the
  renderer can compare freshness instead of treating every poll as new.
- Native CEF close hides the view before requesting browser close and defers
  slot deletion out of `OnBeforeClose`, avoiding use-after-free crashes when
  switching away from the Browser pane.

## Compatibility

The daemon and Tauri command names are unchanged. Existing tab, state, and
file-read payloads remain accepted, but the renderer now tolerates incomplete
or malformed file-read data and ignores stale browser tab metadata during a
local navigation command.
