# Lazy Model Picker Discovery

## Summary

The chat model picker now avoids fetching every authenticated provider's model
list when the picker opens.

## Design

Opening `ModelPicker` loads only the currently selected provider's models. The
provider switcher still lists every authenticated agent provider, but a
provider's discovery request is deferred until the user selects that provider.
This keeps real provider discovery from blocking or janking the common path of
checking the current model.

## Contracts

- Opening the picker does not request inactive provider model lists.
- Selecting another provider still fetches that provider's models on demand and
  applies its default model.
- Existing stale-response guards continue to prevent late provider-switch
  responses from overriding the final selection.
- Provider filtering remains restricted to authenticated agent providers.
