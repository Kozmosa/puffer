# Desktop Browser Copy Failure Scoping

## Context

Browser copy selection is asynchronous. The copy shortcut sent a
`browser_copy_selection` request using the current backend browser session id,
but the failure callback wrote directly to the pane error state. If the user
pressed copy in one agent and switched agents before a delayed failure returned,
the new agent could display the old copy error.

## Change

- Capture the active Browser command target before sending the copy request.
- Reuse the Browser target-liveness check before applying copy success or
  failure results.
- Suppress copy-selection failures when the pane has been disposed, the
  generation changed, or the active backend browser session no longer matches.

## Compatibility

Current-tab copy failures still surface in the Browser pane. The change only
drops stale copy results from an older active tab or older agent session.

## Verification

- Added a Playwright regression that delays an Alpha agent copy-selection
  failure, switches to Beta before it resolves, and asserts Beta remains
  connected without rendering Alpha's stale error.
