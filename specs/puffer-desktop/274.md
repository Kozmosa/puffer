# 274. Deployment Secret Reveal Controls

## Summary

Deployment secret rows now expose per-secret reveal controls instead of generic
duplicate "Reveal" buttons. Each control names the target secret key, reports
pressed state, and switches between reveal and hide semantics after activation.

## Design

- Secrets remain masked by default.
- The reveal button accessible name is `Reveal <SECRET_KEY>` while masked.
- After reveal, the same control becomes `Hide <SECRET_KEY>` and sets
  `aria-pressed="true"`.
- The control returns to `Reveal <SECRET_KEY>` with `aria-pressed="false"` after
  hiding the value.

## Compatibility

This changes only button metadata and the reveal icon in the desktop
Deployments mock surface. Secret data, masking behavior, and deployment state
contracts are unchanged.

## Tests

- Playwright verifies the `DATABASE_URL` row can be revealed and hidden through
  a uniquely named control.
- The regression also asserts no exact generic `Reveal` button remains in the
  deployment secrets pane.
