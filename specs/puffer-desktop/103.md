# Duplicate Prompt Submit Guard

## Scope

The desktop chat view can briefly combine a locally submitted optimistic user
row with the same prompt after the transcript reloads from the daemon. It must
also reject duplicate submit attempts while the current turn is still being
started.

## Change

- The app filters rendered transient chat rows against the persisted transcript
  before combining the timeline for the chat view.
- Optimistic user rows carry a creation timestamp, and transient dedupe only
  matches the same kind, same body, and nearby timestamps.
- The parent submit handler rejects reentrant submit calls while a turn start is
  in flight or an existing turn is active.

## Compatibility

Legitimate repeated prompts remain visible when they belong to older turns. The
change only collapses near-simultaneous transient/persisted duplicates for the
same submitted prompt.
