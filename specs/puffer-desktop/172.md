# Desktop Empty Workspace Search State

## Context

When the workspace has no projects or sessions, the workspace board renders the
baseline "No sessions yet" empty state. If the user types into Search workspace
from that state, the search-empty state also renders because there are no
visible projects. The UI therefore showed two contradictory empty states and
two competing actions at once.

## Change

- Hide the baseline empty workspace state while a search query is active.
- Let the search-empty state own the workspace list until the query is cleared.

## Compatibility

The default empty workspace CTA still appears when there is no active search.
Existing no-results search behavior with non-empty workspaces is unchanged.

## Verification

- Added a Playwright regression covering an empty workspace, entering a search,
  seeing only "No workspace results", and returning to "No sessions yet" after
  clearing the query.
