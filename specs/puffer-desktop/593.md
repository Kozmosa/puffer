# ToolCard visual alignment

## Design

ToolCard terminal-style bodies now share the same visual surface as the rest of
the ToolCard catalog. The change is CSS-only and keeps the existing Svelte
template, story fixtures, prompt controls, and render selection logic intact.

The Bash renderer still uses `.pf-fake-pty`, and generic raw output still uses
`.pf-tool-body .terminal`. Both surfaces now use the shared ToolCard
background, border, font scale, foreground text, muted secondary text, and
restrained accent colors instead of the previous dark terminal treatment.

## Compatibility

Runtime transcript data and Storybook stories are unchanged. The update does
not add new tool-specific branches, helper components, or global chat CSS
changes.

## Tests

`npm run check` validates the Svelte and TypeScript surface. Storybook docs
verification should compare `BashOutput`, `ProcessControl`, `CronCreate`,
`Agent`, `ImageGeneration`, and `Running` against existing reference cards such
as `ReadFile`, `GrepResults`, and `McpResults`.
