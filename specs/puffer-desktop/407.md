# 407. Cancel State During Delayed Turn Start

Cancel state now survives the race where a `turn-start` event reaches the UI,
the user requests cancellation, and the original `run_agent_turn` RPC resolves
afterward. The late RPC result updates the current turn id without clearing a
matching in-flight cancel request.

This keeps the Stop control disabled while `cancel_turn` is still pending and
prevents duplicate cancellation requests for the same agent turn.
