# Generated Image Placement And Local Links

## Scope

Desktop chat should render generated image previews below the assistant text
that announces them. Local file paths in that assistant text should remain
clickable and should open the target in the Files tab.

This change is limited to `puffer-desktop`. CLI and TUI rendering are out of
scope.

## Design

- `ConversationView` renders a message body before its attachment preview strip
  for assistant and user messages.
- `visibleMessageBody` keeps suppressing synthetic attachment summary lines so
  generated media previews do not duplicate `[Image: ...]` text.
- `MessageBody` remains responsible for parsing Markdown links, `file://`
  links, and bare absolute local paths into file-open intents.
- `MessageAttachmentPreviewStrip` remains responsible for loading generated
  image preview bytes and rendering thumbnail buttons.
- `AgentDetail` continues routing file-open intents to the Files tab and image
  attachment intents to the preview overlay.

This keeps the durable contract simple: text owns links, attachments own
previews, and the conversation layout owns ordering.

## Local File Targets

The desktop backend already validates file reads against canonical allowed
roots. Those roots include the user home directory and known session
workspaces, which covers generated paths under `~/.puffer/media/images` without
adding broad filesystem access.

Clicking a local file path opens the Files tab with the requested path and
optional line number. Clicking a generated image thumbnail opens the attachment
preview overlay instead of the Files tab.

## Error Handling

If a local file target is missing or outside allowed roots, the Files tab shows
the existing read error state. If a generated preview is missing, the attachment
strip shows the existing unavailable thumbnail state. The message text remains
visible in both cases.

## Testing

Add Playwright coverage that renders an assistant message containing generated
image paths and generated image attachments, then verifies:

- the path text appears before the image thumbnail in DOM order;
- each local path is exposed as a link;
- clicking a local path opens the Files tab through `read_file`;
- clicking the image thumbnail opens the preview overlay.

Existing local-link and generated-media preview tests should continue to cover
line-qualified paths, missing previews, and tool-card file rows.
