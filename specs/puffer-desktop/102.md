# Tauri Provider Alias Compatibility

## Scope

The desktop frontend and CLI daemon use canonical provider ids such as
`openai` and `anthropic`, while the Tauri fallback backend launches local
`codex` and `claude` CLIs. Creating or running a session through the fallback
backend must not fail with `unknown provider` for frontend-canonical ids.

## Change

- The Tauri backend maps `openai`/`codex` to the Codex CLI backend.
- The Tauri backend maps `anthropic`/`claude` to the Claude CLI backend.
- Session creation, default-provider updates, turn launch routing, provider
  validation, and default-model lookup all use the alias mapping.

## Compatibility

The CLI daemon API remains unchanged. This only makes the legacy Tauri fallback
backend accept the same provider ids that the desktop frontend already sends.
