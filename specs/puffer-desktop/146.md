# Pin Save Busy Controls

## Scope

Desktop pinning already rejects duplicate `set_desktop_pin` requests in the app
state layer. The visible sidebar and workspace pin controls should also lock
while their save is in flight so the UI does not look retryable during an
optimistic pin update.

## Change

- Active agent rows now carry a per-agent `pinBusy` flag derived from the
  desktop pin in-flight key set.
- Sidebar agent pin buttons disable while their agent pin save is pending.
- Workspace project rows receive per-path pin busy state and disable their pin
  button while the workspace pin save is pending.
- Playwright coverage now asserts both agent and workspace pin controls are
  disabled while delayed `set_desktop_pin` requests are in flight.

## Compatibility

The pin persistence protocol is unchanged. Existing optimistic pin updates still
render immediately, but the matching control stays disabled until the daemon
acknowledges or rejects the save.
