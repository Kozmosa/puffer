# Stop Button Cancelability

## Scope

The desktop chat composer should not expose an actionable Stop button before
the daemon has returned the turn id required to cancel the running turn.

## Change

- `turnRunning` still covers both "starting" and "running" states so the
  composer stays locked as soon as the user submits.
- A separate `turnCancelable` flag is passed from `App.svelte` once
  `currentTurnId` is known.
- The Stop button remains visible during startup but is disabled until
  cancellation can actually be sent.

## Compatibility

The daemon `cancel_turn` API and turn event stream are unchanged. This only
prevents a clickable no-op in the renderer while `run_agent_turn` is still
returning its turn id.
