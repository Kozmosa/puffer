# puffer-desktop update 698

The Contacts screen now preserves an explicit collapsed detail pane. Closing
the selected contact marks the selection as dismissed so the auto-selection
effect does not immediately reopen the first visible contact.

Saving or deleting a contact clears that dismissed state because those actions
intentionally move the user to the updated contact set. New-contact selection
now compares canonical sorted contact id sets, matching the daemon's normalized
saved-contact ordering even when a user typed ids in a different order.

The contact RPC shapes and task-monitor contact picker are unchanged.
