# Browser Address Bar Edit Guard

## Summary

The Browser pane address bar no longer loses user-typed text when background
state events arrive from the daemon or recording playback.

## Behavior

- While the address input is focused (user is editing), incoming state events
  update `currentUrl` but skip overwriting `urlDraft`.
- On form submit, the address input is blurred so that subsequent state events
  (e.g., server redirects) can update the displayed URL normally.
- Tab switches still update the address bar regardless of focus, because
  `syncFromActiveTab` already had this guard.

## Compatibility

No daemon protocol changes. The address bar still reflects the canonical URL
from state events whenever the user is not actively editing.

## Tests

`browser-ui.spec.ts` covers:
- Background state event does not overwrite focused address bar
- Submitted URL allows post-navigation state updates
- Tab switch updates address bar even if previously focused
