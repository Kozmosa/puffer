# Desktop Early Turn Completion Race

## Context

The desktop composer starts a turn through `run_agent_turn` while also listening
to session stream events. Under a fast daemon or delayed RPC response, the
session event stream can deliver `turn-complete` before the `run_agent_turn`
request resolves.

## Change

- Keep the event stream as the source of truth for already-settled turns.
- When `run_agent_turn` resolves with a turn id that is already settled, do not
  write that id back into `currentTurnId`.
- Clear the local running indicators so the composer returns to send mode.

## Compatibility

Normal turn startup is unchanged. The guard only applies when a completion or
error event for the same turn id has already been processed before the RPC
response returns.

## Verification

- Added a Playwright regression that delays `run_agent_turn`, emits
  `turn-start` and `turn-complete` first, then verifies the composer can send a
  follow-up instead of staying in `Stop turn`.
