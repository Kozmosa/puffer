# Tauri Mac Drag Chrome

## Summary

The desktop shell now renders a real Tauri titlebar drag region instead of
placing the sidebar directly under macOS overlay traffic-light controls.

## Design

`TitleBar.svelte` remains mounted for all shells, but `.pf-titlebar` is only
visible when platform detection adds an `is-tauri` class. The bar is 38px tall
to provide a forgiving drag target. On macOS, the existing `is-tauri-mac`
padding reserves 84px at the left edge so native traffic-light controls do not
overlap Puffer branding or interactive sidebar controls.

## Contracts

- Web previews keep their edge-to-edge layout because `.pf-titlebar` remains
  hidden without Tauri globals.
- Tauri desktop windows expose a visible `data-tauri-drag-region` strip before
  the main app body.
- Sidebar branding starts below the native traffic-light safe area on macOS.
- Interactive sidebar controls are not marked as drag regions.
