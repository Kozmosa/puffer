# Pipeline Monitor Binding Controls

## Summary

Corbina's Pipelines screen can now show and pause/resume connector monitor
workflow bindings from the GUI.

## Behavior

- The workflow snapshot type accepts `workflow_bindings` and
  `workflow_binding_error` from the daemon.
- Connector search indexes monitor bindings, so queries such as `monitor`,
  `telegram`, or `triage` reveal active monitor workflows.
- The connector picker renders a Connection monitors panel with binding status,
  connection, connector, action type, and Pause/Resume controls.
- The top workflow toolbar also exposes a Pause/Resume action for the selected
  native workflow when there are no unsaved local edits.

## Compatibility

The existing connector picker, monitor task action panel, and workflow save flow
keep their current behavior. Pause/Resume calls the daemon directly and does not
mutate unsaved draft edits.
