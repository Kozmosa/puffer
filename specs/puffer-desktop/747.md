# Turn-Bound Transcript Activity Grouping

## Summary

Desktop chat timelines now carry stable turn identifiers from the transcript to
the renderer so intermediate assistant messages, tool activity, permission
prompts, and questions stay attached to the user prompt that produced them.

## Architecture

- Session transcripts may include `TurnBoundary` events with a `turn_id` and a
  `started` or `finished` state.
- Desktop timeline DTO builders keep the active boundary `turn_id` and stamp
  visible user, assistant, system, command, tool, permission, question, and diff
  rows with that value.
- Live desktop rows created before transcript reload also carry the current
  turn id, including submitted user messages once `run_agent_turn` returns.
- The chat row builder groups explicit-turn activity by `turnId` instead of
  only by adjacent item order. Untagged legacy rows keep the existing sequential
  grouping behavior.

## Rendering Contract

If activity for an older turn arrives after a newer user message, the renderer
must place that activity under the older turn's agent row. Explicit-turn rows
order activity by `createdAtMs` when timestamps are available, then select the
chronologically last assistant or command message as the visible final response.
Timestamp ties and missing timestamps preserve arrival order.

## Compatibility

Older transcripts without turn boundaries remain supported. Their timeline
items have no turn id and continue to render with adjacency-based grouping.
Turn boundaries are structural transcript metadata and do not appear as visible
chat messages.
