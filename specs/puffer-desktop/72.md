# Stable chat rows after turn reload

Desktop keeps live chat rows stable when a completed turn reloads the
persisted transcript.

Compatibility contract:

- Local submitted user messages and live assistant messages remain visible
  while the daemon reloads the saved transcript.
- If the reloaded transcript contains the same user or assistant message with
  a different persisted event id, the UI reuses the transient message id for
  that render pass.
- The reuse is scoped to the completed turn's submitted and live messages, and
  duplicate message text is matched from the end of the transcript so earlier
  repeated messages are not stolen.
- Persisted tool, permission, question, diff, and command ids remain untouched.

Regression coverage lives in
`apps/puffer-desktop/tests/chat-session-ui.spec.ts`.
