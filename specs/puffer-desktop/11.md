# Browser Reconnect Toolbar Guard

## Purpose

The managed Browser toolbar must not expose daemon-backed navigation controls
while an active tab exists but its backend session is disconnected or
reconnecting.

## Design

Browser controls that send backend commands now depend on a connected active
tab instead of only checking for any active tab. The external-open and tab
management controls remain available when they do not require an active daemon
connection.

## Logic

- `browserControlsEnabled` is true only when the pane has an active tab and the
  active backend session is connected.
- Back, forward, reload, and the URL input are disabled while the active tab is
  disconnected.
- URL form submission also returns early when there is no connected active tab
  so programmatic submits cannot bypass disabled controls.

## Contracts

Daemon-backed toolbar actions must not be clickable during a reconnect window.
The Playwright desktop harness delays the reconnecting `browser_open` response
and asserts that history, reload, and address controls are disabled before the
backend session becomes available again.
