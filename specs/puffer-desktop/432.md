# Files Preview Reliability

## Summary
- PDF previews no longer depend on the WebView's built-in PDF plugin. The Files pane extracts readable text from PDF content streams and renders it in an in-app document page.
- Legacy binary Office formats (`.doc`, `.ppt`, `.xls`) now attempt readable text extraction from UTF-16 and ASCII runs instead of showing a conversion-only placeholder.

## Compatibility
- OOXML previews for `.docx`, `.pptx`, and `.xlsx` keep their existing ZIP/XML extraction paths.
- PDF extraction supports plain streams and Flate-compressed streams; non-text PDFs fall back to a visible empty-text message.
- Legacy Office extraction is best-effort and does not mutate or convert the source file.
