# Corbina Workflow Run Search

The Pipelines screen now filters recent runs for the selected workflow.

## Contract

- The selected workflow run panel includes a run search input and `N/T runs`
  result count.
- Run search matches run index, run id, workflow slug, status, error text,
  trigger payload, trigger key, node ids, node status, node output, and node
  errors.
- Empty filtered run lists render `No matching runs.` while preserving the
  currently selected trajectory details.
- Switching workflows resets the run search alongside the selected run and step
  state.

This only changes Corbina's Pipelines screen. It does not change workflow run
storage, workflow execution, connector auth, connector runtime contracts, TUI
overlays, or webhook behavior.
