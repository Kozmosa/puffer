# Browser Registry Focus Guard

## Purpose

The desktop browser registry must not mutate tab-set active state when a focus
request targets a tab id that does not exist.

## Design

`BrowserTabRegistry::focus_tab` now validates that the requested tab is present
before updating `active_tab_id` or refreshing active flags.

## Logic

- Missing focus requests return `None` without changing the tab set.
- Existing focus requests keep the previous behavior: update `active_tab_id`,
  refresh flags, and return the focused tab.
- The renderer-side active-id normalization remains a defense against stale or
  inconsistent snapshots from older daemon states.

## Contracts

Failed focus calls must be side-effect free. The Rust unit test opens a tab,
focuses a missing id, and asserts that the original active tab remains active in
the listed state.
