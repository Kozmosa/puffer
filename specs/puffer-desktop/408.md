# 408. Catalog Failure Send Fallback

The chat composer now distinguishes an empty provider catalog from a catalog
request failure. If model validation fails because the provider catalog cannot
be fetched, an existing session with an explicit model id remains sendable
instead of getting stuck behind a disabled Send button.

A successful empty catalog still blocks submission and asks the user to pick or
switch models. Successful catalog refreshes continue to replace stale models
with the provider's current default when one is available.
