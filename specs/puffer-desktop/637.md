# CEF BrowserView Deferred Browser Pointer Patch

## Summary

The release pipeline now extends the CEF BrowserView compatibility patch to
remove Chromium's `const` qualifier from `BrowserView::browser_` when restoring
CEF's deferred `InitBrowser(Browser*)` initialization path.

## Design

CEF constructs hosted `BrowserView` instances before the backing `Browser`
pointer is available, then injects that pointer through `InitBrowser`. Chromium
146 made the stored `raw_ptr<Browser>` member const because upstream initializes
it directly in the constructor. The CEF compatibility patch already restores the
default constructor and `InitBrowser`; it now also restores mutability for the
stored browser pointer so delayed assignment can compile.

## Compatibility

This patch is scoped to the release-time Chromium/CEF source patcher. It only
changes a Chromium checkout when the exact const member declaration is present,
and it remains idempotent when rerun against an already patched tree.

## Verification

Linux CEF packaging should progress past `browser_view.cc` compilation after
this patch is applied to `/mnt/lvm_data/chromium_tintin/src`.
