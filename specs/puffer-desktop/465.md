# Claude-Planned OpenRouter Small-Model Fuzz Campaign

## Summary

The UI fuzz harness now includes a separate AgentFlow campaign for evaluating
cheaper OpenRouter-backed worker models on small UI-tree shard batches while a
Claude Opus planner owns the global campaign plan.

## Behavior

- `agentflow_puffer_openrouter_campaign.py` uses a Claude `plan` node to define
  scope boundaries, false-positive filters, and report requirements.
- Scheduled UI-tree shards run through deterministic AgentFlow shell nodes that
  execute the assigned trigger, report, replay, and feedback commands.
- `puffer-openrouter-triage.mjs` sends each shard's bounded replay artifacts to
  the configured OpenRouter model and writes the shard `findings.md`.
- The triage script has a deterministic replay gate. When bounded replay has no
  new candidate, product candidate, stable failure, or actionable failure, it
  writes a no-finding report and suppresses `BUG_LIST_APPEND` without asking the
  model to invent a finding.
- The campaign defaults to two shards and two-way concurrency so model behavior
  can be evaluated before scaling.
- Workers may only write under their shard-specific `.runs/` namespace and must
  report findings through `BUG_LIST_APPEND` blocks.
- `puffer-openrouter-aggregate.mjs` scans the OpenRouter run namespace, reports
  replay summary counts, and extracts `BUG_LIST_APPEND` blocks for the main
  agent to review.

## Configuration

- `OPENROUTER_API_KEY` supplies credentials without storing secrets in the repo.
- `ANTHROPIC_BASE_URL`, `ANTHROPIC_AUTH_TOKEN`, and an explicitly empty
  `ANTHROPIC_API_KEY` route the Claude planner through the Infer-compatible
  Anthropic CLI flow.
- `PUFFER_OPENROUTER_PLANNER_MODEL` selects the planner model, defaulting to
  `claude-opus-4-6`.
- `PUFFER_OPENROUTER_MODEL` selects the model, defaulting to
  `inclusionai/ling-2.6-flash`.
- `PUFFER_OPENROUTER_SHARD_LIMIT`, `PUFFER_OPENROUTER_CONCURRENCY`, and
  `PUFFER_OPENROUTER_AREAS` control campaign size and scope.
- `OPENROUTER_BASE_URL` can override the default OpenRouter API base URL for
  compatible gateways.

## Compatibility

This only adds fuzz harness entrypoints and documentation. Product desktop code,
Playwright regression tests, and the existing Claude/Infer AgentFlow campaign
are unchanged.
