# Desktop ACL permission prompts

## Purpose

Match desktop permission prompt copy and action mapping to the project ACL model.

## Design

Desktop approval choices now display:

- `Approve once`
- `Always allow`
- `Always allow browser context`
- `Deny`

The frontend maps `always` to `allow_session`, matching the core behavior where
always-allow writes a bounded project ACL rule for the current request or
browser context. The normal desktop prompt no longer exposes the broad
all-actions session grant.

## Logic

Browser-specific prompts keep a dedicated always-allow browser-context action.
Generic permission prompts keep one-shot, always, and deny choices.
Conversation-pane permission icons treat the new `always` action as a session
allow action.

## Compatibility

The desktop API defaults and mock data were updated to use the new labels and
action names while preserving the existing permission-request response shape.
