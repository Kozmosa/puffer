# Same-session canceled turn reload

- `openSession` reconciles canceled local turn state with inactive daemon session details even when the requested session is already selected.
- A canceled turn can leave `currentTurnId`/`turnStartedAtMs` set while the daemon session list is already idle if no completion event reaches the selected detail view.
- Reloading that same session now clears the canceled runtime state whenever the loaded `activityStatus` is inactive, so the detail header, composer, and stop control match the idle workspace card.
- Active daemon states (`running`/`awaiting`) and normal first-turn title reloads still preserve live-turn state during detail reloads.
