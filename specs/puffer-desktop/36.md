# Replay-safe chat turn starts

## Purpose

Desktop chat must not blank already visible streaming text when the daemon
replays older turn events after a reconnect.

## Design

Fresh `turn-start` events still clear live rows for a new run. Replayed
`turn-start` events update turn status but keep existing live rows intact,
because replay ordering can arrive after text deltas in reconnect scenarios.

## Logic

- `turn-start` always refreshes the active turn id and thinking status.
- `turn-start` clears `liveStreamItems` only when `replay` is not set.
- A Playwright regression emits a text delta followed by replayed `turn-start`
  and verifies the streamed text remains visible.

## Contracts

This does not deduplicate replayed deltas; tool and assistant live rows keep
their existing id/upsert behavior.
