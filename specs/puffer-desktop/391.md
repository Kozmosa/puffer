# 367. Session-Scoped Model Picker Fetches

Provider model discovery can be slow, especially for remote model catalogs, and
must not let a stale response rewrite another session's routing controls.

## Contract

- The model picker is scoped to the active session identity supplied by the
  conversation view.
- Switching sessions cancels pending provider switches, pending model loads,
  transient busy state, filter text, open menus, and load errors.
- A late model list response from an old session cannot call the model-change
  callback for the newly selected session.
- Existing same-session stale response handling remains in place for repeated
  provider changes.

## Verification

`tests/chat-session-ui.spec.ts` delays the OpenRouter model list response,
switches from one session to another while the provider change is pending, and
asserts that the second session still sends with its Codex provider/model
routing after the delayed response resolves.
