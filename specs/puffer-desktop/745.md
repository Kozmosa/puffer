# Telegram User ID Contact Validation

## Summary

The contact editor now accepts Telegram private-user contact ids and explains
invalid contact-id input inline.

## Design

- Frontend contact-id normalization includes the backend-supported
  `telegram-user-id` prefix.
- Typographic dash characters in the prefix are normalized to ASCII hyphens, so
  pasted values such as `telegram-user-id@123` remain usable even when the dash
  was converted by another editor.
- Numeric Telegram user ids reject zero and strip leading zeroes before save.
- The Create and Update contact dialog shows a compact invalid-id hint when the
  entered contact ids normalize to an empty list.

## Compatibility

Existing `telegram@username`, `google@email`, Slack, Discord, Matrix, and Lark
contact ids keep their previous normalization behavior.
