# Active Project Collapse Persistence

## Summary

The Active agents sidebar now respects manual collapse toggles even when the
collapsed project contains the currently open agent.

## Design

`Sidebar.svelte` still auto-expands an active agent's project when the active
agent changes, so newly opened sessions remain discoverable. The auto-expand
effect now records the last active agent/project pair it handled and returns
before reading `collapsedProjects` for repeats. That removes the feedback loop
where a user click updated `collapsedProjects`, retriggered the effect, and
immediately reopened the same project.

## Contracts

- Opening a different active agent may expand that agent's project once.
- Manually collapsing the current active project keeps it collapsed.
- Filter reset behavior is unchanged: if the active agent is hidden by a state
  filter, the filter returns to `All states`.
- Collapsed project state is still persisted in local storage.
