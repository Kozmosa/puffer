# 284. Browser Tab Open Pending Guard

## Summary

The Browser pane now treats new-tab creation as a single in-flight action.
While the backend is opening a tab, repeated clicks on the new-tab controls are
ignored and the controls are disabled.

## Behavior

- The tab-strip `New tab` button disables immediately after a tab open starts.
- The empty-state `New tab` button uses the same pending guard.
- The pending state clears when the open request settles for the current
  browser session.
- Switching sessions clears stale pending state from the previous session.

## Compatibility

This preserves the existing `browser_agent` open contract. It only prevents
duplicate open requests from the same Browser pane while the first request is
still in flight.
