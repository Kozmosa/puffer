# 235. Review State In Active Agents

## Summary

The desktop active-agents sidebar preserves the `review` session activity state
instead of folding it into `idle`.

## Behavior

- Sessions whose `activityStatus` is `review` render a `review` state chip in
  the active-agents list.
- The state filter includes `review` and can isolate review-ready agents.
- Runtime-only turn cleanup still treats only `running` and `awaiting` as live
  turn activity; `review` is a persisted session state, not a running turn.

## Compatibility

No daemon protocol changes are required. The GUI already receives
`activityStatus: "review"` in `SessionListItem`.

## Tests

`plus-121.spec.ts` covers a review-ready session and verifies both the visible
state chip and the review filter.
