# Terminal Resize Observer Scope

## Context

Terminal tabs recreate the xterm instance when the active PTY changes. The
previous `ResizeObserver` is disconnected during cleanup, but browsers may
still deliver a callback that was already queued before disconnect.

The callback captured the old PTY id but read the current global xterm and fit
addon. If the user opened or selected another terminal before that stale
callback ran, the pane could send the new terminal dimensions to the old PTY.

## Change

- Bind terminal resize work to the attach generation that created the observer.
- Drop resize callbacks when their attach generation is stale, the pane is
  disposed, or the callback's PTY is no longer the active PTY.
- Keep normal first-fit and active-terminal resize behavior unchanged.

## Compatibility

The `pty_resize` RPC shape is unchanged. This only suppresses resize requests
from stale observers after the terminal attachment has moved on.

## Verification

- Added a Playwright regression that manually fires the first terminal's stale
  `ResizeObserver` after opening a second terminal and verifies no extra
  `pty_resize` is sent for the previous PTY.
