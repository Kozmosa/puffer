# Pipeline Append Binding UX

## Design

Corbina's pipeline connector picker can now create a simple workflow action
binding directly from the selected connector. Users choose or type the
connection name, set the same optional draft pattern used by workflow drafts,
enter an append file path, and create a binding without copying a command or
asking an agent to produce YAML.

## UI Contract

- Trigger-capable selected connectors show an `Append path` input.
- The create action sends `workflow_binding_create` with deterministic slug
  `append-<connection>-<path-leaf>`, connector slug, optional pattern,
  `file_append_path`, and `enabled: true`.
- Relative paths and `/tmp/...` paths are accepted client-side; obvious parent
  traversal and non-`/tmp` absolute paths are rejected before the daemon call.
- Non-monitor workflow bindings render in a `Workflow actions` panel with
  action type, path, format, pattern, connection, status, and pause/resume
  controls.
- Connector search indexes action bindings by slug, description, connection,
  connector, action type, path, format, and pattern.

## Compatibility

The existing workflow draft command, connector setup command, connection
monitor controls, and monitor workflow/task panels remain unchanged.
