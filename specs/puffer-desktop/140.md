# Permission Save Edit Lock

## Scope

Permissions settings save is a daemon round trip that rebuilds the permissions
rows from the saved snapshot. Users must not be able to edit the table while
that save is in flight, because the late save response can otherwise overwrite
new unsaved edits.

## Change

- Permission tool inputs, mode selects, remove buttons, and `Add rule` are now
  disabled while permissions are loading, saving, or the daemon is unreachable.
- The existing single-flight save guard remains the source of truth for
  duplicate save RPC prevention.
- A Playwright regression test delays `save_permissions` and verifies the
  editable permission controls lock until the save completes.

## Compatibility

The permissions daemon protocol is unchanged. Completed saves still refresh rows
from the daemon snapshot; the UI simply prevents edits during the save window.
