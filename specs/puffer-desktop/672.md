# CEF Browser Creation Pump

The macOS native CEF host uses CEF's external message pump integration for the
Tauri/Cocoa event loop, with a small app-side bootstrap only while a browser is
being created.

- `settings.external_message_pump` is explicitly tied to
  `kUseExternalMessagePump = true`.
- App-side scheduled pump callbacks execute `CefDoMessageLoopWork()` through
  CEF's `OnScheduleMessagePumpWork` contract.
- Active pump windows call `CefDoMessageLoopWork()` only while at least one
  slot is waiting for `OnAfterCreated` or still reports page loading.
- Page loading uses the same bounded active pump window as browser creation.
- Once `OnAfterCreated` and `OnLoadingStateChange` leave no active slot, the
  active pump timer is invalidated.
- CEF host logs still report initialization, browser creation, loading, render
  termination, and the applied `use-mock-keychain` switch when enabled.

Compatibility: this avoids a long-running unscheduled manual pump while still
letting CEF drive page loading, DevTools HTTP, and CDP workers from the host
application's event loop.
