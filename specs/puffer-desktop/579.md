# Monitor Deletion Controls

- The task configuration dialog includes an Active monitors section listing each monitor binding with its connection, connector, model, rule count, and enabled state.
- Deleting a monitor uses a two-step row confirmation so the first click only arms the destructive action and the second click calls the existing workflow binding delete RPC.
- The desktop API exposes `deleteMonitor(slug)` as a monitor-specific wrapper over workflow binding deletion, preserving the refreshed workflow snapshot contract used by monitor create, memory save, and task ignore actions.
- Dialog close, monitor create, memory save, and monitor delete operations block each other while an async monitor operation is in flight to avoid stale snapshot writes.
