# Raw Editing For Previewed Text Files

## Scope

Markdown and CSV previews must not remove the existing UTF-8 editing path. These
files now default to the richer preview surface, but the Files viewer exposes a
`Preview` / `Raw` mode switch whenever the active file is both previewable and
editable text.

## Behavior

- `Preview` renders the structured Markdown or CSV view.
- `Raw` restores the same editor used by ordinary UTF-8 source files, including
  dirty tracking, save/cancel controls, and LSP-friendly cursor handling.
- Switching files resets the viewer to `Preview`, so documents open in their
  readable form first.
- Binary previews such as PDF, DOCX, PPTX, and XLSX do not expose the raw
  editor because their daemon payload is base64 and cannot be safely saved
  through the UTF-8 `write_file` path.

## Tests

Playwright coverage opens a Markdown file, verifies the preview, switches to
raw mode, edits and saves through `write_file`, then switches back to preview
and verifies the saved Markdown content is rendered.
