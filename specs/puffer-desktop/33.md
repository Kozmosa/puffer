# Remote scratchpad routing

## Purpose

Settings remote tools must execute on the configured SSH target. A remote bash
or file operation must never silently fall back to local shell or local file IO
while the UI says remote mode is active.

## Design

The Tauri remote scratchpad commands now accept the remote target, cwd, and
password fields already sent by the frontend. They bypass the legacy local
backend methods and call the SSH remote client directly.

## Logic

- Empty remote targets are rejected before any shell operation starts.
- Remote bash executes through `ssh <target> bash -lc ...`.
- Remote read runs `cat` on the remote host with the requested path quoted.
- Remote write validates the base64 payload locally, then decodes it on the
  remote host into the quoted destination path.

## Contracts

Remote scratchpad operations are best-effort SSH operations. The result object
continues to expose `success`, `stdout`, and `stderr`; command failures are
reported as remote operation failures instead of local execution.
