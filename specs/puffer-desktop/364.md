# 364. WebKit Modal Focus Restore

Modal dialogs must restore focus to their trigger after they close in both
Chromium and WebKit.

## Contract

- A modal captures the previously focused element before moving focus inside the
  dialog.
- Closing or submitting the modal restores focus to that trigger when it remains
  connected.
- WebKit timing must not leave the document with no focused element after the
  dialog is removed.
- Existing focus-trap tab behavior inside the modal remains unchanged.

## Verification

`tests/shell-chrome-ui.spec.ts` creates a new deployment draft and checks that
focus returns to the `New deployment` trigger after the dialog unmounts. Running
that test with `--browser=webkit` caught the missing focus restore.
