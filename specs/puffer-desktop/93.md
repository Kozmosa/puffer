# Smooth optimistic chat send

The desktop chat composer now treats send as an immediate UI transition. When
the user submits a prompt, the draft is cleared right away and an optimistic
user message is inserted into the thread before `run_agent_turn` returns.

If the turn cannot be started, the optimistic user message is removed and the
composer restores the draft so the user can retry without losing text. This
avoids a transient empty-thread or typing-only frame after pressing Send, which
made the conversation appear to flash.

Playwright coverage delays `run_agent_turn` and asserts the prompt is already
in the thread while the turn start request is still pending.
