# Active Agent Filter Visibility

## Scope

The sidebar's Active agents list can be filtered by project and state. Those
filters are useful while browsing the workspace, but the currently opened agent
must remain visible in the sidebar so users can keep their orientation after
opening a session from workspace history or project detail.

## Change

- Sidebar filters now clear conflicting project or state selections when they
  would hide the active agent.
- Opening an agent from outside the current sidebar filter restores the relevant
  filter to `All` and leaves the active row visible and highlighted.
- A Playwright regression test selects one project in the Active agents filter,
  opens an agent from another project through Session history, and asserts the
  filter resets and the opened agent is highlighted.

## Compatibility

Filtering behavior is unchanged when no agent detail is active, and filters
still reset if their selected project disappears from the session list. The
change only prevents filters from hiding the active session.
