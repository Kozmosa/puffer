# App Round Two State Guards

## Summary

Hardens renderer state cleanup and timer behavior discovered in the second UI/UX report pass.

## Contracts

- Auto-recap does not arm for empty sessions and is cancelled when switching sessions or resetting daemon-scoped state.
- Logging out the final available agent provider clears session, live-turn, transient, and subscription state through the daemon-scoped reset path.
- Default workspace discovery completes before the first settings/group refresh sequence uses workspace fallback labels.
- Submitted-message baseline cleanup uses immutable reassignment, and completion fallback assistant IDs are deterministic per turn.
- Native agent providers are treated as runnable without stored credentials.
- Focus restoration after closing a modal does not steal focus from a newly opened modal.

## Compatibility

The changes are renderer-only and preserve existing daemon request and event payload contracts.
