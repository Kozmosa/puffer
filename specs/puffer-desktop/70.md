# Browser pointer ownership across tab changes

BrowserPane clears active pointer state when the daemon changes the active tab
during a drag.

Compatibility contract:

- A pointer press is owned by the browser tab that was active when the press
  started.
- If a `browser:<session>:tabs` event switches the active tab before pointer
  release, the local drag state is reset.
- A later global `pointerup` from the old drag cannot emit `mouseReleased` into
  the newly active tab's backend session.
- Existing empty-tab handling still clears pointer state, and ordinary pointer
  movement still probes the cursor once no drag is active.

Regression coverage lives in `desktop-ui.spec.ts`.
