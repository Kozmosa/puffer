# Browser Tab Close Semantics

## Scope

Browser tab headers need valid, keyboard-stable interactive markup.

## Change

- Browser tab headers are now `role="tab"` elements inside a `tablist`.
- The close affordance is a native `button` with an accessible name.
- Keyboard selection remains available on the tab header with Enter or Space.
- The e2e suite asserts that close controls are real buttons and can close a
  secondary browser tab.

## Compatibility

The visual classes and click behavior stay the same, but the DOM no longer
nests one interactive button-like control inside another.
