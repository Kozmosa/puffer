# Puffer Desktop Update 255

## Summary
- Tauri dev and bundle startup now build the repo-local `puffer-cli` before launching the desktop shell.
- The desktop daemon resolver already prefers the repo-local `target/<profile>/puffer`; this update keeps that binary fresh for the active branch instead of silently reusing an older build.

## Compatibility
- `npm run tauri -- dev` builds a debug `puffer-cli` and then starts Vite.
- `npm run tauri -- build` builds a release `puffer-cli` before the frontend bundle.
- Users can still override the daemon executable with `PUFFER_BINARY`.
