# Desktop Recap Card

## Design

The agent conversation view recognizes recap system messages by the `\u203B recap: ` prefix and renders them as a compact box. The box shows only the short recap sentence by default; when expanded, it reveals up to three detail paragraphs.

## Logic

The desktop row builder filters `/recap` command timeline items so auto-recap does not show the slash command above the recap. Recap parsing is local to the conversation view and falls back to normal system-message rendering for non-recap system messages.

The Playwright config accepts `PUFFER_DESKTOP_TEST_PORT` so worktree-local UI tests can run without reusing another checkout's Vite server.

The desktop fake daemon mirrors `list_grouped_sessions_page` so focused UI tests exercise the same paged session-loading path as the app.

## Compatibility

The Tauri DTO shape is unchanged. Existing prefixed one-line recaps render as a non-expandable recap box, while structured recap text renders with the capped details behind the expand control.
