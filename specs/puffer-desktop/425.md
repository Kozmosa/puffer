# Workspace Switch Session Boundary

## Context

Switching the local daemon workspace should be a hard boundary for session
lists and live agent overlays. A running agent from the old workspace could
remain in the Active agents list because daemon-scoped reset cleared selected
conversation state but left grouped sessions and live sidebar fallback state
intact.

## Change

- Daemon-scoped reset now clears grouped sessions, group loading state, and
  live sidebar agents.
- In-flight group refresh generations are advanced so stale grouped-session
  responses from the old workspace are ignored.
- The regression switches from a workspace with a live agent to an empty
  workspace and asserts the old agent disappears from Active agents.

## Compatibility

Reconnect behavior still refreshes groups after the switch. The reset only
removes state that belongs to the previous daemon/workspace boundary.
