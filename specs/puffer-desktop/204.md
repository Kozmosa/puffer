# New Agent Creation Errors

## Context

The New agent modal stayed open when `create_session` failed, but the failure
was only written to the global status strip. Completed onboarding normally hides
that strip, so users saw the Start agent button re-enable without any visible
reason the agent was not created.

## Change

- Track the latest New agent creation error alongside the modal state.
- Clear stale creation errors when opening, closing, retrying, or successfully
  creating a session.
- Render the failure inside the modal with `role="alert"` so it remains visible
  even when the global status strip is hidden.

## Compatibility

Successful session creation and the existing global status message are
unchanged. The inline alert only appears after a failed `create_session` call.

## Verification

- Added a Playwright regression that fails `create_session` under the default
  skipped-onboarding test path and asserts that the modal stays open, Start
  agent is re-enabled, and the inline error remains visible.
