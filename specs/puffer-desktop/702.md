# Contacts Inference Modal Rerun

Opening the Contacts inference modal no longer starts a new inference request.
The modal preserves the last proposal and trace state so users can inspect prior
results without spending another model call.

The modal action label is now `Rerun`, and it is the only UI action that calls
`contacts_infer`. This keeps the backend contract unchanged while making model
usage explicit.

Reruns keep the previous proposal list visible while the new inference is in
flight. Successful reruns replace the proposals with the new response, while
failed reruns surface the error without blanking the prior results. Regression
coverage verifies that opening and reopening the modal do not send
`contacts_infer`, clicking `Rerun` does, and delayed reruns keep prior proposals
on screen.
