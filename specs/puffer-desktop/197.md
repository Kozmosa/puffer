# Session-Scoped Approval Dismissal

## Summary

Desktop now scopes dismissed approval and question ids by session so resolving
or dismissing an item in one transcript cannot hide an item with the same id in
another transcript.

## Design

`App.svelte` derives a scoped key from the selected session id plus the
permission or question id. Pending approval/question filters and duplicate
response guards use that scoped key, while daemon resolution still uses the
original request id mapped from the live turn event.

## Contracts

- Transcript permission ids only dismiss pending approvals in the active
  session where the user acted.
- Question ids follow the same scoping rule as permissions.
- In-flight duplicate-click guards remain active within a session without
  blocking matching ids in another session.

## Verification

- `npm run test:desktop-ui -- tests/chat-session-ui.spec.ts -g "dismissed transcript permissions stay scoped to their session"`
