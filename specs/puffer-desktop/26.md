# puffer-desktop Workspace Daemon Switch Commands

## Purpose

Workspace switching and SSH remote connection controls must call registered
Tauri commands instead of failing at runtime with missing invoke handlers.

## Design

The Tauri shell now manages a shared `DaemonLauncher` alongside the in-process
desktop backend. `restart_local_daemon` restarts a local `puffer daemon` rooted
at the selected cwd, and `start_ssh_daemon` starts the remote daemon/port-forward
flow. Both return the daemon handshake consumed by the existing frontend
`switchDaemonClient` path.

## Logic

- The existing `daemon_launcher.rs` lifecycle owns child-process teardown.
- Command arguments use the same camelCase bridge already used by other Tauri
  commands.
- A static bridge contract test extracts direct frontend `invoke(...)` calls
  and asserts every command is listed in the Tauri registration table.

## Contracts

If frontend code adds another direct Tauri `invoke`, the registration contract
test must fail until the command is registered or routed through
`backend_request`.
