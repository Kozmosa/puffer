# Contact Infer Request Timeout

## Summary

Contact inference now uses a longer daemon request timeout in the desktop
WebSocket client.

## Design

- The shared `DaemonClient.request` method accepts an optional `timeoutMs`
  override.
- Generic daemon calls keep the existing 30 second timeout.
- `inferContacts` uses a 180 second timeout because `contacts_infer` can call a
  model and emit progress events before returning the final snapshot.

## Compatibility

Existing callers do not need to change. The timeout override is opt-in, and the
Tauri invoke fallback keeps the same behavior as before.
