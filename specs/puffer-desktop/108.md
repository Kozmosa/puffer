# Connect Project Create Guard

## Scope

The desktop Connect Project modal starts new local or remote sessions from a
single Start agent action. Repeated activation while the first create request
is still pending must not create duplicate sessions.

## Change

- The modal submit handler now exits when the form is not currently
  submittable, including the in-flight busy state.
- A Playwright regression test delays `create_session`, double-activates Start
  agent, and asserts only one daemon request is recorded.

## Compatibility

The guard preserves the existing local, clone, and remote flows. It only
rejects repeated or invalid submit events that could previously bypass the
button disabled state before the DOM update settled.
