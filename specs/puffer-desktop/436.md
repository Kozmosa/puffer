# Provider connection summary

## Intent

Make the Providers settings pane immediately show which provider credentials are
available before the user scans the full provider catalog.

## Behavior

- When the settings snapshot is loaded, the Providers pane renders a
  `Connected providers` status summary above the provider search box.
- The summary shows the connected provider count and compact pills containing
  each provider display name plus credential kind.
- The existing per-provider card state remains authoritative for connect,
  reconnect, and API key actions.
- Empty auth state is explicit: the summary says that no providers are connected
  and prompts the user to connect one before starting an agent.

## Compatibility

Provider matching continues to use provider-id alias equivalence, so stored
credentials for aliases such as `openai` and `codex` are displayed with the
registered provider name when possible.
