# Default Model Save Edit Lock

## Scope

Default routing save is a daemon config update followed by a parent settings
refresh. Provider/model controls must not stay editable while that update is in
flight, because a delayed refresh can otherwise overwrite a new selection made
during the save window.

## Change

- The Default routing provider and model selects now disable while `update_config`
  is saving or the daemon is unreachable.
- The existing save button single-flight guard remains the source of truth for
  duplicate `update_config` prevention.
- A Playwright regression test delays `update_config` and verifies the
  provider/model controls are locked while the save request is pending.

## Compatibility

The settings daemon protocol is unchanged. Successful default model saves still
refresh settings through the parent; the UI now prevents mid-save edits that
would be lost by that refresh.
