# Monitor Rule RPC Editor Contract

## Design

The Tasks settings `Rules and memory` tab now edits monitor filters through the
daemon rule RPCs. Rules are keyword-only for the selected monitor connection;
the UI does not expose connector-specific payload-field selectors.

## Controls

- The editor presents the selected monitor's include and exclude keyword sets.
- Keyword text is submitted to the daemon as literal keyword intent, not as raw
  regex or connector-specific field filters.
- The residual text input remains borderless and compact around keyword chips.

## Display

Installed keywords render as removable chips. Deleting a chip sends the
original displayed rule JSON to `task_monitor_rule_delete`; multi-keyword rules
are rebuilt with the remaining keywords.

## Tests

Playwright tests cover compact chip-input layout, keyword-only include/exclude
RPC params, and rendering installed include/exclude keyword chips.
