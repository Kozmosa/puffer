# Hidden Completion State Reconciliation

## Summary

Reopening a session whose turn completed while another session was selected no
longer restores stale running controls.

## Design

`App.svelte` now reconciles cached transient turn state after a session detail
reload. If the daemon reports the loaded session as no longer active, and all
locally cached submitted/live timeline items are already represented by the
persisted transcript, the desktop UI clears the cached turn runtime and sidebar
live state.

## Contracts

- A turn that is still missing local submitted or live items stays running.
- A daemon-reported `running` or `awaiting` session keeps its active controls.
- A completed hidden turn with a persisted transcript reopens with normal send
  controls, not a stale Stop button.
