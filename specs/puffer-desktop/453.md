# Question response switch cleanup

- A successful `resolve_user_question` response is scoped to the session that owned the question, even if the user switches sessions before the daemon replies.
- The answered question id is recorded in the dismissed set before checking the current selection, so reopening the original session cannot show a stale question prompt.
- Cached transient state for that session drops the resolved question item and clears its waiting-for-answer hint while preserving the active turn.
- Failed question responses still stay scoped to the original response path and leave the prompt retryable when appropriate.
