# Live Fallback Agent Reopen

## Summary

Active Agents rows backed only by live sidebar state can now reopen their
session before grouped history catches up.

## Design

`App.svelte` already keeps a live sidebar overlay for running sessions. The
sidebar open handler now resolves an agent id from grouped sessions first, then
falls back to the remembered live sidebar session. This keeps the workspace
history model unchanged while making the visible live row actionable.

## Contracts

- A live fallback Active Agents row remains clickable after the user opens a
  different session.
- Grouped session entries remain authoritative when present.
- The selected-session fallback still works when no grouped or live fallback
  session exists.
- No daemon protocol changes are required.

## Verification

- `playwright test tests/workspace-ui.spec.ts --grep "active agents can reopen a live session"`
