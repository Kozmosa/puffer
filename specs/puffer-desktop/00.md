# Desktop UI Automation Harness

## Scope

The desktop app needs repeatable interaction tests for the Svelte/Tauri shell,
with special coverage for the managed Browser tab and daemon event bridge.

## Design

- Playwright owns the browser-level e2e runner.
- The Vite dev server is started from Playwright config and shared across tests.
- Tests use `page.routeWebSocket` to mock the Corbina daemon protocol at the
  WebSocket boundary instead of standing up a real backend process.
- The fake daemon returns realistic session, settings, pin, browser tab, frame,
  state, and devtools payloads so UI tests exercise the production adapters.

## Contracts

- UI tests enter through `/?skipOnboarding&corbinaBackend=...`.
- Browser tab tests assert daemon RPC method names and payloads, not component
  internals.
- Browser frame and devtools streams are delivered as daemon event envelopes
  with the same channel names the production client subscribes to.

## Compatibility

The harness does not replace Tauri or Rust tests. It covers renderer behavior
that unit tests cannot see, and it keeps the existing Codex Playwright MCP test
available through `npm run test:codex-playwright`.
