# Session Event Subscription Ordering

## Summary

The desktop chat view now ignores stale session-event subscription completions
that resolve after the user has already switched sessions.

## Design

`ensureSessionSubscription` assigns each subscription attempt a generation id.
When `subscribeSessionEvents` returns, the disposer is installed only if the
generation and selected session still match. Stale attempts immediately dispose
their listener and leave the current session's listener intact.

The session event API exposes a narrow test hook so Playwright can delay one
subscription and reproduce the stale-disposer ordering deterministically.

## Contracts

- Switching sessions while a subscription is still pending cannot overwrite the
  active session's disposer.
- Reopening a session after a stale subscription completes does not leave two
  listeners for the same session.
- Streaming text deltas continue to render once per daemon event.

## Verification

- `playwright test tests/chat-session-ui.spec.ts -g "late session subscription does not leave duplicate event listeners"`
