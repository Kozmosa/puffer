# Desktop Composer Model Picker Race Scoping

## Context

The composer model picker lets users switch providers before sending a turn.
Provider model loading is asynchronous, and provider buttons remained clickable
while a previous provider's model request was still pending. A slow provider
response could therefore apply after a later, faster provider choice and make
the composer send the turn through the wrong provider/model.

## Change

- Track a local provider-switch generation in the model picker.
- Ignore stale provider model responses and errors when a newer provider click
  has already happened.
- Apply `onChange(provider, model)` only for the latest provider-switch request.

## Compatibility

Provider model lists are still lazy-loaded and cached per provider. The change
only suppresses stale provider choices after the user has clicked a newer
provider.

## Verification

- Added a Playwright regression that makes Anthropic model loading slow, clicks
  Anthropic and then Puffer, and asserts the composer still sends the turn with
  Puffer after Anthropic's delayed response returns.
