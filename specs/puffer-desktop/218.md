# Force Onboarding Provider Gate

## Problem

Forced onboarding treated the debug/deep-link flag as equivalent to an agent
provider login. A new desktop window could show the workspace-ready step and
Continue action even when no Anthropic/OpenAI/Codex credential was available.

## Design

- Keep `forceOnboarding` as a way to display the onboarding shell.
- Do not let `forceOnboarding` satisfy provider authentication.
- Continue to require at least one authenticated agent provider before showing
  the workspace-ready step.
- Preserve unauthenticated workspace edge-case tests through an explicit
  Playwright-only window harness instead of a URL-driven production bypass.

## Test Coverage

- `onboarding-ui.spec.ts` now opens forced onboarding with an empty auth list
  and verifies the provider login form remains visible while the workspace-ready
  heading and Continue action stay hidden.
- Existing no-provider workspace tests now opt into the explicit harness so they
  can keep covering disabled New Agent, Connect Project, and empty-session
  composer states without making `forceOnboarding` a login substitute.
