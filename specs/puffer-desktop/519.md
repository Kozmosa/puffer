# Pipeline Trigger Pattern Defaults

## Scope

Keeps Corbina's Pipelines editor aligned with the native workflow trigger
pattern contract when users create or save workflow drafts.

## Design

- New workflow drafts use `.*` as the default connection or subscription
  trigger pattern.
- Trigger pattern inputs show `.*` as the placeholder and continue to save user
  edits through `workflow_save`.
- The editor normalizes blank trigger patterns to `null` and `*` to `.*` before
  saving a workflow, so daemon-side validation receives either no prefilter or a
  regex-compatible wildcard.
- Playwright coverage verifies a new Telegram-backed draft exposes the default
  pattern and persists a user-entered message match pattern.

## Compatibility

No daemon API changes. Existing workflows keep their stored pattern values until
they are edited and saved through the Pipelines editor.
