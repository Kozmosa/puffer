# Remembered Session Workspace Scope

## Scope

Desktop session restore must not apply legacy remembered-session records to unrelated workspaces.

## Behavior

- Remembered-session records are valid only when they include a non-empty `workspaceRoot`.
- Legacy rootless records are cleared and ignored during startup.
- The app must not call `load_session_detail` for a rootless remembered session id.
- Empty workspaces remain on the workspace board instead of opening a stale detail view from another workspace.

## Compatibility

Current remembered-session records are unchanged because `rememberSession` already writes `workspaceRoot`. The stricter loader only rejects older or malformed records that cannot be safely scoped.
