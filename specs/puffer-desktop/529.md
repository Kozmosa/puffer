# Corbina Workflow List Search

The Pipelines screen now mirrors the terminal workflow dashboard's searchable
workflow list behavior.

## Contract

- The workflow sidebar includes a workflow search input and `N/T workflows`
  result count.
- Workflow search matches workflow slug, pipeline name, working directory,
  enabled/disabled state, trigger type and fields, node ids, node agents, node
  models, node tools, node prompts, dirty local drafts, and latest run metadata.
- Empty filtered lists render an explicit `No matching workflows.` row while
  leaving the current editor selection intact.
- Creating a local workflow draft clears the workflow search so the new draft is
  immediately visible in the sidebar.

This only changes Corbina's Pipelines screen. It does not change daemon workflow
storage, workflow execution, connector authentication, connector runtime
contracts, TUI overlays, or webhook behavior.
