# 334. Deployment Redeploy Feedback

Redeploy controls must produce visible deploy activity instead of acting like
dead buttons.

## Contract

- The detail-header `Redeploy` button and Deploys-pane `Trigger deploy` button
  run the same local redeploy flow.
- Starting a redeploy switches to the Deploys tab, disables redeploy controls,
  marks them busy, and announces the target deployment in a live status.
- A manual deploy-history row is inserted immediately with a `deploying` state,
  branch, deployer, and running duration.
- Completion updates the same history row to `healthy`, clears busy state, and
  announces completion.
- This remains local mock state until provider deployment RPCs exist.

## Coverage

Playwright clicks `Redeploy`, verifies the busy state, live status, Deploys tab,
manual history row, and completion state, then clicks `Trigger deploy` to ensure
the pane action uses the same flow.
