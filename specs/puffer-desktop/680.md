# Media Settings Quiet States

## Scope

Image and video generation settings should avoid transient chrome while their
provider capabilities are loading or unavailable.

## Change

- The media settings modal no longer renders a loading status panel, spinner,
  or loading copy while `list_media_capabilities` is pending.
- No-provider and no-capability messages now render as plain inline text
  instead of using the bordered media state container.
- Error and stale saved-model warnings still use the bordered warning state so
  actionable problems remain visually distinct.

## Compatibility

The media capability API, saved media config shape, provider selection logic,
and image folder behavior are unchanged. This is a renderer-only presentation
change.
