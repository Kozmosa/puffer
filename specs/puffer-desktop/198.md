# Session-Scoped Transient Chat State

## Summary

Desktop now preserves local pending chat state when a user switches away from a
session and returns before the daemon has returned the turn id.

## Design

`App.svelte` captures submitted prompt rows, live stream placeholders, turn
metadata, replay buffers, and pending approval/question lookup maps per session
before switching sessions. Opening a session restores that cached transient
state instead of starting from an empty local overlay.

## Contracts

- Pending submitted prompts must not leak into another selected session.
- Returning to the original session before `run_agent_turn` resolves must keep
  the optimistic prompt row visible.
- If the delayed turn id returns while the original session is no longer
  selected, the cached session state records that running turn for later
  reopening.
