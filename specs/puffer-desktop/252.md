# 252. Office Preview Inflate Fallback

## Summary

Office document previews no longer depend on the WebView exposing
`DecompressionStream("deflate-raw")`.

## Design

- DOCX, PPTX, and XLSX ZIP entries use `fflate` for raw DEFLATE inflation.
- The existing Markdown, CSV, and PDF preview paths are unchanged.
- ZIP entry size validation now checks the inflated byte count against the
  central-directory size when one is provided.
- Text-preview raw editing remains available only for Markdown and CSV files.

## Compatibility

Chromium previews continue to work, while macOS desktop WebViews that do not
implement Compression Streams can still render common Office Open XML files.
Legacy `.doc`, `.ppt`, and `.xls` files keep their explanatory fallback panel.

## Tests

The common file-format Playwright regression disables `window.DecompressionStream`
before the app loads and still verifies Markdown, CSV, PDF, DOCX, PPTX, and
XLSX previews. The raw-editing regression for previewed text files remains
covered separately.
