# Live prompt replay de-duplication

The Desktop chat view now treats live stream item ids as stable keys:

- Appending a live item with an existing id replaces the prior item in place.
- Replayed permission prompts with the same daemon `requestId` do not render
  duplicate approval cards.
- Replayed user-question prompts with the same daemon `requestId` do not render
  duplicate question forms.

This preserves prompt retryability while making WebSocket reconnect/replay
semantics idempotent in the UI.
