# MiniCPM5 One-Click Setup

- The desktop signed-in onboarding and settings provider surfaces now expose a local `MiniCPM5` setup card.
- `LocalModelSetupCard` shows platform support, install state, endpoint, provider registration path, progress messages, and concrete diagnostics from the daemon.
- The install action calls `install_local_model` and returns immediately with a job id while the daemon emits `local-model:minicpm5:event` progress updates.
- The status action is read-only. It reports platform support, Python environment readiness, model files, provider YAML, and the `/v1/models` health response.
- The Tauri local-model backend creates `~/.puffer/venvs/minicpm5`, downloads `openbmb/MiniCPM5-1B-MLX` into `~/.puffer/models/minicpm5-1b`, writes `~/.puffer/resources/providers/minicpm5.yaml`, and starts the OpenAI-compatible shim on `127.0.0.1:8088/v1`.
- The setup is idempotent: an already-installed model verifies registration and starts the local server instead of redownloading weights.
- MiniCPM5 remains an auxiliary local behavior-analysis provider. The card does not change the user's default coding provider or model.
