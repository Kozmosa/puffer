# Settings persistence e2e coverage

## Purpose

Settings panes for permissions and MCP servers are normal Personal Agent setup
flows. They need daemon-backed interaction tests so the desktop UI cannot drift
into "looks saved" states that never reach the backend.

## Design

The Playwright fake daemon now implements the same settings RPCs used by the
Permissions and MCP panes. Tests drive the real Svelte controls and assert the
exact daemon request payloads.

## Logic

- `list_permissions` and `save_permissions` round-trip an editable tool policy
  map through the fake daemon.
- `list_mcp_servers` and `add_mcp_server` round-trip MCP manifest data through
  the fake daemon.
- Settings tests cover adding a permission rule and adding a stdio MCP server.

## Contracts

This is test harness coverage only. It does not change the production settings
RPC contract; it makes the existing daemon contract observable from desktop UI
tests.
