# Desktop Pin Save Guard

## Scope

Desktop agent and workspace pin buttons save through the daemon. Repeated
activation of the same pin while the first save is still in flight must not send
duplicate `set_desktop_pin` requests.

## Change

- `App.svelte` tracks in-flight pin saves by `kind:id`.
- Repeated pin toggles for the same agent or workspace return before touching
  the daemon until the first save settles.
- Session and daemon-scope resets clear pending pin guards.
- The fake desktop daemon now implements `set_desktop_pin` so pin interactions
  are covered by Playwright tests.

## Compatibility

The daemon protocol is unchanged. Normal pin and unpin behavior remains
optimistic; only duplicate in-flight saves for the same pin target are ignored.
