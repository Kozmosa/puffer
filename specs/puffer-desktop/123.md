# Session-Scoped Pending Submit

## Scope

Sending a prompt starts with a daemon `run_agent_turn` request. That request can
remain in flight while the user switches to another session.

## Change

- The composer now tracks pending submit state by session id instead of one shared boolean.
- The desktop shell also tracks `run_agent_turn` submit requests by session id.
- Switching from one session with a pending turn-start request to another session no longer leaves the second session's Send button disabled.
- A Playwright regression test starts a delayed turn in one session, switches to another, and verifies the second session can submit immediately.

## Compatibility

Repeated sends in the same session are still blocked while that session's submit
request is pending. The change only removes accidental cross-session blocking.
