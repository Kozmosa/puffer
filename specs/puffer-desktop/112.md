# Clear Awaiting Prompt Hints

## Scope

Live permission and user-question prompts pause a running turn until the user
responds. Once the response is accepted by the daemon, the chat typing/status
hint should leave the awaiting state so the agent looks active again.

## Change

- Successful permission responses now set the selected turn hint back to
  `Running`.
- Successful user-question responses use the same transition.
- A Playwright regression test approves a live permission request and asserts
  the stale `Awaiting approval` hint is removed.

## Compatibility

Failed responses still keep the prompt retryable and preserve the awaiting
state. Later stream events may replace the generic running hint with more
specific tool, retry, thinking, or completion state.
