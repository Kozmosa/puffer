# Canonical Media Generation Settings UI

Desktop media settings use daemon-normalized capability axes as the single UI
contract.

## Storage And DTO Shape

Saved image and video settings persist:

- `providerId`
- `logicalModelId`
- `selections`

Old desktop-only fields such as `modelId`, `adapter`, `operation`, `parameters`,
and `defaults` are not migrated or retained.

## Modal Behavior

- Image settings render Provider, Model, Mode, Ratio, and Output when supplied
  by the selected capability.
- Video settings render Provider, Model, Mode, Ratio, and Duration when
  supplied by the selected capability.
- Output folder remains separate from generation option axes.
- The modal renders normalized axes generically; it does not branch on provider
  ids or provider request field names.
- Switching models normalizes selections against the new axes, drops stale keys,
  and refreshes the image output maximum.

## Backend Contract

Generation requests pass logical model ids and logical selections to the media
resolver. The desktop backend does not reconstruct provider adapter parameters
from saved settings.
