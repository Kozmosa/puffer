# Linux CEF Window Creation Hook Compatibility

## Summary

Linux CEF release builds now patch the Chromium browser-client API expected by
CEF branch 7680. CEF injects a `CreateWindowResult(...)` call after
`RenderFrameHostImpl::CreateNewWindow` completes and implements that hook in
`ChromeContentBrowserClientCef`, but the local Chromium 146
`ContentBrowserClient` base class no longer declares it.

## Design

`apply_cef_compatibility_patches` adds an idempotent no-op virtual method to
`content/public/browser/content_browser_client.h` when the method is missing:

`virtual void CreateWindowResult(RenderFrameHost* opener, bool success) {}`

This keeps CEF's subclass override valid and lets CEF receive the success or
failure notification it expects. Non-CEF embedders inherit the no-op behavior,
matching Chromium's behavior when the hook is absent.

## Compatibility

The patch only mutates the local Chromium checkout during release builds. It is
guarded by an exact grep check and fails early if the insertion does not take,
so future Chromium API movement is visible during release preparation instead
of surfacing as a late Ninja compile failure.
