# Agent-Capable Default Models

## Scope

The desktop Settings -> Providers default model picker must not save models
that cannot run agent tools.

## Change

- Provider model lists are still cached per provider, but the default route
  picker filters them to models whose `supportsTools` value is not `false`.
- If the configured default model is not agent-capable, Settings selects the
  provider's first agent-capable model instead.
- If a provider has no agent-capable models, the model picker shows a clear
  empty state and disables Save default.

## Compatibility

The daemon model-list API is unchanged. Existing configs that point to a
non-agent model are corrected only when the user saves a valid replacement from
Settings.
