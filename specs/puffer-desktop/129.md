# Permissions Save Duplicate Guard

## Scope

The Settings Permissions pane saves tool permission policies through the daemon
`save_permissions` endpoint.

## Change

- The permission save handler now ignores calls while a save is already in
  flight, while permissions are still loading, or when there are no dirty edits.
- A Playwright regression test delays `save_permissions`, triggers the Save
  button twice in one browser task, and verifies only one permission save
  request reaches the daemon.

## Compatibility

Normal first-click save behavior is unchanged. The guard only removes duplicate
permission writes caused by repeated activation while the first save is still
pending.
