# Browser Pane Daemon-Owned CEF Tabs

- CEF browser tabs are now restored from the daemon tab list before any local
  browser tab cache is used. Saved local tabs are only a migration fallback
  when the daemon tab list cannot be fetched.
- When a live daemon session reports no tabs, the browser pane opens a fresh
  daemon-owned blank tab instead of resurrecting stale local storage or old
  screencast recordings from a prior run.
- Daemon-owned CEF tabs use their daemon-provided native CEF slot id for native
  open, hide, navigation, reload, history, input, resize, and health probes.
  The backend session id is only used as a native CEF id for explicitly
  local-only migration tabs.
- Closing a daemon-owned CEF tab hides its reusable native slot and then closes
  the daemon tab record. Local-only CEF tabs may still close their owned native
  browser directly.
- Native CEF failures now fall back per backend tab. A single failed native slot
  no longer forces unrelated CEF tabs into screencast mode.
