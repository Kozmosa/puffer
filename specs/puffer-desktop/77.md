# Files save completion after tab switch

The Files pane allows users to switch tabs while a save request is still in
flight. The pending save belongs to its original path, not to the tab that is
active when the daemon responds.

Compatibility contract:

- A successful `write_file` response updates the saved file cache for the
  original target path even when another tab is active.
- The dirty marker for the saved tab is cleared after the response.
- The global save-in-flight guard is always released when the request finishes.
- Save errors are shown only when the failing file is still active; background
  save failures must not overwrite the visible tab's header.

Regression coverage lives in
`apps/puffer-desktop/tests/files-workspace-ui.spec.ts`.
