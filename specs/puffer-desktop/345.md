# 345. Compact PDF Preview Controls

PDF previews must keep long-document status and zoom controls visible and usable
when the Files pane is opened in a compact desktop window.

## Contract

- The PDF controls row always owns the first grid row, and the PDF page scroller
  always owns the flexible content row.
- The long-document status badge uses an explicit warning-tone background in
  both light and dark themes, distinct from the preview and control surfaces.
- Zoom buttons and the range control remain hit-testable in compact previews.
- Zooming changes the rendered canvas dimensions without rendering past the
  first 20 pages.

## Coverage

Playwright opens a dark-theme compact window, selects a 29-page wide PDF, checks
the page-limit badge contrast, verifies the page scroller has real height and
overflow, hit-tests the zoom-in button, and confirms zoom changes page width.
