# Pipelines Connector Filter Presets

## Summary

Corbina Pipelines now exposes connector search presets above the connector
result counts.

## Behavior

- Presets write directly into the existing connector search input.
- `All` clears the query.
- `Trigger` uses `trigger-ready` to avoid matching `no-trigger` rows.
- `Actions`, `Serve`, `Subscriber`, `Internal`, and `No trigger` use the same
  terms shown in TUI connector output.
- Connection search text now indexes `actions` and `has-actions` when the
  owning connector has at least one action, so the `Actions` preset covers both
  connector rows and configured connection rows.
- The `Actions` preset writes `has-actions`, which avoids matching connector
  descriptions that merely mention actions.

## Compatibility

The presets do not add a second filtering model. Manual search input continues
to use the same all-terms substring matcher, and missing connector metadata is
still tolerated.
