# Two-Line Monitor Keyword Rules

## Design

The Tasks settings `Rules and memory` tab presents monitor filtering as two
fixed keyword rows under `Ignore rules`: `Only includes` and `Exclude`. Users
edit rules in the same place they read them.

## Behavior

- Each row renders the selected monitor's installed keywords as compact chips.
- Each chip has an `x` button that removes only that keyword. When the keyword
  came from a multi-keyword rule, the UI deletes the old rule and re-adds the
  remaining keywords.
- Typing in a row and pressing Enter, typing comma, or leaving the field adds
  the draft keywords to that row's rule mode.
- The UI does not show commas between keywords or render duplicate filter
  summaries outside the keyword rows.

## Compatibility

The desktop UI still calls the existing keyword-only
`task_monitor_rule_add` and `task_monitor_rule_delete` daemon RPCs. No
connector-specific scope fields are exposed.

## Tests

Playwright coverage verifies the two fixed rows, exclude/include keyword adds,
installed keyword chip rendering, and one-keyword removal from a multi-keyword
rule.
