# Browser late frame invalidation

BrowserPane invalidates pending frame image loads whenever the canvas is
cleared.

Compatibility contract:

- Each rendered frame still owns a draw serial and may paint only while it is
  the latest canvas operation.
- Clearing tabs, switching to an empty tab set, or otherwise clearing the
  canvas advances the draw serial.
- A frame image whose asynchronous `onload` fires after the canvas was cleared
  is ignored and cannot repaint stale page content behind the empty-tab state.
- Existing frame rendering and saved-tab restoration behavior remain
  unchanged for the active tab.

Regression coverage lives in `desktop-ui.spec.ts`.
