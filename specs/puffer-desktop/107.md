# Project Detail Fallback Sessions

## Scope

Workspace board and session history render a selected fallback session while
the grouped daemon history is temporarily stale. Project detail must use the
same fallback group data so the session does not disappear when the user opens
the project board.

## Change

- Project detail lookup now uses the workspace group list that includes the
  selected-session fallback.
- The workspace regression test opens project details after creating a fallback
  session and asserts the new session can be opened from that detail view.

## Compatibility

Persisted groups still replace fallback data once grouped history catches up.
This only changes project-detail rendering while the selected session is absent
from the latest grouped daemon snapshot.
