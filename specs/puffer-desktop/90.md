# Desktop provider helper canonicalization

The desktop provider-id helper now canonicalizes both aliases and canonical
display names before comparing or sending provider ids to the daemon.

`codex` and `OpenAI` resolve to `openai`; `claude` and `Anthropic` resolve to
`anthropic`. This keeps New Agent, reconnect, and turn-send flows aligned with
the daemon registry even when UI labels or persisted values use display-case
provider names.

Unknown provider ids are preserved for custom providers.
