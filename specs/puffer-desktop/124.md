# API Key Login Duplicate Guard

## Scope

Provider API key login can be started from the Settings Providers pane by
clicking Connect or pressing Enter in the API key field.

## Change

- API key fields are disabled while their provider login is busy.
- Enter-submit now ignores repeats for the provider already logging in.
- The desktop shell also ignores duplicate API key login calls while any auth
  request is in flight.
- A Playwright regression test delays the daemon login response, presses Enter
  twice, and verifies only one `login_with_api_key` request is sent.

## Compatibility

Normal first-submit behavior is unchanged. The guard only removes duplicate
auth writes caused by rapid keyboard activation while the first request is still
pending.
