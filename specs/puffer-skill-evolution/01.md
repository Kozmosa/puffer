# puffer-skill-evolution update 01: verifier-first skill generation

The GenSkill GEPA prompts now bias generated skills toward task-fit and
non-interference.

- Candidate generation asks for conditional skills with concrete task/domain
  triggers and explicit "when not to use" guidance.
- Generated skills must preserve verifier-facing details from the trace:
  required files, schemas, tests, permissions, success signals, and minimal
  verification steps.
- Judge prompts penalize broad workflow advice, missing verifier contracts, and
  skills that could increase repeated exploration before reading the current
  task prompt.
- Mutation prompts preserve the same verifier-first and non-interference
  constraints while improving weak candidates.

The public GEPA API is unchanged. The selection loop still uses novelty,
reproducibility, structure, and conciseness scores, but the scoring prompt now
defines reproducibility as reproducing the approach without ignoring the current
task contract.
