# puffer-provider-registry 12: Gemini generateContent media adapter

`MediaExecutionKind` now includes `gemini_generate_content` for image
generation models that do not use the OpenAI-compatible Images JSON schema.

The descriptor contract remains static and typed:

- providers can attach `adapter: gemini_generate_content` at model level;
- `path` may include a `{model}` placeholder resolved by the media runtime;
- batch metadata remains required and uses the existing image batch policy.

This keeps Gemini image routes distinct from `images_json`, avoiding an
incorrect OpenAI request/response shape for native `generateContent` calls.
