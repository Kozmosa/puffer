# OpenRouter Discovery Compat

## Summary

OpenRouter model discovery now preserves the OpenRouter Chat Completions
reasoning wire shape for discovered models.

## Design

- Discovered OpenRouter models keep `api: openai-completions`.
- When discovery returns an OpenRouter model, the registry attaches
  OpenAI-completions compat with `thinking_format: openrouter`.
- The bundled static OpenRouter fallback model carries the same compat block.

## Compatibility

This keeps OpenRouter and Gemini requests on Chat Completions while ensuring
reasoning effort is sent as `reasoning: { effort: ... }` instead of the
canonical OpenAI top-level `reasoning_effort` field.

## Tests

Provider-registry discovery tests cover OpenRouter model compat assignment, and
core reasoning tests cover provider-level OpenRouter compat inference.
