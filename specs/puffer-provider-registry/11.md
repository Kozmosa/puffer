# Canonical Media Settings Descriptor Fields

Provider media model descriptors now carry only the static metadata needed to
turn product-level media settings into provider request fields.

## Descriptor Contract

- `MediaModelDescriptor.max_outputs` is optional and caps image output choices.
  Validation rejects values above `9`.
- `MediaModelDescriptor.media_map` is optional and supports two narrow lookup
  tables:
  - `ratio`: maps canonical ratio labels to one provider request field.
  - `size`: maps `mode + ratio` pairs to one provider request field.
- `mode` and `ratio` are reserved axis ids. They may omit `request_field` only
  when the corresponding provider request value is supplied by `media_map`.
- Ordinary `param` axes still require `request_field`.
- `ratio` axis values must use the canonical ratio vocabulary accepted by the
  registry validator.

## Design Boundaries

The registry remains declarative catalog storage. It does not evaluate
predicates, infer fallbacks, run expressions, or contain a generic media rule
engine. Missing or invalid mappings fail descriptor validation instead of being
handled later by provider adapters.

## Compatibility

Old descriptors that exposed raw image fields such as `size`,
`aspect_ratio`, `response_format`, or `output_format` as user-facing axes are
not preserved. Providers must declare canonical product axes plus `media_map`
when provider wire fields differ from the UI settings.
