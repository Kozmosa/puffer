# Authoritative Model Discovery

## Summary

Runtime model discovery now reconciles bundled provider models against the
provider's current availability response instead of only appending new ids.

## Design

`merge_discovered_models` treats a non-empty discovery response as the
authoritative set of selectable models for that provider. Matching bundled
models keep their curated metadata, newly discovered ids are appended with
discovery defaults, and bundled ids omitted by the provider are removed.

Empty discovery responses remain a no-op so transient endpoint failures or
malformed empty caches do not erase the local provider list.

## Contracts

- Desktop and daemon model pickers no longer expose stale bundled ids after a
  successful discovery refresh.
- Provider-specific metadata remains stable for ids that still exist in both
  the bundled descriptor and discovery response.
- Alias routing such as `codex/<model>` continues to resolve after discovery,
  but only for models that the provider currently returns.
