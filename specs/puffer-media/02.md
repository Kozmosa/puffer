# Canonical Media Settings Resolution

The media layer resolves desktop and tool selections from canonical product
axes into provider request parameters.

## Capability View

- Image capabilities expose normalized `mode`, `ratio`, and synthesized
  `output` axes when static mappings exist.
- `output` is derived from `max_outputs`, capped globally at `9`, and is never
  emitted as a provider request parameter.
- Video capabilities expose product labels `Mode`, `Ratio`, and `Duration`.
- Provider request fields, wire types, and mapping details stay out of desktop
  capability payloads.

## Request Resolution

- `ResolvedMediaRequest` includes `count`.
- Ordinary `param` axes are emitted first, excluding reserved `mode`, `ratio`,
  and synthesized `output`.
- `media_map.ratio` maps canonical ratio selections to one-dimensional provider
  request fields.
- `media_map.size` maps the selected `mode + ratio` pair to size-like provider
  request fields.
- `Auto: null` mapping values mean the provider field is omitted.
- Unknown stale saved selections are dropped by normalized-axis validation.
- Image `output` becomes runtime count and is capped by the selected model and
  the global cap of `9`.
- Video generation remains fixed at count `1`.

## Compatibility

Saved or per-call provider wire settings are no longer accepted as a parallel
path. Image and video generation must go through the same canonical resolver.
