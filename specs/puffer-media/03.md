# puffer-media 03: Gemini generateContent image execution

The media runtime now supports a `gemini_generate_content` image adapter.

Execution behavior:

- builds `contents` plus `generationConfig.responseModalities = ["TEXT", "IMAGE"]`;
- passes only `imageConfig.aspectRatio` and `imageConfig.imageSize`;
- resolves `{model}` in the descriptor path before submitting;
- reads generated image bytes from
  `candidates[].content.parts[].inlineData.data`;
- persists the same job and artifact sidecars used by existing image adapters.

The adapter accepts only per-image call plans. Multi-output requests are
fulfilled by repeated single-image calls through the existing planner.
