# puffer-genskill-eval Update 03

## Summary

Adds synthetic-expert input generation for end-to-end benchmark runs when human
expert recordings are unavailable.

## Design And Architecture

`record_codex_expert_run.sh` starts the same Ladybird sandbox image used by
replay, resets it to the corpus `base_commit`, copies the frozen target tests,
and runs `puffer non-interactive` as the `ladybird` user. It mounts the release
Linux Puffer binary, the host repo, and host Codex credentials read-only when
present. The resulting session transcript is saved as `expert_run.jsonl` and
converted to `expert_run.md`.

`build_linux_puffer.sh` builds that sandbox-compatible binary with a Rust Docker
image and writes it to `benchmark/genskill/ladybird/.bin/puffer-linux`. It
installs native build helpers needed by the CLI dependency graph, including
`protoc`, OpenSSL headers, pkg-config, and SQLite headers. This is required on
macOS because `target/release/puffer` is a host Mach-O binary and cannot
execute inside the Ubuntu Ladybird image.

`generate_skills.sh` consumes both transcript forms. The direct arm reads the
markdown transcript as prompt context and writes `skills/direct/SKILL.md`.
The GEPA arm loads the typed JSONL transcript into Puffer state, runs
`/genskill`, and copies the generated skill into `skills/gepa/SKILL.md`.
It runs from a temporary working directory so `/genskill` does not leave
generated skills under the repo's top-level `resources/skills`.

`prepare_synthetic_eval_inputs.sh` builds the required binaries, validates the
corpus, then records and generates inputs for every frozen PR.

## Logic

The transcript converter now accepts both the older flat `role` JSONL shape and
Puffer's typed `TranscriptEvent` JSONL shape. Tool invocations render with the
tool id, parsed input JSON when available, success state, and a bounded output
excerpt for direct-skill prompting.

## Contracts

- Synthetic expert files are model-generated and must not be represented as
  human expert recordings.
- Existing generated files are reused unless `FORCE=1` is set.
- Model selection follows `PUFFER_EVAL_PROVIDER`, `PUFFER_EVAL_MODEL`, and
  `PUFFER_EVAL_EFFORT`, with OpenAI/GPT-5.4 defaults.
- Replays can override the mounted Linux agent binary with `PUFFER_REPLAY_BIN`.
