# puffer-genskill-eval Update 04

## Summary

Adds shared vcpkg and compiler caches for Ladybird replay containers and
completes the sandbox image package set needed by older corpus commits.

## Design And Architecture

The Ladybird Docker image now installs autotools packages used by vcpkg ports
such as gperf: `autoconf`, `autoconf-archive`, `automake`, `libtool`, and
`libtool-bin`. It also installs Ubuntu's `generate-ninja` package, which
provides the `gn` binary used by Skia's vcpkg overlay port, and `ccache`,
which lets repeated replay builds reuse identical C/C++ compilations.

Each replay sandbox mounts a host cache directory at
`/work/vcpkg-binary-cache` and sets
`VCPKG_BINARY_SOURCES=clear;files,/work/vcpkg-binary-cache,readwrite`.
The default host path is
`benchmark/genskill/ladybird/.cache/vcpkg-binary`; callers can override it
with `PUFFER_LADYBIRD_VCPKG_BINARY_CACHE`.

Each replay sandbox also mounts a host compiler cache directory at
`/work/ccache` and sets CMake's `CMAKE_C_COMPILER_LAUNCHER` and
`CMAKE_CXX_COMPILER_LAUNCHER` to `ccache`. The default host path is
`benchmark/genskill/ladybird/.cache/ccache`; callers can override it with
`PUFFER_LADYBIRD_CCACHE_DIR`.

## Logic

The sandbox still starts from a fresh container and resets the Ladybird checkout
to the corpus `base_commit` before copying tests. Only vcpkg binary packages
and compiler cache entries are shared across replays. Source files, build
trees, test files, and agent edits remain container-local.

The cache directory is created by the host harness before `docker run`. On Unix
hosts it is made world-writable so the non-root `ladybird` container user can
write binary packages through Docker Desktop bind mounts.

## Contracts

- Replay correctness must not depend on files written by a previous agent run.
- The vcpkg and compiler caches are optimizations only; deleting `.cache/`
  must not change benchmark semantics.
- The cache is ignored by git.
