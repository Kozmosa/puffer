# puffer-genskill-eval Update 00

## Summary

Documents the Ladybird replay harness integration with
`puffer non-interactive`.

## Design And Architecture

The eval crate remains a black-box harness under
`benchmark/genskill/ladybird`. It owns corpus validation, replay execution,
sandbox orchestration, metrics, and report rendering. The replay path launches
a Docker container, mounts the release `puffer` binary, copies corpus tests,
and invokes `puffer non-interactive` inside Ladybird.

## Logic

Replay now passes `--artifact-pr` and `--artifact-arm` to the non-interactive
CLI so the emitted JSON can deserialize directly as a `ReplayArtifact`.

The sandbox mounts:

- the `puffer` binary at `/usr/local/bin/puffer`
- corpus tests at `/work/test_files`
- the host repo at `/host` for direct/GEPA skill files

It forwards common provider environment variables so containerized runs can use
the same auth/model selection as the host.

## Contracts

- The crate is tested and built with `-p puffer-genskill-eval`; the full
  workspace build is not required.
- Replay artifacts remain comparable across the frozen 10-PR corpus.
- `direct` and `gepa` arms read skills from
  `/host/benchmark/genskill/ladybird/pr_corpus/<pr>/skills/<arm>/SKILL.md`.
- The Docker image tag contract remains `puffer-genskill-eval-ladybird`.

## Verification

- `cargo test -p puffer-genskill-eval`
- `cargo build -p puffer-genskill-eval --release`
