# puffer-genskill-eval Update 08

## Summary

Adds a bounded-parallel full benchmark runner for the 30-PR Ladybird corpus.

## Design And Architecture

The replay subcommand accepts an optional `--run-date` argument so independent
worker processes can write artifacts into the same explicit report directory.
When omitted, replay keeps the previous behavior of using the current UTC date.

`scripts/run_all_parallel.sh` mirrors the guarded full-run workflow from
`run_all.sh` but generates a PR/arm task list and executes it with bounded
`xargs -P` parallelism. The default is `JOBS=4`.

## Logic

The parallel runner:

- verifies every corpus entry has `expert_run.md` and direct/gepa skills
- clears stale artifacts from the selected report directory
- builds the puffer CLI, eval binary, and Linux puffer replay binary
- validates the corpus
- runs all `30 x 3` replay jobs with the configured parallelism
- aggregates the selected run directory after workers finish

## Contracts

- Full eval still refuses to start when required inputs are missing.
- Parallel workers must use one explicit report directory, not each worker's
  local default date.
- Existing serial `run_all.sh` behavior remains available.
