# puffer-genskill-eval Update 02

## Summary

Replay target-test selection now comes from the frozen corpus test files instead
of `meta.json` source-file metadata. A guarded full-run script records the
single command that launches the 30 replay jobs once expert runs and generated
skills exist.

## Design And Architecture

Each replay derives target filters from `pr_corpus/<pr>/tests`. Runnable input
files, crash tests, and C++ test sources are used directly. LibWeb text/layout
expected-output files are mapped back to their corresponding input HTML files,
then filters are sorted and deduplicated. This keeps PRs with multiple test
cases, such as canvas transform and CSS Typed OM coverage, from collapsing to a
single broad PR-id fallback.

`scripts/run_all.sh` remains an orchestration wrapper only. It validates that
every PR has `expert_run.md`, `skills/direct/SKILL.md`, and
`skills/gepa/SKILL.md` before building release binaries, validating the corpus,
running `no-skill`, `direct`, and `gepa` replays, and aggregating the report.

## Logic

Target tests run every derived filter in one shell command and preserve the
first non-zero state as the command exit status. The artifact therefore still
has one `test_outcome`, but the command text records the exact filters used.

## Contracts

- Replays no longer rely on `meta.files_changed` containing test paths.
- `run_all.sh` refuses to spend replay/model time when prerequisite expert
  transcripts or generated skills are absent.
- Full eval remains 10 corpus PRs times three arms: `no-skill`, `direct`,
  and `gepa`.
