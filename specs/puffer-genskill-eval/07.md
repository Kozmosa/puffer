# puffer-genskill-eval Update 07

## Summary

Expands the frozen Ladybird replay corpus from 10 to 30 entries with 20
lightweight PRs focused on AK, LibCore, LibJS, LibRegex, and LibWasm tests.

## Design And Architecture

The added entries avoid broad LibWeb layout, ref, and screenshot tests. Each
new corpus entry includes:

- `meta.json` with PR metadata, base and merge commits, changed files, area,
  and a task prompt
- `reference_fix.patch` copied from the upstream merged PR patch
- a minimized `tests/` subtree containing only the target test files and
  required small fixtures
- `test_filters.txt` selecting the focused target tests used by replay

Replay now recognizes `Libraries/LibJS/Tests/*.js` and
`Libraries/LibWasm/Tests/Executor/*.js` filters in addition to existing
LibWeb and C++ unit filters. These run through the `test-js` and `test-wasm`
executables after building only the relevant target with low parallelism.

## Logic

LibJS filters are converted from repository paths to `test-js -f` globs
relative to `Libraries/LibJS/Tests`. LibWasm executor filters are converted in
the same way relative to `Libraries/LibWasm/Tests`. `.wasm` fixture files are
kept in the corpus for test execution but are not treated as runnable filters.

Unsupported filter shapes continue to fail visibly with status 127.

## Contracts

- The validate subcommand must report 30 corpus entries.
- Lightweight JS/Wasm PRs must not require LibWeb test targets.
- Fixture files may exist under `tests/`, but only runnable `.js`, `.cpp`, and
  LibWeb `.html` filters are passed to target runners.
- Existing 10-entry corpus paths and generated synthetic eval artifacts remain
  untouched by this corpus expansion.

## Verification

- `cargo test -p puffer-genskill-eval` passes 29 tests.
- `cargo run -p puffer-genskill-eval -- validate` reports `OK: 30 entries`.
