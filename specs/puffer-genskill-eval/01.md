# puffer-genskill-eval Update 01

## Summary

Tightens the Ladybird Docker image, replay test execution, and replay model
selection so corpus base commits, target test results, and agent identity are
real eval inputs.

## Design And Architecture

The eval image keeps the full Ladybird commit graph instead of a depth-1 clone.
Replay sandboxes can therefore reset to the historical `base_commit` recorded
in each corpus entry. The image creates a non-root `ladybird` user, and replay
commands execute as that user because Ladybird's build helper refuses root-owned
build directories. It also installs GCC/G++ 14, Python venv support, and Mesa
development headers required by the frozen corpus commits' vcpkg setup.

Sandbox command execution now has two forms: `exec`, which still treats
non-zero exit status as an error, and `exec_status`, which returns stdout,
stderr, and the actual exit code for commands whose failure is meaningful eval
data.

Replay uses the OpenAI/Codex provider path by default. The eval CLI passes
`--provider openai --model gpt-5.4` to `puffer non-interactive`, unless
`PUFFER_EVAL_PROVIDER`, `PUFFER_EVAL_MODEL`, or `PUFFER_EVAL_EFFORT` override
those values. The sandbox forwards those eval-specific environment variables
and mounts the host `~/.codex` directory read-only when present. Non-interactive
Puffer imports Codex OpenAI credentials in-memory when no `OPENAI_API_KEY` is
set, preserving any Codex base URL, headers, and query params.

## Logic

Target tests run through the Ladybird wrapper available at the checked-out
commit:

- `python3 Meta/ladybird.py test <filter>` when present
- `./Meta/ladybird.sh test <filter>` for older commits
- `ladybird-test --filter=<filter>` as a final fallback

The replay artifact records the real target-test exit code. A non-zero test
result changes an otherwise successful replay to `wrong-fix`.

## Contracts

- Docker image tag remains `puffer-genskill-eval-ladybird`.
- The corpus continues to define base commits and test filters.
- Replay does not assume the latest Ladybird checkout has the same build/test
  wrapper as older corpus commits.
- Replays default to OpenAI/Codex-compatible model execution. Operators may pin
  another Codex-accessible model with `PUFFER_EVAL_MODEL`.
