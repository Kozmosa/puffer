# puffer-genskill-eval Update 09

## Summary

Switches Ladybird skill generation to reference-fix-derived skill inputs and
keeps generated replay artifacts out of version control.

## Design And Architecture

`scripts/generate_skills.sh` now reads each corpus entry's `meta.json` and
`reference_fix.patch` directly. It builds a temporary reference-solution
context that direct generation and `/genskill` can use to produce comparable
direct and GEPA `SKILL.md` files.

The replay phase still loads only the generated skill file for the selected
arm. It never receives the raw patch.

## Logic

The reference context asks skill generation to distill:

- bug root cause
- key files, functions, classes, and symbols
- failing-test meaning
- correct fix shape
- common wrong fixes

`run_all_parallel.sh` runs all three arms for a single PR serially inside each
worker slot. This avoids triple-building the same Ladybird target at once while
still allowing bounded PR-level parallelism.

## Contracts

- `generate_skills.sh` fails fast if `meta.json`, `reference_fix.patch`, or the
  puffer binary is missing.
- Direct and GEPA skill generation use the same reference source.
- Generated `expert_run.*`, `expert_run_artifact.json`, and `skills/`
  directories are local eval artifacts ignored by git.
