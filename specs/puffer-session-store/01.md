# Transcript Message Actors

## Purpose

Transcript events can now carry optional actor metadata so external runtimes can
distinguish root agent output from subagent output without inferring ownership
from message text.

## Design

`MessageActor` records a required actor kind and id plus optional agent
id/type/name, team name, session id, and parent session id. User, assistant, system,
command, and tool invocation transcript events include an optional `actor`.
Tool invocation events also include an optional `subject` for future cases
where one actor reports on another actor.

## Compatibility

Actor fields on transcript events are optional and default to `None` during
deserialization. When an actor object is present, `kind` and `id` are required.
Unknown actor kinds deserialize to `unknown` so future or external kinds do not
make a transcript unreadable. Existing JSONL transcript lines remain valid and
continue to replay. Serialization omits missing actor fields, preserving the old
shape for callers that do not set actor metadata.
