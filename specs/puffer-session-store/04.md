# JSON-Compatible Read Timestamps

## Purpose

Keep historical session transcripts readable when state snapshots contain
Claude read-state timestamps.

## Design

`ClaudeReadSnapshotEvent.timestamp_ms` remains a `u128` in Rust so existing
runtime APIs do not change. The session-store serde layer now encodes values
that fit in JSON numbers as `u64` and falls back to decimal strings for larger
values.

## Logic

Deserialization accepts non-negative JSON numbers and decimal strings, then
normalizes them into `u128`. Serialization preserves numeric JSON for ordinary
file modification timestamps, avoiding `serde_json`'s unsupported `u128`
number path.

## Contracts

- Existing JSONL state snapshots with numeric `timestamp_ms` fields load.
- New ordinary timestamps continue to be written as JSON numbers.
- Extremely large timestamps round-trip as decimal strings.
- Session listing and detail loading must not fail solely because a transcript
  contains Claude read-state timestamps.
