# Session Summary Paging

## Purpose

Allow desktop clients to load recent sessions in small windows instead of
forcing the store to parse every transcript JSONL before the first render.

## Design

`SessionStore::list_sessions_page` returns sorted session summaries plus page
metadata. The store still reads session metadata for sorting, but it computes
event counts only for the requested window.

## Logic

Session metadata loading and summary construction are split internally. Full
listing maps every metadata record into a summary. Paged listing slices the
sorted metadata first, then builds summaries only for that slice.

## Contracts

- Full `list_sessions` behavior and sort order are unchanged.
- Paged results use the same newest-first ordering as full listings.
- Page metadata reports the total session count and whether more sessions
  remain after the returned window.
- Offset and limit are interpreted against the sorted global session list.
