# Tool Invocation Metadata Persistence

## Purpose

Persist structured metadata on transcript tool invocation events so runtime
trace annotations, including Lambda Skill host-call facts, survive session
storage and export.

## Design

`TranscriptEvent::ToolInvocation` gains an optional `metadata` JSON field.
Writers omit it when the invocation metadata is `null`.

## Logic

Tool invocation appenders copy non-null `ToolInvocation.metadata` into the
session event. Readers treat missing metadata as `None`, preserving old JSONL
transcripts.

## Contracts

- Existing transcript lines without `metadata` continue to deserialize.
- New metadata is stored as structured JSON, not stringified text.
- Session storage does not interpret or validate metadata contents.
- Tool output text remains unchanged.
