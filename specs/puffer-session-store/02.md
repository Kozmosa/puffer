# Legacy-Compatible Session Permission Snapshot Fields

## Purpose

Extend `TranscriptEvent::StateSnapshot` so session resume can recover the
legacy-compatible subset of session permissions without changing the
append-only event model.

## Design

`StateSnapshot` gains two optional fields:

- `session_allow_all: bool`
- `session_tool_permissions: map<string, string>`

These fields are compatibility fields, not a durable typed permission schema.

## Logic

Writers may include the compatibility fields when persisting a runtime state
snapshot.

Readers must treat both fields as optional and default them to:

- `session_allow_all = false`
- `session_tool_permissions = {}`

This keeps older snapshot lines valid while allowing newer runtimes to rebuild
typed session permission state from the compatibility projection.

## Contracts

- `StateSnapshot` remains append-only and backward-compatible.
- Older snapshot lines missing the new fields must continue to replay.
- The store does not serialize full typed permission-grant structure yet.
- The compatibility fields represent only allow-all plus whole-tool approvals.
