# Background Session Visibility

- Background agent sessions use the internal `puffer:background` session tag.
- `SessionStore::create_session_with_tags` creates a session with normalized initial tags so non-interactive runtimes can persist their provenance before any transcript events are written.
- `SessionStore::list_sessions` and `SessionStore::list_sessions_page` hide sessions tagged `puffer:background` by default.
- Empty interactive sessions remain visible so newly-created user sessions can still be resumed before their first transcript event.
- Explicit `*_excluding_tags` listing APIs remain available for callers that need a custom visibility policy or an unfiltered listing by passing an empty exclusion list.
- Pagination totals are computed after hidden sessions are removed, so desktop session pages count only visible user sessions.
