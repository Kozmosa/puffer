# Vendored Protoc Build

## Problem

The gRPC runner build script required a host-level `protoc` binary. On machines
without protobuf installed, Rust tests that transitively compile
`puffer-runner-grpc` failed before daemon and desktop integration regressions
could run.

## Design

- The build script still honors an explicitly configured `PROTOC` binary.
- When `PROTOC` is absent, it uses `protoc-bin-vendored` to provide a pinned
  protoc executable for `tonic-build`.
- Proto compilation arguments and generated client/server bindings stay
  unchanged.

## Test Coverage

- `cargo test -p puffer-cli create_session_ -- --nocapture` can compile through
  the gRPC runner crate on hosts without a system `protoc`.
