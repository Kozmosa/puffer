# puffer-runner-grpc Proto Build Compatibility

## Purpose

The gRPC runner crate compiles `proto/tool_runner.proto` at build time through
`tonic_build`. The proto schema uses proto3 optional fields, which require an
explicit compatibility flag on older `protoc` releases such as 3.12.x.

## Design

`build.rs` keeps the existing tonic client/server generation path and adds
`--experimental_allow_proto3_optional` to the `protoc` invocation. Newer protoc
versions accept the flag, and older versions need it to parse the schema.

## Logic

The build script still watches only `proto/tool_runner.proto`, builds both
client and server bindings, and writes generated Rust into Cargo's output
directory. No runtime code path changes.

## Contracts

This change is build-only. It does not alter the proto schema, generated API
shape, wire format, runner capabilities, or MCP/tool execution behavior.
