# gRPC E2E Listener Reservation

## Summary

The gRPC runner e2e tests now reserve their loopback listener before starting
the server thread instead of selecting a free port and rebinding it later.

## Behavior

- Normal test servers bind `127.0.0.1:0`, keep the listener open, and pass that
  listener into the tonic server.
- Tests that intentionally restart on a fixed port still bind the requested
  port at server creation time.
- The server endpoint is derived from the already-bound listener address.

## Compatibility

This is a test harness change only. Runtime gRPC server behavior, remote runner
client behavior, authentication, and MCP transport contracts are unchanged.
