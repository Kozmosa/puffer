# Required tmux coverage gate

Tmux-backed integration tests now use `require_tmux_or_skip` instead of
returning directly from `tmux_available`.

Local runs without tmux still skip gracefully, but they print a clear message
that the test did not exercise the TUI. CI or focused TUI validation can set
`PUFFER_REQUIRE_TMUX_TESTS=1` to make missing tmux fail the test instead of
reporting a green no-op. The legacy `PUFFER_REQUIRE_TMUX=1` spelling is also
accepted.

This keeps the tmux harness useful for local development while giving release
and deep-test workflows a hard signal that real TUI interaction coverage
actually ran.
