# Telegram list-message terminal event matching

## Scope

Subscription command matching now distinguishes sender-filtered Telegram message
list requests.

## Design

`command_matches_terminal_event` compares a `TelegramListMessages` command's
optional `sender` against the `message_list.sender_filter` payload in addition
to the existing peer and cursor checks. It deliberately does not use
`scan_limit` for event identity because Telegram subscribers clamp requested
scan windows before emitting `message_list`.

## Compatibility

Unfiltered commands omit both optional fields and match as before. The added
checks prevent concurrent filtered list-message requests on the same peer and
cursor from consuming each other's terminal events.
