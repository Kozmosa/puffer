# puffer-subscriptions update 68

Connector stream contact scope now distinguishes all contacts, selected
connector-owned contacts, and no connector-owned contacts.

Workflow monitor filters are normalized through the connection's connector
family before starting a command-backed subscribe stream. If every enabled
workflow filter for a connection contains only contact ids owned by other
connector families, the manager does not start that connector stream instead of
omitting `contact_ids` and accidentally subscribing to all contacts.

Agent proxy bindings still request an all-contact stream because they must see
unscoped inbound commands and bound-agent messages. Explicit all-contact
workflow monitors also keep the existing omitted-`contact_ids` subscribe shape.
