# History-Backed Contact Standard Methods

- Subscriber-backed connectors without command executors still implement the
  standard contact list, search, and context methods by reading workflow history.
  This covers Telegram personal-account, Gmail browser, and Google Calendar
  browser connections that ingest events through host-managed subscribers.
- History fallback only reads runs whose stored `connection_slug` matches the
  requested connection, preventing older or malformed history rows from leaking
  contacts across configured accounts.
- History fallback contact ids are extracted from connector trigger payloads,
  normalized through the shared contact id helpers, and filtered by connector
  family before being returned to callers.
- Connector-native contact list and search results are normalized before family
  filtering, so casing differences such as `Telegram@Alice` do not cause valid
  connector-owned contacts to be dropped.
- Contact context fallback returns high-entropy deduplicated history snippets for
  the requested ids, preserving connector-specific limits while returning `None`
  when a request has no ids owned by that connection's connector family.
- The manager contact method implementation lives in a dedicated child module so
  the process manager remains below the repository file-size limit.
