# Gmail Browser User Action Request

## Summary

The built-in `gmail-browser` connector template now advertises a
`requestuserbrowseraction` connector action for interactive browser steps.

## Action Contract

The action accepts a required `description` and an optional `url`. Its permission
category is `user_browser_action`, and it is marked as an external side effect
because it asks the user to complete an action in a browser profile that may
change account state.

The Gmail-browser template description now refers to the global Puffer browser
profile rather than a configured Chrome profile. The existing email list, search,
draft, send, and delete actions remain unchanged.

## Runtime Expectations

The subscriptions catalog only declares the action and permission metadata.
Actual prompting is handled by `ConnectorAct`, which routes the action to the
internal Puffer workflow instead of dispatching it to a subscriber process.
