# Workflow Trigger Readiness

## Summary

Subscription connector metadata now has shared helpers for determining whether
a connector or concrete connection can produce workflow trigger events.

## Behavior

- `connector_workflow_trigger_supported(roots, template)` returns true when the
  template has an event-capable command-backed stream, a shared subscriber
  manifest, or a direct subscriber manifest.
- `connection_workflow_trigger_supported(roots, connection, template)` returns
  true when the concrete connection has an event-capable command-backed stream
  or manifest-backed subscriber source.
- Commandless event placeholders without a subscriber manifest are treated as
  unavailable for workflow triggers.

## Compatibility

This is metadata only. It does not change persisted connection records,
workflow binding formats, subscriber startup, or action dispatch.
