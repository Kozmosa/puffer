# Opsgenie Connector Catalog Preset

The built-in connector catalog now includes `opsgenie-webhook`, a setup-only
serve-mode preset for outgoing Opsgenie alert action callbacks.

## Design

- The webhook catalog templates were moved into `catalog/webhooks.rs` so the
  root catalog module stays under the 1000-line repository limit.
- `opsgenie-webhook` advertises `puffer connector webhook`, requires no auth at
  catalog time, has no subscriber, has no actions, and does not claim workflow
  trigger capability.
- The deterministic connection slug is `opsgenie-webhook`.
- The output schema uses `kind=opsgenie_alert` with action, alert id, alias,
  tiny id, message, priority, entity, source, and integration fields.

## Compatibility

Serve-mode subscriber manifest synthesis treats Opsgenie like the other webhook
presets. The catalog split preserves existing webhook template ordering and
public lookup behavior.
