# Contact Header Display Names

## Design

Subscription-backed contact history now derives cleaner display names from
email-style address headers.

## Architecture

`contacts` exposes a shared payload display-name helper. It preserves explicit
name fields, extracts the display portion from single address headers, and
returns no display name for bare addresses instead of treating the address as a
person name. `contact_history` uses that helper for connector contact
list/search/context data.

## Compatibility

Contact ids, filters, connector method shapes, and history storage are
unchanged. Existing events with plain non-email names still surface those names;
only email header display-name cleanup changes.
