# Telegram Self Sender Extraction

## Design

Telegram contact-id extraction now treats direct user chats and group sender
events differently. Direct chats produce the peer `chat_username`; group chats
produce the `sender_username` only for non-outgoing messages.

## Logic

Outgoing direct messages no longer add the local account's `sender_username` as
a contact id. Outgoing group messages are also skipped as contact ids because
they represent the local account, not a candidate contact.

## Compatibility

Incoming direct and group messages keep the same normalized `telegram@...`
ids. Existing bot filtering remains in place.
