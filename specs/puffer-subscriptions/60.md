# Connector Monitor Routing Diagnostics

- Connection consumer refresh logs workflow and proxy consumer inputs, previous
  consumer state, and previous connection state before updating a connection.
- Subscriber startup logs subscriber id, topic, manifest directory, and declared
  state directory; repeated starts log that the subscriber is already running.
- Workflow routing logs control-event skips, binding skip reasons, dispatch
  starts, batch dispatch starts, no-action outcomes, and hashed dedup keys.
- Triage-agent actions log start, completion, failure, batch size, model label,
  prompt length, and event text length without adding connector payload content.
