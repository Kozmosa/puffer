# PagerDuty Webhook Connector Preset

## Change

The built-in connector catalog now includes `pagerduty-webhook`, a setup-only
serve-mode preset backed by `puffer connector webhook`.

## Contracts

- The deterministic connection slug is `pagerduty-webhook`.
- The preset has no connector protocol command, subscriber manifest, proxy
  capability, or outbound actions.
- `requires_auth` is false at the catalog layer because transport protection is
  configured by the generic webhook connector rather than by connector auth.
- Runtime hints identify the preset as `serve` so `/workflows connectors`,
  `/connect`, and desktop catalog search can find it with setup-only terms.
- The output schema uses `kind=pagerduty_event` with `event_type`,
  `resource_type`, `incident_id`, `service`, `status`, `urgency`, `message`, and
  `url` fields.

## Compatibility

Existing connector slugs and suggested connection names are unchanged. The
catalog count increases by one, so snapshot and UI count assertions move from
20 to 21 connectors.
