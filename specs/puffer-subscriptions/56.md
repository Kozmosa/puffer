# Monitor History Running Rows

- Direct workflow history supports a `running` status for matched actions.
- The router writes a started history run before dispatching a matched action, then completes the same run in place with the final status, summary, and token usage.
- Skipped monitor outcomes still write single completed rows and do not affect matched/acted counters.
- Existing completed/failed history remains compatible; new rows may temporarily have `running` in both run status and action log status.
