# Monitor Event History

- The workflow history store can append router outcomes that did not execute an action.
- Monitor bindings record muted/silent skips, installed ignore-filter skips, trigger-filter skips, and classifier skips as completed history runs with `monitor_*` action names.
- Regular action history remains unchanged: dispatched actions still record their original action kind, status, summary, trigger payload, and timing.
- Agent-backed workflow actions may attach provider token usage to their action log entries.
- Router counters and match semantics are unchanged for skipped monitor events; skip history is observability only and does not count as an acted workflow.
