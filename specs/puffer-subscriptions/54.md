# Google Calendar Browser Catalog

The connector catalog now includes `gcal-browser` as a first-class built-in
browser-backed connector.

- Default connection slug is `gcal-browser`.
- The connector can subscribe and uses the `gcal-browser` subscriber manifest
  with state rooted at `gcal-browser-accounts`.
- The output schema remains the generic message schema so monitor routing can
  consume event text like other connectors.
- Host-enforced actions are:
  - `get_detail` with `external_calendar_read` and no external side effect.
  - `accept` with `external_calendar_rsvp` and an external side effect.
  - `deny` with `external_calendar_rsvp` and an external side effect.
- `gcal_browser_act` terminal events are matched by action name when waiting
  for subscriber command completion.
