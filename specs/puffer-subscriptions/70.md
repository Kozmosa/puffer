# Monitor Keyword Filters

## Design

`FilterSpec` supports tagged `any` compounds for OR-style filter fanout.
Monitor rule RPCs compile keyword chips into literal regex text contains
filters. Rules do not combine keyword matching with connector payload fields,
so the subscription layer does not need connector-specific field semantics.

## Contracts

- Empty `any` filters are invalid.
- Regex keyword matching remains literal when callers escape keywords before
  constructing the filter.
- Router prefilters and ignore filters evaluate keyword filters before
  dispatching triage or workflow actions.
- Non-matching include filters skip events before any LLM call.

## Tests

- Unit tests cover multiple keyword OR behavior and literal text matching for
  regex-special input.
- Router tests use Given/When/Then structure for keyword ignore behavior and
  include-filter skip behavior.
