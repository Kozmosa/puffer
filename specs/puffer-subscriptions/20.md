# Restart-Aware Subscriber Commands

`SubscriptionManager::send_command_and_wait` now watches the subscriber command
sender generation while waiting for a terminal event. If the child process
restarts after a command is written, the manager resends the same command after
stdin reconnects instead of waiting for an event that can no longer arrive.

The retry is bounded to avoid hiding a crash loop. Existing terminal-event
matching remains unchanged, and control events still stay out of workflow
routing. This primarily protects internal tools such as Telegram peer lookup
and auth checks from transient subscriber restarts.
