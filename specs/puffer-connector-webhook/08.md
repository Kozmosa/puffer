# Shopify Webhook Normalization

## Summary

Adds Shopify-specific handling to the generic webhook connector. Incoming
Shopify JSON payloads are normalized into Puffer inbound messages using Shopify
HTTP webhook headers and common resource fields.

## Routing

Shopify conversion recognizes:

- `X-Shopify-Topic`
- `X-Shopify-Shop-Domain`
- `X-Shopify-Webhook-Id`
- `X-Shopify-Triggered-At`
- `X-Shopify-Event-Id`
- JSON bodies that include `topic` or `shop_domain` fields

The preset reuses the generic webhook connector's existing origin and optional
bearer-token checks.

## Message Shape

Puffer groups the conversation by shop, topic, and the first stable Shopify
subject in the delivery, for example
`shopify:<shop>:orders_create:order:<order-id>`. Messages include topic, shop,
subject, status, amount, URL, email, timestamp, event id, and note text when
present.

## Compatibility

Native Puffer JSON, GitHub, Linear, Asana, Jira, Stripe, Trello, and GitLab
payload routing is unchanged. Shopify HMAC headers are used only as provider
metadata; the preset does not verify signatures until the setup flow collects an
application secret.
