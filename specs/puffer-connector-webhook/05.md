# Stripe Webhook Payload Normalization

The webhook connector recognizes Stripe Event payloads and converts them into
stable inbound Puffer messages.

## Behavior

- Stripe payloads are detected from `Stripe-Signature` headers or Event-shaped
  JSON with `object = "event"` and `data.object`.
- Conversation ids use `stripe:<account>:<object-type>:<object-id>` when
  account metadata is present, otherwise `stripe:<object-type>:<object-id>`.
- Messages include event type, account, live/test mode, subject, related object
  references, amount, URL, request id, and event id when available.
- Nested Stripe object references such as customer objects are reduced to their
  `id` values for readable summaries.

## Compatibility

Native Puffer JSON, GitHub, Linear, Jira, and GitLab webhook payload handling is
unchanged. Stripe signature verification remains the responsibility of the
deployment edge or future connector-specific auth work.
