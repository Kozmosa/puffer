# New Relic Webhook Normalization

The webhook connector now recognizes New Relic issue and alert webhook
payloads before the generic fallback path.

## Design

- Provider detection accepts explicit `X-NewRelic-Webhook` headers, source
  fields equal to `newrelic` or `new_relic`, modern issue payload fields such
  as `issueId`, `issueTitle`, `triggerEvent`, and classic migrated alert
  fields such as `incident_id`, `condition_name`, `current_state`, and
  `incident_url`.
- Detection is conservative: generic incident payloads with only
  `incident_id`, state, and details are not claimed unless a New Relic-specific
  identifier, link, account, condition, policy, or explicit source hint is
  present.
- Conversation ids use the strongest stable identifier:
  `newrelic:issue:<issue-id>`, then `newrelic:incident:<incident-id>`, then
  `newrelic:event:<trigger-event-or-title>`.

## Message Contract

Normalized messages include the headline, title, state, priority, condition,
policy, entities, incident ids, account id, details, runbook URL, chart URL,
and New Relic issue or incident link when present.
