# Linear Payload Normalization

## Summary

The webhook connector accepts Linear webhook JSON payloads in addition to
native Puffer and GitHub payloads.

## Logic

- Native Puffer payloads and GitHub payloads retain precedence and behavior.
- Linear payloads are recognized from `Linear-Event`, `Linear-Delivery`, or
  body markers such as `webhookId`, `webhookTimestamp`, and `organizationId`.
- Issues use stable `linear:issue:<identifier>` conversation ids when Linear
  identifiers are available.
- Comments are threaded to their issue with `linear:issue:<issueId>`.
- Generated messages include entity type, action, actor, subject, URL, body
  snippet, and sorted `updatedFrom` field names when present.

## Compatibility

Bearer-token auth, origin checks, response shaping, and unsupported payload
errors remain unchanged. Linear signature verification is not added here; this
change is transport-level normalization for the existing webhook endpoint.
