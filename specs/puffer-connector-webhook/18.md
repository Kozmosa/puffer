# Figma Webhook Preset

## Scope

Adds first-class normalization for Figma Webhooks V2 payloads received through
the generic `puffer serve` webhook connector.

## Design

- A new provider normalizer recognizes Figma payloads with supported
  `event_type` values and one of `webhook_id`, `file_key`, or `passcode`.
- Supported events are `PING`, `FILE_UPDATE`, `FILE_DELETE`,
  `FILE_VERSION_UPDATE`, `LIBRARY_PUBLISH`, `FILE_COMMENT`, and
  `DEV_MODE_STATUS_UPDATE`.
- Conversation IDs prefer stable Figma identifiers:
  - comments use file key plus parent or comment id
  - version updates use file key plus version id
  - library publishes use file key plus webhook or event id
  - Dev Mode updates use file key plus node id
  - file updates and deletes use the file key
  - pings use the webhook id
- Rendered messages include file identity, actor, comment fragments and
  mentions, version details, library publish counts, Dev Mode status, related
  links, timestamp, and URL where present.

## Compatibility

The normalizer is additive. Unknown Figma-like events and unrelated payloads
fall through to later provider normalizers or the generic webhook formatter.
