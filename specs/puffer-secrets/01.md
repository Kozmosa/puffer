# Chrome Secret Sync Metadata

`puffer-secrets` tracks richer non-secret metadata and can sync all discoverable
Chrome profile credentials into the encrypted vault.

- Secret summaries, upserts, resolved secrets, and store records include an
  optional `description` field. Existing stores deserialize with `description`
  absent.
- Chrome sync scans every direct profile directory under the macOS Chrome user
  data root that contains a `Login Data` database instead of relying on a fixed
  profile-name list.
- Chrome-imported secrets use the site host as `description`, keep the full
  origin URL in `origin`, and keep the login username in `username` when present.
- Metadata search matches terms across id, name, description, username, origin,
  and source without decrypting secret values.
- `import_chrome_saved_credentials` remains as a compatibility wrapper around
  the new sync behavior.
