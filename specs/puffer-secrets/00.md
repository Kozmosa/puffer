# Encrypted Secret Vault

`puffer-secrets` owns local encrypted secret storage for agent-requestable
credentials.

- The vault stores non-secret metadata and AES-256-GCM ciphertext in
  `secrets.json` under the user config directory.
- On macOS, the data-encryption key is created and loaded from the macOS
  Keychain generic-password item `Puffer Code Secrets` / `puffer-code`.
- On other platforms, the key is stored beside the vault as `secrets.key` with
  private file permissions. `PUFFER_SECRET_STORE_KEY` can override the key for
  tests and controlled automation.
- Chrome import is supported on macOS by reading copied `Login Data` SQLite
  databases, decrypting `v10`/`v11` values with Chrome Safe Storage, and writing
  the imported passwords back through the same encrypted vault path.
- Public vault APIs expose only summaries unless `reveal` is called by the
  runtime after user approval.
