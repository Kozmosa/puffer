# ConnectorAct Email Operations

## Summary

The email subscriber now handles `SubscriberCommand::Custom` with
`op = "email_act"` so model-facing `ConnectorAct` calls execute through the
configured IMAP/SMTP account instead of a browser workflow.

## IMAP Actions

- Listing selects the requested mailbox/category/label, searches `ALL`,
  `UNSEEN`, or `SEEN`, fetches recent RFC822 messages with flags, and filters
  locally by sender, recipient, subject, and keywords.
- `mark_read` resolves `uid`, `uids`, `id`, or `message_id` selectors and
  applies `+FLAGS.SILENT (\Seen)`.
- `delete` applies `+FLAGS.SILENT (\Deleted)` and expunges by default.
- Draft actions append RFC822 plain-text messages to a draft mailbox, trying
  the requested mailbox, `Drafts`, and `[Gmail]/Drafts`.

## SMTP Actions

`send_email` builds a plain-text SMTP message with `to`, optional `cc` and
`bcc`, subject, body, `In-Reply-To`, and `References`. The existing
`SendMessage` compatibility path delegates to the same SMTP helper.

## Events

Successful actions emit `email_action_complete` with the action slug, summary,
and action payload. Failures emit `email_action_error` with the same action slug
and a human-readable error.
