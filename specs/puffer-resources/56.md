# External Lambda Skill Libraries

External Lambda Skill libraries can be imported without vendoring the library
into Puffer.

## Design

Resource layers may declare library manifests under
`lambda_skill_libraries/*.yaml`. Each manifest names an `id` and a filesystem
`root`. Puffer recursively discovers skill directories below that root by
looking for `skill.lskill` or `main.lskill`, skipping dot directories and build
output directories.

Imported Lambda Skills use the generated prompt descriptor at
`out/GENERATED.SKILL.md` by default. The `SkillSpec.source_info.path` remains
the formal `.lskill` source path so `${CLAUDE_SKILL_DIR}` resolves to the
external skill directory rather than to generated output. The imported prompt
body includes a short verification header naming the formal source and generated
descriptor.

`PUFFER_LAMBDA_SKILL_ROOTS` may also name one or more external roots for local
experimentation. Explicit resource manifests should be preferred for durable
workspace configuration.

## Contracts

Imported Lambda Skills are user-invocable by default and model invocation is
disabled by default, avoiding a 249-skill model prompt expansion unless a
workspace explicitly opts into it.

The loader preserves verification metadata on `SkillSpec.verification`,
including the Lambda Skill system name, source path, generated descriptor path,
optional host catalogue path, and generated stats when present. The library
itself remains outside the Puffer repository.

This update covers resource discovery and verified prompt provenance only.
Runtime fill and tool-call gate enforcement remain separate integration steps.

## Verification

`cargo test -p puffer-resources` covers importing an external Lambda Skill
library from a workspace resource manifest, using the generated descriptor body,
retaining the formal source path, and preserving verification metadata.
