# RequestSecret Tool Resource

`resources/tools/request_secret.yaml` declares the model-facing `RequestSecret`
workflow tool.

- Aliases `request-secret` and `request_secret` are accepted.
- The schema accepts `id`, `label`, and an optional non-secret `reason`.
- The description states that output is a `PUFFER_SECRET_...` placeholder and
  that Puffer performs execution-time replacement plus result redaction.
- The handler is `runtime:workflow:request_secret`; approval is handled inside
  the runtime so the prompt can offer per-secret or all-secret session scopes.
