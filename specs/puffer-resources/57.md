# puffer-resources update 57

Imported Lambda Skills now include bridge guidance in their generated prompt
header.

## Design

`render_lambda_skill_content` keeps the existing provenance header and adds a
short runtime bridge line directing strict executions to use `LambdaHostCall`
before the concrete Puffer tool that implements a formal host operation.

The source library remains external. Puffer still loads `skill.lskill`,
`out/GENERATED.SKILL.md`, and `out/stats.json` from the configured external
root without copying the library into the repository.

## Contracts

The guidance is informational for prompt-only execution and operational for
strict Lambda Skill runs where the active gate and `LambdaHostCall` bridge are
available.

## Verification

`cargo test -p puffer-resources lambda_skill_library_imports_generated_external_skills`
continues to cover imported skill prompt rendering and verification metadata.
