# Secret Collection Guidance

Bundled resource guidance now routes reusable credentials through
`RequestSecret` instead of plain user questions.

- `RequestSecret` documents `action: "collect"` for asking the user to enter a
  new masked secret and saving it to the encrypted vault.
- Browser login guidance searches stored secret metadata first, requests an
  unambiguous match when available, and uses `collect` when a required
  credential is missing.
- Browser agents must use the returned `PUFFER_SECRET_...` placeholder for form
  filling and must not ask users to paste passwords, API keys, cookies, or tokens
  into chat or `AskUserQuestion`.
- `AskUserQuestion` guidance now limits direct input to non-reusable values and
  points credential collection to `RequestSecret collect`.
