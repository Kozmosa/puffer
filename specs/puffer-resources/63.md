# Per-Skill Lambda Host Tool Bindings

`lambda_skill_libraries/*.yaml` manifests may now provide
`skill_host_tool_bindings` in addition to library-wide `host_tool_bindings`.
The new map is keyed by normalized skill name and carries the formal host tool
to concrete Puffer tool bindings for that one imported Lambda Skill.

This keeps multi-skill external libraries configurable without vendoring the
library into Puffer and without forcing every skill in the library to share the
same formal host tool names. Library-wide bindings remain useful for common
tools; per-skill bindings override or extend them before verification metadata
is stored on the loaded `SkillSpec`.

Focused verification:

- `cargo test -p puffer-resources lambda_skill_library_imports_generated_external_skills`
