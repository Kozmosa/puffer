# BytePlus Seedream Response Defaults

## Summary

BytePlus native Seedream image descriptors now prefer stable synchronous image
bytes over expiring image URLs.

## Catalog Contract

- `seedream-5-0-260128` remains labeled `Seedream 5.0 Lite`; it is not exposed
  as a generic Seedream 5.0 model.
- Seedream 5.0 Lite keeps `output_format` because BytePlus documents it for
  that model. The default is `jpeg`, matching the documented default.
- Seedream 4.5 and 4.0 continue to omit `output_format`.
- All native Seedream image models declare `response_format=b64_json` by
  default so the adapter can persist image bytes directly without downloading a
  short-lived URL.

## Non-Goals

Boolean and object-valued API fields such as `watermark`, reference `image`,
streaming, and sequential generation are intentionally not modeled yet. The
current provider parameter system is select-string based, and adding those
fields should wait for typed media parameters rather than stringly-typed
workarounds.
