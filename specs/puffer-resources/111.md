# Skill Action Obligation Frontmatter

Skills can declare that activation must be followed by tool-driven work before
the turn may complete.

## Contract

- `requires-action: true` marks a skill as action-obligated.
- `requiresAction: true` is accepted as the camelCase spelling of the same
  field.
- Omitted fields default to `false`.
- The field is resource metadata only. Runtime enforcement is owned by
  `puffer-core`.

## Loading

Both embedded bundled skills and filesystem skills parse the field from
`SKILL.md` frontmatter into `SkillSpec.requires_action`. The loader does not
infer action obligation from skill names, allowed tools, model settings, or
prompt text.

## Compatibility

Existing skills without the field keep default-off behavior and can still finish
normally without follow-up tool calls.
