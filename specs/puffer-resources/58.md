# Lambda Skill Formal Args Guidance

Imported Lambda Skill prompt headers now describe the full strict bridge shape:
`LambdaHostCall` should be called with the formal `host_tool`, formal `args`,
concrete Puffer `tool`, and concrete `input`.

This keeps generated skill libraries external while making the runtime contract
visible to the model that executes the skill. The loaded `SkillSpec`
verification metadata is unchanged; the updated header only clarifies how the
strict harness expects formal catalogue arguments to be supplied.

Focused verification:

- `cargo test -p puffer-resources lambda_skill_library_imports_generated_external_skills`
