# Canonical Media Provider Resources

Bundled media provider descriptors now expose product-level settings to the
desktop and tool resolver.

## Image Providers

- OpenAI image models expose `mode` and `ratio` axes, hide fixed
  `quality` and `output_format` values in `base_params`, and use `media_map`
  to emit provider sizes.
- MiniMax image models expose canonical `ratio`, hide fixed
  `response_format`, and map ratios to the provider `aspect_ratio` field.
- BytePlus image models expose canonical `mode` and `ratio`, hide fixed output
  fields in `base_params`, and map `mode + ratio` to exact provider sizes.
- Image descriptors set `max_outputs` so the UI can synthesize the `Output`
  setting without exposing provider request fields.

## Video Providers

Video descriptors keep provider-specific values where they are real product
choices, but user-facing labels are normalized to `Mode`, `Ratio`, and
`Duration`.

## Governance

Resource tests reject raw provider image settings for the canonical image
providers. Future provider YAML should declare visible image axes in product
terms and move fixed provider fields into hidden base parameters or static
maps.
