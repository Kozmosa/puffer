# Lambda Skill host tool bindings

`lambda_skill_libraries/*.yaml` manifests may now include
`host_tool_bindings`, with `tool_bindings` accepted as an alias. The mapping is
from formal Lambda host tool names to concrete Puffer tool ids:

```yaml
host_tool_bindings:
  gh_auth_status:
    - Bash
```

The loader copies this mapping into `SkillVerificationSpec` for each imported
Lambda Skill. Runtime gate construction uses it together with any
`concreteTools` emitted by the host catalogue. Missing bindings are reported as
readiness failures by `/skills`, `/doctor`, and the model-invocable skill list.
