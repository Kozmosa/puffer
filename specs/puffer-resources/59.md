# Lambda Skill Compiler Path Manifests

## Design

`lambda_skill_libraries/*.yaml` manifests may now include `compiler_path`.
The path uses the same resolution rules as the library root: absolute paths are
used directly, `~` expands to the user home directory, and relative paths are
resolved from the manifest file.

Imported skills copy the resolved compiler path into their verification
metadata. The runtime can then enforce strict host catalogues without guessing
from the external library's repository layout.

## Contracts

The Lambda Skill library remains outside the Puffer repository. The resource
manifest is the durable installation record for both the library root and its
optional compiler.

Libraries that ship checked-in host catalogues do not need `compiler_path`.
Libraries that rely on source-level `lskillc export-json` may configure
`compiler_path`, set `PUFFER_LSKILLC`, or install `lskillc` on `PATH`.

## Verification

`cargo test -p puffer-resources lambda_skill_library_imports_generated_external_skills`
checks that manifest `compiler_path` is preserved in loaded verification
metadata.
