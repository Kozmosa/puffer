# Browser Login Secret Lookup

- The bundled `browser` skill now allows `RequestSecret` in addition to `Bash`
  so browser-login turns can inspect the encrypted secret vault before asking
  the user for credentials.
- Browser login workflows must search stored secret metadata by site name,
  domain, origin, username/email hint, or login URL before manual credential
  collection.
- Agents may request a secret only when the metadata match is unambiguous; the
  returned `PUFFER_SECRET_...` placeholder is used in browser commands and raw
  secret values must not be printed.
- The `RequestSecret` tool description now advertises this login-first search
  behavior for all site/service sign-in tasks.
