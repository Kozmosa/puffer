# Browser skill and internal tool resources

## Purpose

Update bundled resources so Browser is taught as an internal CLI-backed skill
instead of a model-callable tool.

## Design

The Browser skill remains in Anthropic skill format and now declares Bash as the
allowed execution surface. Its instructions tell agents to call either the
`browser` shell alias or `puffer internal-tool browser ...`.

The Browser tool resource remains bundled for provenance and local runtime
registration. Its description now marks Browser as an internal CLI-only runtime
capability, not a provider-advertised tool.

## Logic

Agents learn Browser usage through the skill, then invoke the CLI through Bash.
The internal Browser CLI sends a structured permission preflight to the parent
runtime before dispatch, so Browser domain/action ACL rules are evaluated
without parsing shell command text.

Bash and PowerShell resource descriptions no longer describe commands as
running in a sandbox. Their model-facing schemas no longer include a sandbox
override input.

## Contract

Bundled resource provenance is unchanged. Browser still has a resource entry so
local registries and diagnostics know where it came from, while the tool
registry decides whether it is visible to model providers.

## Verification

- Resource-backed registry tests cover Browser registration and model-visible
  filtering.
- Tool visibility tests cover updated Bash field descriptions.
