# Browser skill command coverage

## Purpose

Keep the model-facing Browser skill aligned with the internal Browser CLI after
Browser moved out of provider-visible tool calls.

## Design

The Browser skill is now the exhaustive agent instruction surface for Browser
CLI methods. It documents the common workflow, global options, tab/session
commands, navigation commands, inspection commands, and interaction commands.

The skill continues to route Browser usage through Bash and the
`puffer internal-tool browser` CLI. Browser remains an internal tool resource,
not a provider-visible tool definition.

## Contract

The Browser skill must not describe Browser as a model tool call. It must list
the supported Browser CLI commands and aliases accurately enough that agents do
not need to infer command shapes from unrelated shell conventions.

The canonical scroll examples use positional pixel distances, matching the
current CLI parser contract.

## Verification

Resource contract tests parse the bundled skill resource and enforce repo
resource invariants. Manual comparison against `BrowserCommand` in
`puffer-cli` verified that the skill covers all current Browser subcommands,
aliases, and command-level options.
