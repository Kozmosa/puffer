# RequestSecret Action Schema

`resources/tools/request_secret.yaml` advertises explicit actions for secret
metadata search, encrypted creation, and masked-token request.

- `action` accepts `request`, `search`, or `create`; omission defaults to
  `request`.
- `name`, `description`, `query`, `limit`, `value`, `username`, `origin`, and
  `reason` are declared so model calls can search and create secrets without
  unknown fields.
- `value` is marked `x-puffer-secret: true`.
- The tool metadata declares file writes because `create` mutates the encrypted
  user secret vault.

Compatibility aliases `request-secret` and `request_secret` remain unchanged.
