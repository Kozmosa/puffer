# Browser Skill Readiness Guidance

- The bundled `browser` skill now states that DOM readiness plus a fresh
  snapshot is the default page-ready signal for managed browser work.
- The skill tells agents not to wait for full network idle unless the user
  explicitly asks for network-quiescence debugging or a page-specific workflow
  requires all meaningful background requests to stop.
- Existing CLI command examples and ref-based interaction guidance remain
  unchanged.
