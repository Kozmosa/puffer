# Media Generation Internal Tool Resources

Image and video generation resources are internal tools only.

- `resources/internal_tools/image_generation.yaml` owns canonical id
  `ImageGeneration`, handler `runtime:workflow:image_generation`, and aliases
  `image-generation` and `imagegen`.
- `resources/internal_tools/video_generation.yaml` owns canonical id
  `VideoGeneration`, handler `runtime:workflow:video_generation`, and aliases
  `video-generation` and `videogen`.
- The moved manifests no longer live under `resources/tools/`, so resource
  loading places them in `LoadedResources.internal_tools` and excludes them from
  `LoadedResources.tools`.
- The media skills expose `allowed-tools: Bash` and instruct foreground Bash
  commands against `puffer internal-tool ...`; that field remains guidance, with
  enforcement coming from internal tool permission checks.
