# puffer-resources 117: WorldRouter callable video catalog

WorldRouter video resources now expose only the public Seedance model ids that
Puffer can call directly:

- `seedance-2.0`;
- `seedance-2.0-fast`.

`seedance-2.0-asset`, `seedance-2.0-asset-fast`, `kling-v3`, and
`kling-v3-omni` are not exposed as video generation settings because the
WorldRouter docs do not describe them as direct task models for Puffer's current
task route. Asset references require the separate asset upload helper and still
use the public Seedance model id.

WorldRouter Seedance settings now expose only documented request controls:
resolution mode and numeric duration. The catalog no longer sends a ratio field
for WorldRouter Seedance tasks.
