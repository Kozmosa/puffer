# Lambda Skill Config-Only Loading

Lambda Skill libraries are now loaded only from declarative
`lambda_skill_libraries/*.yaml` manifests in the normal resource layers.
`PUFFER_LAMBDA_SKILL_ROOTS` is no longer a loading path.

This keeps installation explicit: a user or workspace resource manifest points
at the external skill library root and carries any runtime metadata such as
`generated_subpath`, `host_catalogue_subpath`, `compiler_path`, `allowed_tools`,
model, effort, and context overrides. The skill library itself remains outside
the Puffer repository.

The loader still accepts relative manifest paths and resolves them against the
manifest file, so relocatable package-style installs work without hard-coded
absolute paths.

## Validation

- `cargo test -p puffer-resources lambda_skill_library_imports_generated_external_skills`
