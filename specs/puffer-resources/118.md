# Media generation skill action obligation

Image and video generation skills now require follow-up tool work after
activation.

## Resource Contract

- `resources/skills/image-generation/SKILL.md` declares
  `requires-action: true`.
- `resources/skills/video-generation/SKILL.md` declares
  `requires-action: true`.
- Both skill bodies state that progress-only or promise-only replies are not
  completion.

## Runtime Implication

The existing `puffer-core` skill-action obligation guard now applies to these
skills. After a successful `Skill(image-generation)` or
`Skill(video-generation)` activation, the agent loop continues past the first
no-tool text response and asks the model to start tool-driven work. A second
consecutive no-tool response fails plainly instead of presenting a promise as
completion.

## Compatibility

The internal tool descriptors, Bash helpers, schemas, permission flow, and media
runtime execution path are unchanged. The change only prevents activated media
generation skills from completing without starting a real tool call.
