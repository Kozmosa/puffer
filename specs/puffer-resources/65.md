# Lambda Skill Nested Library De-Dupe

The resource loader now ignores nested Lambda Skill library roots when a broader
configured root already contains them. Workspace-scoped roots win over user
roots at the same path, and broader roots win over narrower roots before skills
are loaded into the model-invocable skill list.

This keeps runtime skill discovery aligned with the Desktop Settings snapshot:
adding a top-level verified skill library does not duplicate skills that were
previously configured through one of its child folders.

Validation:
- `cargo test -p puffer-resources lambda_skill -- --nocapture`
