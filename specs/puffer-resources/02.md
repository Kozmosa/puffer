# Internal tool resources

## Purpose

Represent first-party CLI-only tools as declarative resources without making
them provider tool calls.

## Design

`LoadedResources` now includes `internal_tools: Vec<LoadedItem<ToolSpec>>`.
The resource loader reads `resources/internal_tools/` from embedded,
filesystem-builtin, user, and workspace layers with normal provenance and
override diagnostics under the `internal_tool` resource kind.

Browser moved from `resources/tools/browser.yaml` to
`resources/internal_tools/browser.yaml`. The Browser skill remains the
model-facing instruction surface and tells agents to invoke the internal CLI
through Bash.

## Contract

Normal provider tool resources live under `resources/tools/`. Internal CLI-only
tools live under `resources/internal_tools/`. Loading an internal tool must not
place it in `LoadedResources::tools`.

`PUFFER_NO_BROWSER=1` removes the Browser internal tool, the Browser skill, and
plugin skill references to Browser.

## Verification

Embedded tool YAML parsing covers both `tools` and `internal_tools`. Runtime
resource filter tests assert that the no-browser flag removes the internal
Browser resource alongside the skill references.
