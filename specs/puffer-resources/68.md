# Tool-Only Resource Loading

`puffer-resources` now exposes `load_tool_resources` for callers that only need
to construct a `ToolRegistry`. It uses the normal embedded, filesystem builtin,
user, and workspace resource layering for `tools/` and `internal_tools/`, then
applies the same runtime resource filters as the full loader.

This avoids forcing UI preflight paths such as Verified Skills import readiness
to scan skills, plugins, providers, prompts, or Lambda Skill libraries just to
answer whether a concrete tool id is executable.

The full `load_resources` path remains unchanged for runtime turns and normal
resource inventory surfaces.
