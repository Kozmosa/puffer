# Internal Request User Browser Action Resource

## Summary

`RequestUserBrowserAction` is registered as an internal tool resource under
`resources/internal_tools`.

## Contract

The resource has the alias `requestuserbrowseraction`, uses the
`runtime:workflow:request_user_browser_action` handler, and accepts a required
`description` plus optional `url`, `connector_slug`, and `connection_slug`
fields.

The resource description states that it is not a provider tool call. Model-facing
flows should use `ConnectorAct` with a connector-defined
`requestuserbrowseraction` action, which the runtime maps to this internal
workflow.

## Compatibility

The `ConnectorAct` prompt guidance documents the email and Gmail-browser action
surface, including the internal browser-action handoff. The public
`ConnectorAct` schema remains unchanged.
