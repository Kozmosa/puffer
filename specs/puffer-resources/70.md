# WorldRouter builtin provider

## Design

WorldRouter is now part of the bundled provider catalog. Desktop provider
selection, daemon settings snapshots, CLI doctor output, and non-interactive
provider resolution no longer depend on a workspace-local
`.puffer/resources/providers/worldrouter.yaml` file.

## Contracts

- Provider id: `worldrouter`
- API shape: `openai-completions`
- Base URL: `https://inference-api.worldrouter.ai/v1`
- Chat completions path: `/chat/completions`
- Auth mode: API key
- Fallback model: `worldrouter/auto`
- Discovery: OpenAI-compatible `/models`

## Compatibility

Existing user or workspace resource files with the same provider id still
override the builtin descriptor through the normal resource provenance rules.
