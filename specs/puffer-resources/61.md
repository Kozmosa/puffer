# Lambda Skill Model Invocation Default

External Lambda Skill library manifests load generated verified skills as
model-invocable by default. This matches the product path where users configure
one or more verified skill libraries once, and the model selects the right
skill through the `Skill` tool during ordinary prompts.

Manifests may still set `disable_model_invocation: true` for a specific library
when the operator wants those skills to be user-command-only. The runtime gate
remains fail-closed: model invocation only succeeds when puffer-core can load a
host catalogue and a non-empty concrete `allowed_tools` scope from config.

Tests:

- `cargo test -p puffer-resources lambda_skill_library_imports_generated_external_skills`
