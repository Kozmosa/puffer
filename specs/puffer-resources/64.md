# Lambda Skill Loader Requires Precompiled Host Catalogues

External Lambda Skill library manifests now load generated skill descriptors and
precompiled host catalogue paths without carrying a runtime compiler path into
the active verification metadata.

Older manifests that mention `compiler_path` continue to parse, but loaded
skills receive `compiler_path: None`; strict runtime gating depends on
`host_catalogue_subpath` and the catalogue file it resolves to for each skill
directory.

Validation:
- `cargo test -p puffer-resources lambda_skill_library_imports_generated_external_skills`
