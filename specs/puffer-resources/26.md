# Connect Prompt Resource Contract

Bundled resources now include a `connect` prompt for connector authentication
and repair.

- `resources/prompts/connect.yaml` defines the `/connect` model instructions,
  accepted variables, and allowed tool set.
- The prompt requires `AskUserQuestion` for missing auth methods, credentials,
  phone numbers, login codes, passwords, passcodes, workspace URLs, server
  hosts, and sender lists.
- Auth method selection uses `AskUserQuestion` choice questions, while typed
  auth values use `AskUserQuestion` input questions instead of an "Other"
  workaround.
- The prompt forbids Bash, WriteStdin, shell stdin, and connector
  `--*-stdin` flags for the auth flow.
- Connector-specific method guidance is encoded for `telegram-login`,
  `slack-app`, `slack-login`, `lark-app`, `lark-login`, and `email`.
- The Telegram, Slack, Lark, and Email internal tool descriptions now state
  that `/connect` may expose them as scoped runtime auth tools while the
  existing skills keep using the internal CLI for lookup-oriented work.
- The connector skills now route setup and auth repair to
  `/connect <connector-slug> <connection>` and no longer document stdin-based
  secret collection as the normal auth path.

The resource contract keeps auth UX consistent across connectors while
preserving existing lookup skills and connector action guidance.
