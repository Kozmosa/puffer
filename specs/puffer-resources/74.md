# ConnectorAct Email Guidance

## Summary

The `ConnectorAct` tool description now explicitly tells agents to use the
configured `email` and `gmail-browser` connectors for email operations instead
of opening an independent browser session.

## Actions Documented

The resource lists `list_emails`, `list_inbox`, `list_category`,
`search_emails`, `mark_read`, `draft_reply`, `draft_forward`, `send_email`, and
`delete`, and distinguishes the transport: IMAP/SMTP for `email`, and the
daemon-managed Chrome profile for `gmail-browser`.

## Compatibility

The tool schema remains unchanged. This is prompt/tool metadata guidance only;
permission enforcement still comes from connector action metadata.
