# puffer-resources update 05

Telegram login and Email configuration moved from model-visible tool resources
to internal-tool resources.

Removed model tool YAMLs:

- `EmailConfigure`
- `TelegramLoginStart`
- `TelegramLoginSubmitCode`
- `TelegramLoginSubmitPassword`

Added internal tool YAMLs:

- `Email`, alias `email`, handler `runtime:workflow:email`
- `Telegram`, alias `telegram`, handler `runtime:workflow:telegram`

The built-in plugin now advertises the `email` and `telegram` skills. These
skills instruct agents to use Bash-injected internal CLI aliases, keeping
subscriber setup out of provider tool schemas while preserving user-invocable
documentation for the workflow.
