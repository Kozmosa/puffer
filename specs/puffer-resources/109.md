# Media Generation Helper-First Skills

Image and video generation remain internal tools. The model-facing path is the
media skill plus Bash, but the skill must invoke the stable helper aliases that
Puffer injects into every Bash command.

## Problem

The current media skills instruct the model to run `puffer internal-tool ...`.
That depends on a `puffer` binary being present on `PATH`, which is not true for
all app/session launches. Bash already injects internal helper functions backed
by the current executable path, so the raw `puffer` command is the wrong model
protocol.

## Decision

- Keep `ImageGeneration` and `VideoGeneration` in `resources/internal_tools/`.
- Keep both tools out of the model-facing tool registry.
- Keep `image-generation` and `video-generation` skills model-invocable with
  `allowed-tools: Bash`.
- Make `imagegen` the only documented image command in the image skill.
- Make `videogen` the only documented video command in the video skill.
- Treat `puffer internal-tool ...` as an implementation detail of the helper,
  not as a prompt or skill contract.

## Non-Goals

- Do not promote media generation to model-facing workflow tools.
- Do not add a generic model-facing internal-tool dispatcher.
- Do not add a shell-level `puffer()` helper or PATH shim.
- Do not remove the internal CLI subcommands; the helpers still execute through
  the current executable's `internal-tool` command.

## Resource Changes

- Update `resources/skills/image-generation/SKILL.md` to describe and use
  `imagegen --prompt ... --count ...`.
- Update `resources/skills/video-generation/SKILL.md` to describe and use
  `videogen --prompt ...`.
- Keep `resources/internal_tools/image_generation.yaml` aliases
  `image-generation` and `imagegen`.
- Keep `resources/internal_tools/video_generation.yaml` aliases
  `video-generation` and `videogen`.
- Keep registry visibility expectations unchanged: internal definitions are
  present, model-facing definitions are absent.

## Verification

- Resource tests assert the media skills contain the helper aliases and do not
  contain `puffer internal-tool`.
- Registry tests continue to assert internal-only visibility for
  `ImageGeneration` and `VideoGeneration`.
- System prompt tests continue to list media skills when `Skill` is enabled,
  because the model still reaches media generation through skills.
