# Internal CLI tool descriptors

## Purpose

Keep internal CLI tool metadata in one shared crate so Puffer CLI command setup
and Bash helper injection do not maintain separate hardcoded lists.

## Design

`puffer-tools::internal_tools` exposes static descriptors for CLI-only internal
tools. Each descriptor has a stable internal id, shell helper aliases, and the
skill resource name that teaches agents how to use the tool.

The module also renders shell functions for every alias. The generated function
calls the current Puffer executable with `internal-tool <id>` and forwards all
arguments.

## Contract

Adding a new CLI-only internal tool should add one descriptor entry. Bash helper
injection and `puffer internal-tool aliases` should both read from that shared
descriptor list.

## Verification

Unit tests verify that Browser helpers render from the descriptor list and that
shell words are quoted safely.
