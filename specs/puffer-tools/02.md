# Internal Permission Wire Types

## Purpose

Provide a shared typed protocol for first-party internal tools that need a
runtime permission decision before executing.

## Design

`puffer_tools::internal_permissions` defines the wire contract:

- `InternalToolPermissionRequest`
- `InternalToolPermissionEnvelope`
- `InternalToolPermissionResponse`
- `InternalToolPermissionDecision`

The protocol uses JSON lines over a loopback TCP connection. The child process
discovers the callback through environment variables and sends the structured
request with the per-invocation bearer token supplied by the parent runtime.

## Logic

`request_internal_tool_permission_from_env()` is optional: it returns `None`
when no callback is present, which preserves direct human CLI usage.

`require_internal_tool_permission_from_env()` is mandatory: it errors when the
callback is missing. Hidden internal-tool commands use this path.

`PUFFER_INTERNAL_PERMISSION_REQUIRED=1` makes even the optional helper fail
closed if a parent runtime marked the child process as permission-managed but
could not expose an active callback.

## Contract

- Protocol version is currently `"1"`.
- All request input is structured `serde_json::Value`; callers must not encode
  permission intent inside shell command text.
- Missing callback is only acceptable for non-internal direct CLI use.
- Denials carry a human-readable reason when available.

## Verification

- Unit coverage round-trips an allow decision over the localhost JSON-line
  protocol.
