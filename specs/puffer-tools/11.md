# Internal Browser Action Visibility

## Summary

Tool registry visibility tests now cover the internal
`RequestUserBrowserAction` resource.

## Contract

The registry must load `RequestUserBrowserAction` and its
`requestuserbrowseraction` alias as internal definitions. Neither name may appear
in the model-facing tool definition map.

## Compatibility

This preserves the distinction between connector-visible actions and
provider-visible tools: connector metadata may mention
`requestuserbrowseraction`, but normal provider tool discovery must not expose it
as a direct LLM tool call.
