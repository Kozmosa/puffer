# Lambda Host Call Args Schema

The declarative `LambdaHostCall` tool schema now includes required `args`.
These are the formal host-call arguments keyed by the active Lambda Skill
catalogue parameter names.

`host_tool` continues to identify the formal catalogue operation, while `tool`
and `input` continue to bind the exact concrete Puffer tool call that must
follow. The runtime handler remains `runtime:lambda_host_call`; schema loading
and runtime-local dispatch behavior are unchanged.

Compatibility implication: strict Lambda Skill executions should include
`args: {}` for zero-argument host tools and a populated object for host tools
with parameters.
