# BrowserAction Readiness Guidance

- The `BrowserAction` descriptor now documents DOM-ready navigation followed by
  `snapshot` or an action-specific assertion as the preferred readiness pattern.
- `waitNetworkIdle` remains available, but the tool description and timing
  parameter descriptions now frame it as an explicit meaningful-network quiet
  wait rather than a default page-ready primitive.
- `openScreenshot` and `openConsoleLogs` are documented as waiting for DOM
  readiness rather than full network idle.
