# MCP Tool Name Budgeting

## Summary

MCP tool qualification now enforces the 64-byte Responses API callable-name
limit across the full generated name.

## Contract

Qualified MCP names keep the `mcp__<server>__<tool>` shape after sanitization.
When the sanitized server and tool components do not fit, both components share
the remaining byte budget and stay non-empty when possible. Collision suffixes
are included in the same budget, so disambiguated names must also stay within
the limit.

Repeated identical `(server, tool)` rows must receive distinct names after the
first collision rather than reusing the same hash-suffixed callable name.

## Compatibility

Normal MCP names and short sanitized collisions preserve their existing shape.
Only names that previously exceeded the 64-byte limit, or duplicate collisions
that previously reused a generated name, change behavior.
