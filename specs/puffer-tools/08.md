# Lambda Host Call Runtime Tool

## Design

`runtime:lambda_host_call` is classified as a runtime-local tool handler. Tool
resource loading can therefore advertise the declarative `LambdaHostCall`
schema, while execution remains owned by `puffer-core`.

## Architecture

- `puffer-tools` only recognizes the handler as runtime-local.
- `puffer-core` parses the call, checks the active Lambda Skill gate, and
  binds the admitted formal host call to one exact concrete Puffer tool call.
- The handler is not executable through external tool backends.

## Contracts

- Resources may define a `LambdaHostCall` tool with handler
  `runtime:lambda_host_call`.
- `ToolRuntime::RuntimeLocal` is returned for this handler.
- Direct execution through `puffer-tools` returns the normal runtime-local
  execution error.

## Compatibility

This preserves the existing declarative resource model for runtime tools and
does not change built-in tool dispatch.
