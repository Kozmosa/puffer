# puffer-tools update 05

Telegram and Email are now first-party internal CLI tools, not model-visible
provider tools.

`internal_tools` registers three shared descriptors: `browser`, `email`, and
`telegram`. Bash helper generation emits one function per alias, so agents run
`email ...` and `telegram ...` through Bash without hardcoded alias plumbing in
the CLI or core runtime.

The internal callback protocol now has two envelope types over the same
loopback endpoint:

- `request` for permission-only preflights, used by Browser before daemon
  actions.
- `execution` for parent-runtime execution, used by Email and Telegram so
  subscriber state stays in the long-lived Puffer process.

Execution responses carry `success`, optional `output`, and optional `reason`.
The protocol remains JSON-lines over localhost with the existing bearer token
and protocol version.
