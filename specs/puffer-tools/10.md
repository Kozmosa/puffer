# BrowserAction Concrete Tool

Puffer now exposes `BrowserAction` as a model-facing typed concrete tool for
managed browser operations. The existing internal `Browser` helper remains
CLI-only and hidden from provider tool lists; `BrowserAction` is the structured
surface intended for verified Lambda Skill contracts.

The tool uses the existing `runtime:browser` handler and the existing daemon
browser action set. Its schema covers tab management, navigation, snapshots,
screenshots, element-ref input, scrolling, uploads, and evaluate. Contracts bind
directly to JSON fields such as `action`, `url`, `ref`, `text`, and `key`, so the
Lambda gate can compare exact concrete inputs without relying on shell command
strings.

When `PUFFER_NO_BROWSER` disables the browser runtime, `BrowserAction` is
filtered out through the same handler-level rule as the hidden internal browser
tool.
