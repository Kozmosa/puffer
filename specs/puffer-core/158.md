# Lambda Skill URI Refinement Semantics

## Summary

`valid_uri` is now distinct from `valid_url` in the Lambda Skill gate. URL
predicates remain strict HTTP(S) URL checks, while URI predicates accept general
URI strings and the Spotify identifier forms used by translated Lambda skills.

## Runtime Contract

- `is_url` and `valid_url` continue to require an HTTP(S) URL with a host.
- `valid_uri` accepts:
  - HTTP(S) URLs accepted by `valid_url`,
  - Spotify URIs such as `spotify:track:<id>`,
  - Spotify bare identifiers,
  - JSON string arrays whose elements all satisfy `valid_uri`.
- Empty strings and strings containing control characters are rejected.

## Compatibility

This preserves existing URL behavior and only broadens `valid_uri` to match
the generated skill contracts that use Spotify URIs and URI-list strings.
