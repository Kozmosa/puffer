# Computer Use MCP Bridge

Puffer now exposes `ComputerUseAction` as a typed concrete tool for verified
Lambda Skill contracts that operate on macOS app state through the configured
`computer_use` MCP server.

Actions with an explicit `app` argument establish the current app binding in
the workspace runtime store. Actions without an app, such as `doubleClick`,
`typeText`, or `keyCombo`, fail closed until `capture`, `clickElement`, or
`clickCoord` has set the current app. This gives Lambda skills with
current-capture semantics a concrete, auditable app target rather than relying
on implicit UI focus.

The bridge maps only MCP-supported actions: capture/focus, element and
coordinate clicks, scroll, text input, and key combos. It does not translate
element-index drags into coordinates, does not expose focus/raise-window
semantics that the lower MCP schema cannot express, and rejects click modifiers
until the underlying MCP server supports them.
