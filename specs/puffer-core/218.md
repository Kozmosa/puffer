# Telegram QR Password Step

The Telegram workflow tool now waits for `login_awaiting_password` during
`login_qr_wait` and returns `status: "awaiting_password"` instead of surfacing
the subscriber event as a QR approval failure. The response instructs the agent
to ask for the Telegram cloud password and call `login_submit_password` on the
same connection.

Interactive connector setup also routes a QR login that reaches the password
step through the same password prompt used by phone login, so QR and phone
setup share the final 2FA completion path.
