# puffer-core update 08: AutoDream memory consolidation

Adds AutoDream as a conservative background memory-consolidation layer. The
feature reuses project memory and does not auto-install generated skills.

## Design

- `/autodream [status]` is a local command. Without arguments it runs one
  restricted side turn; with `status` it reports config and project-memory
  availability.
- `memory.autodream_enabled` controls automatic background execution and
  defaults to `false` to avoid surprise provider traffic.
- `memory.autodream_interval` defaults to `16` completed assistant turns.
- `memory.autodream_genskill_suggestions` controls whether command output
  recommends `/genskill` when the side turn marks the trace skill-worthy.
- Background AutoDream uses only `Skill`, `Read` scoped to the active
  `MEMORY.md`, and `Memory`. It cannot read arbitrary files or mutate anything
  except the configured project memory tool path.

## Contracts

- AutoDream consolidates durable facts into project memory and explicitly
  rejects transient task progress, temporary paths, and unverified guesses.
- The side turn must include `AUTODREAM_GENSKILL: yes|no`; Rust parses this as
  a suggestion only.
- Skill generation remains owned by `/genskill`. AutoDream may recommend
  `/genskill`, but it does not create or enable skills automatically.

## Compatibility

- Existing project-memory review and flush-on-compact behavior is unchanged.
- Automatic AutoDream is opt-in, so current sessions do not gain extra
  background model calls unless users enable it in config.
