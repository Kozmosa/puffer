# McpToolCall Verified Contracts

## Summary

`McpToolCall` provides a typed concrete execution surface for verified Lambda
Skill host operations whose intended semantics are already an MCP server tool.
It carries the MCP server id, tool name, and exact JSON arguments through
Puffer's existing MCP runner.

## Runtime Contract

- The concrete input must include `server` and `tool`.
- `args` defaults to `{}` when omitted.
- The runtime uses `AppState.tool_runner.call_mcp_tool`, so configured MCP
  subprocess and HTTP transports keep their existing lifecycle, OAuth,
  elicitation, and error behavior.
- Failed MCP results fail closed; successful metadata is returned before stdout.

## Permission Model

The tool is network/process-capable and requires approval. It should only be
used in verified skill contracts when the host catalogue names a specific MCP
server tool and argument shape.

## Compatibility

This does not replace per-server model-visible MCP tools. It gives the Lambda
Skill gate a stable concrete tool name for exact host-call bindings such as
AgentMail and TouchDesigner MCP operations.
