# puffer-core Update 07

## Summary

Lets `/genskill` run on explicit reference-solution context used by the
Ladybird evaluation harness.

## Design And Architecture

`/genskill` still requires a substantive transcript by default. The only
exception is a user message containing the eval harness's reference-solution
markers, including `Reference solution context` and `Reference fix patch:`.
This exception is intentionally narrow so normal interactive use still nudges
users to run `/genskill` after real tool-using work.

## Logic

The command counts tool-call transcript messages as before. If fewer than five
tool calls exist, it now checks for the reference-solution markers before
returning the "needs a substantive transcript" message.

## Contracts

- Existing `/genskill --candidates` and `--rounds` behavior is unchanged.
- Ordinary short conversations without the eval markers are still rejected.
- The replay agent does not see the reference patch; only skill generation uses
  the marked context.

- `cargo test -p puffer-genskill-eval`
