# LambdaInternal Readiness Boundary

Puffer's Lambda gate treats `LambdaInternal` as a pure verified-skill planning
step only. Any host tool with runtime effects, including `proc`, must bind to a
real concrete tool with an exact `concreteInputContracts` entry. This prevents a
proc-only internal placeholder from being model-invocable as if the harness had
a concrete execution contract.

Runtime validation rejects `LambdaInternal` whenever a host tool declares any
effect other than `pure`. Empty effect lists remain allowed for pure internal
proof/planning steps. The diagnostic names the host tool and the runtime
effects that require a real concrete binding.

Compatibility: generated skill catalogues that still use `LambdaInternal` for
classification, rewriting, patching, local process work, or other non-pure
steps are fail-closed until their skill library supplies a concrete binding or
the host adds an explicit concrete adapter.
