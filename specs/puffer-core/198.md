# Lambda Host Catalogue Binding Introspection

`puffer-core` now exposes `lambda_host_catalogue_concrete_tool_bindings` and the
`LambdaHostConcreteToolBinding` DTO. The helper parses the same host catalogue
shape used by the Lambda gate runtime and returns each formal host tool with its
declared concrete Puffer tool ids.

Import and readiness surfaces can use this helper for lightweight concrete-tool
preflight checks without carrying daemon-local host catalogue schemas. Runtime
contract validation still stays in `validate_lambda_host_catalogue_runtime`.
