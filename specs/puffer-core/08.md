# Optional Claude Reference Parity Tests

## Purpose

Allow the normal `puffer-core` test suite to pass in a standard checkout
that does not include the external Claude Code reference tree.

## Design

Claude parity tests now use a small runtime guard before reading files from
`references/claude-code`. When that directory exists, the tests run the
same prompt and tool-description comparisons as before. When it is absent,
the individual parity test prints a skip message and returns early.

## Logic

The reference tree is not tracked by this repository, but several unit
tests read it with `unwrap()`. That made ordinary local test runs fail
with `No such file or directory` before reaching the actual assertion. The
guard keeps the parity tests opt-in by fixture presence while preserving
automatic coverage on machines or CI jobs that materialize the reference
checkout.

## Contracts

- No product code, bundled resources, prompt templates, or tool schemas
  change.
- Parity assertions still execute automatically whenever
  `references/claude-code` is present.
- Non-parity tool visibility tests continue to run unconditionally.

## Compatibility

This is a test-harness-only change. Developers can keep running the full
core suite without a private or separately checked-out reference tree, and
dedicated parity environments keep their existing coverage.
