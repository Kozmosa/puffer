# Workflow Connector Filter Presets

## Summary

`/workflows connectors` now prints stable connector filter terms and supports
`trigger-ready` as an unambiguous query for connectors that can start workflow
triggers.

## Behavior

- The connector catalog output includes a `filters:` line before result counts.
- The all-connectors view lists `trigger-ready`, `no-trigger`, `has-actions`,
  `serve`, `subscriber`, `internal-tool`, and `command`.
- Default `/workflows` summaries point users to `/workflows connectors ...`
  for the same filters because the default summary only includes trigger-ready
  connector rows.
- Trigger-capable connectors index `trigger-ready`; setup-only connectors keep
  `no-trigger`, `no trigger`, and `setup-only`.
- Connectors with actual action definitions index `has-actions` so preset
  filtering does not match descriptive prose that merely mentions actions.
- Action summaries show `send_message` first when present, then remaining action
  slugs in stable order, so the primary outbound action stays visible in
  capped connector rows.

## Compatibility

The filter line is additive. Existing connector rows, deterministic `/connect`
commands, action summaries, connection summaries, and result counts remain.
