# Parallel Test Robustness Updates

## Summary

Workspace test verification exposed two stale or timing-sensitive puffer-core
checks. The AskUserQuestion parity test now accounts for Puffer's documented
markdown-image preview extension, and the Anthropic SSE error mock accepts
connections for longer under highly parallel test load.

## Design

- Preserve Claude-reference comparison as the base for AskUserQuestion prompts.
- Apply Puffer-specific replacements for typed questions, searchable choices,
  and markdown-image previews before comparing the bundled tool resource.
- Keep the SSE error mock nonblocking, but extend its accept deadline from 5 to
  30 seconds so CPU contention does not produce a false "no client connected"
  failure.

## Compatibility

No runtime provider behavior changes. The updates are limited to test
expectations and mock-server timing.
