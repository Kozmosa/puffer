# Process Store Test Synchronization

Process-store command execution tests now wait for their expected buffered output
with a short deadline instead of sampling once after a fixed sleep. The runtime
contract is unchanged: pipe and PTY processes still stream output into the
shared head/tail buffer and expose it through `collect_output`.

This keeps the tests aligned with the asynchronous reader-thread architecture
and avoids treating normal scheduling delay as a process-store regression.
