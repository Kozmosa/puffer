# Telegram account state path

The Telegram internal workflow helper now rewrites the subscriber manifest state
directory for every personal-account connection, including the default
`telegram-user` slug.

Before this change the default slug kept the bundled manifest's relative
`state` directory, which could make phone-code login save and later read
`telegram.session` under the subscriber manifest/runtime state instead of the
account-scoped `telegram-accounts/telegram-user` directory. Alternate
connections already used the account-scoped path; the default connection now
matches that contract.

This preserves the existing subscriber id/topic behavior while ensuring that
login, reconnect, and auth probes all share the same durable account directory.
Regression tests cover both the default and alternate connection manifests.

The related one-shot CLI connect path now uses the same account directory
contract, and the Telegram subscriber login saver now creates the first session
file atomically.
