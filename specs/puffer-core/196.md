# Lambda Bash Input Canonicalization

Verified Lambda Skills still validate concrete Bash calls against the
precompiled host catalogue input contract, but the pending bridge now stores
the Bash runtime input in canonical tool form. `description` is treated as
UI-only metadata, and omitted `tty` is normalized to `false`; `tty: true`
continues to fail unless a contract explicitly allows it.

This keeps the formal binding strong while avoiding false rejections caused by
OpenAI adding harmless Bash display fields after `LambdaHostCall` returns the
materialized command.

Lambda result checking now unwraps Bash's structured tool output and validates
the inner stdout payload. Result refinements such as `[Paper]{parsed_ok(p)}`
therefore see the JSON emitted by the verified command instead of the
Puffer-side `{stdout, stderr, exit_code}` wrapper.

Validation:
- `cargo test -p puffer-core arxiv_search_materializes_bash_input_when_omitted -- --nocapture`
- `cargo test -p puffer-core arxiv_search_accepts_noop_bash_ui_fields_in_host_call_input -- --nocapture`
- `cargo test -p puffer-core lambda_pending_bash_bridge_ignores_description_and_default_tty -- --nocapture`
- `cargo test -p puffer-core lambda_bash_result_check_uses_inner_stdout_payload -- --nocapture`
