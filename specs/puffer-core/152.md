`LambdaInternal` is an auto-approved, read-only concrete tool for verified
Lambda Skills whose formal host steps are internal computation only. Runtime
catalogue validation rejects a `LambdaInternal` binding when the formal host
step declares any effect other than `proc`, so filesystem, network, secret,
memory mutation, delegation, spawning, signing, and user-interaction steps must
bind to their real concrete tools instead. It records the exact skill-qualified
step and formal arguments declared by the precompiled concrete input contract.

The tool does not bypass the verified skill gate. During an active Lambda Skill
turn, it is still rejected unless a preceding `LambdaHostCall` admitted the same
formal host tool, formal args, concrete tool id, and concrete input. Successful
execution returns the declared `result` field when present, otherwise a JSON
echo of the recorded step and args; normal Lambda fact commitment still happens
only after this concrete execution succeeds.

This gives generated catalogues an honest binding target for internal
model-side steps without pretending they are shell commands. External effects
must continue to bind to their real concrete tools, and readiness checks fail
closed if a catalogue attempts to route them through `LambdaInternal`.
