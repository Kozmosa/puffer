# Workflow Append Slash Command

## Design

The `/workflows` TUI command now exposes deterministic file append workflow
bindings. `/workflows append <connection-slug> <file-path> [pattern]` creates an
enabled binding that filters connection events with an optional
case-insensitive regex and appends matching text to a safe file path.

## Contract

- Existing connections are validated against their connector template before a
  binding is written.
- Planned connections are supported with
  `--connector <connector-slug>`; the command writes the binding and prints the
  matching `/connect` command the user must run before events flow.
- `--paused` creates the binding without activating it. `--slug` overrides the
  deterministic `append-<connection>-<path-leaf>` slug.
- `*`, `.*`, and empty patterns mean no filter. Other pattern values become
  case-insensitive regex filters.
- File append action validation remains owned by `puffer-subscriptions`, so
  unsafe absolute paths are rejected by the existing workflow binding store.

## TUI Discovery

`/workflows` now renders workflow bindings in a `Workflow actions` section.
Connection and connector rows include copyable `append=/workflows append ...`
commands, and search terms include `append`, `file`, and `save` so users can
discover this path from the existing workflow dashboard.
