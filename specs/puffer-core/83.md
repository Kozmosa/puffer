# Workflow Dashboard Monitor Tasks

## Scope

The terminal `/workflows` dashboard now includes read-only connector monitor
task visibility alongside workflows, connections, connectors, and runs.

## Behavior

- The workflow header reports active and total monitor task counts as
  `monitor_tasks=<active>/<total>`.
- The default `/workflows` view includes a `Monitor tasks` section.
- `/workflows tasks <query>` filters monitor tasks by task id, subject,
  description, status, connection, connector, action labels/prompts, ignore
  reasons, and command hints.
- Monitor task rows show the deterministic `/tasks show <task-id>` command and
  an ignore command using the first suggested ignore reason when one exists.
- Ignored monitor tasks are still visible through `/workflows tasks ignored`
  but are excluded from the active count.
- Missing monitor task storage is treated as zero tasks. Unreadable or invalid
  storage keeps the dashboard usable and prints a single diagnostic line.

## Compatibility

This is a read-only dashboard change. It does not mutate monitor tasks,
workflows, connector subscriptions, or `/tasks` action handling.
