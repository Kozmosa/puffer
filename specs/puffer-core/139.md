# Mandatory Lambda Host Bridge

When a verified Lambda Skill is active, concrete Puffer tool calls no longer
commit directly through matching host catalogue names. The active gate accepts
only:

- `LambdaHostCall`, which validates the formal host tool and arguments before
  recording the exact next concrete tool/input.
- The one pending concrete call declared by the accepted `LambdaHostCall`.
- `Skill`, which remains a meta-operation so the model can switch to another
  skill and clear or replace the active gate.

This makes shared concrete tools safe in mixed skill libraries. If two skills
both use `Read`, `ToolSearch`, or another concrete tool, the currently active
Lambda Skill still requires an explicit formal host-call bridge before that
tool can run.

Focused verification:

- `cargo test -p puffer-core lambda_gate_`
- `cargo test -p puffer-core model_invoked_plain_skill_clears_active_lambda_gate`
