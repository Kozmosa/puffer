# 17. Serial Tool Error Recovery

## Summary

Serial tool execution now reports tool failures back to the provider as failed
tool outputs instead of aborting the entire agent turn.

## Design

- Serial execution matches the existing parallel batch behavior for failed
  tool invocations.
- Tool executor errors are converted into a bounded
  `Tool execution failed: ...` output.
- The failed invocation keeps the original call id, tool id, and input so the
  next provider request can continue the same turn.
- Observability records the failed tool output and marks the tool span failed.

## Compatibility

Successful serial tool execution is unchanged. Providers that require every
tool call to receive a corresponding tool output can recover from missing,
denied, or otherwise failed local tools without showing the raw executor error
as a top-level system failure.

## Tests

OpenAI runtime tests cover a serial missing-tool error followed by a successful
second provider response, including the function-call output sent back to the
mock Responses endpoint.
