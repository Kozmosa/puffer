# puffer-core update 212: AutoDream verifier-first memory

AutoDream memory consolidation now narrows external and benchmark workflow
memory before it suggests GenSkill.

- Benchmark-style workflow memories must include a domain trigger instead of a
  broad recipe.
- External trace memories should preserve verifier-facing contracts such as
  required artifacts, schemas, permissions, focused tests, and success signals.
- Failed or unsolved traces may produce anti-pattern memory only when the
  avoided failure mode and verification signal are durable.
- GenSkill suggestions for external traces require non-interference: the saved
  workflow must say when to use it, when not to use it, and that current task
  prompts/verifiers override the workflow when they differ.

This keeps useful AutoDream memory while reducing the chance that a generated
skill causes extra exploration or overrides task-specific instructions.
