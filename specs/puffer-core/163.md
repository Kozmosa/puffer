# TaskFlow Concrete Tool

## Summary

`TaskFlow` is a typed workflow-state concrete tool for verified Lambda Skill
contracts. It persists managed flow records under the existing workspace
workflow runtime directory, enforces expected-revision checks before mutating a
flow, exposes compact summaries, and supports named local buffers.

## Contracts

- The tool is model-visible as `TaskFlow` with handler
  `runtime:workflow:task_flow`.
- `create_managed` creates a local flow with `revision = 1`, `status =
  running`, controller, goal, current step, and JSON state.
- `set_waiting`, `resume`, `finish`, and `fail` require `flowId` plus
  `expectedRevision`; mismatches return `applied: false` without changing the
  stored flow.
- `request_cancel` and `cancel_flow` update only the stored flow status and do
  not claim to stop child agents.
- `append_buffer` writes an item into a named local buffer. The namespace must
  be a safe identifier.

## Boundaries

`TaskFlow` does not spawn child agents, execute shell commands, send network
messages, or access external services. Lambda skills that require those effects
must stay fail-closed until a concrete tool with matching semantics exists.
