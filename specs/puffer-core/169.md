# BrowserAction Lambda Contracts

## Summary

`BrowserAction` now exposes the browser QA operations needed by verified Lambda
Skill host contracts without routing through free-form shell execution.

## Runtime Contract

- `waitNetworkIdle` waits for the managed Chrome session to have no active CDP
  network requests for the requested idle window, then fails closed on timeout.
- `domInspect` evaluates a fixed DOM-inspection program against the requested CSS
  selector and returns bounded element metadata.
- Both actions accept the existing Browser tab context, including `page.tabId`,
  so Lambda contracts can bind the formal page argument into a concrete browser
  target.

## Permission Model

Both actions are classified as Browser `Inspect` operations. They remain subject
to the same Browser permission path as snapshot, screenshot, and console-log
inspection, while verified Lambda execution still requires exact concrete input
matching through the host-call bridge.
