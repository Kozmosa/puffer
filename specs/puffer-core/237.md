# Move soul.md / user.md out of the cached system prompt into the per-turn reminder

## Why

The Anthropic system prompt is sent with `cache_control: { type: ephemeral }`
(`anthropic.rs`), i.e. it is the **cached prefix**. PRs #275/#276 appended
`soul.md` + `user.md` into that prompt (`render_runtime_system_prompt`). Because
`user.md` is **agent-mutable** (the `Remember` tool / `user_memory_*` RPC rewrite
it), every change rewrote the cached prefix and **busted the prompt cache**.

This mirrors shipped Claude Code, which has `excludeDynamicSections`: *"omit
per-user dynamic sections from the cached system prompt and re-inject them as the
first user message … lets cross-user prompt caching hit on a static system prompt
prefix."*

## Change

`soul.md` and `user.md` are no longer appended to the system prompt. They are
injected per-turn inside `build_system_reminder` (`openai::conversation`), which
is emitted as the `<system-reminder>` block **after** the cache breakpoint and is
rebuilt every turn without rewriting prior messages.

- `render_runtime_system_prompt` keeps only the project memory append
  (AGENTS.md/CLAUDE.md) — unchanged, pre-existing behavior.
- `load_soul_prompt` / `load_user_prompt` are now `pub(crate)` and called from
  `build_system_reminder`.
- All three provider paths get it: Anthropic (`anthropic.rs` -> build_system_reminder),
  Chat Completions (`completions_session.rs`), and Responses
  (`openai.rs::build_context_reminder_message` -> build_system_reminder).

## Result

- System prompt prefix is static again -> prompt cache stays warm across turns
  and across `user.md` edits.
- soul/user context is still present every turn (in the reminder), and a mutating
  `user.md` only changes the uncached tail. Prior messages are never rewritten.

## Tests

- `system_prompt::render_system_prompt_keeps_memory_but_excludes_soul_and_user`:
  the rendered system prompt contains project memory but NOT soul/user markers.
- `openai::conversation::build_system_reminder_carries_soul_and_user_not_system_prompt`:
  the reminder carries soul.md + user.md content.
