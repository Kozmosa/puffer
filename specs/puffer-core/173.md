# DiscordAction Runtime

## Intent

Add a typed Discord concrete tool for verified Lambda Skill contracts that need
Discord Bot API reads or narrowly scoped writes. This avoids binding effectful
Discord host tools to generic Bash or HTTP surfaces.

## Architecture

- `DiscordAction` is registered as a workflow runtime tool with
  `approval_policy: ask` and `sandbox_policy: network`.
- Runtime execution lives in `puffer-core` under
  `runtime/claude_tools/workflow/discord_action.rs`.
- Desktop Verified Skills import recognizes `DiscordAction` as a supported
  concrete tool when inferring allowed tools from precompiled host catalogues.
- Credentials are read from `PUFFER_DISCORD_BOT_TOKEN`, falling back to
  `DISCORD_BOT_TOKEN`.

## Contract

Supported actions are intentionally small:

- `sendEmbeds`: posts `body` plus JSON Discord embeds to `channelId`.
- `readMessages`: reads recent messages from `channelId` with `limit` in
  `1..=100`.

The runtime rejects non-Discord service discriminators, non-snowflake channel
ids, invalid embed JSON, empty embed lists, oversized embed lists, and
out-of-range read limits. It does not implement Discord search or presence,
because those require separate semantics from a single Bot REST call.
