# Searchable Monitor Selection

## Summary

The `/monitor` command can now choose a connector connection interactively
instead of requiring users to know a connection slug.

## Behavior

- Running `/monitor` with no arguments opens a searchable `AskUserQuestion`
  choice listing event-capable connections.
- Running `/monitor <query>` resolves a unique connection match using slug,
  connector slug, description, and state text; ambiguous matches open the same
  searchable choice.
- If more than 50 connections match, `/monitor` asks the user to narrow the
  search instead of surfacing the lower-level tool schema limit.
- Explicit multi-connection invocations, such as `/monitor telegram-user
  slack-team`, still create monitors for each named connection.
- `/workflows connections` prints `monitor=/monitor <connection>` beside
  trigger-ready connections so the terminal workflow dashboard teaches the
  monitor path.

## Compatibility

Existing exact `/monitor <connection>` calls keep their behavior. The new
selection path uses the existing `AskUserQuestion` TUI machinery and does not
change workflow storage or monitor task metadata.
