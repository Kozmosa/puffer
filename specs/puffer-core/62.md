# Local Telegram slash command

- `/telegram` is now a built-in local command instead of a prompt-backed skill
  command. It dispatches directly to the `Telegram` workflow tool and therefore
  avoids provider streaming failures and model latency.
- The local parser accepts `--connection`/`--account`, `search-peers`,
  `list-peers`, `search-messages`, and `list-messages`. Bare text is treated
  as a peer search query.
- Local message search defaults to `--context 0` and succinct output to keep
  the common lookup path fast. Callers can request previous-message context
  explicitly with `--context`.
- The command registry moved from `command.rs` into `command/registry.rs` so
  the command dispatcher remains below the repository line-count ceiling.
- Telegram succinct result formatting moved into `workflow/telegram_format.rs`
  so the Telegram workflow module remains focused on tool dispatch.
