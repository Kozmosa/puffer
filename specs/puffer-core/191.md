# Shopify Inventory Adjust Bridge

`ShopifyAction` now supports `inventoryAdjustSingleLocation` for verified
Lambda Skill contracts that expose `shopify.inventory_adjust(item_id, delta)`.

The action binds `itemId` to the formal `item_id` and `delta` to the formal
`delta`. Puffer validates that `itemId` is a Shopify `InventoryItem` gid, queries
the Admin GraphQL API for the item's inventory levels, and fails closed unless
there is exactly one location. It then calls `inventoryAdjustQuantities` with
that single `locationId`, the bound `itemId`, and the bound `delta`.

The bridge treats top-level GraphQL errors, non-empty `userErrors`, missing
adjustment groups, missing locations, and multi-location inventory items as
runtime failures. This preserves the Lambda host contract's limited parameter
surface without silently selecting a location that the verified host call did
not specify.
