# Workflow action shorthand validation

- `WorkflowValidate` and `WorkflowCreate` still accept canonical
  `ActionSpec` JSON/YAML, including `type: file_append`.
- User-facing YAML parsing now also accepts step-style shorthand such as
  `steps: [{ write_file: { path: /tmp/hi, content: "{{ event.text }}" } }]`.
- The shorthand normalizes `write_file`, `write`, `append_file`, and
  `file_append` into `file_append` actions because workflow triggers should
  append each matched event rather than overwrite the destination file.
- `write_file` shorthand accepts only event-text content templates; static
  custom content is rejected with an actionable error instead of being silently
  ignored.
