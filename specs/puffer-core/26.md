# 26. Tool-Free Responses Requests

## Summary

OpenAI Responses-shaped Codex requests no longer advertise tool use when the
current turn has no enabled tools.

## Design

- `tools`, `tool_choice`, and `parallel_tool_calls` are emitted only when the
  request has at least one tool definition.
- Tool-enabled turns still use `tool_choice: "auto"` and parallel tool calls.
- Reasoning, streaming, response storage, prompt cache keys, and structured
  text configuration are unchanged.

## Compatibility

OpenAI-compatible providers that reject tool-selection fields without tools can
now route normal chat turns without reporting tool-use endpoint errors. Tool
turn behavior is unchanged.

## Tests

Rust request-body coverage asserts that tool-selection fields are omitted from
tool-free Responses requests.
