# Lambda Skill concrete binding enforcement

Verified Lambda Skill execution now requires each formal host tool to have an
explicit concrete Puffer tool binding. Bindings can come from `concreteTools`
entries in the host catalogue or from `host_tool_bindings` carried by the
loaded verification metadata. Gate readiness fails closed when any host tool
lacks a binding.

`LambdaHostCall` admission checks three contracts before installing the pending
bridge: the formal host call is accepted by the catalogue, the concrete target
is in the active skill tool scope, and the selected formal host tool is bound to
that concrete target. `Skill` cannot be used as a bridged concrete target.

Active Lambda gates also block switching to ordinary skills. The Skill tool may
only replace an active Lambda gate with another verified Lambda Skill gate.

Gate facts are committed only after the bridged concrete tool returns success.
If the concrete tool fails, the pending bridge remains available for retry and
registered facts are not added.
