# Resource Watcher Directory Creation

## Change

The resource watcher now treats creation or deletion of a watched top-level
resource subdirectory, such as `resources/skills`, as a reload-worthy event.

## Contract

- Events for `skills`, `mcp_servers`, `plugins`, and other recognized resource
  subdirectories flip the shared reload signal even when the event path names
  the subdirectory itself.
- Events under unrelated directories, dotfiles, swap files, and temporary files
  remain ignored.

## Compatibility

This preserves existing file-level hot reload behavior and makes first-time
resource directory materialization reliable on backends that report the parent
directory event before nested file events are watchable.
