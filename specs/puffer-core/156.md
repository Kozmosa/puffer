# Lambda Concrete Tool Effect Capability

## Summary

Lambda Skill readiness now checks whether a bound concrete tool can perform the
formal host tool effects declared by the generated catalogue. Concrete input
contracts still bind arguments exactly, but readiness also rejects impossible
tool/effect pairings before the model can invoke the skill.

## Runtime Contract

- `Read` may satisfy `fs_r` and its own bounded `proc` parsing behavior.
- `WebFetch` and `WebSearch` may satisfy `net_r` and their own bounded `proc`
  summarization/parsing behavior.
- `LambdaInternal` remains limited to `proc` unless future runtime support gives
  it an explicit external capability.
- `Bash` and unknown custom tools are not narrowed by this check; they remain
  governed by exact concrete input contracts and the existing pending bridge.

This prevents catalogues such as `effects=["net_r","proc"]` bound only to
`WebFetch` from appearing model-invocable when the concrete tool cannot perform
the declared process effect.
