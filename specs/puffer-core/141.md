# Lambda Skill Recoverable Gate Denials

Lambda Skill gate denials are recoverable tool results, not fatal runtime
errors. When a model calls a concrete tool without the required
`LambdaHostCall` bridge, Puffer returns a failed tool result with explicit retry
guidance and `lambda_skill` metadata:

- `event=gate_rejected`
- `recoverable=true`
- `retry_tool=LambdaHostCall` for missing bridges
- `retry_tool=<pending concrete tool>` for mismatched pending bridges

The model should retry through the indicated tool instead of asking the user to
manually intervene. User approval is still preserved: after `LambdaHostCall`
admits a bridge, the next concrete tool call proceeds through the normal
permission pipeline and prompts the user when that concrete tool requires
approval.

Focused verification:

- `cargo test -p puffer-core lambda_gate_rejects_direct_concrete_tool_without_host_bridge`
- `cargo test -p puffer-core lambda_host_call_rejects_mismatched_concrete_input`
- `cargo test -p puffer-core lambda_bridge_preserves_concrete_tool_approval_prompt`
