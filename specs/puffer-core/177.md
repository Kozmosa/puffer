# TaskFlow Run Task Bridge

Puffer's verified Lambda Skill runtime now accepts `TaskFlow` concrete input
with `action = "run_task"`.

The action requires a managed `flowId`, `runtime`, `childSessionKey`, and
`taskDesc`. `runId` is optional; when omitted Puffer allocates a local child id.
The concrete call appends a running child record to the stored TaskFlow, bumps
the flow revision, and returns the linked child record. Missing or unknown flow
ids fail closed.

This keeps Lambda Skill taskflow contracts on the typed `TaskFlow` bridge rather
than routing through a generic shell or unstructured agent prompt. The bridge is
state-local: it records the child linkage and does not send network messages or
execute shell commands.
