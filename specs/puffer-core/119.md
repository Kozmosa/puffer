# 119. Verified Skill Provenance

## Summary

Verified skill provenance is now visible on core skill surfaces.

## Design

`/skills` appends verification details for skills whose `SkillSpec` carries a
verification record. The display names the verification system and includes
generated tool/action counts when available.

The model-facing `Skill` tool output preserves Lambda Skill provenance by
emitting `verified-skill`, `formal-source`, `generated-descriptor`, and
`verified-stats` lines before the expanded skill body. This lets downstream
runtime logic and the model distinguish a generated verified skill from an
ordinary markdown skill.

The runtime system prompt marks model-invocable verified skills with
`[verified <system>]` in the available-skill list.

## Contracts

Verification metadata is informational in this update. It does not widen tool
permissions and it does not replace the existing `allowed_tools` request filter.
Tool-call gate enforcement for Lambda Skill host catalogues remains a separate
runtime integration step.

## Verification

`cargo test -p puffer-core runtime::claude_tools::skill` checks the model-facing
Skill tool provenance output.

`cargo test -p puffer-core command_helpers::common::tests::render_skills_panel_marks_verified_skills`
checks the `/skills` panel detail line.

`cargo test -p puffer-core runtime::system_prompt::tests::runtime_system_prompt_lists_model_invocable_skills`
checks the runtime system prompt marker.
