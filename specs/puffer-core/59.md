# Connect Prompt Auth Flow

Puffer now exposes `/connect <connector-slug> <connection-name>` as a
prompt-backed slash command for connector authentication and repair.

- The command is registered as `CommandKind::Prompt` with a deterministic
  two-token argument contract: the first token is the connector slug and the
  second token is the stable connection name/slug.
- Prompt specialization computes `CONNECTOR_SLUG`, `CONNECTION_NAME`,
  `CONNECT_EXTRA_ARGUMENTS`, and `CONNECT_PARSE_STATUS` before the model turn.
  Missing values or extra trailing tokens are handled by the prompt through
  `AskUserQuestion` rather than implicit defaults.
- `/connect` gets a scoped resource view that promotes only the `Telegram`,
  `Slack`, `Lark`, and `Email` internal auth helpers into model-facing tools
  for that request. Other prompt commands keep the normal resource view and
  cannot call those internal helpers.
- The prompt-level tool filter still restricts `/connect` to
  `AskUserQuestion`, connector listing/registration tools, and the scoped
  connector auth helpers. `Bash` and `WriteStdin` are intentionally absent.
- `AskUserQuestion` now accepts first-class `type: "input"` questions with no
  options for typed credential and login values. Existing choice questions
  remain the default and continue to require two to four options.
- Connection auth failure messages now tell users to run
  `/connect <connector-slug> <connection-name>` instead of re-running an
  unspecified connector skill.

This keeps connector auth interactive and typed without making every connector
invent a separate stdin-based credential path.
