# Image Generation Bridge

Puffer now exposes `ImageGeneration` as a typed concrete tool for verified
Lambda Skill contracts that produce image artifacts. The tool calls OpenAI's
image generation endpoint with an exact prompt and aspect ratio, then writes
the returned image bytes into a workspace-relative output path.

The prompt may be literal text or a workspace-relative prompt file. Contracts
may also pass `promptReference` when a skill has both a generated prompt
library and a per-image prompt; Puffer reads the reference and prepends it as
generation context. Output paths must be relative and cannot escape the
workspace. When no output path is provided, generated images are stored under
`.puffer/workflows/images/`.

`OPENAI_API_KEY` or `PUFFER_OPENAI_API_KEY` supplies the image backend key.
`PUFFER_IMAGE_MODEL` can override the default model. The bridge fails closed on
unsupported aspect labels, missing credentials, invalid output paths, HTTP
errors, and responses that do not contain either `b64_json` or an HTTPS image
URL.
