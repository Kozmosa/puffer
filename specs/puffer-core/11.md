# Slash Auth Provider Aliases

## Summary

Core slash-command auth now shares desktop provider alias behavior.

## Design

`/login codex` and `/logout codex` are canonicalized before descriptor lookup,
login-flow spawning, auth-store checks, and active-selection cleanup.
The same applies to `claude`, which targets the canonical `anthropic`
provider.

Provider cleanup compares active selections by canonical provider id, so stale
session state containing `codex` or `claude` is cleared when the matching
canonical provider is logged out.

## Contracts

- Slash login uses the canonical provider id when invoking the nested CLI auth
  flow.
- Auth completion checks look for the canonical stored credential.
- Logout removes canonical credentials and clears active selections that use an
  alias provider prefix.

## Verification

- `cargo test -p puffer-core login_command_accepts_desktop_provider_aliases`
