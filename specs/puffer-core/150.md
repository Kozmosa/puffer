# Lambda Semantic Predicates

Verified Lambda type checking now routes direct value predicates through a
central semantic predicate registry. The registry covers predicates that can be
checked from the JSON argument itself, such as URLs, arXiv IDs, repository or
channel identifiers, JSON workflow shapes, ISO-8601 timestamps with timezone,
bounded numeric ranges, and known format/provider enums.

Custom fact predicates remain separate from semantic predicates. Approval,
confirmation, and workflow-completion facts such as `plan_approved`,
`mail_approved`, or `speaker_confirmed` are not registered as semantic
predicates, so catalogues still fail readiness until a trusted producer or
approval bridge supplies those facts.

The type checker uses the registry for normal and negated refinements. The
readiness path can therefore distinguish unsupported syntax from supported
semantic predicates while preserving fail-closed behavior for unknown predicate
names.
