# BrowserAction Contract Question Field

## Summary

`BrowserAction` accepts an optional `question` field for verified Lambda Skill
contracts that bind visual-inspection host calls to concrete browser
screenshots.

## Runtime Contract

- `question` is preserved in the concrete tool input so LambdaHostCall gate
  admission can bind every formal host argument exactly.
- The browser daemon does not treat `question` as an imperative browser action.
  Screenshot behavior is still controlled by `action`, `annotate`, dimensions,
  and screenshot options.
- This field is metadata for the model-facing verified step and does not create
  a fallback execution path.

## Compatibility

Existing BrowserAction calls are unchanged. Skills that previously had an
unrefined textual visual-inspection prompt can now bind that prompt into the
declared concrete input instead of misusing `proofParams`.
