# Lambda Skill Config Panel

## Scope

The `/skills` slash command now has a config view for external Lambda Skill
libraries. The library contents remain outside Puffer; Puffer only owns the
manifest locations and schema guidance.

## Change

- `/skills config` renders the workspace and user
  `resources/lambda_skill_libraries` manifest directories.
- The panel includes a YAML template for `root`, `compiler_path`,
  `allowed_tools`, `host_tool_bindings`, and per-skill
  `skill_host_tool_bindings`.
- The normal `/skills` panel links to `/skills config` so users can discover
  the install path from the TUI.

## Compatibility

Existing `/skills` and `/skill:<name>` behavior is unchanged. Config discovery
uses the same `ConfigPaths` layout as resource loading, so workspace and user
manifests resolve through the normal resource stack after reload or restart.
