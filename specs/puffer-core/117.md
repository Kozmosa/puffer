# Telegram slash command removal guard

- Hidden skills such as the bundled Telegram helper must not create direct
  slash-command fallbacks.
- Dispatching `/telegram ...` with the hidden Telegram skill loaded now has
  explicit regression coverage and remains on the normal unknown-command path.
- The internal `telegram ...` CLI helper and Telegram connector workflows remain
  available to agent/runtime code; only the user-facing slash-command surface is
  guarded.
