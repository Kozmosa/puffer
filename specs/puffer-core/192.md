# Native MCP Verified Skill Bridge

## Scope

This update adds a concrete runtime bridge for the translated
`native-mcp` Lambda skill. The bridge is exposed as the guarded
`NativeMcpAction` tool and is allowed as a verified Lambda concrete tool.

## Runtime Contract

- `validateStdioConfig` and `validateHttpConfig` parse the formal Lambda
  arguments into redacted server configs. Secret map values are stored in the
  session secret store and are not returned to the model.
- `applySecurityPolicy` confirms credentials are redacted and records the
  exact env/header keys reviewed by the skill.
- `configureSampling` returns a bounded policy, but `addStdioServer` and
  `addHttpServer` reject `enabled=true` because Puffer does not yet implement
  MCP `sampling/createMessage`.
- `addStdioServer` and `addHttpServer` write workspace MCP manifests under
  `.puffer/resources/mcp_servers/` only after the config, security review, and
  sampling policy agree on transport and reviewed keys.
- `discoverTools` reads the live runner MCP roster and tool list; it does not
  synthesize discovery through web search.

## MCP Manifest Semantics

- `McpServerSpec` now supports explicit stdio `env`, `inherit_env`, `timeout`,
  and `connect_timeout` fields. Existing manifests keep inheriting the full
  environment and default to 120 second tool calls and 60 second connection
  setup.
- Verified stdio manifests set `inherit_env: false`, restore only the safe
  baseline environment in the launcher, and pass explicit env entries after
  env-var expansion.
- Raw secret material is never persisted into generated manifests. Stdio
  secret values must be env-var references, or match an existing process
  environment variable so the manifest can store `${KEY}`. HTTP secret headers
  must use env-var references.
- MCP runner timeouts are enforced for HTTP client requests, MCP initialize
  handshakes, tools/list discovery, and tools/call requests.

## Compatibility

Existing MCP YAML remains valid because new fields have serde defaults and
skip serialization when unset. Fixture updates set the new fields to `None`
so existing tests continue exercising legacy behavior.
