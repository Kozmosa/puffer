# Browser Console Verified Contracts

## Summary

`BrowserAction` now exposes a `consoleLogs` inspect action for verified Lambda
Skill contracts that need live browser console output. The action reads the
managed desktop browser's DevTools console buffer instead of routing through a
generic shell or fetch command.

## Runtime Contract

- The desktop daemon records `Runtime.consoleAPICalled` and `Log.entryAdded`
  DevTools events per backend browser tab.
- `BrowserAction` accepts `consoleLogs` with an optional `page` handle and
  optional `clear` boolean.
- `page` may be a tab id string or an object containing `tabId`/`tab_id`.
- The response is `{ logs, count, cleared }`.
- `clear=true` returns the current entries and removes them from the buffer.
- Console buffers are capped per backend tab and removed when the tab/root
  session is closed or pruned.

## Permission Model

`consoleLogs` maps to the Browser inspect action set. It does not evaluate page
JavaScript, navigate, click, type, or read files.

## Compatibility

Existing browser actions are unchanged. The runtime input bridge now preserves
all declared `BrowserAction` fields so verified contracts can bind upload and
screenshot options without losing parameters before daemon dispatch.
