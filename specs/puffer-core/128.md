# Lambda Skill Doctor Status

`/doctor` and the public CLI doctor helper now report verified Lambda Skill
harness status. The summary includes loaded Lambda Skills, checked-in host
catalogues, compile-on-demand sources, known compiler stats, and the resolved
`lskillc` path used by strict catalogue compilation.

When imported Lambda Skills rely on source-level compilation instead of a
checked-in `host.json`, doctor surfaces emit an explicit warning unless
`PUFFER_LAMBDA_SKILL_GATE=compile` is enabled. If compile mode is enabled but
no compiler can be resolved through `PUFFER_LSKILLC`, the Lambda workspace
candidate paths, or `PATH`, `/doctor` reports that as a separate warning.

This keeps the external library non-vendored while making the harness boundary
observable: users can tell whether loaded verified skills are prompt-only or
strictly backed by the Lambda Skill host catalogue gate before running a
verified skill turn.

The runtime gate behavior is unchanged. This update only exposes existing
gate-mode and compiler-resolution checks to the diagnostics paths.
