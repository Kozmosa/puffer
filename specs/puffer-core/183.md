# 183. MCP status bridge

Puffer adds an `McpStatus` workflow tool for Lambda Skill contracts that need
to report MCP readiness. The tool asks the configured runner for known MCP
servers and each server's tools, then returns a JSON status report with server
metadata, tool counts, optional tool names, and per-server errors.

This bridge is read-only. It does not add MCP servers, edit configuration, or
call MCP tools. Config mutation remains fail-closed until Puffer has a typed
config action that can preserve transport exclusivity, secret redaction, and
sampling bounds.
