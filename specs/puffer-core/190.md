# Debugpy Attach Bridge

Puffer adds `DebugpyAction` as a narrow verified-skill bridge for the
`python-debugpy` Lambda host tool `dap_client`.

The `attach` action accepts a `port`, connects only to `127.0.0.1:port`, sends
a minimal Debug Adapter Protocol `initialize`, `attach`, and
`configurationDone` handshake, and fails closed unless the adapter returns
successful responses. It returns JSON `null`, which satisfies the Lambda
`unit` result type and lets the gate commit the `attached(port)` register fact
only after the concrete DAP attach succeeds.

This keeps `dap_client` semantically guarded by both Lambda fact flow and the
runtime bridge: the model must first have a `debugpy_listening(port)` fact, then
the next concrete `DebugpyAction` call must bind the exact integer port, and the
runtime must prove the listener accepted the attach.
