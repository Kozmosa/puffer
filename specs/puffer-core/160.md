# ProcessControl Concrete Tool

## Summary

`ProcessControl` is a typed workflow tool for controlling process/session
handles that already exist in Puffer. It exists so verified Lambda skill host
calls can bind process-control effects without receiving an arbitrary shell
surface.

## Runtime Contract

The tool supports only these actions:

- `list`: return tracked interactive processes and background tasks.
- `poll`: return status and exit metadata for an existing `sessionId`.
- `log`: return buffered output for an existing `sessionId`.
- `kill`: stop an existing interactive process or background task.
- `interrupt_or_kill`: send Ctrl-C before terminating an interactive process.

The tool does not launch commands and does not accept shell text. `sessionId`
is required for all actions except `list`; background task identifiers still go
through the safe-identifier check used by the workflow task store.

## Architecture

Interactive processes are served through `ProcessStore`, which now exposes
read-only snapshots and stores the spawned OS process id so typed termination
can stop the underlying child before closing stdin or the PTY master fd.
Background tasks reuse the existing workflow task store, refresh, output-read,
and termination helpers.

The tool definition is declarative in `resources/tools/process_control.yaml`
and is registered in both the workflow executor and deferred tool search list.

## Compatibility

Existing Bash-backed process commands continue to work. `ProcessControl` only
adds a narrower concrete target for verified skill contracts that need process
introspection or termination, keeping command launch and arbitrary stdin outside
this bridge.
