# 14. Browser Tools Use Visible Session Roots

## Summary

Implicit Browser tool session targeting now resolves to the desktop-visible root
session when a nested agent runs inside a parent session.

## Behavior

- Top-level Browser tool calls still use the current session id.
- Nested-agent Browser tool calls with missing, empty, or `current`
  `sessionId` use the parent session id.
- Explicit non-current `sessionId` values are preserved and continue to count as
  cross-session Browser access.
- Session-scoped Browser approval grants are recorded against the same visible
  root session id that runtime Browser execution uses.

## Compatibility

The daemon `browser_agent` protocol is unchanged. The change only affects the
default session id selected before runtime-local Browser calls and Browser
permission categories are evaluated.

## Tests

`puffer-core` state tests cover nested AppState Browser root resolution and
session-scoped Browser grants for a nested agent.
