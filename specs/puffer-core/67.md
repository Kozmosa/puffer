# Workflow Trigger Readiness UX

## Summary

Workflow connector surfaces now distinguish event-capable templates from
connectors that can authenticate or act but cannot start workflow triggers.

## Behavior

- `ConnectorList` rows include `can_trigger_workflow`.
- `/workflows` default connector summaries show only connectors that can start
  workflow trigger events.
- `/workflows connectors` still shows every connector template, with a
  `trigger` capability only when the connector can produce trigger events.
- `/workflows connections` annotates each connection as `trigger=ready` or
  `trigger=unavailable`.
- Native workflow creation keeps using the same shared readiness predicate that
  powers these display surfaces.

## Compatibility

The new fields and annotations are additive. Existing connector fields and
commands are unchanged, and unsupported connectors remain visible through the
explicit `/workflows connectors` catalog view.
