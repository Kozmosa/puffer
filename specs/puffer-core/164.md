# HttpRequest Concrete Tool

## Summary

`HttpRequest` is a typed concrete network surface for verified Lambda Skill
contracts that already specify exact HTTP semantics. It sends one request with
declared method, URL, headers, and either JSON or raw body content, then returns
status, response headers, and response body.

## Contracts

- The model-visible tool id is `HttpRequest`, handled by
  `runtime:workflow:http_request`.
- Supported methods are `GET`, `POST`, `PUT`, `PATCH`, and `DELETE`.
- URLs must be absolute `http` or `https` URLs.
- `json` and `body` are mutually exclusive. JSON bodies use reqwest JSON
  encoding; raw bodies are sent as UTF-8 text.
- `timeoutMs` defaults to 60000 and is capped at 300000.
- `failOnHttpError` defaults to true, so 4xx/5xx statuses fail the tool instead
  of silently producing a successful host result.

## Boundaries

`HttpRequest` is not a semantic fallback for app skills. A Lambda host tool may
bind to it only when the generated contract fully fixes method, endpoint, auth
headers, and payload shape. MCP sessions, browser state, workflow state,
streaming handles, secret preparation, and app-specific local config remain
outside this tool.
