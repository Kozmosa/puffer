# Video Runtime Review Cleanup

Date: 2026-06-09

## Summary

The exact video generation runtime is split into a focused
`media_runtime_video` module. Relaydance and BytePlus adapters now share common
video job helpers for polling, failed-job persistence, and MP4 artifact
storage.

## Behavior

- Exact video routing behavior is unchanged for Replicate, Relaydance, and
  BytePlus.
- BytePlus poll parser failures now persist the local media job as failed with
  provider, adapter, and task diagnostics.
- Relaydance-specific parsing remains provider-specific and fixture-backed.

## Maintainability

- `media_runtime.rs` stays under the 1000-line source limit without compressed
  one-line control flow.
- Shared video job mechanics live in `runtime/media/video_jobs.rs`.
- Provider-specific request and response parsing remains in each adapter.
