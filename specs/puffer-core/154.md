# Lambda proof-only parameters

Verified Lambda host catalogues may declare `proofParams` (alias
`ghostParams`) on a host tool. A proof parameter is still required in the
formal `LambdaHostCall` args and is still checked by the Lambda type/refinement
gate, but it is not required to appear in the concrete tool input contract.

This prevents fake concrete bindings where a refined approval or witness value
is injected into a shell command only to satisfy contract completeness. Runtime
validation only accepts proof parameters that are declared tool parameters and
whose type carries an explicit refinement segment. Concrete input contracts
must bind exactly the remaining non-proof formal parameters.

The temporal bridge is unchanged: `LambdaHostCall` records the formal host
tool, formal args, concrete tool, and rendered concrete input, and the next
concrete tool call must match that input exactly before normal permission
approval and execution continue.
