# Media Internal Runtime Execution

Parent-side internal tool execution now runs media generation with the same
runtime context as the previous workflow path.

- `execute_internal_tool_request` and its result helper receive the provider
  registry, auth store, and exact media discovery cache.
- The Bash execution branch clones `state.exact_media_discovery_cache` and
  falls back to `ExactMediaDiscoveryCache::empty` before draining internal tool
  broker requests.
- Internal permission resolution still happens before backend execution.
- Canonical internal tool names `imagegeneration` and `videogeneration` call
  `execute_image_generation` and `execute_video_generation` directly with
  `ImageGenerationMediaContext` and `VideoGenerationMediaContext`.
- Email, Telegram, and request-user-browser-action internal execution mappings
  remain unchanged.
