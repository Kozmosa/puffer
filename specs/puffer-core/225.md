# puffer-core update 10: AutoDream original-mechanism gates

Adds original-mechanism-style scheduling around automatic AutoDream while
preserving Puffer's restricted Rust-native consolidation turn.

## Behavior

- Manual `/autodream` still runs immediately for inspection and debugging.
- Automatic AutoDream remains opt-in through `memory.autodream_enabled`.
- Automatic AutoDream now requires all gates:
  - project memory is enabled and configured
  - completed-turn interval is reached
  - `memory.autodream_min_hours` has elapsed since the last successful automatic
    consolidation
  - at least `memory.autodream_min_sessions` other sessions were updated since
    the last successful automatic consolidation
  - no other process currently owns the AutoDream consolidation lock
- If the time gate passes but the session gate does not, session scans are
  throttled for ten minutes to avoid repeated disk walks.
- When automatic AutoDream runs, Puffer collects a bounded recent-session
  context pack from the same project:
  - excludes the current session
  - excludes sessions from other working directories
  - includes at most five sessions
  - includes at most sixteen recent transcript events per session
  - redacts secret-like words and truncates each snippet before prompt injection

## State And Locking

AutoDream stores scheduler state under the user Puffer home next to the session
store:

- `autodream/state.json`
  - `last_consolidated_at_ms`
  - `last_session_scan_at_ms`
- `autodream/consolidation.lock`
  - cross-process lock used only by automatic background runs
- `autodream/run_status.json`
  - diagnostic status for the latest automatic background run
  - records `running`, `completed`, `failed`, or lock-contention `skipped`
  - includes sessions reviewed, Memory/tool-call count, GenSkill marker, a
    short summary, and a short error when applicable
  - includes sanitized structured Memory changes observed during the run
  - includes the latest pending GenSkill suggestion when the marker is positive

The last-consolidated timestamp is updated only after a successful automatic
review. Failed or skipped reviews leave it unchanged so a future turn can retry.
The run-status file is visible through `/autodream status` but does not affect
scheduler gates.

## Structured Outcome And Suggestions

Completed background runs summarize each `Memory` tool invocation as an action,
short content, short old text, success flag, and Memory tool message. The
summary is redacted and truncated before persistence; raw transcripts remain out
of the AutoDream state file.

Positive GenSkill markers create a pending suggestion record with an id,
timestamp, rationale, memory-change count, and `pending_review` status.
`/autodream suggestions` renders the latest record for human review. AutoDream
does not create skill files directly.

## Recent Session Context

The automatic background path now injects a "Recent session context" section
after the core AutoDream prompt. The section is built from structured
`TranscriptEvent` values loaded through `SessionStore::load_session`; it does
not read JSONL as raw text. The packer summarizes user, assistant, system,
command, tool, rename, and git-diff events, while skipping state snapshots and
transcript rewrite events.

The injected context is supporting evidence only. AutoDream still has to write
durable project memory through the restricted `Memory` tool and must not store
raw transcript snippets as long-term memory.

## Configuration

New memory settings:

- `memory.autodream_min_hours`, default `24`
- `memory.autodream_min_sessions`, default `5`

Existing settings are unchanged:

- `memory.autodream_enabled`
- `memory.autodream_interval`
- `memory.autodream_genskill_suggestions`

## Compatibility

- The restricted side-turn tool surface is unchanged: `Skill(project-memory)`,
  `Read` scoped to the active memory file, and `Memory`.
- Puffer does not copy Claude Code telemetry, GrowthBook dependency, or broad
  memory-directory write permissions.
- GenSkill handoff is reviewable state only; `/genskill` remains the command
  that actually writes a skill.
- Existing tests that call the simple turn counter can keep using
  `autodream_turn_completed`; TUI code uses the gated
  `autodream_turn_completed_with_store` path.

## Validation
