# Lambda Skill Compile Cache

Compile-gated Lambda Skills now reuse host catalogues within the current
process. When `PUFFER_LAMBDA_SKILL_GATE=compile` is enabled and a verified
skill has a formal `skill.lskill` source but no checked-in `host.json`, the
runtime caches the `lskillc export-json` output by source path and compiler
path.

The cache is invalidated when either the formal source file or the compiler
binary changes according to filesystem metadata. It is intentionally
process-local and does not write generated catalogues into Puffer or into the
external skill library.

Checked-in host catalogues still take precedence and are read directly. The
gate semantics are unchanged: each skill invocation still receives a fresh
`LambdaGateState`, so facts and pending bridged host calls cannot leak across
skill turns.

Focused verification:

- `cargo test -p puffer-core runtime::lambda_gate::tests::compile_gate_reuses_catalogue_until_source_changes`
