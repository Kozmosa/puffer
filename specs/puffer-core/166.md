# SlackAction Verified Contracts

## Summary

`SlackAction` provides a typed concrete tool for verified Lambda Skill contracts
that need Slack Web API reads. The concrete input names the Slack operation and
its domain arguments; Puffer owns credential lookup and HTTP execution.

## Runtime Contract

- Credentials are read from `PUFFER_SLACK_BOT_TOKEN` or `SLACK_BOT_TOKEN`.
- Supported read actions:
  - `reactions` -> `reactions.get`
  - `readMessages` -> `conversations.history`
  - `listPins` -> `pins.list`
  - `memberInfo` -> `users.info`
  - `emojiList` -> `emoji.list`
- `readMessages.limit` is clamped to Slack's bounded history-read range.
- Slack responses with `"ok": false` fail closed and surface the Slack error.

## Permission Model

The tool is network-capable and uses `approval_policy: ask` because the Slack
surface may later grow to include narrowly declared writes. The model never
receives or supplies the Slack bearer token.

## Compatibility

This does not replace connector Slack ingestion. It is a concrete execution
surface for verified skills whose host contracts already describe specific
Slack API reads.
