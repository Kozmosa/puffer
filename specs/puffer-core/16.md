# 15. Current Responses Include Selectors

## Summary

OpenAI Responses-shaped requests no longer send stale include selectors that
the current API rejects before model execution.

## Design

- Reasoning continuation requests use the accepted
  `reasoning.encryptedcontent` include selector.
- Native Responses web-search calls omit the old
  `web_search_call.action.sources` include selector. Source extraction remains
  best-effort from whatever fields the provider returns.

## Compatibility

The runtime request contract is otherwise unchanged. Chat Completions providers
continue to use their existing route, while Responses requests avoid 400
`invalidprompt` errors caused solely by unsupported include values.

## Tests

Rust request-body tests assert the compact reasoning include selector and that
web-search source includes are not emitted.
