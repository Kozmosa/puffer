# RequestSecret Search And Create

## Summary

`RequestSecret` is now a small secret-management tool instead of only a reveal
tool.

## Behavior

- Omitted `action` still means `request`, preserving the existing masked-token
  reveal flow and approval prompt.
- `action: "search"` returns non-secret metadata only and never decrypts secret
  material.
- `action: "create"` asks for user approval, writes the encrypted vault through
  `puffer-secrets`, and returns metadata without echoing the raw value.
- `name` is accepted as the user-facing secret name; `label` remains an alias for
  compatibility.
- Tool output includes `description` so model calls can disambiguate similarly
  named secrets without requesting raw values.

## Compatibility

Existing request calls by id or label continue to work. Create is a new write
path guarded by the same permission prompt surface used for secret reveal.
