# Lambda Integer Argument Binding

## Summary

Lambda concrete input contracts can now use `$int_arg` to bind a formal
argument into a JSON integer field.

## Runtime Contract

`$int_arg` accepts either a JSON integer argument or a string argument whose
trimmed content parses as an integer. Rendering fails closed for missing,
non-integer, or non-string/non-number values, so the declared concrete input
will not match.

This is intended for typed concrete tools such as `WriteStdin`, where the
formal host API may call a handle `session_id` while the concrete Puffer tool
schema correctly requires an integer `process_id`.

## Compatibility

Existing `$arg`, `$template`, `$shell_json`, `$concat`, and skill-path contract
forms are unchanged. `$int_arg` still counts as a reference to the formal
argument for catalogue completeness checks.
