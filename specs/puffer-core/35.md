# Fast Resume Project Scoping

## Scope

Interactive startup and `/resume` now avoid running `git` for every historical
session while deciding which conversations belong to the current project.

## Design

- Puffer still resolves the current launch directory to a Git repository root
  once when the current directory is inside a repository.
- Historical session paths are normalized and compared to that repository root
  with component-aware path containment.
- Sessions whose cwd is the repository root or a descendant path are included
  in the picker. Sessions outside that tree are rejected without launching a
  child process.
- Directory-scoped matching for non-Git workspaces keeps the existing symmetric
  nested-directory behavior.

## Compatibility

The picker continues to group conversations by project repository for normal
repo-root and subdirectory sessions. Deleted historical cwd paths under the
same repository also continue to match by lexical containment after
normalization fallback.

This intentionally drops per-session `git rev-parse` probes. That avoids
startup latency when the session store contains many stale temp directories and
prevents unrelated historical sessions from spawning thousands of Git
processes before the first TUI frame.

## Verification

- Unit coverage asserts repository scopes match root, live descendant, and
  deleted descendant paths without requiring session cwd Git metadata.
- Existing resume-launch tests continue to cover sibling directories inside the
  same repository and exact query resolution.
