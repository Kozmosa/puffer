# SlackAction Message Send

## Summary

`SlackAction` now supports the verified Lambda Skill concrete action
`sendMessage`, backed by Slack Web API `chat.postMessage`.

## Runtime Contract

- `sendMessage` requires a concrete Slack `channel` and message `text`.
- `threadTs` is optional and maps to Slack's `thread_ts` field for threaded
  replies.
- Puffer still owns Slack credential lookup through `PUFFER_SLACK_BOT_TOKEN` or
  `SLACK_BOT_TOKEN`; the model never supplies the bearer token.
- Slack `"ok": false` responses continue to fail closed before returning a
  tool result to the Lambda gate.

## Compatibility

Existing Slack read actions are unchanged. Verified skills can now bind narrow
Slack write host calls to `SlackAction` instead of effectful `LambdaInternal`
when the formal host call is exactly a Slack message send.
