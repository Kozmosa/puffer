# Monitor Task Threshold

## Summary

Monitor triage prompts now define a task as an explicit request, schedule, or
service notification that clearly requires user action.

## Behavior

- Triage agents must not create tasks for standalone greetings, thanks,
  acknowledgements, reactions, FYI/status-only updates, generic bot summaries,
  newsletters, training logs, GitHub-style notifications, or repeated source
  updates unless they contain an explicit request, assignment, review request,
  approval request, or schedule.
- The task threshold is connector-independent. Telegram, email, Slack, and
  GitHub-like notifications use the same criteria.
- Triage agents may check surrounding message, thread, linked task, or
  connector-provided context when an event is ambiguous.
- Triage exits early for non-actionable events and must not call task tools
  when the event lacks a clear next action for the user.
- Backfill agents use the same task threshold as live triage.

## Compatibility

Only monitor prompt text changes. Existing workflows, task metadata, and
subscriber event contracts remain compatible.
