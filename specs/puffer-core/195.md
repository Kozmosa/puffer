# Lambda Host Input Binding

## Scope

Verified Lambda Skills no longer require the model to hand-author the concrete
tool input for `LambdaHostCall`. The model supplies the formal `host_tool`,
formal `args`, and target concrete `tool`; Puffer materializes the exact
concrete input from the precompiled host catalogue contract.

## Contract

`LambdaHostCall.input` is optional. When omitted, Puffer renders the verified
`concreteInputContracts` template against the formal host arguments. When
present, the supplied input must still match the rendered contract exactly.
The pending bridge records the materialized input, and the next concrete tool
call must match both the concrete tool id and that exact input.

## Tool Scope

An active Lambda Skill gate only requires `LambdaHostCall` for concrete tools
inside the active skill request scope. Unrelated exploratory tools continue
through normal permission checks, while in-scope concrete tool calls still fail
closed until admitted through the bridge.

## Compatibility

`McpToolCall` keeps runtime validation for its two invocation modes, but its
model-visible schema uses an object root without top-level union keywords so
OpenAI function schema validation accepts the bundled tool list.
