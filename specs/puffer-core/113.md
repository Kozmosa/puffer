# Workflow action deletion command

`/workflows` now exposes deterministic workflow action removal for bindings
created by monitors, append actions, and other subscription-backed workflow
actions.

## Contract

- `/workflows delete <binding-slug>` deletes the matching workflow binding from
  the subscription workflow binding store.
- `/workflows remove <binding-slug>` and `/workflows rm <binding-slug>` are
  aliases for the same operation.
- The workflow action list shows a copyable `delete=/workflows delete <slug>`
  hint on each binding row.
- Workflow action search indexes `delete`, `remove`, and the full delete
  command so users can find cleanup actions from the dashboard.
- After deletion, connection consumer state is refreshed so idle/active status
  reflects the removed binding.

Native workflow definitions are not deleted by this command; it is scoped to
subscription workflow bindings because those are the actions created by
`/workflows append` and monitor setup.
