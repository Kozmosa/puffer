# New Relic Webhook Connect Preset

`/connect newrelic-webhook <connection-name>` now runs the deterministic
serve-mode setup for New Relic issue and alert webhooks.

## Behavior

- Resolve `newrelic-webhook` from exact slug or searchable terms such as
  `new relic alert`.
- Ask for bind address and URL path through the existing user-question flow.
- Default the webhook URL path to `/newrelic`.
- Write `[connectors.webhook]` in `.puffer/connectors.toml` with the requested
  display name, bind address, path, and welcome message.
- Report the connection as setup-only in `/workflows connectors` with
  `runtime=serve`, `[no-trigger]`, and the deterministic command
  `/connect newrelic-webhook newrelic-webhook`.

## Compatibility

The preset continues to use the generic webhook connector runtime and does not
add a connector protocol subscriber, workflow trigger runtime, or outbound
action.
