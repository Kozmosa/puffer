# ImageGeneration Optional Count

`ImageGeneration` count is optional so persisted image settings can provide the
default output count.

## Tool Contract

- The internal tool schema no longer requires `count`.
- When `count` is omitted, image generation uses the saved canonical `output`
  selection from media settings.
- When `count` is supplied, it overrides the saved `output` for that call.
- Explicit count is validated by the canonical media resolver and cannot exceed
  the selected model cap or the global image cap of `9`.
- Per-call aspect overrides are routed through canonical `ratio` selections
  instead of directly writing provider `size` or `aspect_ratio` fields.

## Compatibility

The old required-count behavior and old raw aspect-to-provider-field override
path are removed. Video generation keeps fixed count `1`.
