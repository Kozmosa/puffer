# Lambda Refinement Expression Coverage

## Summary

`puffer-core` expands Lambda refinement checking beyond simple predicate calls
and scalar comparisons. Formal argument and result checking now support:

- object-field comparisons such as `sec = secret`
- boolean literals in comparisons such as `w = true`
- implication expressions such as `a ==> b`

## Contracts

- Field comparisons read identifiers from the current JSON object before
  falling back to the whole current value or symbolic literals.
- String fields can be compared with symbolic enum literals, preserving the
  existing `mode = private` style.
- Implication is evaluated as fail-closed logical implication: when the
  antecedent holds, the consequent must also hold.

## Compatibility

Existing numeric, cross-argument, string, semantic predicate, and custom fact
refinements are unchanged. The added cases cover generated Lambda catalogues
that use field-style security tags, boolean guards, and budget implications.
