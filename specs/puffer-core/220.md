# Recap Brief Shape

## Design

`/recap` now asks the model for one short first paragraph plus at most three optional detail paragraphs. The prompt explicitly rejects markdown, bullets, file inventories, exhaustive logs, and root-cause dumps so recap output stays glanceable.

## Logic

The recap side turn normalizes model output before rendering. Normalization strips common labels, removes list markers, uses the first sentence as the visible summary, moves extra same-paragraph sentences into details, caps details at three paragraphs, and bounds detail paragraph length. `render_recap` applies the same normalization before adding the `RECAP_DISPLAY_PREFIX`.

## Compatibility

The transcript contract remains a transient system message prefixed with `RECAP_DISPLAY_PREFIX`; no persisted session format changes are introduced. Older one-line recap text still renders as a single summary paragraph.
