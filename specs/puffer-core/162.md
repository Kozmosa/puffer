# Lambda Skill Loader Binding

## Summary

Lambda host catalogues may now bind a formal host tool to the concrete `Skill`
tool. This covers verified workflows whose formal step means "load this other
verified skill" rather than run a shell command.

## Runtime Contract

`LambdaHostCall` admits `Skill` only when the active verified catalogue declares
`Skill` as the concrete tool and the concrete input matches the catalogue's
`concreteInputContracts.Skill` pattern exactly.

During the concrete `Skill` call, Puffer lets the skill loader install the next
verified skill gate, then temporarily restores the previous gate to commit the
formal host call. After the commit succeeds, Puffer reinstalls the newly loaded
skill gate. This keeps the current formal transition guarded while allowing the
workflow to continue under the loaded skill.

## Compatibility

Unverified skill loading is still rejected while a Lambda gate is active.
Direct model-selected verified skills remain available for normal skill choice;
the new path is only for host tools that explicitly bind to `Skill`.
