# Workflow Contact Defaults

- Workflow binding constructors in command helpers and Claude-compatible workflow
  tools initialize `contact_ids` to an empty list.
- Empty contact ids preserve the existing workflow behavior: subscriptions and
  generated workflow bindings receive events from all contacts unless a higher-level
  caller explicitly scopes the binding.
- This is a schema compatibility update only; slash-command and tool callers do not
  gain contact-selection UI or prompt surface in `puffer-core`.
