# Anthropic Iteration Test Stream Blocking

## Summary

The Anthropic iteration-behavior test mock server now switches accepted TCP
streams back to blocking mode before reading request bytes.

## Behavior

- The listener remains nonblocking so the test can enforce its request deadline.
- Each accepted connection is made blocking before `read`, avoiding transient
  `WouldBlock` failures when the client has connected but has not delivered the
  full request yet.

## Compatibility

This is a test harness change only. Runtime Anthropic iteration behavior,
provider request construction, and tool-loop contracts are unchanged.
