# Agent Cwd Permission Prompt

## Context

Desktop turns can expose the runtime `Agent` tool to provider models. The tool
accepts an optional `cwd` override so a subagent can run from a different
directory.

## Change

- Treat `Agent.cwd` as a filesystem path when pre-resolving tool permissions.
- If the requested cwd is outside the session workspace roots, emit the normal
  runtime permission prompt before the subagent is launched.
- A denied prompt returns the standard blocked tool result instead of allowing
  the turn to continue into a later filesystem error.

## Compatibility

Existing in-workspace Agent calls are unchanged. `Allow once` grants the target
directory only for that tool call; `Allow for session` and `Allow all` continue
to use the existing session grant machinery.
