# Connect Action Search

## Summary

`/connect` now uses the same skill and action metadata that `/workflows
connectors` and Corbina Pipelines expose, so the terminal connector picker can
be searched by what the connector does instead of only by connector slug.

## Behavior

- Built-in connector picker rows include concise `skill=` and `actions=`
  metadata in the searchable option description.
- `ConnectorList` rows with action definitions or `action_slugs` contribute
  those action slugs to `/connect` search.
- Partial connector resolution uses the enriched metadata, so commands such as
  `/connect vote telegram-user` resolve to `telegram-login`; the interactive
  connector picker can also filter by multi-term action names such as
  `vote poll`.
- Search remains term-based and case-insensitive; each query term must match
  somewhere in the connector slug, description, skill, traits, or action
  summary.

## Compatibility

Exact `/connect <connector-slug> <connection-name>` commands keep the existing
fast path. The visible picker text is more descriptive but still returns the
same connector slug labels.
