# Verified Lambda Skill Model Activation

The model-visible `Skill` tool may activate a verified Lambda Skill directly.
This is the normal user path: users can preload many verified skills through
resource config and ask ordinary questions; the model chooses the matching
skill by calling `Skill`.

Activation uses the same Lambda gate as `/skill:<name>`. A verified Lambda
Skill must have an active host catalogue and a non-empty `allowed_tools` list.
Puffer appends `LambdaHostCall` to that scope, installs the gate, clears any
pending bridge, and returns the rendered skill prompt to the model.

The active gate is turn-local. `execute_user_prompt*` restores the prior Lambda
gate and pending host-call bridge after the provider loop finishes, so selecting
a skill cannot leak restrictions into later user prompts. `Skill` calls force
serial tool-batch execution; if a model emits `Skill` alongside other tools,
Puffer loads the skill first and then applies the gate to subsequent calls.

`LambdaHostCall` admission also checks the active skill tool scope before it
records a pending bridge. It can only target a concrete Puffer tool/input that
the active skill config permits. The concrete tool still passes through normal
Puffer permission and approval handling before the Lambda gate commits.
