# Workflow connector action detail

`/workflows connectors <query>` now expands action visibility when a search
narrows the terminal result to a single connector.

## Contract

- The default connector list remains compact and keeps the existing
  `actions=<first-three>,+N` row summary.
- When a non-empty connector query returns exactly one connector, the row is
  followed by `actions_all=<slug,...>` with every action slug in deterministic
  display order.
- `send_message` remains first when present, followed by the remaining action
  slugs in lexical order.
- The detail line is terminal-only rendering metadata. It does not change
  connector catalog loading, workflow creation, auth, action execution, or
  webhook behavior.

This keeps terminal workflow discovery aligned with the Corbina connector
detail panel, where selecting a connector already exposes the full action
surface.
