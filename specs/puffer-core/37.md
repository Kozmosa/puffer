# Internal Browser permission lookup

## Purpose

Route Browser permission checks through a structured internal-tool channel
instead of allowing Browser to exist as a normal model tool call.

## Design

The internal permission resolver canonicalizes the requested internal tool id
and resolves Browser through `ToolRegistry::internal_definition`. The resolver
enriches Browser input with current-tab context, evaluates the normal browser
ACL profile, and prompts with `BrowserInternalTool` provenance when needed.

Normal model tool-call execution still resolves definitions through
`ToolRegistry::definition`, so Browser calls are unknown unless a test or custom
resource intentionally registers Browser as a normal tool.

## Contract

Browser ACL behavior is driven by the internal Browser definition. Shell command
text is not parsed to infer Browser actions; the Browser CLI sends structured
permission requests before executing the action.

## Verification

Runtime permission tests cover Browser ACL loading from `[browser]` config,
session grants, deny rules, and internal permission request handling against the
internal registry definition.
