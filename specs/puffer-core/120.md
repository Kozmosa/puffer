# 120. Lambda Skill Host Gates

Puffer now has a typed Rust Lambda Skill gate for host catalogues emitted by
`lskillc export-json`, and skill side turns can install it as the active
host-call monitor.

## Design

`runtime::lambda_gate` parses the JSON host catalogue shape used by LambdaW:
declared effects, domains, tools, effect rows, registered facts, and optional
context requirements. It mirrors the executable gate described in the paper and
the LambdaW runtime:

- reject unknown host tools
- reject calls whose effect row exceeds the gate capabilities
- reject calls whose `contextReq` fact is absent
- commit a tool's `registers` facts after accepted calls

Runtime wiring installs the gate with the catalogue's full effect alphabet and
an empty fact set. The gate implementation has test coverage for narrower
capability sets and initial facts, which leaves a small typed path for resumed
or pre-authorized contexts without changing the active runtime contract.

When a user invokes a skill whose verification metadata points at a host
catalogue, Puffer reads that catalogue before starting the skill side turn and
stores a gate on `AppState` for the duration of the turn. Tool execution
preflights the gate immediately after resolving the tool id so unknown or
forbidden host calls do not open permission prompts. It then re-checks and
commits registered facts after normal permission and filesystem access checks,
but before hooks and actual tool dispatch. A rejected gate call returns a failed
tool result instead of running the host tool.

OpenAI and generic batched tool execution fall back to serial execution while a
Lambda Skill gate is active so fact registration stays ordered.

## Contracts

The gate does not execute tools and does not trust model text. Runtime callers
must present the host tool name from a verified skill catalogue and only run the
real tool after `step_call` accepts.

The gate intentionally does not re-prove argument refinements. That obligation
remains with the Lambda Skill compiler and generated host catalogue; the runtime
gate enforces the stateful premises it can observe at the host-call boundary.

## Verification

`cargo test -p puffer-core runtime::lambda_gate` covers parsing the exported
catalogue shape, unknown-tool rejection, capability rejection, context
precondition rejection, fact registration, and initial facts.

`cargo test -p puffer-core lambda_gate_` covers active-gate enforcement during
tool execution and fact commitment after accepted calls.
