# Plan Cache Test Synchronization

Plan slug tests now serialize access to the process-wide plan slug cache. The
runtime behavior is unchanged: plan slugs remain cached per session id, resume
still reuses the source slug, and fork still assigns a distinct target slug when
the target metadata would collide.

This only hardens the unit tests against parallel `clear_all_plan_slugs` calls
that could invalidate another test's source or target path between assertions.
