# Monitor Task Timestamp Contracts

Monitor triage now has explicit text timestamp and expiry contracts.

- The model-facing system reminder includes the computer's current RFC3339
  timestamp in addition to the current date.
- `TaskCreate` accepts optional RFC3339 `receivedAt` and `expiresAt` fields,
  stores them under those exact JSON keys, and returns them in task
  create/get/list output.
- Monitor tasks must provide both `receivedAt` and `expiresAt`; invalid
  RFC3339 timestamps are rejected before writing task state.
- The monitor triage prompt instructs agents to use the workflow trigger's
  `receivedAt` value and set a concrete `expiresAt` for every monitor task.
