# OpenAI Tool Parameter Schema Compatibility

OpenAI tool serialization now normalizes function parameter schemas before
building Responses or Chat Completions requests.

- Tool parameter roots are forced to `type: object`, with a `properties` map
  present even when the declarative schema omitted one.
- Root-level `oneOf`, `anyOf`, `allOf`, `enum`, and `not` are removed because
  OpenAI rejects these keywords at `tools[*].parameters`.
- Nested schema combinators are preserved. Expressive bundled schemas such as
  `WorkflowCreate` can keep Anthropic/local validation semantics while OpenAI
  receives an acceptable request shape.
- Runtime tool handlers remain responsible for semantic validation, including
  mutually exclusive inputs such as `action` versus `yaml_action`.

This is provider serialization compatibility only; it does not change
declarative resource loading, Anthropic tool schemas, permissions, or local tool
execution contracts.
