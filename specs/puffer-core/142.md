# OpenAI Lambda Skill Batch Ordering

OpenAI tool-call execution now falls back to serial execution whenever a batch
contains `Skill`, matching the generic provider batch path. This prevents
parallel-safe concrete tools in the same model response from running before a
verified Lambda Skill installs its gate.

The serial path preserves the original call order: the `Skill` call installs
the Lambda gate first, and following concrete tool calls are rejected with the
recoverable `LambdaHostCall` retry guidance unless they have a matching pending
bridge.

- `cargo test -p puffer-core openai_skill_batch_serializes_before_parallel_safe_concrete_tools`
