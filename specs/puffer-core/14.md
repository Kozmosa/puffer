# Explicit Add-Dir Roots

## Purpose

Keep `/add-dir` state changes distinct from the default scratch roots that are
available for tool writes.

## Design

Workspace path resolution still treats the primary workspace, explicit
`/add-dir` roots, `/tmp`, and `$TMPDIR` as writable roots. `/add-dir`
validation, however, checks duplicates only against the primary workspace and
directories the user already added explicitly.

## Logic

Default scratch roots are ambient tool-write permissions, not session working
directories. A path under `/tmp` or `$TMPDIR` can therefore be added explicitly
so the session state, UI, and permission summaries reflect the user's intent.
Duplicate prevention still rejects paths inside the primary workspace or an
already-added working directory.

## Contracts

- Tool path resolution continues to allow `/tmp` and `$TMPDIR`.
- `/add-dir` records an existing directory under a scratch root when it is not
  already inside the primary workspace or an explicit working directory.
- `/add-dir` still rejects missing paths, files, and directories already covered
  by explicit roots.

## Compatibility

This preserves the scratch-root behavior introduced for Claude-compatible tool
writes while restoring the previous visible `/add-dir` command behavior for
temporary and generated workspaces.
