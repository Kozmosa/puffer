# OpenAI Responses reasoning include selector

- OpenAI Responses requests use the compact `reasoning.encryptedcontent` include selector for public OpenAI and Codex-compatible backend routes.
- The selector is emitted only when reasoning config is present, preserving existing non-reasoning request bodies.
- Response parsing still reads the returned `encrypted_content` field; the wire selector spelling does not change the response item shape.
- This keeps desktop OpenAI sessions from failing immediately with `include[0]` `invalid_value` errors when the selected model supports reasoning.
