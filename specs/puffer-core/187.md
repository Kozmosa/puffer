# Shopify Verified Skill Bridge

## Context

Some translated Lambda skills expose Shopify host tools whose formal API has
more semantic information than a raw shell command. `fulfillment_create`
declares both the Shopify order id and the fulfillment input. A safe concrete
bridge must use the order id as a guard instead of merely forwarding the input
to GraphQL.

## Design

`ShopifyAction` is a workflow runtime tool with one supported action:
`fulfillmentCreate`. The input contains `orderId` and `inputJson`. `inputJson`
must be the Shopify GraphQL variables object with a `fulfillment` field and an
optional `message` field.

Before running the mutation, Puffer queries the order's fulfillment orders and
extracts their ids. It then compares those ids with every
`fulfillment.lineItemsByFulfillmentOrder[].fulfillmentOrderId` in `inputJson`.
The mutation is only sent when all requested fulfillment order ids belong to
the declared order.

## Runtime Contract

Credentials come from `SHOPIFY_STORE_DOMAIN`, `SHOPIFY_ACCESS_TOKEN`, and
optional `SHOPIFY_API_VERSION`; the API version defaults to `2026-01` to match
the generated host catalogue. The store domain must be a bare domain. GraphQL
HTTP errors, top-level GraphQL errors, and `fulfillmentCreate.userErrors` fail
closed.

## Compatibility

The bridge is exposed as a normal runtime workflow tool and participates in the
existing LambdaHostCall pending bridge. The concrete call still requires user
approval through the regular tool approval path because the resource is marked
with `approval_policy: ask`.
