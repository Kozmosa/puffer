# 182. Vision analysis bridge

Puffer adds a `VisionAnalyze` workflow tool for Lambda Skill contracts that
need a concrete screenshot or image analysis step. The tool accepts an image
path and a question, reads the image bytes locally, and sends a Responses API
request with one `input_text` item and one `input_image` item.

The image path may be absolute or workspace-relative. Relative paths cannot
escape the workspace. Supported image extensions are PNG, JPEG, GIF, and WebP,
and the bridge rejects empty or oversized files before making a network call.

`OPENAI_API_KEY` or `PUFFER_OPENAI_API_KEY` supplies the backend key.
`PUFFER_VISION_MODEL` can override the default model. The bridge returns only
the text report from the model response, so Lambda Skill result values remain
plain report text rather than synthetic host metadata.
