# puffer-core update 09: AutoDream external trace boundaries

Refines AutoDream behavior for noisy external trajectories and separates
durable workflow memory from GenSkill suggestion decisions.

## Design

- AutoDream now treats external benchmark traces as valid memory signal when
  they show a reusable diagnose-filter-verify workflow.
- External memories must generalize away dataset names, task ids, exact paths,
  commands, secrets, payloads, temporary artifacts, model names, and benchmark
  noise.
- GenSkill suggestions are parsed only from the explicit
  `AUTODREAM_GENSKILL: yes` marker. Writing workflow memory no longer promotes
  the trace to skill-worthy by itself.

## Contracts

- AutoDream may write one durable workflow memory entry for a failed, unsolved,
  or noisy trace when the reusable method is clear.
- Clean/control-style external tasks default to `AUTODREAM_GENSKILL: no` unless
  the trace also shows repeated exploration failures, recovery pivots, or
  multi-stage methodology with future transfer value.
- A positive GenSkill marker requires durable workflow memory plus stronger
  evidence than ordinary memory consolidation.

## Compatibility

- `/genskill` remains manual and separate from AutoDream.
- Existing project-memory storage and command output shape are unchanged.
- The change only narrows the suggestion signal; it does not suppress durable
  memory extraction.
