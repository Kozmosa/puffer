# Browser Tool Daemon Discovery

## Purpose

Allow chat-driven Browser tool calls to reuse the desktop daemon advertised in
the user-level daemon handshake file when the active chat session uses a
temporary or non-repository working directory.

## Design

Browser tool daemon discovery checks the workspace handshake first and only
accepts it when its `workspaceRoot` matches the current workspace root. It then
checks the user-level handshake and treats it as the global desktop daemon
fallback without applying the workspace filter.

## Logic

Desktop sessions can create chat turns whose runtime `cwd` differs from the
desktop daemon workspace. In that case a workspace-scoped handshake must not be
borrowed from another repository, but the user-level handshake is the daemon
advertised by the running desktop app and remains valid for Browser tab
control.

## Contracts

- Matching workspace handshakes take priority over user-level handshakes.
- Mismatched workspace handshakes are ignored rather than used.
- User-level handshakes are accepted as global desktop daemon fallback.
- Missing handshake files still produce the existing "running Puffer daemon"
  error.
- Malformed or unreadable handshake files still surface decode or read errors.

## Compatibility

This keeps repository-local daemon isolation for workspace handshakes while
restoring Browser tool availability for desktop chat sessions launched from
temporary working directories.
