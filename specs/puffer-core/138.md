# Gate-Ready Lambda Skill Discovery

Model-facing skill discovery now includes verified Lambda Skills when they are
ready to install a runtime gate. A verified Lambda Skill is model-invocable only
when `disable_model_invocation` is false, `allowed_tools` is non-empty, and its
verification metadata has either a host catalogue path or a compiler path plus a
formal source path.

Prompt-only verified Lambda Skills remain hidden from the model-invocable list.
They still fail closed if invoked directly, matching the runtime gate contract.

Focused verification:

- `cargo test -p puffer-core runtime_system_prompt_lists_model_invocable_skills`
