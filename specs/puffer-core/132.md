# Explicit Lambda Skill Compiler Configuration

## Design

Strict Lambda Skill compilation no longer probes directories above a skill
source for repository-local `lskillc` binaries. Runtime compiler resolution is
limited to explicit configuration:

- `compiler_path` carried by the loaded Lambda Skill verification metadata
- `PUFFER_LSKILLC`
- `lskillc` installed on `PATH`

`compiler_path` is fail-closed. If a manifest points at a missing compiler,
strict mode reports that configured path instead of silently trying unrelated
local layouts.

## Contracts

Translated skill checkout layouts are not part of Puffer's product contract.
Users install a Lambda Skill library by declaring the library root in resources,
and they install the compiler through either the same resource manifest, an
environment override, or their shell path.

Compile-cache keys still include the resolved compiler path and source path, so
changing either input invalidates cached host catalogues.

## Verification

`cargo test -p puffer-core runtime::lambda_gate::tests::compiler_resolution_ignores_unconfigured_neighboring_compilers`
checks that unconfigured neighboring binaries are ignored.

`cargo test -p puffer-core runtime::lambda_gate::tests::compile_gate_reuses_catalogue_until_source_changes`
checks that strict compilation uses a configured compiler path and preserves the
existing cache invalidation behavior.
