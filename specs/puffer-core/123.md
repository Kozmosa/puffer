# 123. Lambda Host Bridge Skill Filters

Verified Lambda Skill side turns preserve the host-call bridge under
request-scoped tool filters.

## Design

When a Lambda Skill has an explicit `allowed_tools` list, Puffer appends
`LambdaHostCall` to the side-turn filter. Empty `allowed_tools` remains
unrestricted.

## Architecture

`execute_skill_command` now builds the side-turn filter from a helper that
recognizes `verification.system = "lambda-skill"`. The helper only changes
request filtering; it does not alter the skill resource, persisted metadata, or
normal Puffer permissions.

## Contracts

- A filtered verified Lambda Skill can always admit formal host calls through
  `LambdaHostCall`.
- Non-Lambda skills keep their declared `allowed_tools` unchanged.
- Unfiltered Lambda Skills keep the existing unrestricted tool visibility.

## Compatibility

This keeps existing skill manifests valid while allowing strict Lambda gate runs
to compose with Puffer's request-scoped tool filtering.
