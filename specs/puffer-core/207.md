# Recall — auxiliary context with replay

## Summary

`Recall` is a retrieval layer over the session's own transcript. Long agent runs
compact/summarize their context and lose exact detail; Recall lets the model ask
for relevant earlier content and **replays the most relevant past messages back
verbatim** — an earlier diff, a tool output, a path the user gave, a decision —
that has scrolled out of the live window.

Grounded in the long-horizon agent-memory literature (MemoRAG / Memex(RL)
indexed-experience replay, episodic context reconstruction): keep the session's
full history addressable and re-inject exact past content on demand, rather than
relying only on a lossy compacted window. puffer already has a *live* replay ring
in the daemon (recent events → reconnecting clients); Recall is the durable,
query-driven counterpart available to the model.

## Design

- Tool `Recall` (`resources/tools/recall.yaml`, handler `runtime:workflow:recall`).
  Input `{ query: string, k?: number (default 5, max 20) }`.
- Handler (`crates/puffer-core/runtime/claude_tools/workflow/recall.rs`) ranks
  `AppState.transcript` (the in-memory message history, incl. tool name + input)
  with a **BM25-lite** lexical score (df-weighted term overlap, mild length
  normalization), excludes the current call, and returns the top-k matches with
  their verbatim content — replayed into the turn as the tool result.
- Offline, zero-dependency, no network. The current call's own message is
  excluded so Recall doesn't echo itself.

## Why lexical first
A dependency-free lexical index works immediately and is good for the common case
(recovering a path / error / symbol the user or a tool mentioned). A semantic
index (embeddings) is the natural upgrade and can run **on-device** via the local
MiniCPM5 provider — keeping the auxiliary memory private and free. Deferred.

## Follow-ons
- Read the full **session.jsonl** (lossless, survives compaction) instead of only
  the in-memory transcript, so detail dropped from memory is still recallable.
- Embedding-based semantic recall (on-device).
- Auto-recall: inject likely-relevant context at compaction boundaries without an
  explicit tool call.

## Testing
`cargo test -p puffer-core recall::` — ranks the relevant earlier message to the
top; empty query errors.
