# TouchDesigner SafeScript Bridge

Puffer adds `TouchDesignerAction` as a narrow verified-skill bridge for the
TouchDesigner Lambda host tool `make_safe_td_script`.

The `validateScript` action accepts `code`, rejects empty snippets, rejects
quoted hardcoded absolute paths, and rejects simple Python snippets that map to
native TouchDesigner MCP operations such as direct parameter setting, operator
creation, or connector wiring. Accepted code is returned as the same JSON string
so the Lambda gate can commit the `SafeScript` result facts for that exact value.

This keeps `td_execute_python` guarded by normal Lambda fact flow: the concrete
MCP execution can only be admitted after the exact script value has passed the
SafeScript bridge. The bridge is read-only and does not contact TouchDesigner or
run Python; execution remains with the separately admitted `McpToolCall`.
