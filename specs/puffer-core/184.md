# ComputerUse Focus App Bridge

## Scope

`ComputerUseAction` now supports `focusApp` for verified Lambda Skill host
contracts that bind subsequent app-less computer-use actions to a named macOS
application.

## Contract

- Input: `{ "action": "focusApp", "app": string, "raiseWindow"?: bool }`.
- With `raiseWindow=false` or omitted, Puffer calls the `computer_use`
  `list_apps` MCP tool, records the requested app as the active app, and does
  not raise a window.
- With `raiseWindow=true`, Puffer calls `computer_use.get_app_state` for the
  app, records the requested app as active, and returns the capture metadata.
- Later app-less actions such as `typeText`, `keyCombo`, `scroll`,
  `doubleClick`, and `rightClick` use the recorded active app.

## Compatibility

This is a narrow extension of the existing typed computer-use bridge. Existing
actions and input shapes are unchanged. Unsupported computer-use actions still
fail closed before reaching MCP.
