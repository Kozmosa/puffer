# Gmail Browser Connect Auth State

- The deterministic `/connect gmail-browser` helper now removes stale
  `auth_state.json` after writing a new per-connection config.
- This aligns the manual connect path with daemon-owned Gmail setup, which
  already clears auth-required state when saving config.
- The helper logs config save and connection upsert outcomes using connection
  slug and account count, not raw account diagnostics beyond the existing
  connection description.
- Reconnecting a Gmail browser account can now recover from a prior
  `auth_required` poll without waiting for a later successful poll to clear the
  stale auth-state file.
