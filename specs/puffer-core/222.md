# Recall Self-Filtering And Ranking Improvements

## Summary

The Recall workflow tool keeps the existing transcript replay contract while
making ranking safer and more useful.

## Design

- Recall tool calls and Recall results are excluded from searchable text so the
  tool does not recursively retrieve its own output.
- Ranking scans the whole current transcript instead of dropping the newest
  item by position. This avoids missing the latest real user/tool message when
  the Recall invocation itself is not the final transcript row.
- Query terms are deduplicated, capped, and have document frequency precomputed
  once to avoid repeated full transcript scans per candidate document.
- Replayed snippets now allow longer excerpts while still truncating very long
  messages.

## Compatibility

The public `Recall` input and output JSON shape is unchanged. The change only
affects which transcript rows can rank and how much relevant context is replayed
per hit.
