# Legacy Filesystem Alias Path Approval

Legacy file/search aliases must follow the same manual approval path as the
Claude-compatible filesystem tools when a model targets a path outside the
session's current working directories.

## Requirements

- `read_file`, `list_dir`, and `search_text` inspect their path inputs before
  execution and request path-scoped permission when the target is outside the
  current workspace roots.
- An `Allow once` response extends only the in-flight filesystem policy; it
  must not add a persistent session path grant.
- Approved legacy alias execution must use the granted filesystem roots rather
  than falling through to the older workspace-only builtin path checks.

## Compatibility

This does not widen access automatically. Denied prompts still return a normal
tool-denied result, and calls without an explicit external path keep the
existing workspace behavior.

## Verification

Core runtime coverage runs `search_text` against an external directory, answers
the path approval prompt with `Allow once`, and verifies the search result
contains the external file while session grants remain unchanged.
