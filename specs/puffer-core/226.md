# AutoDream Manual Project Memory Bootstrap

Manual `/autodream` now lazily initializes project memory when memory is enabled
but the current working directory has no registered project memory.

## Behavior

- `/autodream status` and `/autodream suggestions` remain read-only and do not
  create project memory.
- A manual `/autodream` run checks `state.project_memory` before invoking the
  model side turn.
- If project memory is missing and `memory.enabled` is true, Puffer registers
  the current working directory in the user project registry, creates the
  managed project `MEMORY.md`, refreshes `state.project_memory`, and then
  continues the AutoDream review.
- Automatic background AutoDream does not call the bootstrap path. It still
  skips when project memory is unavailable.

## Storage

The bootstrap uses the existing Puffer project-memory storage model:

- registry: `~/.puffer/projects.toml`
- memory file: `~/.puffer/projects/<project-name>-<path-hash>/MEMORY.md`

The project name defaults to the current directory basename. The hash continues
to come from the normalized project root so same-name projects do not collide.

## Compatibility

AutoDream still writes only to Puffer-managed project memory through the Memory
tool. It does not write to repository `CLAUDE.md` files.
