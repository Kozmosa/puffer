# Telegram Muted Monitor Inputs

Telegram monitor prompts now treat muted notifications as ignored input.

- The monitor triage prompt tells agents not to create tasks for events marked
  `notification_muted` or `notification_silent`.
- Telegram monitor backfill instructs agents to call `list_peers` first and
  skip peers where `notification_muted=true` before reading recent messages.
- Task metadata and monitor storage remain unchanged; muted-notification
  suppression happens before monitor triage dispatch in the subscription
  router.
