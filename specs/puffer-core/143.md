# Lambda Skill Gate Readiness Verification

Verified Lambda Skill readiness now uses the actual gate construction path
instead of path-existence checks. A skill is reported as gate-ready and
model-invocable only when Puffer can build a Lambda gate from its configured
host catalogue or compiler-backed `lskillc export-json` output.

This keeps `/skills`, `/doctor`, `puffer doctor`, and the runtime system prompt
aligned with execution. Broken catalogues, failed compiler exports, invalid
generated host JSON, missing `allowed_tools`, and missing gate config all block
model invocation and surface the concrete failure reason in doctor output.

Focused verification:

- `cargo test -p puffer-core render_lambda_status_rejects_invalid_host_catalogue`
- `cargo test -p puffer-core runtime_system_prompt_hides_compiler_backed_lambda_skill_when_export_fails`
