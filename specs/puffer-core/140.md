# Lambda Skill Readiness UX

User-facing Lambda Skill surfaces now report per-skill readiness instead of
only aggregate counts.

`/skills` includes, for each verified Lambda Skill:

- whether the skill is gate-ready and which gate source it uses
- whether model invocation is currently available
- the configured concrete `allowed_tools` scope

`/doctor` and the non-interactive doctor status include the same per-skill
status lines. Doctor warnings now point at the specific non-ready skill and
include the concrete failure reason, such as missing `allowed_tools`, missing
host catalogue or compiler config, or a configured file path that does not
exist.

Focused verification:

- `cargo test -p puffer-core render_skills_panel_marks_verified_skills`
- `cargo test -p puffer-core render_lambda_status_`
