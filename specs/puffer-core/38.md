# Bash internal tool helpers

## Purpose

Make Browser available to Bash without requiring skill prompts to teach shell
alias setup.

## Design

Claude-style Bash foreground and background commands are prefixed with a shell
function for each alias in the shared internal CLI tool descriptors. The
Browser function calls the current Puffer executable with `internal-tool
browser` and forwards all arguments.

The existing internal permission broker environment is still inherited by the
Browser child process. Browser permission decisions continue to come from the
structured internal-tool protocol, not from shell command parsing.

Interactive TTY commands keep their existing command path and are unchanged.

## Contract

Agents can run internal CLI tool aliases such as `browser ...` inside Bash. The
skill prompt should not need to print or install aliases.

## Verification

Unit coverage asserts that a foreground Bash command can resolve the `browser`
helper with `type browser`.
