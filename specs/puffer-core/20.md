# 18. OpenAI Responses Include Dialects

## Summary

OpenAI Responses request bodies now choose the reasoning encrypted-content
include selector for the endpoint they are actually targeting.

## Design

- Public OpenAI `/v1/responses` requests send the official
  `reasoning.encrypted_content` selector.
- ChatGPT/Codex backend requests keep the compact
  `reasoning.encryptedcontent` selector required by that compatibility path.
- The selector decision is centralized in the OpenAI Responses request-body
  builder and is reused by the agent loop, websocket transport, legacy
  streaming fallback, and reflection judge calls.

## Compatibility

OpenAI API-key sessions avoid 400 `include[0]` validation failures caused by
sending the ChatGPT/Codex selector to the public API. OAuth/Codex-compatible
sessions keep their previous wire shape.

## Tests

Rust request-body tests cover both public OpenAI and ChatGPT/Codex backend
reasoning include selectors, while existing agent-loop tests continue to
exercise the shared Responses request path.
