# Precompiled Lambda Skill Gates

Verified Lambda Skills are now gate-ready only when they provide a precompiled
host catalogue through `host_catalogue_subpath`. Runtime gate construction reads
that JSON directly and no longer runs `lskillc export-json`, resolves compiler
paths, or caches compiler output.

`compiler_path` remains a deserializable compatibility field on verification
metadata, but it is ignored by runtime admission. A skill with only
`compiler_path` is treated as not gate-ready and is hidden from model-invocable
skill lists.

`/skills config` and `/doctor` now describe the precompiled-catalogue path:
users install an external library folder that already contains generated skill
descriptors and host catalogues, then Puffer verifies readiness from those files.

Validation:
- `cargo test -p puffer-core lambda_skill -- --nocapture`
- `cargo test -p puffer-core command_helpers::lambda_doctor -- --nocapture`
