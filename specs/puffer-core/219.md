# Internal Browser Interaction Requests

## Summary

`requestuserbrowseraction` is an internal workflow prompt path, not a
model-facing provider tool.

## Behavior

`ConnectorAct` may invoke the `requestuserbrowseraction` action when a connector
needs the user to complete a browser-backed sign-in or approval step. The runtime
injects the connector and connection slugs into the action input, then routes the
request to the internal `RequestUserBrowserAction` workflow. That workflow
validates a non-empty `description`, preserves an optional `url`, and asks the
user a browser interaction question through the existing `AskUserQuestion`
machinery.

The workflow returns structured JSON containing the original description, the
optional URL, and the nested prompt result. `ConnectorAct` wraps that output in
the normal connector action response shape and preserves the action permission
metadata from the connector catalog.

## Compatibility

Provider-visible tool discovery does not include `RequestUserBrowserAction` or
its `requestuserbrowseraction` alias. Agents should reach the behavior through
connector action metadata, while internal runtime callers can still dispatch the
workflow directly by name.
