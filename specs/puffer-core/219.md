# Connector Skill Template Guidance

## Summary

Connector skill generation now produces a fuller implementation guide instead
of a minimal auth/subscribe/action checklist.

## Design

- The generated skill text distinguishes command-backed connectors, reusable
  subscriber manifests, and internal/tool-backed connector runtimes.
- It documents template metadata expectations, including action schemas,
  subscriber selection, output schemas, and permission flags.
- Auth guidance routes setup through standard connection flows and warns against
  importing unrelated live app sessions unless that is explicitly the connector
  design.
- Streaming guidance requires JSONL `event`, `checkpoint`, and `health` frames
  with durable IDs, cursors, resume behavior, ack persistence, and duplicate
  suppression.
- Action guidance requires list/search/read actions when later side-effecting
  actions need stable target identifiers.

## Compatibility

This changes generated skill instructions only. Existing connectors, workflow
stores, and runtime APIs keep their current behavior.
