# VideoGeneration Parameter Resilience

Date: 2026-06-09

## Summary

`VideoGeneration` accepts scalar JSON parameter overrides from tool-calling
models and normalizes them into the existing string parameter map before exact
media execution.

## Behavior

- String, number, and boolean parameter values are accepted.
- Object, array, and null parameter values remain invalid at the workflow
  boundary.
- BytePlus video requests encode `duration` with the provider-native JSON number
  type.
- Other BytePlus request fields continue to be passed as strings.
- The runtime system prompt now explicitly forbids text-only "I will try"
  responses for video-generation requests; the model must issue the
  `VideoGeneration` call in that response.

## Compatibility

- The internal `ExactMediaGenerationRequest` contract remains unchanged.
- Existing saved media defaults continue to use string values.
- The change prevents model-generated `duration: 5` overrides from failing
  before video generation can reach the provider.
