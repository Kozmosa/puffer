# Lambda Host Arguments

Strict Lambda Skill bridge calls now carry the formal host-call arguments from
the verified catalogue. `LambdaHostCall` admission validates that `args` is a
JSON object with exactly the catalogue parameter names for the selected
`host_tool`. The runtime also checks simple base JSON types for primitive
Lambda types such as `str`, `int`, `real`, `bool`, `unit`, and list forms.

The bridge still separates formal host calls from concrete Puffer tools. A
successful `LambdaHostCall` records the formal args in the pending bridge,
admits only the declared next concrete tool call with the declared concrete
input, and commits the host-call facts at the concrete execution boundary.

Trace metadata now includes `lambda_skill.host_args` on both admitted and
committed bridged calls, so transcripts retain the formal argument payload that
was checked against the catalogue.

Focused verification:

- `cargo test -p puffer-core lambda_host_call_`
- `cargo test -p puffer-core runtime::lambda_gate::tests::gate_validates_formal_host_arguments`
