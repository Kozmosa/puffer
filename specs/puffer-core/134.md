# Verified Skill Side-Turn Fail-Closed Gate

## Design

Slash-invoked verified Lambda Skills no longer fall back to prompt-only side
turns when no host catalogue is available. `/skill:<name>` now requires
`gate_for_verified_skill` to return an active gate for `lambda-skill`
verification metadata.

## Contracts

A verified Lambda Skill command is executable only when Puffer can load a
checked-in host catalogue or compile one through explicit `lskillc`
configuration. Missing catalogues are reported before the provider side turn
starts, so the model cannot execute a verified skill without the runtime gate.

## Verification

`cargo test -p puffer-core lambda_skill_command_rejects_prompt_only_verified_skill`
checks that verified Lambda Skill commands fail closed instead of running
prompt-only.
