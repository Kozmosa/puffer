# Telegram Bot /connect Setup

## Summary

`/connect` now supports the built-in `telegram-bot` connector with the same
deterministic AskUserQuestion flow used by other connector setup paths.

## Behavior

- `/connect telegram-bot telegram-bot` asks for a Telegram bot token through an
  input question instead of falling through to the generic unsupported-auth
  message.
- The flow writes `.puffer/connectors.toml` for the `telegram` serve platform
  with `enabled = true`, `display_name = <connection>`, the supplied `token`,
  `require_mention = true`, and `group_key_policy = "per_user"`.
- The success summary names connector `telegram-bot`, the chosen connection
  name, the config path, and the `puffer serve` next step.

## Compatibility

`telegram-login` remains the personal-account auth path. `telegram-bot` remains
unavailable as a workflow trigger until the serve connector exposes a typed
subscriber manifest or command-backed event stream, but its catalog command is
now executable.
