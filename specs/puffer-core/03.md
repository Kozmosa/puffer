# Session allow-all API

## Purpose

Daemon integrations need a typed way to mark a loaded `AppState` as
session-wide allow-all without reaching into `puffer-core` private permission
state.

## Behavior

- `AppState::set_session_allow_all` is public.
- The method sets the legacy `session_allow_all` flag and the typed
  `SessionPermissionState` allow-all bit together.
- Filesystem sandbox breadth is still controlled independently through
  `AppState::sandbox_mode`.

## Compatibility

Existing in-crate permission prompt flows continue to use the same method.
External callers can now apply allow-all consistently with those flows.
