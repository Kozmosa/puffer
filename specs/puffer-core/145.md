# Lambda Host Argument Refinement Guard

`LambdaHostCall` formal argument validation no longer erases Lambda Skill
refinement predicates down to their base JSON type. The checker now lives in a
separate `runtime::lambda_gate::type_check` module so the gate file remains
below the repository file-size limit.

The runtime validator handles the refinements Puffer can check locally:
numeric bounds such as `real{a > 0}`, integer ranges such as
`int{(s >= 0 && s <= 1000)}`, cross-argument equality or inequality such as
`TokenAddr{to != from}`, and string suffix predicates such as
`str{ends_with_pdf(p)}`. Unsupported refinements fail closed instead of being
silently admitted as plain strings, numbers, or domains.

Lambda Skill readiness reporting now formats the full error chain, so
`/doctor`, `/skills`, and `puffer doctor` show concrete causes such as the
specific formal host tool missing a concrete binding.

Focused verification:

- `cargo test -p puffer-core runtime::lambda_gate::type_check`
- `cargo test -p puffer-core runtime::lambda_gate::tests::gate_validates_formal_host_arguments`
- `cargo test -p puffer-core render_lambda_status_reports_missing_host_tool_binding_detail`
