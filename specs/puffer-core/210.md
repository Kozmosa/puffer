# Streaming No-Tool Turns

- `puffer-core` now exports a streaming no-tool prompt entrypoint alongside the existing blocking no-tool entrypoint.
- The entrypoint uses the same tool suppression contract as `execute_user_prompt_without_tools`, so advisory turns can stream provider requests without exposing mutation-capable tools.
- This is intended for daemon and workflow callers that need transport compatibility with providers requiring `stream: true`.
- The normal chat, structured-output, permission, cancellation, and blocking no-tool APIs keep their existing contracts.
