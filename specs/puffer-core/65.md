# Workflow Connector Query Hints

## Summary

The `/workflows` terminal dashboard now supports focused connector, connection,
and run queries, and connector rows include the deterministic `/connect`
command needed to create a matching connection.

## Behavior

- `/workflows connectors <query>` filters connector templates by slug,
  description, or skill.
- `/workflows connections <query>` filters connection records by connection
  slug, connector slug, or description.
- `/workflows runs <query>` filters runs by workflow slug, run id, or error
  excerpt.
- Connector rows render `connect=/connect <connector> <connection>` using the
  shared connector connection hint.

## Compatibility

The default `/workflows` dashboard remains read-only and still shows workflow,
connection, and event-capable connector summaries. The command does not create
connections or mutate workflow state.
