# RequestSecret Collect

`RequestSecret` supports `action: "collect"` for credentials that are not yet
stored in the encrypted Puffer secret vault.

- The action requires `name`/`label` and may carry `description`, `username`,
  `origin`, `id`, and an optional user-facing `prompt`.
- The runtime asks the user a single `secret: true` input question through the
  existing user-question prompt channel.
- The submitted value is saved to `puffer-secrets`, registered as a
  session-masked secret, and returned only as a `PUFFER_SECRET_...` placeholder
  plus non-secret vault metadata.
- Raw collected values must not appear in the tool result.
- Existing `search`, `request`, and `create` behavior is unchanged.
