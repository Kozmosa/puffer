# Verified Lambda MCP bridge

`McpToolCall` now supports the qualified MCP call shape used by translated
Lambda skills. In addition to the existing explicit `server`/`tool`/`args`
form, the tool accepts `qualifiedToolName`, `argsJson`, and the formal
`registry` proof argument.

The runtime resolves `qualifiedToolName` against the live MCP registry before
execution. It accepts both Puffer's `mcp__server__tool` names and Hermes'
`mcp_server_tool` names, but only when exactly one live `(server, tool)` pair
matches. Missing or ambiguous names fail closed. `argsJson` must parse as JSON
and is passed unchanged to `ToolRunner::call_mcp_tool`.

This lets a Lambda catalogue bind `native-mcp.call_mcp_tool` without weakening
the gate into a generic shell or HTTP bridge. The temporal bridge still admits
the formal host call first, and concrete execution must carry the exact
qualified tool name, argument JSON, and registry proof emitted by that host
call.
