# Telegram QR question body

Telegram QR `/connect` approval now embeds the generated QR markdown in the
question body instead of on the Approved option preview. The approval choices
remain plain actions (`Approved` and `Cancel`), which keeps the QR visually
attached to the prompt text and avoids hiding the login artifact inside option
metadata.

The question body still includes the raw `tg://login` URL below the generated
SVG data URI so users can inspect or copy the login link when image rendering is
not available.
