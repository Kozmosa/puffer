# Workflow Slash Summary

## Summary

The TUI command surface now includes a read-only `/workflows` command, with
`/workflow`, `/pipelines`, and `/pipeline` aliases, for inspecting workflow and
connector state without leaving the terminal.

## Behavior

- `/workflows` renders a dashboard with workspace path, workflow count, run
  count, configured connection count, and connector template count.
- Registered native workflows are listed with enabled/paused state, trigger
  label, node count, and latest run status when one exists.
- Connection records are listed with connector slug, lifecycle state, consumer
  state, and description.
- Event-capable connector templates are listed by default; `/workflows
  connectors` includes every connector template.
- `/workflows connections` and `/workflows runs` show focused slices of the
  same state.
- If the subscription runtime is not installed, the command still shows
  workflow definitions and built-in connector templates, plus a single
  diagnostic line for the connector runtime.

## Compatibility

This is a local slash command only. It does not mutate workflow state, does not
start providers, and does not change the existing `/tasks`, `/connect`, CLI
`puffer workflow`, or desktop daemon workflow contracts.
