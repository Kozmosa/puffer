# Workflow Connector Capability Search

## Summary

The `/workflows connectors` command now searches connector action names and
prints a concise action summary for action-capable connectors.

## Behavior

- Connector queries are split into whitespace terms.
- Every query term must match somewhere in the connector slug, description,
  skill, or action slug metadata.
- Action-capable connector rows include `actions=` followed by up to three
  action slugs.
- If the search terms match one or more action slugs directly, those matching
  action slugs are prioritized in the `actions=` summary.
- Rows with more than three actions append `,+N` to indicate hidden actions.
- Connector setup commands keep the existing deterministic
  `/connect <connector-slug> <connection-name>` shape.

## Compatibility

Single-term connector searches keep substring behavior. Existing command output
still includes the capability list and connect command, with the new `actions=`
field inserted before `connect=` when actions are available.
