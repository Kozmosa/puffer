# Telegram QR approval retry

The deterministic `/connect telegram-login` QR path now checks QR approval with
a short bounded wait after the user clicks `Approved`. If Telegram has not
actually approved the login yet, the subscriber returns another `qr_pending`
event and the setup question is shown again instead of leaving the UI in a long
blocked state.

After three unapproved checks, `/connect` reports a clear failure and asks the
user to restart setup. This prevents a pending QR status from being summarized
as a successful connector configuration.
