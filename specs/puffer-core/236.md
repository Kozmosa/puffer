# Global user memory: agent-writable user.md + daemon RPC CRUD

## Summary

Adds a **global, agent-writable user-facts store**: keyed markdown blocks in
`~/.puffer/user.md` (home address, tool/service preferences, timezone, etc.).
The agent records facts with the new `Remember` tool; the daemon exposes CRUD
RPCs for external/UI management; the blocks are injected into every system
prompt. Complements (does not replace) the per-project `MEMORY.md` store in
[`crate::memory`] — that is project-scoped and an opaque entry list; this is
global and block-structured.

## Store (`crate::user_memory`)

- File: `~/.puffer/user.md` (honors `PUFFER_HOME`), format `## <key>\n\n<body>`.
- `UserMemory`: `list` / `get` / `set` (upsert) / `append` / `delete`. Reads are
  lock-free; writes take an exclusive lock and rewrite atomically.
- Keys are normalized to lowercase dash-slugs (`Home Address` -> `home-address`)
  so the same fact is addressed consistently.
- Reuses `memory::FileLockGuard` (atomic cross-process lock) and
  `memory::validate_content` (prompt-injection / exfiltration guards). Caps:
  `MAX_BLOCKS = 200`, `MAX_BODY_CHARS = 4000` (the file rides in every prompt).

## Activation (read) — `runtime::system_prompt`

`load_user_prompt(cwd)` loads `user.md` (project + `~/.claude` + `~/.puffer`,
via the shared `load_context_blocks`) wrapped in `USER_INSTRUCTION_PROMPT`, and
`render_runtime_system_prompt` injects it in the order **soul (identity) -> user
(facts) -> project memory (operating)**. Universal across providers.

## Write paths

- **Agent tool `Remember`** (`resources/tools/remember.yaml`, handler
  `runtime:workflow:remember`, `runtime::claude_tools::workflow::remember`).
  Actions `set` / `append` / `forget` / `list`; `{ action, key?, value? }`.
  Auto-approved (silent write); the call is recorded in the session transcript,
  so every change is **auditable + reversible**. Operating/project facts still
  belong in the project `Memory` tool; `Remember` is for durable *user* facts.
- **Daemon RPC** (`puffer-cli` daemon): `user_memory_list`, `user_memory_get`,
  `user_memory_set`, `user_memory_delete` (over the existing NDJSON dispatch),
  for the desktop UI / external CRUD of the same store.

## Contracts

- `set` requires `{key, value}`; `forget`/`get` require `{key}`; `list` takes
  none. Empty key/value and over-cap/injection content are rejected.
- Missing `user.md` -> empty list / no injection (no behavior change for users
  without one).

## Tests

- `user_memory`: set/get/list/delete round-trip (+ key normalization, upsert
  dedupe), append create-then-extend, on-disk markdown round-trip, injection +
  oversize + empty-key rejection.
- `system_prompt`: `load_user_prompt_loads_user_facts`,
  `render_runtime_system_prompt_orders_soul_user_memory`.
