# BytePlus Response Format Request Contract

## Summary

Exact image generation keeps BytePlus response-format defaults while pruning
stale model-specific parameters.

## Runtime Contract

For BytePlus Seedream requests, the capability descriptor controls which saved
parameters reach the HTTP body and artifact metadata:

- undeclared `output_format` values are pruned for Seedream 4.5 and 4.0;
- missing declared values are filled from the current model defaults;
- declared `response_format=b64_json` remains in the request body;
- output path extension, MIME type, and metadata use the actual resolved
  request parameters;
- the Images JSON adapter continues to decode `b64_json` directly and only
  downloads URLs when the provider returns `url`.

## Compatibility

This favors stable direct image bytes over URL fetches. It does not introduce a
typed parameter model; boolean/object BytePlus options remain out of scope until
media parameters support non-string values.
