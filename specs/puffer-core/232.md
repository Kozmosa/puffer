# Monitor Batch Prompt Contract

## Summary

Monitor triage instructions now describe the runtime as a persistent
per-connection agent that may receive a batch of connector triggers in one turn.

## Design

- The monitor prompt tells the agent to handle every trigger in the current
  turn and to create or update multiple tasks when multiple messages are
  actionable.
- The prompt continues requiring monitor task metadata so downstream task views
  can relate tasks back to connector events.
- The prompt clarifies that `receivedAt` belongs to each individual trigger,
  which matters when batched messages arrive at different times.

## Compatibility

Only the monitor instruction text changes. Existing task tools, metadata keys,
and monitor configuration remain compatible.
