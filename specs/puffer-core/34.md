# Automatic Permission Review Model Selection

## Scope

Puffer now routes permission prompts through a generic automatic reviewer before
falling back to Browser-specific review or the human prompt.

## Design

- The reviewer runs as a locked-down side turn with no model-visible tools,
  no reflection, no project memory side effect, one model round trip, and a
  90-second timeout.
- The side prompt receives a structured permission request containing the tool
  id, prompt summary, reason, raw tool input, cwd, workspace roots, Browser
  context when present, and a compact recent transcript.
- The required output is strict JSON with `decision`, `risk`, and `rationale`.
  Supported decisions are `allow_once`, `allow_session`, `deny`, `needs_user`,
  and `unavailable`.

## Model Selection

- OpenAI sessions first try `openai/codex-auto-review`, matching Codex's
  guardian reviewer preference. The model is inserted only into the cloned
  side-turn provider registry, so it is not exposed as a normal selectable
  model.
- Anthropic sessions first try `claude-haiku-4-5-20251001`.
- If the preferred reviewer is missing or the side turn fails, Puffer retries
  the same review with the current turn's provider/model.
- Reasoning-capable reviewer attempts use `low` effort, matching Codex's
  preference for a cheap reviewer model.

## Permission Effects

- `allow_once` permits only the current tool call or filesystem path access.
- `allow_session` applies the same session grant and ACL persistence as a human
  Allow Session response.
- `deny` blocks the tool call with the existing permission-denied tool result.
- `needs_user` and `unavailable` preserve the previous fallback behavior: Browser
  asks can still use the Browser-specific reviewer, and all remaining asks go to
  the human prompt.

## Compatibility

- Browser-specific auto-review remains available as a fallback so existing
  Browser grant scoping and tests keep working when the generic reviewer is not
  configured or returns `unavailable`.
- Filesystem path prompts outside the workspace now use the same automatic
  review path before prompting the user.
