# Lambda Skill Config-Only Gates

Verified Lambda Skill gates now resolve strict host catalogues only from
configuration carried by the loaded skill metadata:

- `host_catalogue_path` loads a prebuilt host catalogue.
- `compiler_path` compiles the formal `.lskill` source with that configured
  compiler.

Runtime gate construction no longer consults a compile-mode environment flag,
`PUFFER_LSKILLC`, or `lskillc` on `PATH`. Missing catalogue and compiler
configuration leaves the gate absent, and `/skill:<name>` fails closed before a
provider turn starts.

Doctor output reports manifest compilers and skills missing gate config instead
of environment-gate status. Warnings direct users back to the
`lambda_skill_libraries` manifest.

## Validation

- `cargo test -p puffer-core runtime::lambda_gate`
- `cargo test -p puffer-core command_helpers::lambda_doctor::tests::render_lambda_status_reports_missing_gate_config`
- `cargo test -p puffer-core lambda_skill_command_rejects_prompt_only_verified_skill`
