# Resume Picker Pending Prompts

## Design

Interactive resume selection can start from a temporary CLI session when the
user runs `puffer --resume <query> <prompt>` and the query opens the startup
picker. The prompt is stored as an `AppState` pending query while the picker is
visible.

## Logic

When `/resume <session-id>` loads the chosen session, `resume_into_session`
replaces the temporary `AppState` with state reconstructed from the selected
session record. The pending query prompt must be moved across this replacement
so the TUI can submit it after startup overlays close.

## Contract

- Loading the selected session remains append-only and uses the session store
  record as before.
- A pending startup prompt survives same-project resume selection.
- If the selected session cannot be loaded, the existing state and queued
  prompt are left unchanged by the failed load.
