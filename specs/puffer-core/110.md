# Deterministic `/connect` Default Connection Names

## Design

`/connect` keeps the searchable connector chooser, but selecting or typing only a
connector now uses that connector's suggested connection slug automatically. A
custom connection name is still available through the explicit
`/connect <connector-slug> <connection-name>` form.

## Contract

- `/connect` asks one searchable connector question, then proceeds with
  `suggested_connection_slug(connector_slug)`.
- `/connect <connector-slug>` resolves exact or searchable connector terms and
  uses the same deterministic suggested connection slug.
- `/connect <connector-slug> <connection-name>` preserves the caller-provided
  connection name.
- Extra positional words are rejected with the `/connect` usage shape instead of
  opening another prompt.

## Compatibility

The auth questions for connector-specific credentials are unchanged. Existing
two-argument `/connect` commands remain deterministic, while picker-driven setup
now matches the Corbina connector picker behavior.
