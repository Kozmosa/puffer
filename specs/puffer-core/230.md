# RequestSecret Runtime Tool

The runtime now supports `RequestSecret`, a model-facing workflow tool that
returns masked placeholders instead of raw secret values.

- `RequestSecret` resolves one encrypted secret by exact id or unique label from
  the configured `puffer-secrets` vault.
- The permission prompt offers one-secret, session-scoped, and all-secrets
  session approvals. Approvals are kept in `AppState::secret_access_state`.
- Successful requests register the raw value in the session masked-secret map and
  return only `PUFFER_SECRET_<uuid>` plus non-secret id/label metadata.
- Tool inputs are expanded from placeholders to raw values immediately before
  concrete tool execution. The `Agent` tool is excluded so masked values are not
  forwarded to another model.
- Tool stdout/stderr and provider tool-result payloads are redacted back to the
  registered placeholder before model/UI delivery.
