# puffer-core Update 248: Prune Stale Image Parameters

## Summary

Exact image generation now prunes saved parameter names that the current
selected capability no longer declares before dispatching a provider request.

## Runtime Contract

The runtime resolves the current exact image capability from:

```text
provider_id + model_id + adapter
```

It then drops only undeclared saved parameter names. Declared parameter names
still keep existing value validation, so invalid values continue to fail before
network dispatch.

## Compatibility

This lets older BytePlus Seedream 4.5 settings that still contain
`output_format=png` generate successfully after the catalog correction. The
actual HTTP request preserves `size=2K` and omits `output_format`.
