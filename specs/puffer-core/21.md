# Parallel test config isolation

- Core tests that previously changed the process-wide `PUFFER_HOME` now use the thread-local `puffer-config` override.
- Command, prompt, session, workflow, and agent-runtime tests keep their per-test home directories without exposing those directories to unrelated concurrent tests.
- The change preserves the production command and runtime paths because only tests opt into the override guard.
- `puffer-core --lib` is expected to run under the default parallel test harness instead of requiring `--test-threads=1`.
