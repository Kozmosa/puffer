# Reject Model-Invoked Verified Lambda Skills

## Design

The model-visible `Skill` tool no longer expands verified Lambda Skills. That
tool returns prompt text only and has no access to the session `AppState`, so it
cannot install the Lambda gate required for guarded execution.

Runtime system prompt generation now omits verified Lambda Skills from the
model-invocable skill list even if a resource accidentally marks them as model
invocable.

## Contracts

Verified Lambda Skills are invoked through `/skill:<name>` only. That path can
install the Lambda gate before the provider side turn starts; the `Skill` tool
cannot.

## Verification

`cargo test -p puffer-core rejects_verified_lambda_skill_model_invocation`
checks the `Skill` tool rejection.

`cargo test -p puffer-core runtime_system_prompt_lists_model_invocable_skills`
checks that verified Lambda Skills are not advertised as model-invocable.
