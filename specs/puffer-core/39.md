# puffer-core update 39

Email and Telegram internal CLI calls now execute inside the parent agent
runtime over the Bash internal-tool bridge.

The Bash broker accepts both permission and execution envelopes. Permission
envelopes keep the Browser path unchanged. Execution envelopes dispatch through
`internal_tool_permissions::execute_internal_tool_request`, which first resolves
the normal internal tool permission policy, then runs the consolidated workflow
tool (`Email` or `Telegram`) in the parent process.

This parent-process dispatch is required for Telegram: `login-start` stores the
Telegram login token in the live `telegram-user` subscriber process, and
`login-submit-code` / `login-submit-password` must reach that same supervised
process. Running those steps in short-lived CLI-local subscription managers
would lose the token between commands.

Permission prompts redact email passwords, Telegram login codes, and Telegram
2FA passwords before building summaries. The actual execution payload remains
available only inside the parent runtime so subscribers receive the required
secret values.
