# Workflow Trigger Draft Pattern UX

## Scope

Improves `/workflows new` so terminal-created workflow drafts can be aimed at a
specific connection and initial event-text regex in one deterministic command.

## Design

- `/workflows new [slug] [connection-slug] [pattern]` parses quoted arguments
  with shell-style quoting, so patterns containing spaces can be supplied as one
  argument.
- Explicit connection triggers are resolved against configured connections and
  connector trigger support before a draft is written.
- Suggested built-in connection names remain usable as planned connections when
  the matching connector can start workflow triggers; the command output prints
  the required `/connect <connector-slug> <connection-name>` repair step.
- Blank or `*` trigger patterns normalize to `.*`, matching the existing native
  workflow default and avoiding invalid wildcard-shaped regexes.
- Workflow dashboard labels now report subscription triggers as
  `subscription:<topic>` instead of `connection:<topic>`.

## Compatibility

Existing two-argument `/workflows new <slug> <connection>` commands keep
creating disabled native workflow drafts. Custom unconfigured connection names
now fail with a concrete `/connect` hint instead of silently creating a draft
that cannot run.
