# Media Internal Helper Execution Plan

The runtime fix is to rely on the existing Bash helper and internal permission
broker path, not to introduce another media execution route.

## Runtime Contract

- Bash command execution prepends internal helper functions generated from the
  current executable path.
- `imagegen` expands to `current_exe internal-tool image-generation "$@"`.
- `videogen` expands to `current_exe internal-tool video-generation "$@"`.
- Child internal CLI commands use the internal permission environment variables
  to call back into the parent runtime.
- Parent-side internal execution maps image and video requests directly to the
  existing image/video workflow implementations with media context.

## Gaps Found

- The failed session used the raw `puffer internal-tool ...` command from the
  skill text, bypassing the helper alias contract.
- Preview tests still cover raw `puffer internal-tool ...` commands. Those tests
  should no longer define the model protocol.
- The internal CLI commands must remain available because helper aliases execute
  through them.

## Implementation Plan

1. Update image and video skill text to use only `imagegen` and `videogen`.
2. Update resource skill tests to reject `puffer internal-tool` in media skill
   bodies.
3. Add or adjust a Bash helper regression test proving media helpers are
   injected from `current_exe` and do not require `puffer` on `PATH`.
4. Keep internal registry visibility tests unchanged except for any wording
   needed around helper-first skills.
5. Update generated media preview tests so the canonical command examples are
   `imagegen` and `videogen`.
6. Keep existing workflow execution tests for image and video config usage.
7. Run focused tests for `puffer-resources`, `puffer-tools`, `puffer-core`, and
   `puffer-cli` media internal CLI parsing.

## Overdesign Guardrails

- No new dispatcher.
- No new media runtime abstraction.
- No broad shell alias for `puffer`.
- No attempt to make arbitrary internal tools model-facing.
- No cleanup of unrelated internal tools.
