# LSP Shutdown Timeout

## Purpose

Keep cached LSP session shutdown from blocking user-facing flows when a
language server ignores the graceful `shutdown` request.

## Design

Normal LSP operations still use the existing 45 second request timeout.
Shutdown now sends the same JSON-RPC `shutdown` request with a dedicated
one second timeout, then continues to send `exit` and reap the child
process.

## Logic

Some servers and test doubles do not respond to `shutdown`, especially when
they only implement the subset of requests used by interactive tools. Using
the normal operation timeout made session cleanup appear hung. The short
shutdown timeout preserves the graceful path for compliant servers without
letting cleanup stall the app.

## Contracts

- Successful `shutdown` responses are still handled before `exit`.
- Missing or delayed `shutdown` responses are ignored during cleanup.
- Request timeouts for hover, definition, references, symbols, hierarchy,
  and diagnostics are unchanged.

## Compatibility

This keeps LSP protocol ordering intact while making `/clear`, process
shutdown, and cached-session resets responsive when a server does not
complete graceful shutdown.
