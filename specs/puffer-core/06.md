# Branch Session Test Isolation

## Purpose

Keep `/branch` command tests independent from the developer's real Puffer
session store so repeated local runs produce stable branch display names.

## Design

The affected `session_local` tests now acquire the shared `PUFFER_HOME`
environment lock before calling `ConfigPaths::discover`, create a temp
home directory under the test workspace, and install it with
`ScopedPufferHome`. This matches the existing isolation pattern used by
other command tests that touch user-scoped config or sessions.

## Logic

`/branch` intentionally calls the session store to choose a unique
`"<name> (Branch)"` display name. Without a temp `PUFFER_HOME`,
`SessionStore::from_paths` points at the real user session directory, so
old manual or failed test sessions can force suffixes like
`"(Branch 4)"`. The tests now build the store from a per-test user home,
leaving the command's collision behavior unchanged while making the
expected first branch name deterministic.

## Contracts

- Production `/branch` naming remains unchanged and still scans the active
  user session store for collisions.
- Tests that mutate `PUFFER_HOME` hold the process-wide lock, preventing
  concurrent test races over the environment variable.
- The empty-conversation refusal test uses the same isolation even though
  it does not create a branch, because it still constructs a user-scoped
  session store.

## Compatibility

No command output, session metadata format, or persistence location changes.
This is a test-only isolation fix for `puffer-core`.
