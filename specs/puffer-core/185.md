# Computer Use Drag Bridge

## Summary

`ComputerUseAction` supports a `drag` action for verified Lambda Skill contracts
that name source and destination elements from the current macOS capture.

## Runtime Contract

- `capture`, `focusApp` with `raiseWindow=true`, and `clickElement` with
  `captureAfter=true` persist the active app plus element bounds parsed from
  the computer-use MCP capture text.
- `drag` requires `fromElement` and `toElement`. It fails closed unless both
  element indices have captured bounds for the active app.
- When both elements are known, Puffer calls the concrete MCP `drag` tool with
  the active app and the centers of both captured bounds:
  `from_x`, `from_y`, `to_x`, and `to_y`.

## Compatibility

Existing app-targeted actions preserve the current app binding. Actions that do
not produce a capture keep any existing bounds only when they continue targeting
the same app, so app changes cannot reuse stale element geometry.
