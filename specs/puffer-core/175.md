# Lambda Skill MCP Tool-Name Refinement

## Summary

`puffer-core` treats `tool_names_normalized(...)` as an executable Lambda
semantic predicate. The predicate accepts normalized MCP tool names in either
Puffer form (`mcp__server__tool`) or Hermes form (`mcp_server_tool`) and
rejects empty, control-character, dotted, or otherwise unsanitized names.

## Contracts

- The predicate is checked during formal host-call admission, before the
  pending concrete tool bridge is created.
- The check only validates normalized tool-name syntax. It does not decide
  which live MCP server/tool is targeted.
- Concrete dispatch remains owned by `McpToolCall`, which resolves the admitted
  normalized name against the live MCP registry and fails closed if it is
  missing or ambiguous.

## Compatibility

Existing custom fact refinements still work unchanged. This adds a runtime
semantic interpretation for the Lambda MCP tool-name predicate so translated
native-MCP skills can pass admission and still rely on the concrete bridge for
exact registry binding.
