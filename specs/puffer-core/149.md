# Lambda Result-Backed Facts

Verified Lambda host catalogues can satisfy custom fact refinements from two
trusted runtime sources:

- `registers` entries are instantiated from formal host arguments when a
  register slot names a formal parameter, and from the concrete tool's result
  witness when it names any result slot.
- Positive custom refinements on the host tool result type are committed as
  facts for the concrete tool's result witness.

The result witness is derived only after the admitted concrete Puffer tool
successfully runs. Puffer parses normalized stdout as JSON when possible and
otherwise uses normalized stdout as a string. The model cannot provide this
witness through `LambdaHostCall`, so downstream refined parameters only pass
when they match facts produced by an earlier successful concrete execution.

Readiness validation now treats result-backed registers and result refinements
as dynamic fact producers. Unsupported refinement syntax still fails closed.
