# Manifest Env Entry Re-Export

## Summary

`puffer-subscriber-runtime` now re-exports `EnvEntry` from its manifest schema
so host crates can safely add environment variables to loaded manifests.

## Design

- The existing public `manifest::EnvEntry` type is exported from the crate root.
- No manifest serialization or supervisor behavior changes.

## Compatibility

This is additive API surface for existing callers. Existing manifests, state
directories, and subscriber process launches are unchanged unless a host crate
mutates `Manifest.spec.run.env` before starting the subscriber.
