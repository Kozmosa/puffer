# Telegram list-message sender filter command fields

## Scope

`SubscriberCommand::TelegramListMessages` now carries optional sender-filtering
and bounded-scan metadata.

## Contract

- `sender` is optional and accepts JSON aliases `from` and `from_user`.
- `scan_limit` is optional and lets callers bound sender-filtered history
  scanning separately from the number of messages returned.
- Existing `peer`, `limit`, `before_id`, and `succinct` fields retain their
  meaning.

## Compatibility

Older callers that omit `sender` and `scan_limit` serialize and deserialize as
before. Subscribers that ignore unknown command variants are unaffected.
