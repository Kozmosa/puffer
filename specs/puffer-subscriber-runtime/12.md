# Command Sender Restart Generation

`CommandSender` now tracks a monotonically increasing stdin generation. The
generation increments whenever the supervisor replaces the child stdin handle,
including disconnects during process exit and reconnects after restart.

This keeps the runtime protocol unchanged: subscribers still receive one JSON
command per line and emit events on stdout. The generation is an in-process
supervision signal for callers that need to detect whether an in-flight command
may have been lost across a child restart.
