# Tool-Only Empty Assistant Guard

## Problem

Provider turns that terminate after tool execution can legitimately return an
empty final assistant text. The TUI finalized that empty string as a normal
assistant row and persisted an empty `AssistantMessage`, leaving blank history
items after tool-only turns.

## Design

- Treat whitespace-only final assistant text as absent output.
- Preserve tool invocation transcript entries and session snapshots.
- Remove a trailing blank assistant placeholder when finalization discovers
  that the turn produced no visible assistant text.
- Reuse the same finalization path for synchronous and background TUI turns.

## Test Coverage

- `flow_tests.rs` now simulates a successful tool-only terminated turn with an
  empty assistant text and verifies the tool invocation is persisted while no
  empty assistant row or `AssistantMessage` is written.
