# Failed Turn Assistant Draft Cleanup

## Problem

Streaming provider output is appended to the live TUI transcript before the
turn completes. If the provider turn later fails, that partial assistant draft
remained visible in the current UI but was never persisted, so reloading the
same session produced a different transcript shape.

## Design

- Record the transcript index immediately after the user prompt that starts a
  pending provider turn.
- When that pending turn finishes with an error, remove assistant messages
  created after that index before appending the provider-failure system row.
- Keep tool invocation rows and session snapshots intact.
- Leave successful-turn finalization unchanged.

## Test Coverage

- `tests/pending_submit.rs` now simulates a streamed text delta followed by a
  failed pending turn and verifies the partial assistant draft is removed from
  the live transcript and never written as an `AssistantMessage`.
