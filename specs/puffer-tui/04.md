# puffer-tui Slash Popup Ordering Test

## Purpose

Slash-command popup tests should validate the popup ordering contract without
depending on the exact current command registry.

## Design

The popup implementation is unchanged. The focused prefix-ordering test now
uses a local command fixture that includes prefix matches and contains-only
matches.

## Logic

- Hidden commands are still filtered before matching.
- Exact and prefix matches still sort ahead of contains-only matches.
- Broad empty queries still truncate to eight rows.
- The prefix-ordering test asserts the deterministic ordering of a small
  fixture instead of pinning the first commands returned by the production
  registry.

## Contracts

Adding new supported slash commands must not break popup ordering tests unless
the matching or sort contract itself changes.
