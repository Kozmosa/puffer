# Monitor Action Submission Labels

## Summary

Monitor task actions keep using the full generated action prompt for provider
execution, but the TUI records a concise user-facing transcript label when one
of those prompts is submitted from the background task picker.

## Design

- Provider execution still receives the complete monitor action prompt emitted
  by `puffer-core`.
- Transcript user messages and session events use
  `Act on monitored task <task-id>: <action-name>` when the submitted text
  matches the monitor action prompt shape.
- Non-monitor prompts and manually typed text are unchanged.

## Compatibility

This preserves monitor action behavior while avoiding internal task
instructions appearing as the visible user message. The parser is intentionally
shape-specific so ordinary prompts beginning with similar words are not
rewritten.
