# 46. Fast Picker Legacy Model Selectors

## Summary

The `/fast` TUI picker now works for resumed sessions whose current model is
stored as an unscoped legacy selector such as `gpt-5`.

## Design

- Scoped model selectors such as `openai/gpt-5` keep their existing behavior.
- Unscoped selectors are resolved against `current_provider`.
- The picker opens only when the selected provider advertises the unscoped
  model, matching initial provider setup validation.

## Compatibility

Older session snapshots can now open the fast-mode picker without forcing a
model re-selection first. Unknown unscoped model ids still fall through instead
of showing a picker for a model the provider does not expose.

## Tests

`puffer-tui` includes a regression test that sets `current_provider=openai` and
`current_model=gpt-5`, opens `/fast`, and asserts that the fast-mode picker is
shown for `openai/gpt-5`.
