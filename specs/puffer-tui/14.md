# Keep read-only overlays from accepting hidden typing

## Purpose

Scrollable inspection overlays such as `/status`, `/session`, `/config`, and
`/skills` should behave like read-only panels. Printable keys that are not
panel shortcuts must not accumulate in the hidden composer query or block the
next slash command.

## Design

Overlay state now distinguishes text-entry prompts, filterable picker overlays,
and read-only overlays. The key router only forwards printable typing to
filterable pickers, while preserving `/` as an escape hatch that closes the
overlay and starts a new command.

## Logic

- Picker overlays still show and accept `Type to jump` filtering.
- Read-only overlays show a close/command hint instead of a type-to-jump hint.
- Printable non-shortcut keys are swallowed by read-only overlays.
- `/` still closes a read-only overlay and seeds the command composer.

## Contracts

No command dispatch or storage contract changes. This is an interactive TUI
keyboard-routing and prompt-copy fix.
