# Monitor Connector Argument Popup

TUI slash completion now exposes connector-backed monitor commands directly
from `/monitor <query>`.

## Behavior

- `/monitor <query>` suggests trigger-capable connector templates and completes
  to `/monitor <connection>`, using the deterministic suggested connection slug
  from the connector catalog.
- Monitor connector searches match connector slug, suggested connection slug,
  description, skill, binary, monitor/task terms, workflow/event terms, and
  connector action slugs.
- Setup-only connectors are omitted from monitor argument suggestions, while
  remaining visible through `/connect` and `/workflows connectors`.
- Exact monitor commands such as `/monitor telegram-user` continue to submit on
  Enter instead of being rewritten by completion.
- Bare `/monitor` runs on the TUI background command path so its
  `AskUserQuestion` searchable connection picker renders as an overlay instead
  of returning a pending-question failure.

## Compatibility

This is a TUI completion and rendering change only. It does not add connector
templates, alter connector auth, change monitor creation, change workflow
storage, or introduce webhook behavior. The generated command shape and
background command path reuse the existing `/monitor <connection>` terminal
contract.
