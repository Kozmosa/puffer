# Cancelled Pending Submit Persistence

## Scope

The TUI cancel path now drains currently queued pending-submit events before it
records the interrupt message.

## Contract

- Streamed assistant text is persisted as assistant transcript history.
- Completed tool invocation events are persisted before cancellation.
- Requested tool calls that have not produced a tool invocation are recorded as
  failed tool results with `Interrupted by user.` output.
- Pending permission and user-question prompts are denied or answered empty so
  worker-side channels are released during cancellation.

## Compatibility

The existing queued-prompt behavior is unchanged: cancelling the active turn
still records the interrupt and allows the next queued prompt to start.
