# Permission Prompt Guardrails

## Scope

The TUI no longer shows a normal approval option that grants all project
actions.

## Design

Generic permission prompts show only:

- `Approve once`
- `Always allow this request`
- `Deny`

Browser prompts keep `Always allow this browser context` as the durable bounded
option. The `AllowAllSession` runtime action remains understood internally for
compatibility, but it is not offered in the standard approval overlay.

## Copy

Bash permission reasons describe the effective shell program as a "shell
command" rather than using the implementation term `argv`.

## Verification

- Permission prompt render tests assert that the broad all-actions label is not
  present.
