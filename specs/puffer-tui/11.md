# Usage overlay keyboard controls

## Purpose

The `/usage` overlay advertises keyboard actions and can contain scrollable
provider data. Those keys must operate on the usage overlay instead of editing
the composer prompt behind it.

## Design

Usage is handled as a modal overlay with its own retry and scroll controls.
The generic overlay key path now gives usage overlays first chance to handle
`r`, and shared overlay navigation routes arrow and page movement into the
usage scroll state.

## Logic

- `r` and `R` call `UsageOverlay::retry()` while usage is open.
- Up and Down scroll the usage body by one row.
- PageUp and PageDown scroll the usage body by one page.
- Character handling no longer inserts `r` into the hidden composer while
  retry is requested.

## Contracts

Retry remains a no-op for static usage overlays that have no live load request.
Scrollable usage state remains inside the overlay and does not affect transcript
scrolling.
