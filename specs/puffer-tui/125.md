# puffer-tui update 125

TUI flow persistence preserves tool invocation metadata.

## Design

The TUI completed-turn persistence path writes non-null `ToolInvocation`
metadata into session tool invocation events.

## Contracts

- Rendering continues to use only tool id, input, output, success, actor, and
  subject.
- Metadata remains optional and is omitted for ordinary null metadata.
- Existing flow tests and transcript ordering semantics are unchanged.

## Compatibility

Interactive sessions now keep Lambda Skill trace annotations in storage without
changing visible TUI behavior.
