# Prompt overlay draft preservation

## Goal

Permission and user-question overlays can interrupt while a user is already
drafting their next message. Resolving those transient prompts must unblock the
running provider turn without discarding the composer draft.

## Design

Prompt responses now close the active permission or question overlay through a
dedicated path that preserves composer state. The shared overlay replacement
helper still clears the slash/filter input for command-driven overlays, but
provider-interrupt overlays only clear the overlay and reset selection state.

## Contracts

- Answering a permission prompt preserves `input`, `cursor`, and pending paste
  placeholders.
- Answering an `AskUserQuestion` prompt preserves `input`, `cursor`, and
  pending paste placeholders.
- Pending response channels are still resolved exactly once before the overlay
  disappears.
- Command and picker overlays keep using the existing query-clearing behavior.
