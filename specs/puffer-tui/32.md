# Non-Interactive Startup Prompt Output

## Context

When stdin or stdout is not a terminal, `run_app` accepts a non-empty startup
prompt, submits it, and exits. The submission mutates the transcript and session
log, but previously wrote nothing to stdout, so commands such as
`puffer --no-alt-screen "/status"` appeared to do nothing.

## Change

- Capture the transcript length before non-interactive startup submission.
- After `handle_submit` succeeds, print newly produced non-user transcript
  messages to stdout.
- Preserve session logging and snapshot behavior unchanged.

## Compatibility

Interactive TUI rendering is unchanged. Non-interactive output skips the echoed
user prompt and prints system or assistant text, matching the messages the user
would otherwise see in the transcript.

## Verification

- Added a CLI regression that runs `puffer --no-alt-screen "/status"` with
  piped stdio and asserts stdout contains the status summary and provider line.
