# Auth-only login picker

The `/login` picker is an authentication flow, not provider onboarding.

Compatibility contract:

- Selecting a provider from `/login` opens an auth picker with onboarding set
  to false.
- Auth-only `/login` actions do not change the active provider, active model,
  or persisted default provider/model.
- Saving an API key from an auth-only login prompt closes the overlay when the
  current session already has a valid provider/model pair.
- Onboarding provider selection still persists the chosen provider and proceeds
  to model setup.

Regression coverage lives in `crates/puffer-tui/src/tests.rs`.
