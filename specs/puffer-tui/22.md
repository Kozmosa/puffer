# TUI pending-turn tasks overlay

The TUI treats read-only `/tasks` views like other informational overlays while
a provider turn is running.

Compatibility contract:

- `/tasks` opens the Background Tasks picker immediately during a pending turn
  instead of entering the queued prompt list.
- Read-only task subcommands such as `show`, `list`, `path`, `agents`,
  `teams`, `worktrees`, `todos`, `get`, and `output` remain eligible for
  immediate overlay rendering.
- Mutating task commands, especially `/tasks stop <task-id>`, still queue until
  the active provider turn settles.

Regression coverage lives in `crates/puffer-tui/src/tests/panels.rs`.
