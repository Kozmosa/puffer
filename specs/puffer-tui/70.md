# Searchable question match status

The TUI `AskUserQuestion` overlay now reports searchable-choice match counts and
query-specific empty results while the user filters connector-style prompts.

## UX

- Searchable prompts show the total option count before the user types.
- Filtered prompts show `N/T matches`, or `1/T match`, in the footer next to the
  search controls.
- Empty filtered prompts show the exact query that produced no matches and still
  refuse submission until a valid option is selected.

This only changes TUI pending-question rendering and selection feedback. It does
not change connector auth, workflow storage, command execution, Corbina, or
webhook behavior.
