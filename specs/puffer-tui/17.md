# Defer mutating input during active turns

## Purpose

Users should be able to keep typing while a provider turn is running, but
state-changing commands must not race the in-flight turn. Commands such as
`/clear` should wait behind the active turn instead of mutating the transcript
while pending output can still arrive.

## Design

Prompt submission now classifies input before command dispatch whenever a turn
is pending. Provider prompts, shell shortcuts, and mutating slash commands are
queued. Local read-only slash commands without arguments remain available for
inspection during the active turn.

## Logic

- Free-text prompts continue to queue behind the pending provider turn.
- Shell shortcuts such as `!pwd` queue instead of executing immediately.
- Mutating slash commands such as `/clear` queue behind the active turn.
- Read-only no-argument commands such as `/status`, `/diff`, `/doctor`,
  `/session`, and `/remote` still dispatch immediately.

## Contracts

No transcript schema, provider API, or slash-command registry contract changes.
The change only narrows when prompt submission is allowed to execute
state-changing work while another turn is still running.
