# TUI Test Puffer Home Isolation

## Purpose

TUI tests that exercise config persistence must not depend on the caller's
real `PUFFER_HOME` or on another test's temporary home directory.

## Design

The TUI test crate owns a small `#[cfg(test)]` `ScopedPufferHome` guard. Tests
that need config persistence install a unique, stable temporary `PUFFER_HOME`
for the duration of the test and restore the previous value on drop.

## Logic

- The guard serializes `PUFFER_HOME` mutation through one crate-wide lock.
- Generated home directories live under the system temp directory and are not
  tied to a `TempDir` that may be deleted while another parallel test still
  holds a discovered path.
- Model-selection tests use the guard because their Enter path persists the
  chosen provider, model, effort, and fast-mode settings.

## Contracts

Default parallel `cargo test -p puffer-tui` must not fail because a config path
was discovered from a concurrently restored or deleted `PUFFER_HOME`.
