# ACL permission prompt labels

## Purpose

Align TUI permission prompts with ACL persistence semantics: approve once is
transient, and always allow writes a bounded durable project rule for the
current request or browser context.

## Design

The approval overlay now labels choices as:

- `Approve once`
- `Always allow this request`
- `Always allow this browser context`
- `Deny`

The footer uses the same language for keyboard shortcuts. Status and summary
rendering no longer presents an active sandbox mode and instead shows
permissions as project ACL-backed.

## Logic

Generic prompts no longer expose the broad all-actions session grant. The text
is changed so users can distinguish a one-shot approval from a persisted ACL
rule without being offered a project-wide bypass in the normal prompt flow.

## Compatibility

Existing approval action values are preserved, so runtime execution and
connectors do not need a wire-format migration.

## Verification

- TUI permission prompt snapshot/unit tests were updated for the new labels.
