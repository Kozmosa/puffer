# Slash argument popup dismissal

## Purpose

Completing a slash command that expects arguments should leave the composer
ready for argument entry. The popup must not remain open with a misleading
`no matches` row after commands such as `/model `.

## Design

Slash popup matching and rendering now treat the first space as the boundary
between command selection and argument editing.

## Logic

- Popup rows are only produced while the input starts with `/` and contains no
  spaces.
- The inline dropdown follows the same rule, so `/command ` renders as normal
  prompt input without command suggestions.

## Contracts

Argument-bearing commands keep their trailing space completion behavior. The
space transitions the UI out of command-selection mode and into argument-entry
mode.
