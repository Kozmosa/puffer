# Permission Prompt Ctrl+C Dismissal

## Purpose

Permission prompts must not trap common interrupt keys during normal Personal
Agent use. When a pending tool request opens the permissions overlay, `Ctrl+C`
should resolve the prompt instead of being swallowed.

## Design

The permission prompt key handler treats `Ctrl+C` as a deny/dismiss action,
matching the overlay's existing `Esc` behavior. The change stays local to the
permission overlay handler and does not alter global composer or transcript
`Ctrl+C` behavior.

## Logic

- `Esc` and `Ctrl+C` both send `PermissionPromptAction::Deny`.
- The pending permission response channel is resolved immediately.
- The active overlay and pending permission request state are cleared through
  the existing `respond_to_permission_prompt` path.
- Navigation keys and explicit allow shortcuts keep their prior behavior.

## Contracts

Permission prompt keyboard handling must leave the worker unblocked for every
dismissal shortcut. Tests cover the response action and state cleanup so a
future refactor cannot reintroduce a no-op `Ctrl+C` inside the overlay.
