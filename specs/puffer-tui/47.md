# 47. Cost Panel During Streaming Turns

## Summary

`/cost` now opens a read-only TUI text panel instead of writing a system message
into the transcript while a provider turn is still streaming.

## Design

- The existing cost summary renderer is exposed from `puffer-core` so the TUI
  can render the same content without dispatching the slash command.
- The overlay path treats `/cost` like other read-only panels such as `/config`
  and `/context`.
- Opening the panel clears the composer input and leaves the active assistant
  draft as the final transcript item for subsequent stream deltas.

## Compatibility

The `/cost` slash command still writes the cost summary to the transcript in
non-interactive dispatch paths. Interactive TUI users get a panel, which matches
the behavior expected for read-only checks during an active turn.

## Tests

`puffer-tui` includes a regression test that starts a pending turn, seeds a
partial assistant draft, submits `/cost`, and asserts that a text panel opens
without mutating the transcript or queueing a prompt.
