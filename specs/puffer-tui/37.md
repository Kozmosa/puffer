# User Question Prompt Interrupts

## Context

The TUI's global `Ctrl+C` path cancels active provider turns, but
`AskUserQuestion` overlays handled the same key first. While a provider turn was
waiting on a user question, pressing `Ctrl+C` sent an empty answer instead of
interrupting the worker.

## Change

- Treat `Ctrl+C` on a user question prompt with a pending provider turn as an
  interrupt.
- Clear the stale question response channel and close the overlay without
  sending an empty response.
- Share the same pending-turn interrupt path used by permission prompts.

## Compatibility

Question prompts can still use `Esc` to answer with the empty response. The new
behavior only changes `Ctrl+C` when a turn is actively pending.

## Verification

- Added a TUI regression that drives the full `handle_key` path with a pending
  user question overlay, then verifies the cancel token is tripped and no answer
  is sent.
