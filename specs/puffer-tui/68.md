# Deterministic connect question flow

## Scope

The TUI pending-submit path for `/connect` now follows the core deterministic
connection-name contract.

## Design

When `/connect` needs to ask which connector to use, the user answers only that
connector choice. The connection name is derived from the connector's suggested
connection slug, matching the command-helper behavior used outside the TUI.

## Compatibility

The explicit two-argument form, `/connect <connector> <connection>`, remains the
custom-name path. The prompt-driven TUI flow no longer asks a second free-form
connection-name question.
