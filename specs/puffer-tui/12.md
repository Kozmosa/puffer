# Text overlay scroll clamping

## Purpose

Diagnostic text overlays such as `/config`, `/context`, `/doctor`, and `/skills`
must stay readable when users jump to the bottom or repeatedly page through the
panel.

## Design

Text overlays keep their existing scroll state and keyboard controls. Rendering
now clamps the requested scroll offset against the visible body height before
passing it to Ratatui.

## Logic

- `G` still records a large bottom-scroll request so the renderer can resolve it
  with the current viewport height.
- The renderer computes the maximum valid scroll from the overlay line count and
  bordered body height.
- Oversized scroll offsets render the final page of content instead of an empty
  panel.

## Contracts

This is a render-time clamp only. It does not mutate shared overlay state, so
resizing the terminal can re-resolve the same bottom-scroll request for the new
height.
