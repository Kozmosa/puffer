# 48. Queue Diff During Streaming Turns

## Summary

`/diff` is queued while a provider turn is running instead of mutating the transcript during streaming.

## Design

- `/diff` no longer bypasses pending-turn deferral.
- Existing overlay-backed read-only commands still open immediately.
- Once the current turn finishes, queued prompt handling dispatches `/diff` normally.

## Compatibility

Normal `/diff` behavior outside pending turns is unchanged. Startup-safe `/diff` can still run before onboarding because that path is separate.

## Tests

`puffer-tui` has a regression test that seeds a pending turn and assistant draft, submits `/diff`, and asserts the command is queued and transcript unchanged.
