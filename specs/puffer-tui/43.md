# Composer Horizontal Scroll

## Problem

Long single-line prompts in a narrow TUI viewport kept the cursor pinned at the
right edge while the visible text stayed anchored at the beginning. Users could
keep typing, but the tail of the prompt they were editing was hidden.

## Design

- The composer prompt renderer now receives the prompt row width.
- For the active logical input line, it computes a display-width-aware window
  that keeps the cursor tail visible.
- Wide characters are measured with `unicode-width`, so Chinese text and other
  double-width input stay aligned.
- Multi-line vertical scrolling keeps its existing behavior.

## Test Coverage

- `render_long_prompt_keeps_cursor_tail_visible` renders a narrow terminal with
  double-width Chinese text followed by a unique suffix and verifies the suffix
  is visible at the cursor.
