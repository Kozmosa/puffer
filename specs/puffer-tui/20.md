# Queued prompt overlay deferral

## Goal

Queued prompts can become ready while the user is reading or operating a modal
overlay. The TUI should not start the next provider turn behind that overlay.

## Design

The queued-prompt submit helper now treats any active overlay as a temporary
blocker, the same way it already treats an active provider turn. The queued text
stays in order and is retried by the main loop once the overlay has closed.

## Contracts

- A queued prompt is not dequeued while an overlay is active.
- Closing the overlay lets the same queued prompt start normally.
- Provider turns still start immediately when the TUI is idle and no overlay is
  visible.
- User-visible queue ordering is unchanged.
