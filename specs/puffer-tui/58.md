# AskUserQuestion Input Questions

The TUI now renders first-class text input questions from `AskUserQuestion`.

- Choice questions remain the default and continue to render numbered options
  plus the existing free-form "Other" row.
- Questions with `type: "input"` render a single input row and activate the
  composer text field immediately, so digits and other typed characters are
  captured as the answer instead of being interpreted as option shortcuts.
- Empty input questions do not submit on Enter; the overlay stays open until
  the user types a value or cancels.
- Input questions are intended for `/connect` credential and login collection
  such as phone numbers, codes, passcodes, hosts, URLs, and sender lists.

This preserves existing AskUserQuestion choice behavior while making
connector auth collection deterministic and not dependent on stdin.
