# Clamp status and session overlay scroll

## Purpose

`/status` and `/session` are normal inspection flows in the interactive TUI.
Repeated page-down input must leave the bottom of the panel visible instead of
scrolling past the content into a blank overlay.

## Design

The renderers clamp their stored scroll offsets against the rendered content
height at draw time. This matches the existing text-overlay behavior while
keeping keyboard state simple and lossless between viewport sizes.

## Logic

- The session overlay clamps against its text line count and visible body rows.
- The status overlay clamps against Ratatui's wrapped paragraph line count and
  visible body rows.
- Regression tests force large page-down offsets and assert the bottom content
  still renders.

## Contracts

No command or storage contract changes. This is a rendering safety fix for
overscrolled interactive overlays.
