# TUI Transcript Actors

## Purpose

The native TUI persists the same actor metadata as daemon and connector
runtimes, keeping session transcripts consistent regardless of entrypoint.

## Design

User submissions persist user actors, assistant responses and tool invocations
persist assistant actors, and system/auth messages persist system actors.

## Compatibility

The rendered transcript UI is unchanged. Actor metadata is only persisted in
session events for external consumers and future reload/export surfaces.
