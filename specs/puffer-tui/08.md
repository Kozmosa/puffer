# Statusline disable semantics

## Purpose

`statuslineEnabled=false` must fully disable TUI statusline behavior. Users who
turn it off should not run configured statusline commands or see either custom
or built-in statusline text.

## Design

The statusline refresh path treats a disabled statusline the same as an absent
statusline configuration: stale text and signatures are cleared and command
execution is skipped.

## Logic

- `refresh_status_line` returns before reading the configured command when the
  session flag is false.
- Rendering only builds custom status text when the flag is true.
- The footer row renders blank status content while disabled instead of falling
  back to the built-in model/sandbox summary.

## Contracts

Disabling the statusline is session-scoped and does not remove the configured
statusline command from config. Re-enabling can reuse the existing config on a
later refresh.
