# Transcript Actor Compatibility

## Problem

The session transcript schema now records optional message actors, but a few TUI
flow persistence paths and tests still used the older actor-less event shape.
That made `cargo test -p puffer-tui` fail before exercising the TUI flow tests.

## Design

- Resumed-session command events are persisted with the current user actor.
- Pending assistant drafts are persisted with the current assistant actor.
- TUI tests ignore actor fields when the assertion only cares about text.

## Test Coverage

- `cargo test -p puffer-tui` compiles the TUI flow and transcript persistence
  tests against the actor-aware transcript event schema.
