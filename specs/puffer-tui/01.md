# puffer-tui AskUserQuestion Overlay

## Purpose

The TUI renders live `AskUserQuestion` tool prompts as selectable list overlays
so interactive users can answer model clarification questions without leaving
the terminal.

## Design

Background turn execution installs a `with_user_question_prompt_handler` around
the provider turn. When core asks for user answers, the worker sends a typed
pending-submit event to the UI thread and blocks on a response channel.

The UI thread parses the question payload into `UserQuestionOverlay`, stores the
response channel in `TuiState`, and displays the active question as a list. The
same overlay state supports composer-attached inline rendering and modal overlay
rendering through the shared overlay row APIs.

Completed `AskUserQuestion` tool results use a dedicated transcript renderer:
the header is shown as `Ask User Question`, the input summary shows the active
question, and the output body renders selected answers as `question: answer`
instead of exposing the raw JSON payload.

## Logic

- Up and Down move through the active question's options.
- Enter confirms the highlighted single-select answer.
- Multi-select questions use Space to toggle rows and Enter to confirm.
- Multiple questions advance one at a time and send one combined answers map
  after the final confirmation.
- Esc or Ctrl+C dismisses the prompt with an empty response, allowing core to
  keep its existing pending-question fallback behavior.

## Contracts

Answers are keyed by the exact question text. Single-select answers are strings;
multi-select answers are arrays of strings. Annotations are returned as an empty
object because the TUI currently collects only option choices.

Invalid question payloads do not leave the worker blocked: the TUI sends an
empty response and records a visible system message.

The selected-answer transcript rendering is display-only. The persisted tool
invocation remains the original structured `AskUserQuestion` JSON output so
provider replay and session restore preserve the same conversation contract.
