# Monitor Task Overlay Actions

## Summary

The background task overlay now shows monitor task actions as selectable rows.

## Behavior

- Non-slash `/tasks` action prompts are included in the overlay and sent to the
  current agent when selected.
- Monitor action rows use the monitor task id as the selector, matching
  `/tasks show` rows.
- `/tasks ignore <task-id> [reason]` rows remain visible so users can dismiss
  monitored connector tasks directly from the overlay.

## Compatibility

Read-only preview rendering still applies to `/tasks` subcommands. Non-slash
action prompts use their description as preview text because they are intended
to be submitted to the active agent.
