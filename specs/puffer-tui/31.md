# Stale provider fallback

TUI provider setup handles saved provider ids that no longer exist in the
loaded provider registry.

Compatibility contract:

- A normal prompt with a stale `current_provider` must not start a provider
  turn for that missing provider.
- The prompt is deferred and the transcript remains unchanged while setup is
  repaired.
- The user is shown the onboarding provider picker with currently available
  providers.
- Existing configured providers still use the provider-specific auth/model
  setup path.

Regression coverage lives in `crates/puffer-tui/src/tests/login.rs`.
