# Filter Picker Cursor Movement

## Context

Filterable TUI picker overlays render the shared composer cursor, but their
keyboard handler ignored Left, Right, Home, and End. Users could type and delete
filter text, but could not correct text in the middle of the query.

## Change

- Route Left, Right, Home, and End to `TuiState` cursor movement when the active
  overlay accepts filter input.
- Keep picker selection unchanged while moving the cursor; matching is refreshed
  by subsequent edits.

## Compatibility

Non-filterable overlays keep their existing navigation behavior. API-key prompt
overlays already handled cursor movement through their own text-input branch and
are unchanged.

## Verification

- Added a TUI regression for `/login` provider filtering: type `opnai`, move the
  cursor left, insert `e`, and confirm Enter opens the OpenAI auth picker.
