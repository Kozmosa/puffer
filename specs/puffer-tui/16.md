# Let local startup commands bypass auth onboarding

## Purpose

Users should be able to launch the TUI directly into local inspection commands
such as `/diff` and `/session` even when the selected provider still needs
login. Auth onboarding should not block read-only workspace/session inspection.

## Design

Startup prompt bypass now recognizes no-argument local inspection and settings
overlays. Provider-backed prompts and commands with arguments still wait for
onboarding unless they were already explicitly supported.

## Logic

- `/diff` dispatches before missing-auth onboarding and renders the local git
  diff summary.
- `/session` and `/remote` can open their read-only session overlay before
  provider login.
- Read-only panels such as `/config`, `/permissions`, `/skills`, `/mcp`, and
  `/hooks` use the same startup bypass.
- Commands with arguments, free-text prompts, and `/login` remain outside the
  bypass.

## Contracts

No command schema or provider-auth contract changes. This only widens the TUI
startup routing guard for safe local inspection commands.
