# puffer-tui Onboarding Effort Default Test

## Purpose

The onboarding model picker test should track the provider effort policy from
`puffer-core` instead of hard-coding a stale OpenAI default.

## Design

No TUI behavior changes. The test assertion now calls the shared provider
preference helper used by the rest of the model-selection stack.

## Logic

- OpenAI onboarding still opens the effort picker after a model is selected.
- The selected effort row is expected to match
  `default_effort_level(ModelPreferenceFamily::OpenAi)`.
- The picker still exposes OpenAI-specific `minimal` and `xhigh` choices.

## Contracts

Provider effort defaults live in `puffer-core`. TUI tests must assert against
that shared contract so provider-policy changes do not create stale UI fixture
expectations.
