# Pre-Tool Assistant Persistence

## Problem

Successful streaming turns can emit assistant text before a tool call, then
continue with tool output and a final assistant answer. The live TUI transcript
showed that pre-tool text, but only tool rows and final assistant text were
persisted, so resuming the session silently dropped text the user had seen.

## Design

- Track how much of a pending turn's live transcript has been persisted.
- Before appending tool invocation rows, persist any assistant text rows created
  after the previous persisted boundary.
- Advance the persisted boundary after tool rows are written.
- Failed-turn cleanup now removes only still-unpersisted assistant drafts, so
  assistant text that preceded successful tool output remains in history.

## Test Coverage

- `tests/pending_submit.rs` now simulates text before a tool call, streamed
  final text, and a successful completion. It verifies the session log preserves
  assistant text, tool invocation, and final assistant text in the same order
  the live transcript showed.
