# Permission Status Label

## Purpose

Align the native TUI status surfaces with the ACL-based permission model.

## Design

The top context panel and footer status line describe the active permission
system as `permissions ACL`.

## Logic

The label is rendered from the shared summary helpers, so compact panel rows,
footer rails, tmux snapshots, and render snapshots use the same wording.

## Contract

- The status line still includes model, remaining context, current directory,
  cache state when available, and background task count when present.
- The permission suffix is descriptive UI text only; it does not change
  permission evaluation or grant persistence.
- Snapshot fixtures must use `permissions ACL` for TUI-owned status output.
