# Workflow Subcommand Picker

The TUI slash popup now keeps assisting users after they select or type
`/workflows `. It presents deterministic workflow subcommands rather than
dropping the user into an empty argument prompt.

- Typing `/workflows ` lists the supported workflow subcommands for listing,
  creating, appending, deleting, inspecting connections, connector catalog,
  monitor tasks, and run history.
- Typing a partial subcommand such as `/workflows app` filters the picker and
  completion writes `/workflows append ` into the composer.
- Exact subcommands such as `/workflows runs` still submit normally on Enter,
  while Tab or explicit selection can add the trailing space for arguments.
- The popup hides once the user starts a query after the subcommand, such as
  `/workflows connectors telegram`, so free-form workflow searches are not
  interrupted.

This is a TUI-only discoverability change. It does not change workflow command
parsing, connector runtime behavior, or any webhook connector implementation.
