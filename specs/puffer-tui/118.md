# Workflow Connector Argument Popup

TUI slash completion now carries connector catalog metadata into workflow
argument entry for the two connector-backed workflow commands.

## Behavior

- `/workflows new <query>` suggests trigger-capable connector templates and
  completes to `/workflows new <connection>-workflow <connection>`.
- `/workflows append <query>` suggests trigger-capable connector templates and
  completes to `/workflows append <connection> /tmp/<connection>.log
  --connector <connector>`.
- Connector argument searches match connector slug, suggested connection slug,
  description, skill, binary, workflow terms, and connector action slugs.
- Setup-only connectors are omitted from these workflow argument suggestions,
  while remaining visible through `/connect` and `/workflows connectors`.
- The popup hides once the input already looks like a complete workflow command,
  preserving direct command submission for explicit workflow arguments.

## Compatibility

This is a TUI completion and rendering change only. It does not add connector
templates, alter connector auth, change workflow parsing, or modify workflow
storage. The generated command shapes reuse existing `/workflows new` and
`/workflows append` terminal contracts.
