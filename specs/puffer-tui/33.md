# Session Command Overlay Routing

## Context

The TUI intercepted bare `/session` before normal command dispatch and opened
the remote-session overlay. In a local session this showed the remote-mode
notice instead of the same metadata summary produced by `/session show`.

## Change

- Route bare `/session` to the generic scrollable text panel backed by
  `render_session_panel`.
- Keep `/remote` mapped to the dedicated remote session URL/QR overlay.
- Preserve startup/onboarding bypass behavior for read-only session inspection.

## Compatibility

`/session show` and bare `/session` now agree on the visible summary contract.
Remote sharing behavior remains available through `/remote` and the existing
remote-control commands.

## Verification

- Updated TUI panel coverage to assert `/session` renders metadata such as
  `session_id=` and `cwd=`.
- Added coverage that `/remote` still opens the remote session overlay.
