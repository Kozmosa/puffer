# 45. Queued Slash Picker Commands

## Summary

Queued slash commands now preserve the interactive picker path after a running
turn completes.

## Design

- `submit_next_queued_prompt` drains one prompt only when there is no active
  turn and no overlay, as before.
- Before falling back to normal prompt submission, the drained prompt is passed
  through the same overlay-opening path used by direct Enter submissions.
- Empty picker commands such as `/model` and `/resume` therefore open their TUI
  picker instead of degrading into command-summary transcript text.

## Compatibility

Provider prompts, shell shortcuts, and non-picker slash commands still fall
through to normal queued submission.

## Tests

`flow_tests` covers queueing `/model` during a pending turn, clearing the turn,
draining the queue, and asserting that the OpenAI model picker opens.
