# Clear Exits History Navigation

## Context

The TUI lets users recall prior prompts with Up and return through recall
history with Down. Pressing Escape clears the input through `TuiState::clear`,
but the history cursor stayed active after the input was cleared.

## Change

- Make `TuiState::clear` exit history navigation after clearing the input and
  pending paste state.
- Preserve slash-command synchronization after the clear.

## Compatibility

Prompt recall storage is unchanged. The change only affects the state left
behind after clearing a recalled prompt, so Down can return to normal
transcript navigation instead of continuing recall mode.

## Verification

- Added a TUI regression covering recall with Up, clearing with Escape, and
  ensuring Down/history-next is no longer active afterward.
