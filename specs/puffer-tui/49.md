# 49. Cancel Deferred Prompt Onboarding

## Summary

Canceling setup from an onboarding overlay now clears any prompt that was
deferred before provider setup completed.

## Design

- Back navigation through onboarding goes through a shared helper.
- When the active onboarding overlay is closed with no next overlay, the TUI
  clears the deferred prompt before closing the overlay.
- Normal back navigation between onboarding steps keeps the deferred prompt so a
  completed setup can still submit the original message.

## Compatibility

Non-onboarding overlays keep their existing `Esc` behavior. Successfully
completed onboarding still submits a deferred prompt once provider, auth, and
model checks are ready.

## Tests

`puffer-tui` has a regression test that submits a prompt with a stale provider,
backs out of onboarding, and verifies the queued prompt is not sent later.
