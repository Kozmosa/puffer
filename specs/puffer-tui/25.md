# Picker query misses

Filterable TUI picker overlays refuse Enter when the active query does not
match the highlighted item.

Compatibility contract:

- Matching queries still move selection and apply the selected provider, model,
  auth, session, theme, agent, or command entry.
- Empty queries keep the previous picker behavior.
- Non-matching queries leave the picker open and show a transient status hint
  instead of applying a stale highlighted row.
- This guard applies to all filterable picker overlays through shared overlay
  state matching logic.

Regression coverage lives in `crates/puffer-tui/src/tests/model_selection.rs`.
