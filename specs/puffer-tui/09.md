# Slash popup selection clamping

## Purpose

Slash-command completion must remain usable when command resources change while
the composer is open. A stale selection index should not leave the popup with no
highlight or make Enter ignore the visible match.

## Design

Selection clamping now happens at both interaction and render boundaries. The TUI
state resolves the active command through a helper that clamps `slash_selection`
to the current popup row count before applying or completing a command.

## Logic

- Empty popup rows reset `slash_selection` to zero and do not complete anything.
- Non-empty popup rows clamp stale selections to the last available row.
- The renderer mirrors the same clamp so a stale index still highlights the
  selected visible command.

## Contracts

Resource reloads and command-list changes are allowed while a slash query is
active. Completion and highlighting must follow the currently visible row set,
not a selection index from a previous row set.
