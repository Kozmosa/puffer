# Workflow Connector Filter Hints

## Summary

The TUI-visible `/workflows connectors` output now advertises stable connector
filter terms.

## Behavior

- `/workflows connectors` prints the filter terms before result counts.
- `/workflows` default output points users to `/workflows connectors ...`
  because default connector rows remain limited to trigger-ready connectors.
- `trigger-ready` is the preferred trigger filter term because a plain
  substring search for `trigger` can also match `no-trigger`.
- `has-actions` is the preferred action filter term because a plain substring
  search for `actions` can match connector descriptions instead of capability
  metadata.

## Compatibility

The command remains text-only and query-driven. The new line is additive and
does not change slash-command parsing.
