# puffer-tui Status Line Shell Execution

## Purpose

Custom status line commands should refresh predictably without letting slow
login-shell startup files blank the footer or stall interactive rendering.

## Design

The TUI still executes the configured status line command through the user's
detected shell, but it now uses normal shell execution instead of login-shell
execution. The command continues to receive the Claude-compatible status JSON
on stdin and writes the visible footer text to stdout.

## Logic

- Status line refresh builds the same JSON input snapshot as before.
- Refresh signatures still include the command and JSON input, so unchanged
  state does not rerun the command.
- The command runs with `shell -c <command>` in the session cwd.
- Non-empty stdout lines are trimmed and joined into one footer string.
- Failed or timed-out commands remain best-effort and clear the cached footer
  output for that snapshot.

## Contracts

Status line commands must not depend on login-shell profile loading. They may
depend on the normal process environment, current working directory, stdin JSON,
and stdout text contract.

This keeps custom status lines compatible with Claude-style stdin payloads while
avoiding UI instability from slow or interactive login shell configuration.
