# Defer mutating picker selections during active turns

## Purpose

Mutating slash commands must not bypass pending-turn deferral just because the
user reaches them through a picker or overlay. A running provider turn should
finish before commands such as `/rewind` mutate transcript state.

## Design

The Enter path now checks pending-turn deferral before opening mutating
overlays from composer input. Overlay-selected commands use the same deferral
classifier before dispatch: if a turn is active and the selected command is
state-changing, the command is queued and the overlay closes.

## Logic

- `/rewind` typed while a turn is running queues instead of opening the rewind
  picker.
- A rewind picker selection made while a turn is running queues instead of
  truncating the transcript immediately.
- Read-only no-argument overlays such as `/status` still open during active
  turns.

## Contracts

No slash-command schema or transcript format changes. This only routes picker
submissions through the same pending-turn safety policy used by composer
submissions.
