# Thread-local Puffer home override

- `ConfigPaths::discover` still resolves the user config root from `PUFFER_HOME`, `HOME`, then `dirs::home_dir` for normal runtime use.
- Tests and embedders that need an isolated config root can call `set_puffer_home_override(path)` and keep the returned `PufferHomeOverride` guard alive for the scope that calls `ConfigPaths::discover`.
- The override is thread-local, so parallel tests no longer mutate process-wide `PUFFER_HOME` and cannot redirect unrelated sessions or config writes into temporary directories owned by another test.
- Dropping the guard restores the previous thread-local override, preserving nested override behavior.
