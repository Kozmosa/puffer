# 05 - Proxy Selection Normalization

Updates proxy config validation around empty and stale selections.

- `ProxyConfig::normalize_selection` trims selected IDs, disables proxy routing
  when no endpoints exist, and clears `selected` in that state.
- When proxy routing is enabled and endpoints exist but no valid selected ID is
  present, normalization selects the first endpoint.
- Validation still rejects duplicate IDs, empty hosts, zero ports, malformed
  bypass entries, and selected IDs that do not exist.
- Validation no longer emits a missing selected-ID failure for empty enabled
  proxy config; callers normalize that state before persisting or using it.
