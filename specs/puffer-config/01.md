# Project memory canonical roots

Project memory resolution matches the current working directory against the
registered project roots after path normalization. The returned
`ResolvedProjectMemory.root` is the normalized root used for matching and
storage slug generation.

Compatibility contract:

- Registered project paths can be written with platform-specific aliases such
  as `/var` on macOS.
- Resolution still matches nested working directories against the longest
  registered project root.
- Tests compare the returned root against the normalized fixture path so the
  suite is portable across macOS path aliasing.

Regression coverage lives in the project-memory tests in `puffer-config`.
