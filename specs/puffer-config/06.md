# Browser Profile Config Removal

## Summary

The `[browser].chrome_profile` setting is no longer part of the runtime config
contract.

## Behavior

`BrowserConfig` intentionally carries no user-selectable Chrome profile fields.
Unknown browser keys from older config files are ignored by normal deserialization
so prior installs do not fail to load, but new daemon and desktop settings
snapshots do not emit browser profile configuration.

## Compatibility

Browser-backed connectors now use the global Puffer-owned browser profile
documented by the daemon. Config migration is not required because stale keys are
ignored rather than interpreted.
