# Connector Transcript Actors

## Purpose

Connector-driven sessions record actor metadata on ingress and assistant output
so non-TUI runtimes get the same agent/subagent attribution as local sessions.

## Design

Connector user ingress appends user transcript events with `AppState::user_actor`.
Assistant responses append assistant transcript events with
`AppState::assistant_actor`, inheriting root-agent or subagent identity from the
session state. Tool invocations are persisted before the assistant response with
the same assistant actor so connector histories preserve tool/result ordering
and agent provenance.

## Compatibility

Connector command counting and session mapping remain based on transcript event
variants. Actor metadata is optional and does not change routing behavior.
