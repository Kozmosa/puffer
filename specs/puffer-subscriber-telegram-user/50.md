# Account-Scoped Delivery Cursor

- The Telegram user subscriber delivery cursor now records the authenticated
  Telegram `account_user_id` alongside per-chat message watermarks.
- When the update loop authenticates as a different account than the cursor was
  written for, the subscriber clears stale chat watermarks and reruns startup
  hydration for the current account.
- This prevents a cursor from an old Telegram account from treating lower
  message ids in the current account as duplicates.
- Existing cursor files without `account_user_id` are accepted and reset on the
  first authenticated update loop.
