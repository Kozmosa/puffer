# puffer-subscriber-telegram-user update 34

The Telegram subscriber now starts a non-blocking peer-hash priming task after
startup hydration and after live-client replacement. The task walks recent
unsuppressed group and channel dialogs and reads a bounded number of recent
messages without emitting them. Grammers uses those responses to populate its
peer hash cache, which prevents socket updates from being discarded before
Puffer can see them when a live message references a sender that was missing
from the dialog snapshot.

This priming runs alongside the live reader rather than delaying readiness. It
is capped by dialog and message count, skips muted or archived dialogs, and is
aborted whenever the live client is replaced or the subscriber exits.
