# Telegram Peer Cache Avatars

Startup dialog hydration now fetches the small Telegram profile photo for peers
that expose one and stores it in `peer-cache.json` as a data URI on the peer
record's `avatar` field.

Avatar hydration is skipped when a cached avatar already exists, and failures
are logged per dialog without aborting startup hydration. Contact-book hydration
continues to populate names only; avatar fetching stays tied to dialogs so the
subscriber prioritizes peers that can appear in message-based contact ranking.

If Telegram dialog iteration fails after some dialogs were processed, startup
hydration logs the error, exits the dialog scan, and saves the partial peer
cache instead of discarding fetched avatar data.
