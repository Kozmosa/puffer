# Telegram local message listing and faster search context

- The Telegram user subscriber now handles `TelegramListMessages` by resolving
  a peer, reading recent message history, sorting the returned page in
  chronological order, and emitting a `message_list` control event.
- Message-list events include `messages`, `count`, `limit_reached`,
  `before_id`, and `next_before_id`. Callers pass `next_before_id` back as
  `before_id` to fetch older pages.
- Search context no longer scans forward through up to 2000 recent messages
  for every match. Non-zero search context now returns previous messages before
  each hit plus the hit itself, avoiding the slow per-match history crawl and
  the noisy context errors that crawl could trigger.
- Unauthenticated phases now emit `message_list_error` for list requests, using
  the same explicit error style as peer listing and message search.
