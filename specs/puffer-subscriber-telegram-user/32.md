# puffer-subscriber-telegram-user update 32

The Telegram user subscriber no longer asks Grammers to catch up from persisted
MTProto update state on connect. Puffer's durable delivery cursor is the
restart boundary, and the startup dialog hydration pass seeds current
channel/megagroup state before live socket processing begins. This prevents a
stale Grammers session from delaying live delivery behind account or channel
`getDifference` recovery work.

The Grammers update queue remains unbounded so socket updates that do arrive
while commands or startup work are in progress are not dropped by a local
queue cap. The persisted MTProto session is still saved after handled live
updates for compatibility, but it is not treated as the authoritative resume
point for monitor delivery.
