# Telegram session save creation

Telegram login session persistence now writes session bytes through an atomic
create-and-rename helper instead of relying on `grammers-session`'s
`save_to_file` behavior.

The vendored `save_to_file` opens existing files for write and does not create
the target file. A first-time phone-code or 2FA login could therefore fail even
after Telegram accepted the credentials unless `telegram.session` already
existed. The subscriber login path now creates parent directories, writes a
temporary file, and renames it into place.

Regression coverage verifies that a missing `telegram.session` file is created
and no temporary file remains after a successful save.
