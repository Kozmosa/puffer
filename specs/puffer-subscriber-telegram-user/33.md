# puffer-subscriber-telegram-user update 33

The Telegram live update reader now records an `awaiting_raw` diagnostic before
each `next_raw_update` await and a `waiting_raw_timeout` diagnostic when that
await does not yield within the watchdog window. This makes the previously
opaque stuck state visible in `update-diagnostics.ndjson`.

When the reader stalls, the parent subscriber aborts the live reader task,
opens a fresh MTProto client from the persisted Puffer session, reruns startup
dialog hydration, persists the refreshed update state, and resumes live update
streaming. The durable delivery cursor remains the message replay boundary, so
reconnect recovery can emit missed unsuppressed messages without reintroducing
a periodic polling loop.
