# puffer-subscriber-telegram-user update 21

Telegram user events now carry notification mute metadata.

- Dialog catch-up records each dialog's `notify_settings.mute_until` in a
  per-process mute cache.
- Live message events include `notification_muted` from the cache and
  `notification_silent` from Telegram's message-level silent flag.
- Raw Telegram `updateNotifySettings` events refresh the cache for concrete
  users, groups, channels, and forum topics.
- `telegram list-peers` includes `notification_muted` so monitor backfill can
  skip muted chats before reading messages.

This preserves the Telegram event stream for non-monitor workflows while making
notification delivery state available to monitor routing.
