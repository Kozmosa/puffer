# Telegram live update deadline recovery

The Telegram user subscriber depends on Grammers' message box to trigger
`updates.getDifference` and `updates.getChannelDifference` when socket updates
are quiet. The vendored `grammers-session` state machine now records the first
deadline after `updates.getState` and after dialog channel PTS hydration.

Without that deadline, sessions started with `catch_up=false` could load valid
PTS state but never mark any entry as due for difference polling, so channel
messages were only seen when Telegram pushed a socket update. This caused
minute-level gaps and stale task history for active groups.

The subscriber still avoids periodic dialog polling in the hot path. Startup
hydration seeds dialog state once, and steady-state recovery is handled by the
MTProto update state machine. Muted filtering remains payload-specific:
peer mute and archive state suppress delivery, while global notification
settings and silent peer settings do not suppress individual chats.
