# Telegram Peer Metadata Cache

## Summary

The Telegram user subscriber now persists peer metadata to
`peer-cache.json` in the account state directory.

## Architecture

- Startup hydration loads the existing cache once.
- The subscriber refreshes names from Telegram contacts and visible dialogs
  while it is already connected.
- The cache is saved once at the end of hydration when changed.

## Data Contract

- Records include peer id, kind, title, username/usernames, first and last
  names, phone when visible, bot status, source, and update timestamp.
- Contact-book names can override abbreviated profile names when they are more
  complete.
- The cache is best-effort: contact API failures are logged and do not block
  message delivery.

## Compatibility

- No inference-time Telegram calls are introduced.
- Existing session, cursor, and diagnostics files remain unchanged.
