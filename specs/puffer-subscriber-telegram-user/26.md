# Explicit Telegram login session isolation

Explicit phone and QR login commands no longer reuse or short-circuit through
an already-authorized subscriber session. This prevents an existing imported
Telegram Desktop/macOS authorization from being reported as the result of a
new phone or QR login attempt.

Phone login replaces the runtime client with the fresh pre-auth MTProto
session returned by `login::start` and exits the update loop back to the login
finalization phase, where code and password commands complete that same fresh
session.

QR login clears any stale QR state, exports the QR token from a fresh transient
MTProto session, and only replaces the runtime client after Telegram approves
that QR session. The import-desktop path remains the only flow that copies a
Desktop/macOS app authorization into Puffer.
