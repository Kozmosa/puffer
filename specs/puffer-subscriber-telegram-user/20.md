# Telegram sender-filtered message listing

## Scope

The Telegram personal-account subscriber can list messages from one sender
inside a peer without requiring a text query.

## Design

- `handle_list_messages` accepts optional `sender` and `scan_limit` inputs.
- Sender filtering matches outgoing `me`/`self`/`outgoing`, numeric sender ids,
  usernames and `@handles`, and display names through the existing peer-search
  matcher.
- Unfiltered listing scans exactly the requested result limit, preserving the
  prior fast path.
- Sender-filtered listing scans a bounded recent window, defaulting to 200
  messages and capping at 500, while still returning at most the requested
  result limit.
- The response includes `sender_filter`, `scanned`, `scan_limit`, and
  `scan_limit_reached` so callers know whether to page deeper with
  `next_before_id`.

## Compatibility

The `message_list` payload remains append-only. Existing consumers can ignore
the new metadata fields.
