# Bounded live cursor poll

## Summary

The Telegram personal-account subscriber now runs a bounded live cursor poll
alongside the socket update reader.

## Design

- Every three seconds, the main subscriber loop scans a capped set of recent
  dialogs and compares each dialog's latest message against Puffer's delivery
  cursor.
- For chats that already have a cursor, new messages are fetched from Telegram
  history up to a small per-chat cap, sorted oldest-first, and passed through
  the same delivery path used by socket updates.
- Muted and silent messages remain suppressed, but they now produce message
  diagnostics with live lag values before the cursor is advanced.
- New chats without cursor state are marked seen to avoid importing historical
  backlog as live work.

## Compatibility

This path does not change task schemas or filter behavior. It makes the
subscriber resilient when Telegram does not push every personal-account update
over the socket, while keeping the poll bounded by recent dialog activity and
per-chat message count.
