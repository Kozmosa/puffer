You are testing Puffer desktop UI/UX with the local fuzz framework.

Repo: current Puffer repository root. Run commands from the repo root unless a
specific artifact path is requested.

Rules:
- Do not patch product code.
- Do not commit or push.
- Use the local `apps/puffer-desktop/tests/fuzz/` framework and follow its docs/instructions.
- Count only real, user-visible UI/UX bugs triggered by click, type, keyboard, resize, reconnect, stale daemon event, or response ordering.
- Do not count fixture-only, environment-only, or tooling-only failures.
- Focus priority: chat/session/new agent/turn/reload core loop first; then workspace/session switching/provider/model flows; then Files/Terminal/Browser.
- Use fake daemon unless the framework explicitly says otherwise.
- Keep temporary replay specs and Playwright artifacts under `apps/puffer-desktop/tests/fuzz/.runs/`.
- Use scheduler shard boundaries when available. Treat `ownedNodes` as the
  accepted-finding boundary, and use `allowedSetupNodes` only to reach that
  subtree.
- Do not edit `apps/puffer-desktop/tests/fuzz/BUGS.md` directly. For every
  accepted finding, include a `BUG_LIST_APPEND` block in your report so the
  main agent can append it with `puffer-fuzz.mjs bug-list --append`.

Workflow:
1. Read `apps/puffer-desktop/tests/fuzz/README.md`, `apps/puffer-desktop/tests/fuzz/agent_guide.md`, and `apps/puffer-desktop/tests/fuzz/playwright_adapter.md`.
2. Run:
   `node apps/puffer-desktop/tests/fuzz/bin/puffer-fuzz.mjs validate`
3. Generate a scheduler batch and a frontier/plan:
   `node apps/puffer-desktop/tests/fuzz/bin/puffer-fuzz.mjs schedule --limit 4 --namespace <unique-run-name> --out apps/puffer-desktop/tests/fuzz/.runs/<unique-run-name>/schedule.md --json-out apps/puffer-desktop/tests/fuzz/.runs/<unique-run-name>/schedule.json`
   `node apps/puffer-desktop/tests/fuzz/bin/puffer-fuzz.mjs frontier --out apps/puffer-desktop/tests/fuzz/.runs/manual/frontier.md`
   `node apps/puffer-desktop/tests/fuzz/bin/puffer-fuzz.mjs plan --profile core --out apps/puffer-desktop/tests/fuzz/.runs/manual/plan.md`
4. Pick scheduled shards first. If running without scheduler output, pick high-priority core seeds first:
   - `chat-turn-race`
   - `workspace-session-race`
   - `provider-auth-model-race`
   - `modal-focus-race`
5. For each selected seed, run:
   `node apps/puffer-desktop/tests/fuzz/bin/puffer-fuzz.mjs run --seed <seed> --iterations 30 --steps <seed-appropriate-steps> --rng-seed <unique-run-name> --out apps/puffer-desktop/tests/fuzz/.runs/<unique-run-name>/<seed>.json`
   `node apps/puffer-desktop/tests/fuzz/bin/puffer-fuzz.mjs report --input apps/puffer-desktop/tests/fuzz/.runs/<unique-run-name>/<seed>.json --out apps/puffer-desktop/tests/fuzz/.runs/<unique-run-name>/<seed>.md`
6. Pick high-value replay cases with diversity:
   `node apps/puffer-desktop/tests/fuzz/bin/puffer-fuzz.mjs top-cases --input apps/puffer-desktop/tests/fuzz/.runs/<unique-run-name>/<seed>.json --shard <shard-id-if-scheduled> --limit 5 --out apps/puffer-desktop/tests/fuzz/.runs/<unique-run-name>/<seed>-top.json --report-out apps/puffer-desktop/tests/fuzz/.runs/<unique-run-name>/<seed>-top.md`
7. Run isolated bounded replay:
   `node apps/puffer-desktop/tests/fuzz/bin/puffer-fuzz-replay-loop.mjs --seeds <seed> --shard <shard-id-if-scheduled> --limit 5 --attempts 3 --timeout 120 --rng-seed <unique-run-name> --namespace <unique-run-name> --fail-on-new-finding`
8. Record scheduler feedback when the run maps to a shard:
   `node apps/puffer-desktop/tests/fuzz/bin/puffer-fuzz.mjs record-feedback --shard <shard-id> --input apps/puffer-desktop/tests/fuzz/.runs/<unique-run-name>/bounded-replay-report.json`
9. Read `apps/puffer-desktop/tests/fuzz/.runs/<unique-run-name>/bounded-replay-report.md` and classify stable product candidates separately from harness preconditions, flakes, duplicates, and out-of-shard discoveries.
10. If a replay fails, use the repeated attempts in the bounded report to check stability.
11. Shrink the repro mentally or by creating a smaller temporary probe under `apps/puffer-desktop/tests/fuzz/.runs/<unique-run-name>/` only if needed.
